🐳 What is Docker? CONCEPT
🚢

Real World Analogy

Docker = Shipping Container மாதிரி. ஒரு ship-ல different types of goods (electronics, food, clothes) — எல்லாத்தையும் separate container-ல pack பண்ணி ship பண்றோம். எங்கே போனாலும் (Chennai, Singapore, USA) container same ஆ இருக்கும்.

Docker-லயும் உன் app + dependencies எல்லாத்தையும் ஒரு container-ல pack பண்ணி எந்த machine-லயும் run பண்ணலாம்!

Simple Definition

Docker is an open-source platform that allows you to package, ship, and run applications inside lightweight, isolated environments called containers.

Container-ல உன் app-க்கு தேவையான எல்லாமே இருக்கும் — code, runtime, libraries, config எல்லாம்.

❓ Why Use Docker? REASON

❌ Problem Without Docker

Dev machine-ல perfectly work பண்றது → Test server-ல போட்டா crash 😤
"It works on my machine!" — இது மிகவும் common problem.

# Dev machine: .NET 8, SQL Server 2022
# Test server: .NET 6, SQL Server 2019
# Result: App crashes! 💥

✅ Solution With Docker

Docker container-ல எல்லாமே include ஆகும் → எந்த machine-லயும் same ஆ run ஆகும்!

# Docker container: .NET 8 + SQL Server 2022 — fixed versions
# Dev machine ✅ + Test server ✅ + Production ✅
# Same container → Same result everywhere! 🎉
⚡

Consistency

Dev, Test, Prod — எல்லா இடத்திலயும் same environment. "Works on my machine" problem இல்லவே இல்ல.

🚀

Fast Deployment

Minutes-ல build பண்ணி deploy பண்ணலாம். VM மாதிரி hours wait பண்ண வேண்டாம்.

📦

Isolation

ஒவ்வொரு app-உம் separate container-ல. ஒண்ணு crash ஆனாலும் மத்தவை affect ஆகாது.

📈

Scalability

Traffic அதிகம் வந்தா containers இன்னும் add பண்ணலாம். Kubernetes-உடன் auto-scale.

💰

Cost Saving

VM-விட lightweight. ஒரே server-ல நிறைய containers run பண்ணலாம் → cost குறையும்.

🔄

Version Control

Image versions manage பண்ணலாம். Problem வந்தா previous version-க்கு rollback easy.

# Dockerfile example — ASP.NET Core app

FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS base
WORKDIR /app
EXPOSE 80

FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src
COPY . .
RUN dotnet publish -c Release -o /app/publish

FROM base AS final
COPY --from=build /app/publish .
ENTRYPOINT ["dotnet", "ZenCampusWeb.dll"]
🎯 Interview Questions & Answers Q&A

1 Docker என்ன? Simple-ஆ explain பண்ணுங்க.

▼

Docker is an open-source containerization platform that packages an application and all its dependencies together into a standardized unit called a container.

Container-ல app-க்கு தேவையான எல்லாமே இருக்கும் — code, runtime, libraries, config. இதனால எந்த machine-லயும் consistently run ஆகும்.

💡 One liner: "Docker lets you package your application with all its dependencies so it runs the same everywhere — dev, test, or production."

2 Docker Image vs Container — difference என்ன?

▼

Image = Read-only blueprint / template (C#-ல class மாதிரி)

Container = Image-ஓட running instance (C#-ல object மாதிரி)

  • ஒரே image-ல இருந்து multiple containers create பண்ணலாம்
  • Image immutable — change ஆகாது
  • Container mutable — runtime-ல change ஆகும்
💡 "Image is like a class definition, container is like an object created from that class."

3 Docker VS Virtual Machine — என்ன difference?

▼

VM: Full OS தேவை → Heavy (GBs), slow to start (minutes)

Docker: Host OS kernel share → Lightweight (MBs), fast (seconds)

  • Docker containers share the host OS kernel
  • VMs have their own complete OS
  • Docker is more efficient with resources
  • VMs provide stronger isolation
💡 "Docker containers are like apartments in a building (shared structure, separate spaces). VMs are like separate houses (completely independent)."

4 Why Docker? Real world benefit என்ன?

▼
  • Consistency — "Works on my machine" problem solve பண்ணும்
  • Portability — எந்த OS-லயும், cloud-லயும் run ஆகும்
  • Isolation — ஒவ்வொரு app தனியா run ஆகும்
  • Scalability — Kubernetes-உடன் auto-scale பண்ணலாம்
  • Speed — Fast deployment, seconds-ல start
  • Cost — VM-விட resource efficient
💡 Real example: ZenCampus app-ஐ Docker-ல run பண்ணினா — dev laptop-லயும், Azure-லயும் exact same ஆ behave ஆகும்.

5 Dockerfile என்ன? Key instructions என்னென்ன?

▼

Dockerfile = Docker image build பண்ண step-by-step instructions எழுதுற text file.

  • FROM — Base image specify பண்ணு
  • WORKDIR — Working directory set பண்ணு
  • COPY — Files copy பண்ணு
  • RUN — Command execute பண்ணு (build time)
  • EXPOSE — Port expose பண்ணு
  • ENTRYPOINT — Container start ஆகும்போது run ஆகற command
  • ENV — Environment variables set பண்ணு
💡 "Dockerfile is like a recipe — FROM gives the base ingredient, RUN adds steps, ENTRYPOINT defines how to serve it."

6 Docker Volume என்ன? எப்போ use பண்ணுவீங்க?

▼

Container delete ஆனாலும் data survive ஆக Volumes use பண்றோம்.

Container filesystem by default ephemeral — container போனா data போயிடும்.

  • Named Volume — Docker manage பண்ணும் (recommended)
  • Bind Mount — Host folder map பண்ணும்
💡 Use case: SQL Server data files, uploaded images, log files — container restart ஆனாலும் data safe-ஆ இருக்கணும்னா Volume use பண்ணு.

7 Docker Compose என்ன?

▼

Multiple containers-ஐ ஒரே YAML file-ல define பண்ணி, single command-ல manage பண்ண Docker Compose use பண்றோம்.

Example: ZenCampus = Web App + SQL Server + Redis → 3 containers → 1 compose file

  • docker-compose up — All services start
  • docker-compose down — All services stop
  • docker-compose logs — All logs பாரு
💡 "Docker Compose is like an orchestra conductor — it starts all your services (instruments) in the right order at once."

8 Multi-stage build என்ன? Why important?

▼

Build artifacts-ஐ final image-ல include பண்ணாம, only runtime files மட்டும் போடுவது.

  • Stage 1 (Build): SDK use பண்ணி compile பண்ணு (~700MB)
  • Stage 2 (Final): Only runtime use பண்ணு (~200MB)
💡 Benefit: Final image size ~70% குறையும். Smaller image = faster deployment + less storage cost.

9 Docker Hub என்ன?

▼

Docker Hub = Docker images-க்கான public cloud registry. GitHub-மாதிரி but code-க்கு பதிலா images.

  • Official images: nginx, mysql, mcr.microsoft.com/dotnet/aspnet
  • உன் custom images push பண்ணலாம்
  • Private registry alternatives: AWS ECR, Azure ACR
💡 "Docker Hub is like GitHub for Docker images — pull existing ones, push your own."

10 Docker-ல containerize பண்ண steps என்னென்ன?

▼
  • Step 1: Dockerfile எழுது (FROM, COPY, RUN, ENTRYPOINT)
  • Step 2: docker build -t myapp:v1 . — Image build பண்ணு
  • Step 3: docker run -d -p 8080:80 myapp:v1 — Container run பண்ணு
  • Step 4: docker push myrepo/myapp:v1 — Registry-ல push பண்ணு
  • Step 5: Server-ல docker pull பண்ணி deploy
💡 Complete flow: Code → Dockerfile → Image → Container → Registry → Production
⌨️ Must-Know Commands COMMANDS
# ====== IMAGE COMMANDS ======
docker build -t myapp:v1 . # Image build பண்ணு
docker images # All images list
docker pull nginx # Registry-ல இருந்து pull
docker push username/myapp:v1 # Registry-க்கு push
docker rmi myapp:v1 # Image delete

# ====== CONTAINER COMMANDS ======
docker run -d -p 8080:80 myapp:v1 # Container start (detached)
docker ps # Running containers
docker ps -a # All containers
docker stop container_id # Container stop
docker rm container_id # Container delete
docker logs container_id # Logs பாரு
docker exec -it container_id bash # Container-க்குள்ள enter

# ====== COMPOSE COMMANDS ======
docker-compose up -d # All services start
docker-compose down # All services stop
docker-compose logs # All logs
