Q1What is the difference between async/await and Task.Run()? When to use which?▼

async/await is for I/O-bound operations (DB calls, HTTP calls, file reads). It does NOT create a new thread — it releases the thread while waiting.

Task.Run() is for CPU-bound operations. It uses a ThreadPool thread to do heavy computation.

// I/O bound - use async/await (no new thread)
public async Task<string> GetDataAsync()
{
    var result = await _httpClient.GetStringAsync(url); // thread released while waiting
    return result;
}

// CPU bound - use Task.Run
public async Task ProcessImageAsync(byte[] data)
{
    var result = await Task.Run(() => HeavyImageProcessing(data)); // runs on ThreadPool
    return result;
}
💡 In your OTP system — GetAsync for SMS API call = async/await. Never use Task.Run for DB calls in Web API.

DI is providing a class its dependencies from outside instead of creating them inside. .NET Core has built-in DI container.

3 Lifetimes:

  • Transient – New instance every time. Use for lightweight, stateless services.
  • Scoped – One instance per HTTP request. Use for DbContext, services that need request data.
  • Singleton – One instance for app lifetime. Use for caching, configuration.

Q5Explain IEnumerable vs IQueryable. Which did you use with Dapper?▼

  • IQueryable – Builds SQL query, filters happen in database. Used with EF Core.

Q6What is the difference between Value Types and Reference Types? What is boxing/unboxing?▼
  • Value Types: int, double, bool, struct, enum — stored on Stack. Copy on assignment.
  • Reference Types: class, string, array, interface — stored on Heap. Reference copy on assignment.
// Value type - copy
int a = 5;
int b = a;
b = 10; // a is still 5

// Reference type - same object
var s1 = new Student { Name = "Guru" };
var s2 = s1;
s2.Name = "Basha"; // s1.Name also becomes "Basha"

// Boxing - value type → object (heap allocation, slow)
int x = 42;
object obj = x; // boxing

// Unboxing - object → value type
int y = (int)obj; // unboxing
⚠️ Avoid boxing in loops — performance hit. Use generics (List<int>) instead of ArrayList.

Q9What is Middleware in ASP.NET Core? Explain the request pipeline.▼

Middleware = components that handle HTTP request/response in a pipeline. Each middleware can process the request, pass it to next, and process the response.

// Request pipeline order matters!
app.UseExceptionHandler("/error");   // 1. Global error handling
app.UseHttpsRedirection();           // 2. Force HTTPS
app.UseAuthentication();             // 3. Who are you? (JWT decode)
app.UseAuthorization();              // 4. What can you do?
app.UseRateLimiter();               // 5. Throttle
app.MapControllers();                // 6. Route to controller

// Custom middleware
public class RequestLoggingMiddleware
{
    private readonly RequestDelegate _next;

    public RequestLoggingMiddleware(RequestDelegate next) => _next = next;

    public async Task InvokeAsync(HttpContext context)
    {
        Console.WriteLine($"Request: {context.Request.Path}");
        await _next(context); // pass to next middleware
        Console.WriteLine($"Response: {context.Response.StatusCode}");
    }
}

Q17How do microservices communicate? What is the difference between sync and async communication?▼

Synchronous (HTTP/REST)

Service A calls Service B and waits for response. Simple but creates tight coupling and cascading failures.

Asynchronous (Message Queue – RabbitMQ)

Service A publishes a message to queue. Service B picks it up whenever ready. No direct dependency.

// Publisher (StudentService)

});

Q19What is the Circuit Breaker pattern? Why is it important in microservices?▼

When a downstream service fails repeatedly, Circuit Breaker stops calling it for a period (like a electrical breaker). Prevents cascading failures.

// States: Closed (normal) → Open (failing, stop calls) → Half-Open (try one request)

// Using Polly library
var retryPolicy = Policy
    .Handle<HttpRequestException>()
    .WaitAndRetryAsync(3, retryAttempt =>
        TimeSpan.FromSeconds(Math.Pow(2, retryAttempt))); // 2s, 4s, 8s

var circuitBreaker = Policy
    .Handle<HttpRequestException>()
    .CircuitBreakerAsync(
        exceptionsAllowedBeforeBreaking: 5,
        durationOfBreak: TimeSpan.FromSeconds(30)
    );

// Wrap policies
var combined = Policy.WrapAsync(retryPolicy, circuitBreaker);
var result = await combined.ExecuteAsync(() => _httpClient.GetAsync(url));

Q21How do you handle transactions in Dapper?▼
// Payment scenario - debit + create bill must both succeed or both fail
public async Task ProcessPaymentAsync(PaymentDto payment)
{
    using var conn = new SqlConnection(_connString);
    conn.Open();
    using var transaction = conn.BeginTransaction();

    try
    {
        // Step 1: Create payment record
        await conn.ExecuteAsync(
            "INSERT INTO Payments (StudentId, Amount, Date) VALUES (@StudentId, @Amount, @Date)",
            payment, transaction);

        // Step 2: Update bill status
        await conn.ExecuteAsync(
            "UPDATE Bills SET Status = 'Paid' WHERE StudentId = @StudentId",
            new { payment.StudentId }, transaction);

        transaction.Commit(); // both succeed
    }
    catch
    {
        transaction.Rollback(); // either fails → both rollback
        throw;
    }
}
Q22What is N+1 problem in EF Core? How do you fix it?▼

N+1 = 1 query to get N students, then N queries to get each student's class = N+1 total queries. Very bad for performance.

// N+1 PROBLEM - DON'T DO THIS
var students = await _context.Students.ToListAsync(); // 1 query
foreach (var s in students)
{
    var className = s.Class.Name; // EF loads Class for each student = N queries!
}

// FIX 1: Eager Loading with Include
var students = await _context.Students
    .Include(s => s.Class)  // single JOIN query
    .ToListAsync();

// FIX 2: Projection - select only what you need
var students = await _context.Students
    .Select(s => new StudentDto
    {
        Name = s.Name,
        ClassName = s.Class.Name  // EF generates JOIN
    })
    .ToListAsync();
💡 With Dapper, N+1 doesn't happen automatically — you write the JOIN yourself. That's one advantage.

Q25What is the difference between Clustered and Non-Clustered index?▼
  • Clustered Index – Determines physical order of data in table. Only ONE per table. Usually the Primary Key.
  • Non-Clustered Index – Separate structure with pointers to data rows. Can have MANY per table. Used for frequently searched columns.
-- Clustered index (auto-created with PK)
CREATE TABLE Students (
    StudentId INT PRIMARY KEY,  -- clustered index auto-created
    Name NVARCHAR(100),
    ClassId INT
)

-- Non-clustered index for search columns
CREATE NONCLUSTERED INDEX IX_Students_ClassId ON Students(ClassId)
CREATE NONCLUSTERED INDEX IX_Students_Name ON Students(Name)

-- INCLUDE columns to avoid key lookup (covering index)
CREATE NONCLUSTERED INDEX IX_Students_ClassId_Covering
ON Students(ClassId)
INCLUDE (Name, Email, Phone)  -- query can be satisfied from index alone
Q26What are SQL transactions? Explain ACID properties.▼
  • A – Atomicity: All or nothing. Payment debit + bill update = both succeed or both rollback.
  • C – Consistency: DB stays in valid state. Balance can't go negative.
  • I – Isolation: Concurrent transactions don't interfere. Dirty read prevention.
  • D – Durability: Committed data survives crashes. Written to disk.
BEGIN TRANSACTION
    UPDATE Accounts SET Balance = Balance - 5000 WHERE AccountId = 1
    UPDATE Accounts SET Balance = Balance + 5000 WHERE AccountId = 2

    IF @@ERROR <> 0
        ROLLBACK TRANSACTION
    ELSE
        COMMIT TRANSACTION

  • Prevents SQL injection by design

-- Stored procedure from your OTP project
CREATE PROCEDURE sp_ValidateAndExpireOtp
    @Phone NVARCHAR(15),
    @Otp   NVARCHAR(6),
    @Result INT OUTPUT
AS
BEGIN
    SET NOCOUNT ON;

    IF EXISTS(
        SELECT 1 FROM OtpLog
        WHERE Phone = @Phone AND Otp = @Otp
        AND ExpiresAt > GETDATE() AND IsUsed = 0
    )
    BEGIN
        UPDATE OtpLog SET IsUsed = 1 WHERE Phone = @Phone AND Otp = @Otp
        SET @Result = 1 -- success
    END
    ELSE
        SET @Result = 0 -- invalid/expired
END

🔷 JWT Authentication & Security Must Know
Q32How does JWT authentication work? Explain the token structure.▼

JWT (JSON Web Token) = encoded string with 3 parts separated by dots.

// Structure: Header.Payload.Signature
// eyJhbGciOiJIUzI1NiJ9.eyJ1c2VySWQiOiIxMjMifQ.abc123

// Header - algorithm type
{ "alg": "HS256", "typ": "JWT" }

// Payload - claims (user info)
{
  "sub": "1001",          // subject (userId)
  "name": "Gurumoorthy",
  "role": "Admin",
  "exp": 1720000000       // expiry timestamp
}

// Signature - Header + Payload encoded with secret key
HMACSHA256(base64(header) + "." + base64(payload), secretKey)
// ASP.NET Core JWT setup
builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(options =>
    {
        options.TokenValidationParameters = new TokenValidationParameters
        {
            ValidateIssuer = true,
            ValidateAudience = true,

// Base62 encode the DB Id
private static readonly string Chars = "0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ";

public string Encode(long id)
{
    var result = "";
    while (id > 0)
    {
        result = Chars[(int)(id % 62)] + result;
        id /= 62;
    }
    return result.PadLeft(6, '0'); // "abc123"
}

Q39How would you design a notification system that sends SMS, Email, and Push?▼
// Strategy pattern - easily add new channels
public interface INotificationChannel
{
    Task SendAsync(string to, string message, string subject = null);
}

public class SmsChannel : INotificationChannel
{
    public async Task SendAsync(string to, string message, string subject = null)
        => await _smsGateway.SendSmsAsync(to, message);
}

public class EmailChannel : INotificationChannel
{
    public async Task SendAsync(string to, string message, string subject = null)
        => await _emailService.SendEmailAsync(to, subject, message);
}

// Notification service
public class NotificationService
{
    private readonly Dictionary<string, INotificationChannel> _channels;

    public async Task SendAsync(NotificationRequest req)
    {
        var tasks = req.Channels
            .Where(c => _channels.ContainsKey(c))
            .Select(c => _channels[c].SendAsync(req.To, req.Message, req.Subject));

        await Task.WhenAll(tasks); // send to all channels in parallel
    }
}

// Usage
await _notificationService.SendAsync(new NotificationRequest
{
    To = "8122448785",
    Message = "Fee due: Rs.5000 by 30-Jun-2026",
    Channels = new[] { "SMS", "Email" }
});
