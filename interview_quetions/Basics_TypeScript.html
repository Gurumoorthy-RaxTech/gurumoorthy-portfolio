🏗️ Interfaces and Types

// Interface — defines shape of an object
interface Student {
    id: number;
    name: string;
    admNo: string;
    classId: number;
    email?: string;          // optional property
    readonly createdAt: Date; // cannot modify after creation
    address: {
        city: string;
        state: string;
    };
}

// Use the interface
const student: Student = {
    id: 1, name: "Basha", admNo: "ADM001", classId: 5,
    createdAt: new Date(), address: { city: "Chennai", state: "TN" }
};
// student.createdAt = new Date(); ← Error: readonly!

// Interface with methods
interface StudentRepository {
    getById(id: number): Promise<Student | null>;
    getAll(schoolId: number): Promise<Student[]>;
    save(student: Student): Promise<void>;
    delete(id: number): Promise<boolean>;
}

// Extending interfaces
interface HostelStudent extends Student {
    roomNumber: string;
    hostelFee: number;
}

// interface vs type — key differences
// Interface: can be extended with 'extends', can be re-declared (merged)
// Type alias: can use unions, intersections (&), mapped types

// Type intersection — combine types with &
type StudentWithFee = Student & {
    pendingAmount: number;
    lastPaymentDate: Date;
};

// Enums — named constants
enum AttendanceStatus {
    Present = "PRESENT",
    Absent  = "ABSENT",
    Late    = "LATE",
    Holiday = "HOLIDAY"
}
const today: AttendanceStatus = AttendanceStatus.Present;

⚙️ Generics — Reusable Typed Functions

// Generic function — works with any type
function getFirst<T>(arr: T[]): T | undefined {
    return arr[0];
}
getFirst<string>(["Basha", "Priya"]);  // returns string
getFirst<number>([1, 2, 3]);           // returns number

// Generic API response wrapper
interface ApiResponse<T> {
    data: T;
    success: boolean;
    message: string;
    totalCount?: number;
}

// Usage
async function getStudents(): Promise<ApiResponse<Student[]>> {
    const res = await fetch("/api/students");
    return res.json() as ApiResponse<Student[]>;
}
const result = await getStudents();
result.data.forEach(s => console.log(s.name));  // TS knows s is Student!

// Generic with constraint
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
    return obj[key];
}
const name = getProperty(student, "name");    // string
const id   = getProperty(student, "id");      // number
// getProperty(student, "nonExistent"); ← Error! key must exist on Student

// Type Guards — narrow type at runtime
function isStudent(obj: unknown): obj is Student {
    return typeof obj === "object" && obj !== null
        && "admNo" in obj && "name" in obj;
}

const data: unknown = getFromApi();
if (isStudent(data)) {
    console.log(data.admNo);  // TS now knows data is Student
}

🧰 Utility Types — Built-in Type Transformers

interface Student {
    id: number; name: string; admNo: string;
    classId: number; isActive: boolean; fee: number;
}

// Partial<T> — all properties optional (for update payloads)
type UpdateStudentDto = Partial<Student>;
const update: UpdateStudentDto = { name: "New Name" };  // only name, rest optional

// Required<T> — all properties required
type FullStudent = Required<Student>;   // every field must be present

// Readonly<T> — all properties read-only
type ImmutableStudent = Readonly<Student>;
// immutableStudent.name = "x"; ← Error!

// Pick<T, K> — pick specific properties
type StudentCard = Pick<Student, "name" | "admNo" | "classId">;
// Only has name, admNo, classId — for display card

// Omit<T, K> — exclude specific properties
type CreateStudentDto = Omit<Student, "id">;
// Has all Student fields EXCEPT id (DB generates it)

// Record<K, V> — dictionary type
type StudentByAdmNo = Record<string, Student>;
const cache: StudentByAdmNo = {
    "ADM001": bashStudent,
    "ADM002": priyaStudent
};

// ReturnType<T> — extract function return type
async function getStudent(id: number) { return { id, name: "Basha" }; }
type StudentResult = Awaited<ReturnType<typeof getStudent>>;
// StudentResult = { id: number, name: string }

🎯 10 Interview Scenarios

1
What is TypeScript and why use it over JavaScript?Basics

TypeScript = JavaScript + static type system. Compiled to JavaScript. Benefits: 1) Catch errors at compile time (not runtime). 2) IntelliSense autocomplete. 3) Refactoring safety — rename property, compiler finds all usages. 4) Self-documenting code — function signature shows expected types. 5) Easier team collaboration — interfaces define contracts. ZenCampus: student.admNo typo as student.admNO → caught at compile, not in production at 2 AM.

2
What is the difference between interface and type in TypeScript?Types

interface: Object shape definition. Can be extended with 'extends'. Can be re-opened (declaration merging — useful for augmenting library types). type: Alias for any type. Can use unions (A | B), intersections (A & B), conditional types, mapped types. Cannot be re-opened. Both can define object shapes — mostly interchangeable. Prefer interface for object shapes (objects, classes). Use type for unions, primitives, computed types. React components: use interface for Props. API types: use type for union states.

3
What is 'any' vs 'unknown' in TypeScript?Types

any: disables all type checking. You can do ANYTHING with it — no safety. Escape hatch — avoid. unknown: type-safe alternative. You MUST narrow the type before using it (typeof/instanceof check). TypeScript forces you to handle the uncertainty. Rule: never use any. When you receive data from external source (API response, JSON.parse), use unknown and narrow with type guards. ZenCampus: const response: unknown = await fetchApi(); if (isStudentArray(response)) { processStudents(response); }

What is a type guard and how do you write one?Types
// Type predicate: "obj is Student" narrows the type
function isStudent(obj: unknown): obj is Student {
    return typeof obj === "object" && obj !== null
        && "admNo" in obj && "name" in obj;
}

// Built-in narrowing (no custom guard needed)
function processId(id: string | number) {
    if (typeof id === "string") {
        // TypeScript knows: id is string here
        return id.toUpperCase();
    }
    // TypeScript knows: id is number here
    return id * 2;
}

5
Explain Partial and Omit with a ZenCampus example.Utility

Partial: used for PATCH/update endpoints where only some fields change. type UpdateStudentRequest = Partial<Student> → all fields optional. Client sends only changed fields. Omit: used for create DTOs where some fields are server-generated. type CreateStudentRequest = Omit<Student, "id" | "createdAt"> → client doesn't provide id (DB generates) or createdAt (server sets). These prevent type duplication — define Student once, derive all DTOs from it.

6
What is a generic type and when is it useful?Generics

Generic = type parameter that works with multiple types. Without: function getFirstStudent(): Student, getFirstFee(): Fee — duplicate code. With generics: function getFirst<T>(arr: T[]): T | undefined — one function works for any type. Real ZenCampus use: interface ApiResponse<T> { data: T; success: boolean; } → ApiResponse<Student[]> or ApiResponse<Fee> — same wrapper, different data types. Avoids casting with 'as' everywhere.

7
How do you define optional and readonly properties in TypeScript?Interfaces

Optional: append ? to property name. email?: string means email can be undefined. Access safely: student.email?.toLowerCase(). Readonly: prefix with readonly. readonly id: number — cannot assign after creation. Both compile away — JS has no readonly. TypeScript enforces at compile time only. Use readonly for IDs, createdAt, anything set once. Use optional for non-required fields like email, middleName, photo URL.

8
What is the difference between enum and literal union types?Types

Enum: compiled to runtime JS object. Can iterate values. AttendanceStatus.Present. Useful when values needed at runtime (logging, switch statements). type Status = "Active" | "Inactive": pure compile-time, zero runtime cost. Simpler, serializes naturally to string. Prefer literal unions for simple sets. Use enum when: need to iterate all values, computed enum values, or the enum is used in runtime code (not just type annotations). ZenCampus: AttendanceStatus as enum — need to display all options in UI dropdown.

9
How do you type async functions that return Promises?Types
// Explicit return type for async function
async function getStudent(id: number): Promise {
    const res = await fetch(`/api/students/${id}`);
    if (!res.ok) return null;
    return res.json() as Student;
}

// TypeScript can infer return type — explicit is better for documentation
async function getAll(): Promise {
    return (await fetch("/api/students")).json();
}

// void for async functions with no return value
async function deleteStudent(id: number): Promise {
    await fetch(`/api/students/${id}`, { method: "DELETE" });
}
