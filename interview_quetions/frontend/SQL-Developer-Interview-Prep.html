01 · SQL Fundamentals & Querying15 questions
  1. What is the difference between DELETE, TRUNCATE, and DROP?
    Show answer

    DELETE removes rows one at a time, is logged per-row, fires triggers, and can be filtered with WHERE or rolled back. TRUNCATE deallocates all data pages at once (minimally logged), resets identity, skips row-level triggers, and can't use WHERE. DROP removes the table object itself, including its structure, indexes, and constraints.

  2. Explain the logical order of execution of a SELECT statement (FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY).
    Show answer

    SQL Server evaluates clauses logically as: FROM/JOIN → WHERE → GROUP BY → HAVING → SELECT (incl. column aliases) → DISTINCT → ORDER BY → OFFSET/FETCH. This is why you can't reference a SELECT alias in WHERE (not yet resolved) but you can in ORDER BY (resolved last). The written order (SELECT first) is just syntax, not execution order.

  3. What is the difference between WHERE and HAVING?
    Show answer

    WHERE filters individual rows before grouping/aggregation happens; it can't reference aggregate functions. HAVING filters groups after GROUP BY has produced aggregated results, so it can filter on SUM(), COUNT(), etc. Filtering as early as possible with WHERE is usually more efficient since it reduces rows before the (expensive) aggregation step.

  4. What is the difference between UNION and UNION ALL?
    Show answer

    UNION combines result sets and removes duplicate rows, which requires an implicit sort/distinct pass and is more expensive. UNION ALL simply concatenates result sets, keeping duplicates, and is faster since it skips the dedup step. Use UNION ALL whenever you know the sets are already distinct or duplicates are acceptable.

  5. Explain GROUP BY with multiple aggregate functions and how NULLs are grouped.
    Show answer

    You can compute several aggregates (SUM, COUNT, AVG, MAX) in one GROUP BY query as long as every non-aggregated column in SELECT also appears in GROUP BY. For grouping purposes, SQL Server treats all NULL values as equal — they're collapsed into a single group — even though NULL = NULL is normally unknown in a WHERE comparison.

  6. What are window functions? Explain ROW_NUMBER(), RANK(), and DENSE_RANK() with an example.
    Show answer

    Window functions compute a value across a set of rows (a "window", defined by OVER (PARTITION BY ... ORDER BY ...)) without collapsing them into one row like GROUP BY does. ROW_NUMBER() assigns a unique sequential number per partition; RANK() assigns the same rank to ties but skips the next number(s); DENSE_RANK() assigns the same rank to ties without skipping.

    SELECT StudentId, Marks,
       ROW_NUMBER() OVER (PARTITION BY ClassId ORDER BY Marks DESC) AS rn
FROM Results;
  7. Difference between RANK() and DENSE_RANK() when there are ties.
    Show answer

    For scores 90, 90, 80: RANK() gives 1, 1, 3 (it leaves a gap equal to the number of tied rows); DENSE_RANK() gives 1, 1, 2 (no gap). Choose RANK() when the gap should reflect competition-style standings, and DENSE_RANK() when you want compact, gap-free tiers.

  8. How do LAG() and LEAD() work? Give a use case (e.g. month-over-month comparison).
    Show answer

    LAG() reads a value from a previous row and LEAD() reads from a following row, within the same ordered partition, without needing a self-join. A common use is comparing a student's fee payment this month to last month: LAG(Amount) OVER (PARTITION BY StudentId ORDER BY PaidOn) returns the prior payment on the same row so you can subtract directly.

  9. What is a Common Table Expression (CTE)? When would you use a recursive CTE?
    Show answer

    A CTE (WITH name AS (...)) is a named, temporary result set scoped to the statement that follows it — it improves readability over nested subqueries and can be referenced multiple times. A recursive CTE (an anchor query UNION ALL'd with a recursive query referencing itself) is used for hierarchical data like org charts, category trees, or bill-of-materials explosions.

  10. Difference between a temp table (#temp), a table variable (@table), and a CTE.
    Show answer

    A temp table lives in tempdb, supports indexes/statistics, and is best for large intermediate result sets reused across multiple statements. A table variable also lives in tempdb but has limited statistics (historically assumed 1 row by the optimizer) and no user-created nonclustered indexes pre-2014, making it riskier for large data. A CTE has no physical storage — it's inlined into the query and only exists for that one statement.

  11. How do you find and remove duplicate rows in a table that has no primary key?
    Show answer

    Use ROW_NUMBER() partitioned by the columns that define a "duplicate" and delete every row where the row number is greater than 1 (see Coding Problem 02 below for the full pattern). Without a unique key, this window-function approach is the standard way to disambiguate otherwise-identical rows.

  12. Explain PIVOT and UNPIVOT with an example.
    Show answer

    PIVOT rotates unique values from a row-based column into multiple columns, typically alongside an aggregate (e.g. turning a Month column into Jan/Feb/Mar columns of summed amounts — see Coding Problem 11). UNPIVOT does the reverse: it turns multiple columns back into row-value pairs, useful when reshaping a wide report table for storage or further joins.

  13. How does NULL behave in comparisons, aggregates, and ORDER BY?
    Show answer

    NULL represents "unknown," so NULL = NULL and NULL <> x both evaluate to unknown (not true), which is why you must use IS NULL/IS NOT NULL. Aggregate functions like SUM, AVG, COUNT(column) ignore NULLs entirely (though COUNT(*) counts all rows). By default SQL Server sorts NULLs first in ascending order.

  14. What is the difference between CHAR, VARCHAR, NVARCHAR, and TEXT?
    Show answer

    CHAR(n) is fixed-length and space-padded; VARCHAR(n) is variable-length ASCII/single-byte, saving space for variable data; NVARCHAR(n) is variable-length Unicode (2 bytes/char), required for multi-language text such as names in Excel imports. TEXT is a legacy large-object type deprecated in favor of VARCHAR(MAX)/NVARCHAR(MAX), which support normal string functions and better performance.

  15. How would you write a query to find the 2nd or Nth highest salary from an Employee table?
    Show answer

    Use DENSE_RANK() over Salary DESC and filter WHERE rnk = N — this correctly handles duplicate salaries, unlike a plain TOP N approach. See Coding Problem 01 below for the full query, plus a TOP/MIN alternative for versions or contexts where window functions aren't convenient.

02 · Joins & Subqueries10 questions
  1. Explain INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN, and CROSS JOIN with examples.
    Show answer

    INNER JOIN returns only rows matching in both tables. LEFT JOIN returns all rows from the left table plus matches from the right (unmatched right columns are NULL) — e.g. all students plus their payments, including students with none. RIGHT JOIN is the mirror image. FULL OUTER JOIN returns all rows from both sides, matched where possible. CROSS JOIN returns the full Cartesian product with no join condition.

  2. What is a self-join? Give a real scenario (e.g. employee-manager hierarchy).
    Show answer

    A self-join joins a table to itself using two aliases, typically to relate rows within the same table. Classic example: Employees e JOIN Employees m ON e.ManagerId = m.EmployeeId to pair each employee with their manager's name from the same table.

  3. Difference between a correlated subquery and a non-correlated subquery.
    Show answer

    A non-correlated subquery runs independently once and its result is reused for the outer query (e.g. WHERE Salary > (SELECT AVG(Salary) FROM Employees)). A correlated subquery references a column from the outer query, so conceptually it re-executes for every outer row (e.g. finding each employee's department average) — this can be slower unless the optimizer rewrites it as a join.

  4. Difference between EXISTS and IN — which performs better and why?
    Show answer

    EXISTS stops scanning as soon as it finds one matching row (semi-join, short-circuits), and handles NULLs in the subquery correctly. IN materializes the full subquery result list before comparing, and famously misbehaves if that list contains a NULL combined with NOT IN (can silently return zero rows). For large subquery result sets, EXISTS is generally safer and often faster.

  5. What happens when you join two tables where one side has NULL foreign keys?
    Show answer

    An INNER JOIN silently drops those rows because NULL = anything is never true. If you need to keep rows with a NULL foreign key (e.g. students not yet assigned a class), you must use a LEFT JOIN instead — this is a common source of "missing rows" bugs after a bulk import leaves some foreign keys unpopulated.

  6. How do you find rows in Table A that don't exist in Table B (anti-join pattern)?
    Show answer
    SELECT a.*
FROM TableA a
LEFT JOIN TableB b ON a.Id = b.AId
WHERE b.AId IS NULL;
-- or: WHERE NOT EXISTS (SELECT 1 FROM TableB b WHERE b.AId = a.Id)

    Both forms are common in import reconciliation, e.g. finding staged rows that never made it into the production table.

  7. Explain the difference between filtering in the ON clause vs the WHERE clause for an outer join.
    Show answer

    For a LEFT JOIN, a condition in ON is applied while matching rows (unmatched left rows still appear, with NULLs on the right), while the same condition in WHERE is applied after the join completes and can eliminate the unmatched (NULL) rows entirely — silently turning your outer join back into something resembling an inner join. This is a frequent bug when adding a filter on a right-side column.

  8. What is a Cartesian product and how does it accidentally happen in production queries?
    Show answer

    A Cartesian product pairs every row of one table with every row of another, producing rows = (rows in A) × (rows in B). It happens accidentally when a join is missing its ON condition, or when the join condition doesn't uniquely relate rows (e.g. joining on a non-unique column), which can silently multiply row counts and inflate SUM() results.

  9. How do you join more than 2 tables efficiently, and how does join order affect the query plan?
    Show answer

    The SQL Server optimizer generally re-orders joins itself based on cardinality estimates and available indexes, so the order you write joins in usually doesn't dictate execution order. What matters more is that each join has a selective, index-backed predicate — filtering down to the smallest row set as early as possible — and that statistics are up to date so the optimizer's estimates are accurate.

  10. What's the difference between APPLY (CROSS APPLY/OUTER APPLY) and a regular join?
    Show answer

    APPLY lets the right-hand side be a table-valued expression that references columns from the left-hand side — something a normal JOIN's ON clause can't do. It's commonly used to pull "top N related rows per row" (e.g. each student's most recent payment) via CROSS APPLY (SELECT TOP 1 ... ORDER BY ...). OUTER APPLY is the LEFT JOIN equivalent, keeping left rows with no match.

03 · Stored Procedures, Views, Functions & Triggers10 questions
  1. What is the advantage of a stored procedure over an ad-hoc SQL query?
    Show answer

    Stored procedures get a cached, reusable execution plan (avoiding recompilation), reduce network round-trips by bundling logic server-side, centralize business rules so multiple apps stay consistent, and provide a security boundary — you can grant EXECUTE rights without granting direct table access, which also helps prevent SQL injection since parameters are strongly typed.

  2. Difference between a scalar function, an inline table-valued function (iTVF), and a multi-statement table-valued function (MTVF) — and their performance implications.
    Show answer

    A scalar function returns one value and, pre-SQL Server 2019, is notoriously slow because it's invoked row-by-row and blocks parallelism. An iTVF is essentially a parameterized view — a single SELECT the optimizer can inline into the calling query, so it performs like a view/join. An MTVF returns a table built with multiple statements into an internal table variable, which the optimizer treats as a black box (fixed row estimate), often hurting plan quality. Prefer iTVFs where possible.

  3. What is a view? Difference between a regular view and an indexed (materialized) view.
    Show answer

    A view is a stored, named query that's re-executed against live data every time it's referenced — it doesn't store data itself. An indexed (materialized) view actually persists its result set physically via a unique clustered index, so reads are fast, but every underlying data change must maintain that index too, adding write overhead — a trade-off worth making for expensive aggregates queried often but written rarely.

  4. Can you update data through a view? What are the restrictions?
    Show answer

    Yes, if the view is based on a single table (or updates target only one underlying table), doesn't use aggregates, DISTINCT, GROUP BY, or set operators, and includes any NOT NULL columns without defaults. Views joining multiple tables generally aren't directly updatable unless you add an INSTEAD OF trigger to control how the update is distributed across base tables.

  5. What is a trigger? Difference between AFTER/FOR and INSTEAD OF triggers.
    Show answer

    A trigger is code that fires automatically in response to INSERT/UPDATE/DELETE on a table or view. An AFTER (a.k.a. FOR) trigger runs after the data modification has completed and can roll it back; it's used for auditing, cascading logic, or validation. An INSTEAD OF trigger runs in place of the actual modification, letting you redirect the operation entirely — commonly used to make a multi-table view updatable.

  6. What are the risks of using triggers for business logic during bulk imports?
    Show answer

    Triggers fire per statement (not per batch call), but if written naively assuming single-row inserts, they can silently process only inserted.* incorrectly for multi-row bulk operations. They also add hidden overhead to every bulk insert, can cause deadlocks/long transactions on large batches, and make troubleshooting harder since the logic isn't visible in the calling code. For heavy imports it's often safer to disable non-essential triggers and apply that logic explicitly in the import script.

  7. How do you handle errors inside a stored procedure? Explain TRY...CATCH and ERROR_MESSAGE()/ERROR_LINE().
    Show answer
    BEGIN TRY
  BEGIN TRANSACTION;
  -- import statements here
  COMMIT;
END TRY
BEGIN CATCH
  IF @@TRANCOUNT > 0 ROLLBACK;
  INSERT INTO ErrorLog (Message, Line, Proc)
  VALUES (ERROR_MESSAGE(), ERROR_LINE(), ERROR_PROCEDURE());
  THROW;
END CATCH;

    TRY...CATCH traps runtime errors so they can be logged and handled gracefully instead of aborting the batch outright; ERROR_MESSAGE()/ERROR_LINE()/ERROR_PROCEDURE() capture diagnostic detail, and THROW (or RAISERROR) re-raises the error to the caller after logging.

  8. What are table-valued parameters (TVPs) and how are they used to pass bulk data from C# to a stored procedure?
    Show answer

    A TVP is a user-defined table type that lets you pass an entire set of rows into a stored procedure as a single strongly-typed parameter, avoiding row-by-row round trips or string concatenation. In C#, you build a DataTable matching the type's shape and pass it via a SqlParameter with SqlDbType.Structured — ideal for bulk-importing a validated batch of records in one call.

  9. What is SET NOCOUNT ON and why is it recommended in stored procedures?
    Show answer

    SET NOCOUNT ON suppresses the "N rows affected" message SQL Server sends after each statement. In procedures with many statements (common in import logic), this reduces network chatter and can noticeably improve performance, and it prevents client libraries like ADO.NET from misinterpreting extra result sets.

  10. How do you version-control and deploy stored procedure changes safely in a live environment?
    Show answer

    Keep every procedure's CREATE OR ALTER script in source control (Git) alongside application code, run changes through code review, and deploy via a migration tool or CI/CD pipeline rather than manually in SSMS. Test against a staging database first, wrap the deploy in a transaction where possible, and keep the previous version's script handy for a quick rollback.

04 · Indexing & Performance Tuning10 questions
  1. Difference between a clustered index and a non-clustered index. Can a table have more than one clustered index?
    Show answer

    A clustered index physically orders the table's data rows on disk by the key — only one is allowed per table since data can only be sorted one way. A non-clustered index is a separate structure with a copy of key columns plus a pointer (the clustered key, or RID if there's no clustered index) back to the data row; a table can have many.

  2. What is a covering index and how does it avoid a key lookup?
    Show answer

    A covering index contains every column a query needs (in its key or as INCLUDEd columns), so SQL Server can satisfy the query entirely from the index without a "key lookup" back to the clustered data row. This eliminates a costly per-row lookup operation and is one of the highest-leverage tuning techniques for frequently-run reporting queries.

  3. What is index fragmentation and how do you fix it (REORGANIZE vs REBUILD)?
    Show answer

    Fragmentation happens as inserts/updates/deletes cause index pages to split and fall out of logical order, hurting range-scan performance. ALTER INDEX ... REORGANIZE is an online, lightweight defrag of leaf-level pages (good for moderate fragmentation, ~10–30%); ALTER INDEX ... REBUILD fully recreates the index (better for heavy fragmentation, >30%) and can run online or offline depending on edition.

  4. How do you read an execution plan? What operators indicate a performance problem (e.g. Table Scan, Key Lookup, Sort)?
    Show answer

    Read the plan right-to-left, top-to-bottom, and compare estimated vs actual row counts — a big gap signals stale statistics. Watch for Table Scan/Clustered Index Scan on large tables (missing or unused index), Key Lookup (missing covering index), Sort/Hash Match operators with high cost (missing supporting index for ORDER BY/joins), and thick arrows (large row counts flowing between operators).

  5. What is a composite index and how does column order matter?
    Show answer

    A composite (multi-column) index is only efficiently seekable from its leading column(s) inward — like a phone book sorted by last name then first name, useless for searching by first name alone. Put the column with the highest selectivity and most common equality-filter usage first, and order remaining columns to match your most common query patterns and ORDER BY clauses.

  6. What is SARGability — give an example of a non-SARGable predicate and how to fix it.
    Show answer

    SARGable (Search ARGument-able) predicates let the optimizer use an index seek rather than a scan. Wrapping a column in a function breaks that: WHERE YEAR(PaidOn) = 2026 forces a scan since every row must be evaluated. Rewrite it as a range: WHERE PaidOn >= '2026-01-01' AND PaidOn < '2027-01-01', which can seek an index on PaidOn directly.

  7. Difference between statistics and indexes — how does the query optimizer use them?
    Show answer

    An index is a physical structure that speeds up data access; statistics are metadata describing the distribution of values in a column (histograms), which the optimizer uses purely to estimate row counts and pick a plan — they don't store the data itself. Stale statistics (common after a huge bulk import) lead to bad row estimates and poor plan choices even if the right indexes exist, so UPDATE STATISTICS after large loads matters.

  8. How would you diagnose and fix a slow bulk-import stored procedure that runs fine on 10K rows but times out on 5 million?
    Show answer

    Check the actual execution plan at scale for scans instead of seeks, missing indexes on staging/lookup tables, and row-by-row logic (cursors/loops) that doesn't scale — convert those to set-based operations. Also check for parameter sniffing on a plan cached from a small run, excessive logging/triggers firing per row, lock escalation causing blocking, and whether statistics are stale after the data grew. Batch very large inserts (e.g. 10K-row chunks) to avoid one giant transaction log/lock footprint.

  9. What are included columns in a non-clustered index and when should you use them?
    Show answer

    INCLUDEd columns are stored at the leaf level of a non-clustered index without being part of the key (so they don't affect sort order or key size limits). Use them to make an index covering — add columns the query only SELECTs (not filters or sorts on) so the engine can satisfy the whole query from the index alone.

  10. Explain parameter sniffing and how it can cause inconsistent query performance.
    Show answer

    SQL Server caches a query plan based on the parameter values used the first time it compiles. If that first call had an unusual value (e.g. a class with 2 students vs. one with 20,000), the cached plan may be badly suited to typical calls — fast for one caller, slow for another using the same procedure. Fixes include OPTION (RECOMPILE), OPTIMIZE FOR hints, or splitting into separate procedures for very different data profiles.

05 · Transactions, Locking & Concurrency8 questions
  1. Explain the ACID properties with a real example from a payment/fee transaction.
    Show answer

    Atomicity — a fee payment that debits a student's balance and inserts a receipt either both happen or neither does. Consistency — the transaction can't leave the balance in an invalid state (e.g. violating a check constraint). Isolation — two clerks processing payments for the same student concurrently don't see each other's uncommitted changes. Durability — once committed, the payment survives a server crash immediately after.

  2. What is a deadlock? How do you detect and resolve one in SQL Server?
    Show answer

    A deadlock occurs when two transactions each hold a lock the other needs, so neither can proceed. SQL Server's deadlock monitor detects the cycle and automatically kills the "victim" (usually the cheaper-to-rollback transaction), returning error 1205. You diagnose it via the deadlock graph (SQL Server Profiler/Extended Events/the system_health session) and typically fix it by accessing tables in a consistent order across transactions, keeping transactions short, and adding appropriate indexes to reduce lock scope.

  3. Explain SQL Server transaction isolation levels (Read Uncommitted, Read Committed, Repeatable Read, Serializable, Snapshot).
    Show answer

    Read Uncommitted allows dirty reads (no locking). Read Committed (the default) only reads committed data but values can change between reads in the same transaction. Repeatable Read locks read rows so they can't change until the transaction ends, preventing non-repeatable reads but not phantoms. Serializable is the strictest, range-locking to block phantom rows too. Snapshot uses row versioning to give each transaction a consistent point-in-time view without blocking readers/writers against each other.

  4. What is the difference between optimistic and pessimistic locking, and which fits a bulk-import scenario?
    Show answer

    Pessimistic locking acquires a lock upfront and holds it until the transaction completes, blocking other writers — safe but reduces concurrency. Optimistic locking assumes conflicts are rare: it reads without locking and checks a version/timestamp column at write time to detect if the row changed, retrying on conflict. For bulk imports, which are typically the sole writer to staging tables and don't compete with live users, minimal locking (often via batching plus ROWLOCK hints) is usually preferable to broad pessimistic locks that would stall the application.

  5. What is a dirty read, a non-repeatable read, and a phantom read?
    Show answer

    A dirty read sees another transaction's uncommitted change, which might later roll back. A non-repeatable read happens when a transaction reads the same row twice and gets different values because another transaction committed a change in between. A phantom read happens when a transaction re-runs a range query and sees new rows that another transaction inserted in between — the row set itself changed, not just a value.

  6. How do you safely perform an UPSERT (insert-or-update) under concurrent load — is MERGE safe by itself?
    Show answer

    MERGE is not inherently safe under concurrency — a well-documented race condition lets two simultaneous MERGE statements both see "not matched" and both attempt an insert, causing a primary-key violation. The reliable fix is wrapping it in an explicit transaction with a HOLDLOCK (or SERIALIZABLE) hint on the target, or using a unique index plus a TRY...CATCH retry pattern around a simple INSERT/UPDATE as an alternative.

  7. What locking hints (NOLOCK, ROWLOCK, UPDLOCK) do and when NOLOCK is dangerous.
    Show answer

    NOLOCK (equivalent to Read Uncommitted) reads without taking or respecting locks — it's fast but can return dirty, duplicate, or missing rows, and can even error out on an index being rebuilt mid-scan; avoid it for anything financial. ROWLOCK forces row-level (rather than page/table) locking to reduce blocking scope. UPDLOCK takes an update lock on read, preventing another transaction from also acquiring an update lock on the same row — useful to prevent two processes reading-then-updating the same row simultaneously.

  8. How would you design a large batch import so it doesn't block live application traffic?
    Show answer

    Load into a staging table first (no contention with live tables), then move data into production in smaller batches (e.g. 1,000–10,000 rows per transaction) rather than one giant transaction that holds locks for a long duration. Use ROWLOCK hints, schedule heavy imports during low-traffic windows if possible, and consider Snapshot Isolation so readers aren't blocked by the import's writes at all.

06 · Data Import, Migration & ETL12 questions
  1. What approaches do you know for importing large CSV/Excel files into SQL Server (BULK INSERT, BCP, OPENROWSET, SSIS, C# SqlBulkCopy)? Compare them.
    Show answer

    BULK INSERT/BCP are the fastest for plain server-side file loads but offer little validation and need file access on the SQL Server box. OPENROWSET(BULK...) is similar, usable inline in a query. SSIS gives a visual, reusable ETL pipeline with built-in transformation/error-routing but has more setup overhead. SqlBulkCopy in .NET is the best fit when the file arrives via a web upload — it streams rows from the app tier without needing server file-system access, and lets you validate/transform in C# first.

  2. How does SqlBulkCopy in .NET work, and how would you handle partial failures mid-batch?
    Show answer

    SqlBulkCopy streams rows from a DataTable/IDataReader directly into SQL Server using the bulk copy protocol, bypassing per-row INSERT overhead. Set BatchSize to commit in chunks rather than one massive transaction; on failure mid-batch, only the current batch rolls back (earlier committed batches stay), so pair it with a staging table and a resumable "last successful batch" marker so a retry doesn't reinsert already-loaded rows.

  3. How do you validate incoming data (data types, required fields, referential integrity) before loading it into production tables?
    Show answer

    Load raw values as loosely-typed text into a staging table first, then run validation queries using TRY_CAST/TRY_CONVERT for type checks, NOT EXISTS/LEFT JOIN checks against reference tables for FK validity, and simple predicates for required-field/format checks (e.g. email pattern). Route failing rows to an error table with a reason, and only INSERT the rows that pass every check into production.

  4. Describe a staging-table pattern for imports — why load into staging first instead of directly into production tables?
    Show answer

    A staging table has loose types (mostly NVARCHAR) and no constraints, so a raw bulk load never fails on a bad row. From there you validate, clean, and transform in a controlled second step, and only well-formed rows ever touch production — keeping production tables' constraints, triggers, and indexes intact and avoiding partially-corrupted data reaching live reports or application queries.

  5. How do you parse and query JSON data in SQL Server (OPENJSON, JSON_VALUE, JSON_QUERY, FOR JSON)?
    Show answer

    OPENJSON shreds a JSON array/object into a rowset (see Coding Problem 08). JSON_VALUE extracts a single scalar from a JSON column/variable (e.g. JSON_VALUE(Payload,'$.Email')); JSON_QUERY extracts a JSON object/array fragment instead of a scalar. Going the other way, FOR JSON PATH/FOR JSON AUTO serializes a query's result set back into JSON — handy for API responses built straight from SQL.

  6. How do you parse and query XML data in SQL Server (OPENXML, nodes(), value(), FOR XML)?
    Show answer

    The native XML data type's .nodes() method shreds elements into a rowset, and .value() extracts a typed scalar from a node (see Coding Problem 09) — this is the modern, preferred approach. OPENXML is the older, more verbose method requiring an explicit document handle via sp_xml_preparedocument. FOR XML PATH/AUTO/RAW serialize query results into XML for the reverse direction, e.g. generating an XML export file.

  7. How do you identify and handle duplicate records across multiple import files/batches?
    Show answer

    Define a natural business key for "duplicate" (e.g. email, or student admission number + name), and before inserting from staging, check NOT EXISTS against both the production table and rows already processed in the current staging batch (using ROW_NUMBER() to keep only the first occurrence within the file itself). Log skipped duplicates with their source file/row so the business can confirm they weren't legitimate re-registrations.

  8. What's your strategy for re-running a failed import safely (idempotency) without creating duplicate or corrupted data?
    Show answer

    Make the load step check-before-insert (NOT EXISTS by natural key, or a MERGE/upsert) rather than blind INSERT, tag each staged row with a batch/file identifier so a rerun can target only unprocessed rows, and wrap the production move in a transaction so a mid-run failure doesn't leave partial data. See Coding Problem 15 for the concrete pattern.

  9. How do you log and report import failures — row-level error capture vs. batch-level rollback?
    Show answer

    For business-rule failures (bad email, missing field), capture at row level into an error table with the source row and reason, and continue processing the rest of the batch — this gives the business an actionable list to fix and re-upload. For system-level failures (connection loss, constraint violation mid-transaction), roll back the whole batch since partial application-level state could be inconsistent, then surface a single batch-level error/notification.

  10. How do you handle encoding issues, date-format mismatches, and locale differences in imported files?
    Show answer

    Explicitly specify the file's encoding (e.g. UTF-8 with BOM handling) when reading it rather than trusting a default, since mismatches corrupt special characters silently. For dates, never trust implicit conversion — parse using TRY_CONVERT with an explicit style code or a known format string, and standardize on an unambiguous format (ISO yyyy-MM-dd) as early as possible in the pipeline to avoid day/month swaps between locales (e.g. 03/04 as Mar 4 vs Apr 3).

  11. What is the difference between a full load and an incremental/delta load in data migration?
    Show answer

    A full load wipes/reloads the entire target dataset every run — simple, but expensive and disruptive for large tables. An incremental (delta) load only pulls rows that are new or changed since the last run, typically tracked via a ModifiedDate/timestamp column, a change-tracking feature, or a high-watermark value — far cheaper for ongoing syncs, but requires reliable change detection on the source.

  12. How would you migrate data between two databases with different schemas while preserving referential integrity?
    Show answer

    Map old-to-new tables/columns explicitly (a mapping document helps), migrate parent (referenced) tables before child tables so foreign keys always resolve, and build an ID-mapping table (old key → new key) when surrogate keys change, so child-table inserts can translate references correctly. Disable/defer FK constraints during bulk load if needed, then re-enable and validate with a check for orphaned rows before going live.

07 · Database Design & Normalization8 questions
  1. Explain 1NF, 2NF, and 3NF with a practical example.
    Show answer

    1NF: every column holds a single atomic value — no comma-separated subjects in one cell. 2NF: every non-key column depends on the whole primary key, not part of it — in an Enrollment table keyed by (StudentId, CourseId), a StudentName column would violate 2NF since it only depends on StudentId. 3NF: no non-key column depends on another non-key column — e.g. storing both ClassId and ClassTeacherName in Students violates 3NF since ClassTeacherName depends on ClassId, not StudentId; it belongs in a Classes table.

  2. When would you deliberately denormalize a schema, and what's the trade-off?
    Show answer

    Denormalize (duplicate data, pre-aggregate, or flatten joins) when read performance on a hot reporting path matters more than write simplicity — e.g. storing a student's current class name directly on a dashboard table to avoid a join on every page load. The trade-off is update complexity and risk of data drift: every place the duplicated value lives must be kept in sync, usually via triggers or application logic.

  3. Difference between a primary key, a unique key, and a surrogate key.
    Show answer

    A primary key uniquely identifies each row and disallows NULLs; a table can have only one. A unique key/constraint also enforces uniqueness but allows one NULL (or more, depending on config) and a table can have several. A surrogate key is a system-generated identifier (identity column, GUID) with no business meaning, used instead of a natural key (like email) that might change over time.

  4. What is referential integrity and how do foreign keys with ON DELETE CASCADE affect data imports?
    Show answer

    Referential integrity ensures a foreign key value always matches an existing row in the referenced table — no orphaned child records. ON DELETE CASCADE automatically deletes dependent child rows when the parent is deleted, which is convenient but dangerous during imports: a staging cleanup or a mistaken re-import that deletes and reinserts a "parent" row can unintentionally cascade-delete large amounts of unrelated child data.

  5. How do you design a schema to support soft deletes and audit history?
    Show answer

    Add an IsDeleted bit (or DeletedAt nullable datetime) column instead of physically deleting rows, and filter it out of normal queries (often via a view). For audit history, add CreatedAt/CreatedBy/ModifiedAt/ModifiedBy columns, or maintain a separate history/audit table populated by triggers or application code capturing before/after values on every change.

  6. What data type would you choose for money values, and why not FLOAT?
    Show answer

    Use DECIMAL(p,s) (or the MONEY type, though DECIMAL is generally preferred for its explicit precision control). FLOAT is a binary floating-point approximation, so it can introduce tiny rounding errors when representing decimal fractions exactly — unacceptable for financial totals where cents must add up precisely.

  7. How would you model a many-to-many relationship (e.g. students to courses)?
    Show answer

    Create a junction (bridge) table — e.g. Enrollments(StudentId, CourseId, ...) — with foreign keys to both sides and a composite primary key (or its own surrogate key plus a unique constraint on the pair). This lets each student enroll in many courses and each course have many students, and lets you attach relationship-specific attributes like enrollment date or grade.

  8. What is a check constraint, and how does it differ from validating data in application code?
    Show answer

    A CHECK constraint enforces a boolean rule at the database level (e.g. CHECK (Amount > 0)), so it's guaranteed no matter which application, script, or direct query writes to the table. Application-level validation can be bypassed by any other client hitting the database directly (including bulk import tools), so check constraints act as a last line of defense — the two are complementary, not substitutes for each other.

08 · ASP.NET Core / C# / Web API10 questions
  1. How do you call a stored procedure from C# using ADO.NET or Dapper, and how do you pass a table-valued parameter?
    Show answer

    With ADO.NET, create a SqlCommand with CommandType.StoredProcedure, add typed SqlParameters, and call ExecuteReader/ExecuteNonQuery. Dapper simplifies this to one line: conn.Query<T>("dbo.ImportStudents", param, commandType: CommandType.StoredProcedure). For a TVP, build a DataTable matching the SQL user-defined table type and pass it as a parameter with SqlDbType.Structured, giving the stored procedure the whole batch in one round trip.

  2. Difference between Entity Framework Core and Dapper — when would you choose one over the other for a data-heavy import module?
    Show answer

    EF Core is a full ORM with change tracking, migrations, and LINQ-to-SQL translation — great developer productivity for typical CRUD, but its per-entity tracking and generated SQL add overhead unsuited to bulk operations. Dapper is a lightweight micro-ORM that maps raw SQL/stored-procedure results to objects with minimal overhead, giving you full control over the exact query — the better fit for a high-throughput import module where you're hand-tuning performance-critical SQL anyway.

  3. How do you implement dependency injection in ASP.NET Core, and why does it matter for testability?
    Show answer

    Register services against interfaces in Program.cs (e.g. builder.Services.AddScoped<IImportService, ImportService>()), and the framework's built-in container injects them into controller/service constructors automatically. This matters for testability because a unit test can supply a mock/fake implementation of IImportService instead of a real database-backed one, letting you test business logic in isolation without a live SQL Server connection.

  4. How do you build a Web API endpoint that accepts a file upload (CSV/Excel) and processes it asynchronously?
    Show answer

    Accept the file via [FromForm] IFormFile file on a multipart/form-data POST endpoint, save/stream it, then hand off actual processing to a background job (e.g. a hosted service, a queue like Azure Storage Queue/RabbitMQ, or IBackgroundTaskQueue) so the HTTP request returns immediately with a job/tracking ID instead of blocking the client while millions of rows import. The client then polls a status endpoint or receives a notification when processing completes.

  5. Explain the difference between synchronous and asynchronous methods (async/await) and why it matters for a bulk-import API.
    Show answer

    A synchronous call blocks the calling thread until it completes; an async method frees the thread to handle other requests while waiting on I/O (like a database call), and resumes via await when the result is ready. For a bulk-import API handling many concurrent uploads, async I/O lets the server scale to far more simultaneous requests with the same thread pool instead of each blocked import call tying up a thread doing nothing but waiting.

  6. How do you handle and return structured error responses from a Web API (status codes, ProblemDetails)?
    Show answer

    Use meaningful HTTP status codes (400 for validation errors, 404 for not found, 409 for conflicts like duplicate import, 500 for unexpected server errors), and return a consistent body shape — ASP.NET Core's built-in ProblemDetails (RFC 7807) standardizes this with fields like title, status, and detail, so every error the API returns, including for import failures, has a predictable, machine-parseable structure the frontend can render consistently.

  7. What is middleware in ASP.NET Core, and how would you use it for logging or global exception handling?
    Show answer

    Middleware components form a pipeline that every HTTP request passes through in order (e.g. authentication → logging → routing → your endpoint), each able to inspect/modify the request or short-circuit the pipeline. A custom exception-handling middleware wraps await _next(context) in a try/catch, logs unhandled exceptions, and returns a clean ProblemDetails response instead of leaking a stack trace — a single place to catch everything rather than repeating try/catch in every controller.

  8. How do you implement pagination, filtering, and sorting on a large dataset exposed via API?
    Show answer

    Accept query parameters (page, pageSize, sortBy, filter fields) and push them down into the SQL query using OFFSET ... FETCH NEXT (or keyset/seek pagination for very large tables, which scales better than OFFSET on deep pages) so the database returns only the needed slice — never load the full dataset into memory and paginate in C#. Return total count and page metadata alongside the data so the client can render pagination controls.

  9. Difference between IEnumerable, IQueryable, and List<T> — and why the choice matters for query performance.
    Show answer

    IQueryable<T> builds an expression tree that's translated into SQL and only executes when enumerated — chaining .Where()/.OrderBy() on it pushes those operations to the database. IEnumerable<T> operates in-memory over already-materialized objects, so chaining LINQ on it after a premature .ToList() means filtering happens in application memory, not the database — a common accidental performance bug pulling far more rows than needed. List<T> is a concrete, fully-materialized in-memory collection.

  10. How do you secure a Web API (JWT authentication, API keys, role-based authorization)?
    Show answer

    JWT bearer authentication validates a signed token on each request (via AddAuthentication().AddJwtBearer(...) middleware) without server-side session state, embedding claims like user ID and roles. Role/policy-based [Authorize(Roles = "Admin")] attributes then restrict specific endpoints (e.g. only an admin role can trigger a bulk import). API keys are a simpler alternative for service-to-service calls, validated via a custom middleware or handler checking a header against known keys.

09 · Angular & React8 questions
  1. Explain one-way vs two-way data binding — how does Angular's [(ngModel)] compare to React's controlled components?
    Show answer

    One-way binding flows data in a single direction (model → view, or view → model), requiring an explicit event handler to sync the other way. Angular's [(ngModel)] "banana in a box" syntax gives built-in two-way binding — the input updates the model and the model updates the input automatically. React has no built-in two-way binding; a "controlled component" simulates it by setting the input's value from state and updating that state explicitly in an onChange handler.

  2. What are components, and how do Angular components differ structurally from React functional components?
    Show answer

    A component is a self-contained, reusable unit of UI with its own template and logic. An Angular component is a TypeScript class decorated with @Component, pairing a separate HTML template (and often a separate CSS file) with class properties/methods, plus built-in dependency injection. A React functional component is just a JavaScript/TypeScript function that returns JSX, using hooks for state and side effects instead of a class — generally more lightweight with less required boilerplate.

  3. How do you call a REST API from Angular (HttpClient) or React (fetch/axios) and handle loading/error states?
    Show answer

    In Angular, inject HttpClient and subscribe to the Observable it returns (e.g. this.http.get<Student[]>(url).subscribe(...)), tracking a loading flag and catching errors with catchError/in the subscribe error callback. In React, call fetch or axios inside a useEffect, storing the response, a loading boolean, and an error in useState, then conditionally render a spinner, the data, or an error message based on those states.

  4. What are React hooks (useState, useEffect) used for, and what problem do they solve?
    Show answer

    useState lets a function component hold local, re-render-triggering state without needing a class. useEffect runs side effects (data fetching, subscriptions, timers) after render, with a dependency array controlling when it re-runs. Hooks solved the problem of sharing stateful logic between components without the "wrapper hell" of higher-order components/render props that class-based React required, and let related logic live together instead of being split across lifecycle methods.

  5. What is Angular's dependency injection system, and how does a service get shared across components?
    Show answer

    Angular has a hierarchical DI container: services decorated with @Injectable({providedIn: 'root'}) become app-wide singletons automatically injected into any component/service constructor that declares them as a parameter. Because it's the same instance everywhere it's injected at that scope, a service is a natural place to hold shared state (like the current logged-in user or a cached import-progress status) accessible from multiple components.

  6. How would you build a UI to upload a large file and show progress while it's being processed on the backend?
    Show answer

    Use XMLHttpRequest or axios's onUploadProgress (or Angular's HttpClient with reportProgress: true) to show upload-transfer progress via a progress bar. Since server-side processing happens after the upload completes, have the API return a job ID immediately and poll a status endpoint (or use SignalR/WebSockets for push updates) to show processing progress separately from the upload itself.

  7. How do you display and paginate a large grid of data (e.g. thousands of imported records) without freezing the browser?
    Show answer

    Never render all rows at once — request paginated data from the API (server-side paging, not loading everything into the browser) and/or use a virtualized grid component (e.g. Angular CDK's virtual scroll, React's react-window/react-virtualized) that only renders the DOM nodes currently visible in the viewport, recycling them as the user scrolls, which keeps DOM size and render cost constant regardless of total row count.

  8. What is the virtual DOM (React) and change detection (Angular) — how do they affect rendering performance?
    Show answer

    React keeps an in-memory virtual DOM tree, diffs it against the previous version on state change, and applies only the minimal set of real DOM updates needed — avoiding expensive full re-renders. Angular instead walks its component tree on each change-detection cycle (triggered by events/async operations via Zone.js) comparing bound values to previous ones; large component trees can make this costly, which is why OnPush change detection (checking a component only when its inputs change by reference) is used to optimize hot paths.

10 · HTML, CSS & JavaScript5 questions
  1. Difference between <div> and semantic HTML5 tags — why does it matter for accessibility?
    Show answer

    A <div> carries no meaning — it's a generic box. Semantic tags like <nav>, <header>, <main>, <table>, and <button> convey the purpose of the content to the browser, search engines, and screen readers, which lets assistive technology announce landmarks and let users jump directly to a page's navigation or main content instead of reading through unlabeled generic containers.

  2. Explain CSS specificity and the box model.
    Show answer

    Specificity determines which CSS rule wins when multiple rules target the same element, roughly ranked: inline styles > IDs > classes/attributes/pseudo-classes > element/type selectors, with !important overriding all of it. The box model describes how an element's rendered size is built up from content, then padding, then border, then margin — box-sizing: border-box makes width/height include padding and border rather than adding them on top, which avoids a lot of layout surprises.

  3. Difference between var, let, and const in JavaScript.
    Show answer

    var is function-scoped and hoisted (usable before its declaration line, as undefined), which can cause subtle bugs in loops/closures. let is block-scoped and not accessible before declaration (temporal dead zone), and its value can be reassigned. const is also block-scoped but can't be reassigned after initialization (though object/array contents it points to can still be mutated). Modern code should default to const, using let only when reassignment is needed, and avoid var.

  4. What is a Promise, and how does it relate to async/await?
    Show answer

    A Promise represents the eventual result (or failure) of an asynchronous operation, in one of three states: pending, fulfilled, or rejected, and is consumed via .then()/.catch(). async/await is syntactic sugar over Promises — an async function always returns a Promise, and await pauses execution within that function until the awaited Promise settles, letting asynchronous code (like an API call) read like sequential synchronous code instead of nested callbacks.

  5. How would you validate a form client-side before submitting file-import data to the server?
    Show answer

    Check required fields, allowed file types/extensions, and a reasonable file-size limit before upload begins, and give immediate inline feedback (not just an alert on submit). For structured data, you can even parse the CSV/Excel client-side (e.g. with a library) to catch obviously malformed headers or empty files early. Client-side validation is purely a UX convenience, though — the server must always re-validate independently since client checks can be bypassed.

11 · REST vs SOAP APIs5 questions
  1. Key differences between REST and SOAP — protocol, payload format, and when you'd still need to support SOAP.
    Show answer

    REST is an architectural style over HTTP using verbs (GET/POST/PUT/DELETE) with typically JSON payloads — lightweight, cacheable, and the modern default. SOAP is a stricter XML-based protocol with a formal contract (WSDL), built-in error handling, and standards for transactions/security (WS-Security) — heavier but more rigorous. You'd still need SOAP when integrating with older enterprise systems (banking, government, some ERP/legacy college-management platforms) that only expose SOAP endpoints.

  2. What is WSDL, and how do you consume a SOAP service from a .NET application?
    Show answer

    WSDL (Web Services Description Language) is an XML contract describing a SOAP service's operations, message formats, and endpoint address. In .NET, you add a "Connected Service"/"Service Reference" pointing at the WSDL URL, and Visual Studio generates a strongly-typed client proxy class so you can call the SOAP operations like normal C# methods without hand-writing the XML envelope.

  3. Explain idempotency in REST — which HTTP verbs are idempotent and why does it matter for retrying a failed import call?
    Show answer

    An idempotent operation produces the same end state no matter how many times it's repeated. GET, PUT, and DELETE are idempotent by spec; POST is not — calling it twice typically creates two resources. This matters for a flaky import call: if the client times out and retries a non-idempotent POST, it can create duplicate records, so import endpoints should either use an idempotency key or be designed as an idempotent upsert (PUT-style) so a safe retry doesn't duplicate data.

  4. How do you version a REST API without breaking existing consumers?
    Show answer

    Common approaches: URL versioning (/api/v1/students, /api/v2/students), a custom header (Api-Version: 2), or content negotiation via the Accept header. URL versioning is the simplest and most visible. Whichever approach, keep the old version live and functioning until consumers have migrated, and communicate deprecation timelines rather than changing a live endpoint's contract in place.

  5. What is the difference between authentication and authorization, and how is each typically implemented in a REST API?
    Show answer

    Authentication verifies who you are (login, typically producing a JWT or session token); authorization determines what you're allowed to do once identified (role/claim-based access checks). In ASP.NET Core, authentication is handled by middleware validating credentials/tokens, while authorization is enforced per-endpoint with [Authorize]/policy attributes checking the authenticated user's roles or claims before letting the request reach the controller action.

12 · SSIS, SSRS & Power BI5 questions
  1. What is an SSIS package, and what are the core components (Control Flow, Data Flow, Connection Managers)?
    Show answer

    An SSIS package is a self-contained ETL workflow (a .dtsx file). Control Flow orchestrates the overall sequence of tasks (e.g. "truncate staging, then run data flow, then email on failure") including loops and conditional branches. Data Flow tasks handle the actual row-by-row movement and transformation between a source and destination (e.g. CSV → SQL table, with lookups/derived columns in between). Connection Managers define reusable connections (file paths, database connection strings) referenced by tasks in the package.

  2. How would you build an SSIS package to import a CSV file, validate rows, and route errors to a separate table?
    Show answer

    Use a Flat File Source to read the CSV, add Data Conversion/Derived Column transforms to enforce types and required-field checks, and configure the destination component's Error Output to redirect any row that fails conversion or a lookup (e.g. an invalid foreign key via a Lookup transform) to a separate OLE DB Destination pointing at an error-logging table, instead of failing the whole package. Valid rows continue to the normal destination.

  3. What is the difference between SSIS and writing a custom C# import utility — when would you pick each?
    Show answer

    SSIS is best for scheduled, recurring, fairly standard ETL jobs where a visual designer, built-in scheduling (via SQL Agent), and reusable transformation components speed up development and give non-developers visibility into the flow. A custom C# utility (e.g. behind a Web API using SqlBulkCopy) is better when the import is triggered interactively by end users through the application UI, needs custom business logic that's awkward to express visually, or must integrate tightly with the app's existing authentication/logging.

  4. What is SSRS, and how do you build a parameterized report backed by a stored procedure?
    Show answer

    SSRS (SQL Server Reporting Services) is a server-based platform for authoring, publishing, and rendering paginated reports (PDF, Excel, etc.) accessed through a report portal or embedded viewer. In Report Builder/SSDT, you create a dataset whose query calls a stored procedure, map the procedure's parameters to report parameters (which render as filter dropdowns/date pickers on the report page), and the report re-executes the procedure with the user's chosen values each time it's run.

  5. How does Power BI connect to SQL Server, and what's the difference between Import mode and DirectQuery?
    Show answer

    Power BI connects via a native SQL Server connector using a server/database name (and optionally a gateway for on-premises databases). Import mode copies data into Power BI's own in-memory compressed model at refresh time — fast interactive reports, but data is only as fresh as the last scheduled refresh. DirectQuery instead sends live queries back to SQL Server on every interaction — always current, but report performance depends directly on the underlying query/database performance.

06 Bulk-load a CSV with SqlBulkCopy pattern (staging → validate → merge) Medium

Write the T-SQL that validates rows already bulk-loaded into StagingImport and moves only clean rows into Students, logging bad rows separately.

Show solution 
-- 1. Flag invalid rows (bad email format or non-numeric amount)
SELECT * INTO #BadRows
FROM StagingImport
WHERE RawEmail NOT LIKE '%_@__%.__%'
   OR TRY_CAST(RawAmount AS DECIMAL(10,2)) IS NULL;

INSERT INTO ImportErrorLog (RowId, RawEmail, RawAmount, SourceFile, Reason, LoggedAt)
SELECT RowId, RawEmail, RawAmount, SourceFile, 'Invalid email or amount', GETDATE()
FROM #BadRows;

-- 2. Load only clean rows, skipping ones already logged as bad

SELECT s.RawEmail, GETDATE()
FROM StagingImport s
LEFT JOIN #BadRows b ON s.RowId = b.RowId
WHERE b.RowId IS NULL;
07 Upsert (MERGE) incoming payment records Medium

Given a staging table of new/updated payments, insert new ones and update existing ones by PaymentId.

Show solution 
MERGE INTO Payments AS target
USING StagingPayments AS source
   ON target.PaymentId = source.PaymentId
WHEN MATCHED AND target.Amount <> source.Amount THEN
  UPDATE SET target.Amount = source.Amount,
                 target.PaidOn = source.PaidOn
WHEN NOT MATCHED BY TARGET THEN
  INSERT (PaymentId, StudentId, Amount, PaidOn)
  VALUES (source.PaymentId, source.StudentId, source.Amount, source.PaidOn)
OUTPUT $action, inserted.PaymentId;
-- Note: wrap in a transaction with HOLDLOCK/SERIALIZABLE hint under concurrent load
-- to avoid the well-known MERGE race condition.

14 Second highest payment per student (correlated subquery vs window function) Medium

Return each student's second-largest single payment. Show both approaches.

Show solution 
-- Window function approach (preferred, single scan)
SELECT StudentId, Amount
FROM (
  SELECT StudentId, Amount,
         ROW_NUMBER() OVER (PARTITION BY StudentId ORDER BY Amount DESC) AS rn
  FROM Payments
) x
WHERE rn = 2;

-- Correlated subquery approach
SELECT p1.StudentId, MAX(p1.Amount) AS SecondHighest
FROM Payments p1
WHERE p1.Amount < (
  SELECT MAX(p2.Amount) FROM Payments p2
  WHERE p2.StudentId = p1.StudentId
)
GROUP BY p1.StudentId;
