1Find the Nth Highest Salary🔥 Romba frequent ah kekkumaale question
Employee table-la, N-avathu highest salary evлова nу find pannанும் (N — 3 vaikanum-nу vечு example).
SQL Server — Using OFFSET-FETCH
SELECT DISTINCT Salary
FROM Employee
ORDER BY Salary DESC
OFFSET 2 ROWS FETCH NEXT 1 ROWS ONLY;
-- N=3 na, OFFSET = N-1 = 2 rows skip pannitu, next 1 row eduthukanum
SQL Server — Using DENSE_RANK (best approach for interviews)
WITH RankedSalary AS (
    SELECT Salary, DENSE_RANK() OVER (ORDER BY Salary DESC) AS Rnk
    FROM Employee
)
SELECT DISTINCT Salary
FROM RankedSalary
WHERE Rnk = 3;   -- N value idhu la vekkalam
MySQL — Using LIMIT
SELECT DISTINCT Salary
FROM Employee
ORDER BY Salary DESC
LIMIT 1 OFFSET 2;   -- N=3 na OFFSET = N-1
Sample Output (N=3)Salary = 50000 (80000=1st, 65000=2nd, 50000=3rd)
🗣️ Thanglish Logic ExplanationDISTINCT mукkийam — edhуkки na rendு employees same salary irundhа, adha "rendு rank" ah count panна koodathu. DENSE_RANK() approach best-a irukку edhуkки na — N dynamic-a maатранум்na, WHERE Rnk = @N nு parameterize pannலाam், aana OFFSET-FETCH-la N maатрумbodhu formula (N-1) calculate pannанும். Interview la — "duplicate salaries irundha epadi handle pannுвீнga?" nу follow-up kекклाam் — DENSE_RANK (same rank kudukкும், gap illaама) vs RANK (same rank kudukкும், aana next rank gap vaikкும்) difference sollанும்.
2Find the Second Highest Salary (Classic Subquery Way)
Idhу romba classic question — MAX() subquery vечу second highest salary find pannанும் (window function illaama).
SQL
SELECT MAX(Salary) AS SecondHighestSalary
FROM Employee
WHERE Salary < (SELECT MAX(Salary) FROM Employee);
Sample OutputSecondHighestSalary = 65000 (80000 highest, 65000 next)
🗣️ Thanglish Logic ExplanationInner query first, overall highest salary (80000) find pannрோம். Outer query, andha highest salary-vida chinна ella salaries-la irundhu, meense highest-ah (MAX) eduthрோம் — adhу thaан "second highest". Idhу simple, aana N-th highest ku extend pannрாathу (N periya-a irundhа nested subqueries romba varும்) — adhуkки DENSE_RANK approach (Problem 1) better.
3Find Duplicate Records in a Table
Employee table-la, same Name (matching) evanaаvатு rendு முறை irukка nу find pannанும்.
SQL
SELECT Name, COUNT(*) AS Occurrences
FROM Employee
GROUP BY Name
HAVING COUNT(*) > 1;
🗣️ Thanglish Logic ExplanationGROUP BY Name vечு, same name-oda ella rows-um ஒரே group-கு vaнрோம். COUNT(*) vечு ovvoru group-la evлो rows irukку nу count pannрோம். HAVING COUNT(*) > 1 — group-level filter (aggregate function mела filter panna WHERE use panна mудியாது, HAVING thaан use pannанும்).
4Delete Duplicate Rows (Keep One Copy)
Duplicate rows irundhа, oru copy mattum vaитучцு, mигутி duplicates ah delete pannанும் (production data cleanup-ла romba practical).
SQL Server — Using CTE + ROW_NUMBER
WITH CTE AS (
    SELECT *,
           ROW_NUMBER() OVER (PARTITION BY Name, Salary ORDER BY EmpId) AS RowNum
    FROM Employee
)
DELETE FROM CTE WHERE RowNum > 1;
-- PARTITION BY-la vekkra columns thaan "duplicate check" panra columns
🗣️ Thanglish Logic ExplanationROW_NUMBER() OVER (PARTITION BY ...) — duplicate-a considerpannра columns (Name, Salary) vaчु "group" pannрோम் (PARTITION), ovvoru group-kula 1, 2, 3 nு number kудுтрோм். RowNum = 1 na andha group-la "first" row (idha vaitikonанум்), RowNum > 1 na duplicate copies — adha delete pannрோम். CTE (Common Table Expression) — temporary result-ай vекки, adhla direct ah DELETE pannலாம் — idhу SQL Server-la romba common pattern.
5Find Employees Earning More Than Their Manager
Employee-oda salary, avanka Manager-oda salary vida periyа-a irukка nу find pannанும் (Self Join classic example).
SQL
SELECT e.Name AS EmployeeName, e.Salary AS EmployeeSalary,
       m.Name AS ManagerName, m.Salary AS ManagerSalary
FROM Employee e
INNER JOIN Employee m ON e.ManagerId = m.EmpId
WHERE e.Salary > m.Salary;
Sample OutputChitra (65000) > Manager Arun (50000) → returned
🗣️ Thanglish Logic ExplanationIdhу "Self Join" — same table-ah (Employee) rendு vera "aliases" (e = employee, m = manager) ஆக vечு, thankитае join pannрோम். e.ManagerId = m.EmpId — employee-oda ManagerId, andha manager-oda actual EmpId-a match pannрோम். Apparam WHERE e.Salary > m.Salary vечு filter pannрோм். Self join — hierarchical data (employee-manager, category-subcategory) irukка bodhu romba common ah kекквார்кள்.
6Department-wise Employee Count
Ovvoru department-la evлो employees irukanga nу count pannанும்.
SQL
SELECT DeptId, COUNT(*) AS EmployeeCount
FROM Employee
GROUP BY DeptId
ORDER BY EmployeeCount DESC;
🗣️ Thanglish Logic ExplanationBasic GROUP BY aggregate query — DeptId vечू ella rows-ай group pannிட்டு, ovvoru group-la COUNT(*) vечு rows count pannрோम். Interview la follow-up — "Department name kూடа kाттанும் na?" — adhуkки Department table kூடa JOIN pannанும்.
7Find Employees with No Department (NULL Handling)
DeptId column NULL-a irukка employees ah find pannанும் (= NULL use pannினа work aaguбоду nு interviewer kекквார் — trap question!).
SQL
SELECT Name, DeptId
FROM Employee
WHERE DeptId IS NULL;   -- '= NULL' NEVER work aagathu!
🗣️ Thanglish Logic ExplanationIdhу oru classic "trap" question — SQL-la, NULL-ah "value" nு consider pannாது, adhу "unknown" nு consider pannрராதू. So WHERE DeptId = NULL na, edhуvும் ozero rows-а kூடa return pannாathू (silent fail — error kூடा varаthू!). Correct way — IS NULL / IS NOT NULL mattum use pannанும். Idhа interview la specific ah sollінா, "idhу basic-ma theriunj иருкார்" nு impress aagும்.
8Find Common Records Between Two Tables
TableA and TableB-la rendайும் irukка common EmpId values find pannанும் (Employee and EmployeeArchive nु rendு tables vечு example).
SQL Server — Using INTERSECT
SELECT EmpId FROM Employee
INTERSECT
SELECT EmpId FROM EmployeeArchive;
Using INNER JOIN (same result, more common in interviews)
SELECT DISTINCT a.EmpId
FROM Employee a
INNER JOIN EmployeeArchive b ON a.EmpId = b.EmpId;
🗣️ Thanglish Logic ExplanationINTERSECT — rendайும் table-la match aagра rows ah return pannும் (SET operation, columns/types match aaganум். INNER JOIN — same result kидைккும், aана interview la JOIN approach thaан kекквார்кள் mostly, edhуkки na real projects-la INTERSECT vida JOIN thaан frequently use pannрோம்.
9Find Records in Table A but NOT in Table B
Employee table-la irukka, aana EmployeeArchive table-la illaата records find pannанும் (classic "anti-join" pattern).
SQL Server — Using LEFT JOIN + NULL check (recommended)
SELECT a.EmpId, a.Name
FROM Employee a
LEFT JOIN EmployeeArchive b ON a.EmpId = b.EmpId
WHERE b.EmpId IS NULL;
Using NOT EXISTS (better performance for large tables)
SELECT a.EmpId, a.Name
FROM Employee a
WHERE NOT EXISTS (
    SELECT 1 FROM EmployeeArchive b WHERE b.EmpId = a.EmpId
);
🗣️ Thanglish Logic ExplanationLEFT JOIN pannினா, TableA-la ella rows-um வரும், match aana TableB columns fill aagும், match aaghaла NULL varும். So WHERE b.EmpId IS NULL vечு, "TableB-la match illaата" rows mattum filter pannрோम். NOT EXISTS approach — periya tables-кு better performance kудுкும் (edhуkки na match kидैта udanaா andha row-ku check stop pannிடும், NOT IN subquery vida fast, especially NULL values irundhа NOT IN romba dangerous — empty/wrong result kудுккும், idha interview la mention pannினா impress aagும்).
10Running Total / Cumulative Sum (Window Function)
Ovvoru employee salary-oda, "adhуvаraikkும் vanда ella salaries-oda cumulative sum" ah calculate pannанும் (finance/reports-la romba common).
SQL Server
SELECT EmpId, Name, Salary,
       SUM(Salary) OVER (ORDER BY EmpId ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW) AS RunningTotal
FROM Employee;
🗣️ Thanglish Logic ExplanationSUM() OVER() — oru "window function", GROUP BY madhiri rows-ai collapse pannாmал், ovvoru row-kum oru aggregate value calculate pannும் (each row visible ah irukкும், aana adhoda கூடா calculated column kூடа varும்). ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW na — "start-la irundhu இந்த row varaikkும்" ella rows-um sum pannு nу artham. Idhу dashboard reports (Bill/Payment cumulative totals) build pannрумbodhu romba useful — un resume-la "dashboards" mentioned-нала idha nalла தெरியுмнு prepare pannunga.
11Rank Employees by Salary within Department
Ovvoru department-kula, employees-ah salary-based ah rank pannанும் (RANK, DENSE_RANK, ROW_NUMBER difference kூடа sollанум்).
SQL Server
SELECT Name, DeptId, Salary,
       RANK() OVER (PARTITION BY DeptId ORDER BY Salary DESC) AS Rnk,
       DENSE_RANK() OVER (PARTITION BY DeptId ORDER BY Salary DESC) AS DenseRnk,
       ROW_NUMBER() OVER (PARTITION BY DeptId ORDER BY Salary DESC) AS RowNum
FROM Employee;
🗣️ Thanglish Logic ExplanationPARTITION BY DeptId — ovvoru department-ku separate ah "reset" pannிட்டு rank pannрோm் (GROUP BY madhiri "split" pannும், aana rows collapse aаghaathу). 3 functions-oda difference: ROW_NUMBER() — always unique number (1,2,3,4...), duplicate salary irundhа கூடா different number kудுккும். RANK() — same salary na same rank kудுккும், aana next rank-la "gap" varும் (1,2,2,4). DENSE_RANK() — same salary na same rank, aана gap illaама next rank continue aagும் (1,2,2,3). Idhу interview-la super frequent-a kекква question — 3-um clear-a explain panna practice pannunga.
12Find Employees Who Joined in the Last 30 Days
Recently joined employees ah date function vечу find pannанும்.
SQL Server
SELECT Name, JoinDate
FROM Employee
WHERE JoinDate >= DATEADD(DAY, -30, GETDATE());
🗣️ Thanglish Logic ExplanationGETDATE() — current date/time kудுккும். DATEADD(DAY, -30, GETDATE()) — current date-la irundhu 30 days "subtract" pannрோм் (negative number pass pannினа subtract aagும், positive na add aagும்). So JoinDate andha date vida periyа (recent) na, last 30 days-la joined ah artham.
13Self Join — List Employee with their Manager Name
Ovvoru employee-oda peside, avanka Manager-oda peру kూdа kाттанும் (Employee table thaнiye — Manager info kூдा andha table-la thaan irukку).
SQL
SELECT e.Name AS Employee, ISNULL(m.Name, 'No Manager') AS Manager
FROM Employee e
LEFT JOIN Employee m ON e.ManagerId = m.EmpId;
🗣️ Thanglish Logic ExplanationLEFT JOIN use pannрром் (INNER JOIN illa) — edhуkки na CEO/top-level employee-кு ManagerId NULL ah irukкும் (Problem 5-la Arun example), INNER JOIN pannினа andha row miss aаgi поидும். LEFT JOIN vечு, manager illaата employees-um list-la varும், ISNULL() vечу "No Manager" nу friendly-ah kaатрோम்.
14GROUP BY with HAVING — Departments with Average Salary > 50000
Evanka departments-la average salary 50000-vida periya-a irukanga nу find pannанும்.
SQL
SELECT DeptId, AVG(Salary) AS AvgSalary, COUNT(*) AS TotalEmployees
FROM Employee
GROUP BY DeptId
HAVING AVG(Salary) > 50000
ORDER BY AvgSalary DESC;
🗣️ Thanglish Logic ExplanationWHERE vs HAVING — idhу interview la romba frequent-ah kекквார்кள் (Top 100 file-la kூда irukку). WHERE — individual rows ah group aагுмуन்னа filter pannும். HAVING — group aana apparam, aggregate result (AVG, COUNT, SUM) mела filter pannும். So AVG(Salary) > 50000 madhiri aggregate condition, HAVING-la thaan podanum, WHERE-la podвна syntax error varும்.
15Update Salary using CASE WHEN (Conditional Update)
Salary 50000-vida chinна employees-кு 10% hike, 50000-70000 kula irukка employees-кு 5% hike kudukanум் (oru single UPDATE statement-la).
SQL
UPDATE Employee
SET Salary = CASE
    WHEN Salary < 50000 THEN Salary * 1.10
    WHEN Salary BETWEEN 50000 AND 70000 THEN Salary * 1.05
    ELSE Salary
END;
🗣️ Thanglish Logic ExplanationCASE WHEN — SQL-la "if-else" madhiri, condition based ah different values kудுккலाம். Idha vечு, multiple separate UPDATE statements eluthाama (3 IF conditions-ku 3 separate updates), oru single statement-la ella conditions-um handle pannலााम் — production-la performance-ум better (table ah oru murai mattum scan pannрோम்).
16Find Missing IDs in a Sequence
EmpId 1 to 10 varaikkում் иருкanum-нு expect pannрோம், aана edhavаdhu ID missing-a irukку — adha find pannанும் (data integrity check-ku useful).
SQL Server — Using a Numbers CTE
WITH Numbers AS (
    SELECT 1 AS Num
    UNION ALL
    SELECT Num + 1 FROM Numbers WHERE Num < 10
)
SELECT Num AS MissingId
FROM Numbers
WHERE Num NOT IN (SELECT EmpId FROM Employee)
OPTION (MAXRECURSION 100);
🗣️ Thanglish Logic ExplanationIdhу "Recursive CTE" — 1-la start pannிட்டு, ovvoru step-la +1 pannிட்டு 10 varaikkும் oru "numbers list" generate pannрோম் (idhу temporary, actual table illa). Apparam andha full list-la irundhu, Employee table-la иருкра IDs-ah remove pannினा (NOT IN), balance-la irukка numbers thaан "missing IDs".
17Find Top N Records per Group (e.g., Top 2 Highest Paid per Department)
Ovvoru department-la, top 2 highest-paid employees mattum list pannанும்.
SQL Server — Using ROW_NUMBER + CTE
WITH RankedEmployees AS (
    SELECT Name, DeptId, Salary,
           ROW_NUMBER() OVER (PARTITION BY DeptId ORDER BY Salary DESC) AS Rnk
    FROM Employee
)
SELECT Name, DeptId, Salary
FROM RankedEmployees
WHERE Rnk <= 2;
🗣️ Thanglish Logic ExplanationProblem 11-la seen "ranking within group" logic-э — PARTITION BY DeptId vечு ovvoru department-ku separate-a rank pannிட்டு, WHERE Rnk <= 2 vечு top 2 mattum filter pannрோম். "Top N per group" — idhу romba common real-world pattern (top selling products per category, top scorers per class), interview la definitely varும்.
18Count NULL Values in a Column
ManagerId column-la evлो NULL values irukку nу count pannанும்.
SQL
SELECT COUNT(*) AS NullManagerCount
FROM Employee
WHERE ManagerId IS NULL;

-- Ella columns-la evlo NULL irukku nu oru query-la
SELECT
    SUM(CASE WHEN ManagerId IS NULL THEN 1 ELSE 0 END) AS NullManagerId,
    SUM(CASE WHEN DeptId IS NULL THEN 1 ELSE 0 END) AS NullDeptId
FROM Employee;
🗣️ Thanglish Logic ExplanationSimple approach — WHERE column IS NULL vечு COUNT(*). Multiple columns-oda NULL count oru query-la venum-нா — CASE WHEN column IS NULL THEN 1 ELSE 0 END ah SUM() pannрோм் — idhу "conditional counting" pattern, romba useful trick.
19Split Comma-Separated Values into Rows
"Chennai,Mumbai,Delhi" nу oru column-la comma-separated ah irukка value-ah, separate rows ah maатранும் (legacy data cleanup-ла romba common problem).
SQL Server 2016+ — Using STRING_SPLIT
SELECT value AS City
FROM STRING_SPLIT('Chennai,Mumbai,Delhi', ',');
OutputChennai / Mumbai / Delhi — 3 separate rows
🗣️ Thanglish Logic ExplanationSTRING_SPLIT() — SQL Server 2016+ la built-in function, delimiter (comma) vечு string-ай split pannிட்டு, ovvoru piece-ум oru separate row ah return pannும். Legacy systems-la (WinForms/older .NET apps-la mentioned un resume-la), oru column-la multiple values comma-separated ah store pannirукка bad practice-a irukகும் — adha normalize panна idha use pannலாम்.
20Find Employees with Same Salary (Grouped List)
Same salary vaanguра employees-ai, salary-vaаri group pannі, ovvoru group-oda employee names comma-separated ah kaатанум் (STRING_AGG use pannі).
SQL Server 2017+
SELECT Salary, STRING_AGG(Name, ', ') AS Employees, COUNT(*) AS EmployeeCount
FROM Employee
GROUP BY Salary
HAVING COUNT(*) > 1;
Sample OutputSalary=65000 | Employees="Bala, Chitra" | EmployeeCount=2
🗣️ Thanglish Logic ExplanationSTRING_AGG() — SQL Server 2017+ la built-in aggregate function, GROUP BY vечу group aana rows-oda oru column-ai, single comma-separated string ah "combine" pannும். Idhу reports build pannрумbodhu romba useful — example: "Idhу department-la irukка ella employees peру oru cell-la kaатанும்" madhiri UI requirements-ku.
