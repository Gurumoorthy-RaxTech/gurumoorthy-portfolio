F1 HTML5
1. Semantic HTML tags na enna, ethuku use pannanum?
<header>, <nav>, <main>, <section>, <article>, <footer> — idhu ellaam "artham irukka" tags, <div> madhiri artham illaama irukкாthு. Idhu use panрினা — SEO improve aagும் (Google idha better ah understand pannும்), accessibility better aagும் (screen reader users ku easy aagும்), and code படிக்க வேற devs ku easy ah puriyும்.
🗣️ Answer Script"I use semantic tags like header, nav, main, and footer instead of generic divs everywhere — it improves SEO, accessibility for screen readers, and makes the markup self-documenting for other developers."
2. Form validation — HTML5 la built-in ah eppadi pannலாம்?
required, type="email", pattern, minlength/maxlength attributes vechа, JavaScript எழுதாமலேயே basic validation pannலாம். Browser automatic ah error message kaatும் (example: "Please fill this field"). Aana server-side validation um mandatory — client-side ah bypass pannalaам (browser dev tools la).
🗣️ Answer Script"HTML5 gives built-in validation via attributes like required, type='email', and pattern — but I always back it up with server-side validation too, since client-side checks can be bypassed."
3. <div> vs <span> difference?
<div> — block-level element, full width edukкும், next content ah puthu line ku push pannும். <span> — inline element, adha content evlo width edukкும் அத்தனை width mattum edukкும், next content adhoda kூடa same line la irukкும். Example: paragraph kulla oru word ah highlight panna span use pannுவோம்.
4. localStorage vs sessionStorage vs Cookies?
localStorage — browser close pannிட்டாலும் data stay aagும், manually clear pannும் varaikkும். sessionStorage — tab close pannினா data poyيடும். Cookies — server kூடa automatic ah every request la anுppுрும் (so size limit chinnа, 4KB), expiry time set pannலாம். Token store panna localStorage use pannுрого, aana XSS attack risk irukку — HttpOnly cookie safer.
🗣️ Answer Script"localStorage persists until manually cleared, sessionStorage clears when the tab closes, and cookies are automatically sent with every request but have a small size limit. For sensitive tokens, HttpOnly cookies are safer than localStorage since JavaScript — and therefore an XSS attack — can't read them."
F2 CSS3
5. Box Model na enna?
Ovvoru HTML element um oru "box" — content → padding (content-a சுத்தி space) → border → margin (outside space, next element kூடa gap). box-sizing: border-box pota, width/height la padding+border um include aagும் — idhu illna, padding/border extra ah add aagi element expected size vida periya ah kanpiккும்.
6. Flexbox vs Grid — eppo edha use pannanum?
Flexbox — 1-dimensional layout (row illna column, oru direction mattum) ku perfect — example: navbar la items evenly space pannuрது. Grid — 2-dimensional layout (rows AND columns rendайும் ஒரே time la control pannрது) ku perfect — example: dashboard la cards ah rows/columns ah arrange pannuрது.
🗣️ Answer Script"Flexbox is one-dimensional — great for aligning items in a row or column, like a navbar. Grid is two-dimensional — I use it for full page or dashboard layouts where I need control over both rows and columns together."
7. CSS Specificity eppadi calculate pannуவathু?
Priority order: !important (highest, avoid pannрது nalladu) > inline style > ID selector (#id) > class/attribute/pseudo-class (.class) > element selector (div). Rendு selector same specificity irundha, last la eluthina CSS thaan apply aagும்.
8. Responsive Design — Media Queries eppadi work aagும்?
@media (max-width: 768px) { ... } madhiri, screen width based ah different CSS apply pannலாம். Mobile-first approach na — base CSS ah mobile ku eluthिட்டு, apparam min-width media queries vechа tablet/desktop ku styles add pannுрது (opposite ah, desktop-first na max-width vechа kammi pannுрது).
9. position: relative vs absolute vs fixed vs sticky?
relative — normal position-a base ah vachु offset (top/left) pannலாம், aana andha space still reserve aagும். absolute — nearest positioned ancestor-a base ah vachु position aagும், normal flow-la irundhu remove aagும். fixed — viewport-a base ah vachু, scroll pannினalும் same place la stay aagும் (example: sticky header). sticky — normal ah scroll aagும், oru threshold varaikkும், apparam fixed madhiri stick aagும்.
F3 JavaScript Core
10. var vs let vs const?
var — function-scoped, hoist aagும் (top ku move aagும், aana undefined ah), loop kulla use pannினा bugs varும். let — block-scoped, reassign pannலாம். const — block-scoped, reassign panna முடியாது (aana object/array kulla irukka values mாtha முடியும்). Ippa ellame let/const mattum use pannுрோம், var avoid pannுрோм்.
🗣️ Answer Script"var is function-scoped and hoisted, which causes classic loop-variable bugs. let and const are block-scoped — I always use const by default and let only when I know the variable needs reassignment."
11. == vs ===?
== — type coercion pannிட்டு compare pannும், so '5' == 5 true varும் (surprise varும்). === — type um value um rendайும் exact ah match aaganum, so '5' === 5 false. Ella நேரமும் === use pannுрது safe.
12. Closures na enna? Example kudu.
Oru function, adhoda "outer scope" variable ah, adhoda function return aana apparam kூடa access panna mudியும் — idhu thaan closure. Example: counter function — outerCount variable ah private ah vaikka use pannலாம், direct ah global variable ஆக்காம.
function makeCounter() {
  let count = 0;
  return function() {
    count++;
    return count;
  };
}
const counter = makeCounter();
console.log(counter()); // 1
console.log(counter()); // 2 (count value memory la nikkuthu!)
🗣️ Answer Script"A closure lets an inner function remember variables from its outer scope even after the outer function has returned — I use it to create private state, like a counter, without exposing a global variable."
13. Event Bubbling vs Capturing?
Bubbling — event oru child element la trigger aana, adhu parent, grandparent nu "மேலே" propagate aagும் (default behavior). Capturing — opposite, parent-la irundhu child ku "கீழே" varும் (rare ah use pannுрோம், addEventListener(event, fn, true) vechа enable pannலாம்). event.stopPropagation() vechа bubbling ah stop pannலாம்.
14. Event Delegation na enna, ethukku useful?
Ovvoru child element kula separate ah event listener podama, parent element la ஒரே listener vechு, event.target check panни handle pannுрது. Benefit — dynamically add pannра elements (example: AJAX vaзி add aana new row) kூடa automatic ah work aagும், edhуkки na parent-la already listener irukку.
15. Array methods — map, filter, reduce difference?
map() — every element ah transform panни puthu array return pannும் (same length). filter() — condition true aana elements mattum vaiche puthu array (chinna/same length). reduce() — array ah oru single value ku "reduce" pannும் (example: sum, total).
const nums = [1, 2, 3, 4, 5];
const doubled = nums.map(n => n * 2);        // [2,4,6,8,10]
const evens = nums.filter(n => n % 2 === 0); // [2,4]
const sum = nums.reduce((acc, n) => acc + n, 0); // 15
16. Hoisting na enna?
JavaScript engine, code run pannுмуன்னаடி, variable/function declarations ah "top" ku move pannrathu madhiri behave pannும். var declarations hoist aagும் (undefined ah), function declarations (full function) hoist aagும், aana let/const hoist aana kூடa, declare pannுмуன்னа access pannினா error (temporal dead zone).
F4 JavaScript — Async / Promises
17. Callback vs Promise vs Async/Await?
Callback — function ah argument ah pass pannி, velai mudinja apparam call pannрது (nested aana "callback hell" varும்). Promise — future la result varும் nu oru "guarantee object" — .then()/.catch() chain pannலாம். Async/Await — Promise-a synchronous madhiri readable ah eluthрதுкки sugar syntax.
// Promise
fetch('/api/students').then(res => res.json()).then(data => console.log(data));

// Async/Await — same thing, easy to read
async function getStudents() {
  const res = await fetch('/api/students');
  const data = await res.json();
  console.log(data);
}
18. fetch() API eppadi use pannуவathু?
fetch() — native browser API, API call panna use pannுрோм். Important point — fetch network error mattum reject pannும், HTTP 404/500 status வந்தа கூட reject pannাது — so manually response.ok check pannuவathு mukkiyam.
async function loadData() {
  try {
    const res = await fetch('/api/students');
    if (!res.ok) throw new Error('Request failed: ' + res.status);
    const data = await res.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}
19. Debounce vs Throttle — difference?
Debounce — user typing nirutthinappuram (pause aana apparam) mattum function call aagும் (example: search box la API call). Throttle — evlo fast events trigger aanalும், fixed interval ku ஒரு முறை mattum function call aagும் (example: scroll event handle pannுрது).
