1 Delegate என்னன்னு — Simple-a புரிஞ்சுக்கோ
🎬 Dasavathaaram (2008) — Kamal 10 Roles

Kamal — 10 different avatars, but oru director "action" solluvan — yaara-a anupuvan. Delegate = type-safe method pointer. "Which method to call" — runtime-la decide pannalaam. Same signature, different implementations — director maadhiri!

💬 Tanglish — 1 Line Definition

Delegate = method-ku reference vachu vaikka oru type. Variable maari method-a pass panalaam, return panalaam, store panalaam. Method itself = data!

Step by Step — Delegate 3 steps:

// STEP 1: Declare delegate type (signature define)
public delegate decimal FeeCalculator(Student student);
//        ↑ return type  ↑ delegate name  ↑ parameters

// STEP 2: Create methods that MATCH the signature
public decimal CalculateTermFee(Student s) => s.Grade * 500m;
public decimal CalculateBusFee(Student s)  => s.Distance * 20m;
public decimal CalculateHostelFee(Student s) => 3000m;

// STEP 3: Create instance + point to a method
FeeCalculator calc = CalculateTermFee;   // point to method
decimal amount = calc(student);          // INVOKE — like calling method!

// Switch at runtime — no if/else needed!
if (student.HasBus) calc = CalculateBusFee;
if (student.InHostel) calc = CalculateHostelFee;

Multicast Delegate — Multiple methods, one call:

🎬 Bigil (2019) — Coach calls all players at once

Coach whistle adipaan — ellaa players-um action edukaanga. Multicast delegate = oru invoke-la multiple methods run aagum. += add pannu, -= remove pannu!

public delegate void StudentAdmittedHandler(Student student);

StudentAdmittedHandler onAdmit = SendWelcomeSms;
onAdmit += CreateStudentPortal;  // add second
onAdmit += NotifyClassTeacher;  // add third

onAdmit(newStudent);
// Calls: SendWelcomeSms → CreateStudentPortal → NotifyClassTeacher
// All three — ONE invoke!

onAdmit -= CreateStudentPortal; // remove specific
⚠️ Multicast Trap — Return Value Lost!

Multicast delegate with return value → only LAST method's return value kept! Others silently discarded. Void return use pannaa safe. Return value needed-a → single method delegate use pannu.

2 Delegate Evolution — C# 1.0 → 3.0 → Today
🎬 Enthiran → 2.0 → Robot 3.0 (evolution concept)

Chitti 1.0 → 2.0 — same core, better syntax. Delegates-um same — custom delegate → anonymous method → lambda → expression. Interview-la evolution therinjaa senior impression!

C# 1.0 — Named Method (verbose)

public decimal TermFeeCalc(Student s) => s.Grade * 500m;
FeeCalculator calc = new FeeCalculator(TermFeeCalc);

C# 2.0 — Anonymous Method (no separate method needed)

FeeCalculator calc = delegate(Student s) {
    return s.Grade * 500m;
};

C# 3.0 — Lambda Expression (modern, clean) ✅ Use This!

FeeCalculator calc = s => s.Grade * 500m;

// Multi-line lambda:
FeeCalculator calc = s => {
    var base = s.Grade * 500m;
    var discount = s.Scholarship ? 0.1m : 0m;
    return base * (1 - discount);
};
💡 Interview: Closure — Lambda captures outer variable

Lambda outer variable capture pannutu saapududu — closure. Variable value changes → lambda sees new value. Careful in loops!

// Closure trap in loop
var actions = new List<Action>();
for (int i = 0; i < 3; i++)
    actions.Add(() => Console.WriteLine(i)); // captures i ref!

actions.ForEach(a => a()); // prints 3,3,3 NOT 0,1,2!

// Fix: capture local copy
for (int i = 0; i < 3; i++)
{
    int copy = i;                              // local copy!
    actions.Add(() => Console.WriteLine(copy)); // prints 0,1,2
}
3 Built-in Delegates — Func, Action, Predicate Most Asked!
🎬 Baasha (1995) — Ready-made soldiers

Manikkam own army train panna vendam — already trained soldiers irukkaanga. Func, Action, Predicate = ready-made delegates — custom delegate declare pannaa vendam!

🔵 Func<TIn, TOut>

Returns a value. Last type param = return type.

// Func<input, output>
Func<Student, decimal> calcFee
    = s => s.Grade * 500m;

// Multiple inputs
Func<int, int, int> add
    = (a, b) => a + b;

// No input, returns value
Func<string> getTime
    = () => DateTime.Now.ToString();

decimal fee = calcFee(student);

🟢 Action<T>

No return value (void). Side effects only.

// Action<input> — no return
Action<Student> sendSms
    = s => _sms.Send(s.Mobile);

// Multiple inputs
Action<string, string> log
    = (level, msg)
        => _logger.Log(level, msg);

// No input, no return
Action clearCache
    = () => _cache.Clear();

sendSms(student); // invoke

🟡 Predicate<T>

Returns bool. Same as Func<T, bool>.

// Predicate<T> → bool
Predicate<Student> isEligible
    = s => s.Attendance >= 75
        && !s.HasFeesDue;

// Same as Func<Student, bool>
Func<Student, bool> isActive
    = s => s.IsActive;

// Used in List.Find / Where
var eligible = students
    .FindAll(isEligible);

bool ok = isEligible(student);

4 Events — Notification System 🔔
🎬 96 (2018) — Ram waits for Janu's call

Ram phone vanthuchu — "event raised!" Janu ketkudaa? Subscribe! Event raise panna — subscribers (handlers) automatically execute. Publish-Subscribe pattern = Event system!

Delegate vs Event: Delegate = method pointer (anyone can invoke). Event = delegate + restrictions (only publisher can invoke, outsiders only subscribe/unsubscribe). Event = encapsulated delegate!

// ZenCampus — Student Admission Event System
public class AdmissionService
{
    // 1. Define EventArgs (data to pass with event)
    public class StudentAdmittedEventArgs : EventArgs
    {
        public Student Student { get; init; }
        public DateTime AdmittedAt { get; init; }
    }

    // 2. Declare Event (uses EventHandler delegate)
    public event EventHandler<StudentAdmittedEventArgs> StudentAdmitted;

    // 3. Raise event (publisher only)
    public void AdmitStudent(Student student)
    {
        // ... save to DB ...

        // Raise — null check (no subscribers = null)
        StudentAdmitted?.Invoke(this, new StudentAdmittedEventArgs
        {
            Student = student,
            AdmittedAt = DateTime.Now
        });
    }
}

// 4. Subscribers (multiple independent handlers)
public class SmsNotificationService
{
    public void Subscribe(AdmissionService svc)
        => svc.StudentAdmitted += OnStudentAdmitted;

    private void OnStudentAdmitted(object? sender, StudentAdmittedEventArgs e)
        => _sms.Send(e.Student.ParentMobile, "Welcome to ZenCampus!");
}

public class PortalCreationService
{
    public void Subscribe(AdmissionService svc)
        => svc.StudentAdmitted += OnStudentAdmitted;

    private void OnStudentAdmitted(object? s, StudentAdmittedEventArgs e)
        => _portalSvc.CreateLogin(e.Student); // completely independent!
}
🏫 ZenCampus Flow — New Student Admitted:

AdmissionService.AdmitStudent() → raises StudentAdmitted event →
SMS Service handler → sends welcome SMS
Portal Service handler → creates student login
Teacher Service handler → notifies class teacher
All independent, all decoupled, one event!

Event Memory Leak — Unsubscribe pannaa vendam!

// ❌ Leak — subscribe panni forget pannitaa
admissionSvc.StudentAdmitted += OnStudentAdmitted;
// SmsService disposed aana-um, publisher holds reference → GC cannot collect!

// ✅ Fix — implement IDisposable
public class SmsNotificationService : IDisposable
{
    private readonly AdmissionService _svc;

    public SmsNotificationService(AdmissionService svc)
    {
        _svc = svc;
        _svc.StudentAdmitted += OnStudentAdmitted;
    }

    public void Dispose()
        => _svc.StudentAdmitted -= OnStudentAdmitted; // unsubscribe!
}
5 Real Patterns — ZenCampus Usage

Pattern 1: Strategy Pattern with Func — Fee Calculation

🎬 Anniyan (2005) — Multiple personalities, same body

Ambi/Anniyan/Remo — same person, different behavior. Strategy pattern — same interface, different algorithm. Fee calculation — Term/Bus/Hostel/Scholarship — each different, swappable!

public class FeeService
{
    // Dictionary of strategies — type → calculator
    private readonly Dictionary<string, Func<Student, decimal>> _strategies = new()
    {
        ["Term"]        = s => s.Grade * 500m,
        ["Bus"]         = s => s.Distance * 20m,
        ["Hostel"]      = s => 3000m,
        ["Scholarship"] = s => -(s.ScholarshipAmount),
    };

    public decimal Calculate(Student student, string feeType)
    {
        if (!_strategies.TryGetValue(feeType, out var strategy))
            throw new ArgumentException($"Unknown fee type: {feeType}");
        return strategy(student); // invoke the strategy!
    }

    // Add new strategy at runtime — no code change needed!
    public void AddStrategy(string type, Func<Student, decimal> calc)
        => _strategies[type] = calc;
}

Pattern 2: Pipeline with Action — Admission Processing

public class AdmissionPipeline
{
    private readonly List<Action<Student>> _steps = new();

    public AdmissionPipeline AddStep(Action<Student> step)
    {
        _steps.Add(step);
        return this; // fluent!
    }

    public void Execute(Student student)
        => _steps.ForEach(step => step(student));
}

// Usage — fluent chain!
var pipeline = new AdmissionPipeline()
    .AddStep(s => _validationSvc.Validate(s))
    .AddStep(s => _repo.Save(s))
    .AddStep(s => _sms.SendWelcome(s))
    .AddStep(s => _portal.CreateLogin(s));

pipeline.Execute(newStudent);

// Lazy DB connection — only create when first needed
public class ReportService
{

    public ReportService(Func<IDbConnection> connectionFactory)
        => _connectionFactory = connectionFactory;

    public void GenerateReport()
    {

        // use conn...
    }
}

// DI registration
services.AddScoped(sp => new ReportService(

2
New Fee Type Add Panna — Open/Closed with Func Easy
Topic: Strategy pattern with dictionary of Funcs
📋 Scenario: ZenCampus-la new "Lab Fee" add pannanum. Current code-la big if-else chain irukku: 
public decimal GetFee(Student s, string type)
{
    if (type == "Term") return s.Grade * 500m;
    if (type == "Bus") return s.Distance * 20m;
    if (type == "Hostel") return 3000m;
    throw new Exception("Unknown");
    // New type add panna → this method MODIFY pannanum! (violates OCP)
}
Refactor panni SOLID Open/Closed follow panna code ezhudanum.
✅ ANSWER — Dictionary of Funcs (OCP compliant)
public class FeeService
{
    private readonly Dictionary<string, Func<Student, decimal>> _calculators;

    public FeeService()
    {
        _calculators = new()
        {
            ["Term"]    = s => s.Grade * 500m,
            ["Bus"]     = s => s.Distance * 20m,
            ["Hostel"]  = s => 3000m,
        };
    }

    // Register new type — NO existing code change!
    public void Register(string type, Func<Student, decimal> calc)
        => _calculators[type] = calc;

    public decimal GetFee(Student s, string type)
        => _calculators.TryGetValue(type, out var calc)
            ? calc(s)
            : throw new ArgumentException($"Unknown fee: {type}");
}

// Startup-la register new type — zero existing code change!
feeService.Register("Lab", s => s.HasLabSubject ? 500m : 0m);
⭐ SOLID Connection

Open/Closed Principle: "Open for extension, closed for modification." Dictionary of Funcs = extension without modification. Interview-la SOLID + Delegate = double marks!

3
Closure Trap — Capture Value vs Reference Hard
Topic: Lambda closure — classic interview trick
📋 Scenario: ZenCampus developer oru bug report pannuran — "5 buttons create pannom, each click different student ID show pannanum — aaana ellaa buttons-um last student ID show pannuthu!" Code: 
var students = new[] { 101, 102, 103, 104, 105 };
var handlers = new List<Action>();

foreach (var id in students)
    handlers.Add(() => Console.WriteLine($"Student: {id}"));

handlers.ForEach(h => h());
// Expected: 101, 102, 103, 104, 105
// Actual: ???
Output enna varum? Bug enna? Fix pannu.
✅ ANSWER

Actual Output: 105, 105, 105, 105, 105

Why? foreach loop-la id = same variable, value changes each iteration. Lambda captures the variable reference, not the value. All 5 lambdas point to same id. Loop ends → id = 105. All lambdas print 105!

Note: foreach with var in C# 5+ actually creates new variable per iteration — so this may print 101-105 correctly in modern C#. But with for(int i=0...) it's definitely a trap!

// Guaranteed trap with for loop:
for (int i = 0; i < students.Length; i++)
    handlers.Add(() => Console.WriteLine(students[i])); // captures i ref!
// All print students[5] = IndexOutOfRange! OR last value

// ✅ Fix — capture local copy
for (int i = 0; i < students.Length; i++)
{
    int localI = i;  // new variable each iteration!
    handlers.Add(() => Console.WriteLine(students[localI]));
}
4
Event Memory Leak — Object Not GC'd Hard
Topic: Event subscription leak — senior level
📋 Scenario: ZenCampus running for 6 months — memory slowly growing. Profiler-la pathuchu — old AdmissionService objects GC aagala! Code: 
public class SmsDashboard
{
    public SmsDashboard(AdmissionService svc)
    {
        svc.StudentAdmitted += OnAdmitted; // subscribe
    }
    private void OnAdmitted(object? s, EventArgs e) { }
    // ← No unsubscribe!
}
Root cause enna? Fix pannu.
✅ ROOT CAUSE + FIX

Memory Leak Cause:

Event subscription = publisher holds reference to subscriber. AdmissionService.StudentAdmitted holds reference to SmsDashboard.OnAdmitted. SmsDashboard "out of scope" aana-um — publisher still holds it → GC cannot collect → memory leak!

// ✅ Fix — IDisposable pattern
public class SmsDashboard : IDisposable
{
    private readonly AdmissionService _svc;
    private bool _disposed;

    public SmsDashboard(AdmissionService svc)
    {
        _svc = svc;
        _svc.StudentAdmitted += OnAdmitted;
    }

    public void Dispose()
    {
        if (!_disposed)
        {
            _svc.StudentAdmitted -= OnAdmitted; // unsubscribe!
            _disposed = true;
        }
    }

    private void OnAdmitted(object? s, EventArgs e) { }
}

// Usage
using var dashboard = new SmsDashboard(admissionSvc);
// using block ends → Dispose() → unsubscribe → GC can collect!
⭐ WeakReference alternative

WeakEventManager (WPF) or WeakReference pattern — publisher holds weak reference to subscriber. GC can collect subscriber even if subscribed. Advanced — mention in interview for bonus points!

// One method — Func<Student, bool> as filter!
public List<Student> GetStudents(

{
    var query = _students.AsQueryable();

Null Delegate Invoke — Safe Pattern Easy
Topic: NullReferenceException when no subscribers
📋 Scenario: ZenCampus StudentAdmitted event raise panurom — subscriber illai-na crash aaguthu. Production-la NullReferenceException! Bug fix pannu + safe pattern explain.
✅ ANSWER
public event EventHandler<StudentAdmittedEventArgs> StudentAdmitted;

// ❌ Crash if no subscribers!
StudentAdmitted(this, args); // NullReferenceException!

// ✅ Option 1 — Null check (old style)
if (StudentAdmitted != null)
    StudentAdmitted(this, args);

// ✅ Option 2 — Null conditional (modern, thread-safe)
StudentAdmitted?.Invoke(this, args); // ← Best! Use this always

// ✅ Option 3 — Default null pattern
public event EventHandler<StudentAdmittedEventArgs> StudentAdmitted
    = delegate { }; // always has at least one no-op handler
StudentAdmitted(this, args); // never null — safe!

// Protected virtual raise method — subclass can override
protected virtual void OnStudentAdmitted(StudentAdmittedEventArgs e)
    => StudentAdmitted?.Invoke(this, e);

7 ⚡ Quick Quiz

Q1: Func<int, string, bool> — how many input params? Return type?

A) 3 inputs, no return
B) 2 inputs (int, string), returns bool
C) 1 input, returns bool
D) 3 inputs, returns bool
✅ Func — last type param = return type. Func<int, string, bool> = takes int + string, returns bool. "Last is return" — remember!
❌ Func — LAST type param is return. Func<int, string, bool> = int input, string input, bool return. Count inputs = total params - 1!

Q2: Action<string> vs Func<string> — difference?

A) Action takes string param, returns void. Func takes no param, returns string.
B) Both same
C) Action returns string, Func returns void
D) Action = async, Func = sync
✅ Action<T> = takes T as input, returns void. Func<T> = no input, returns T. Key: Action = void always, Func = has return!
❌ Action<string> = input is string, void return. Func<string> = no input, returns string. Action = void always. Func = has return type always!

Q3: Event vs Delegate — main protection difference?

A) No difference — same thing
B) Event restricts invoke — only declaring class can invoke. Delegate anyone can invoke.
C) Event is faster than delegate
D) Delegate supports multiple subscribers, Event doesn't
✅ Event = encapsulated delegate. Outside code can only += (subscribe) or -= (unsubscribe). Cannot Invoke() directly. Publisher controls when to fire!
❌ Key difference: event = restricted delegate. Only publisher class can invoke. Outside world = subscribe/unsubscribe only. Encapsulation!

Q4: Multicast delegate with return value — what happens when 3 methods return different values?

A) Exception thrown
B) All values returned as array
C) Only last method's return value kept
D) First method's return value kept
✅ Multicast with return = only LAST method's value! Others silently discarded. That's why multicast delegates should use void return. Use GetInvocationList() to capture all!
❌ Last one wins! All methods execute but only last return value survives. Multicast = void return recommended. GetInvocationList() for all results.
