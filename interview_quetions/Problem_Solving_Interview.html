Palindrome Check — "madam", "racecar", "ZenCampus"?
Easy
🎯 What interviewer really tests
String manipulation, loop logic, two-pointer technique, ability to handle edge cases (null, spaces, case). Senior answer = give 3 methods + explain trade-offs!
Method 1 — Two Pointer (Best for interviews — O(n/2))
✅ Method 1: Two Pointer — Most Efficient + Explains Well
public static bool IsPalindrome(string s)
{
    if (string.IsNullOrEmpty(s)) return true;

    s = s.ToLower().Replace(" ", ""); // normalize

    int left  = 0;
    int right = s.Length - 1;

    while (left < right)
    {
        if (s[left] != s[right])
            return false;
        left++;
        right--;
    }
    return true;
}

// Test
IsPalindrome("madam")   // true
IsPalindrome("racecar") // true
IsPalindrome("hello")   // false
IsPalindrome("A man a plan a canal Panama") // true (with normalize)
Time: O(n)
Space: O(1)
Best for: Large strings, memory sensitive
Method 2 — Reverse and Compare (One-liner, clean)
Method 2: Reverse Compare — Clean, Readable
public static bool IsPalindromeReverse(string s)
{
    s = s.ToLower().Replace(" ", "");
    string reversed = new string(s.Reverse().ToArray());
    return s == reversed;
}

// One-liner version (LINQ)
public static bool IsPalindromeLinq(string s) =>
    s.SequenceEqual(s.Reverse());
Time: O(n)
Space: O(n) — creates reversed string
Best for: Code readability, short strings
Method 3 — StringBuilder (Interview shows you know StringBuilder)
Method 3: Using StringBuilder — Demonstrates knowledge
public static bool IsPalindromeSB(string s)
{
    s = s.ToLower().Replace(" ", "");
    var sb = new StringBuilder();

    for (int i = s.Length - 1; i >= 0; i--)
        sb.Append(s[i]);

    return s == sb.ToString();
}
🎤 Senior Dev Interview Answer (speak this!):
"I'd use the Two-Pointer approach — O(n) time, O(1) space. Two pointers, one from each end, move towards center comparing characters. If mismatch found, return false immediately — short-circuits, no need to check full string. I'd also normalise — toLower and strip spaces — to handle 'A man a plan a canal Panama' type inputs. In production ZenCampus code, I'd add a null check upfront. The LINQ reverse approach is cleaner for readability but uses O(n) extra space for the reversed copy."

Reverse a String — without built-in Reverse()
Easy
Method 1 — Two Pointer Swap (Most asked!)
✅ Method 1: Two Pointer In-place Swap
public static string ReverseString(string s)
{
    char[] chars = s.ToCharArray();
    int left  = 0;
    int right = chars.Length - 1;

    while (left < right)
    {
        // swap
        (chars[left], chars[right]) =
            (chars[right], chars[left]);
        left++;
        right--;
    }
    return new string(chars);
}
// "ZenCampus" → "supmaCreZ"
Time: O(n/2) = O(n)
Space: O(n) — char array
Method 2 — StringBuilder backward loop
Method 2: StringBuilder Loop
public static string ReverseWithSB(string s)
{
    var sb = new StringBuilder(s.Length);
    for (int i = s.Length - 1; i >= 0; i--)
        sb.Append(s[i]);
    return sb.ToString();
}
Method 3 — Recursion (shows depth)
Method 3: Recursive
public static string ReverseRecursive(string s)
{
    if (s.Length <= 1) return s;

    return ReverseRecursive(s[1..])   // tail
           + s[0];                          // + first char
}
// "abc" → Reverse("bc") + 'a'
//       → Reverse("c") + 'b' + 'a'
//       → "c" + "b" + "a" = "cba"
Time: O(n²) — string concat each level
Space: O(n) call stack
Note: Conceptually clean, not production ideal
Bonus — Reverse words in a sentence
Bonus: Reverse Words (common follow-up!)
// "ZenCampus School ERP" → "ERP School ZenCampus"
public static string ReverseWords(string sentence)
{
    return string.Join(" ",
        sentence.Split(' ',
            StringSplitOptions.RemoveEmptyEntries)
        .Reverse());
}
🎤 Senior Dev Interview Answer:
"Strings in C# are immutable, so I convert to char array first, then do an in-place two-pointer swap — left and right indices walking toward the center. This is O(n/2) swaps, O(n) space for the array. I'd mention that built-in Array.Reverse(chars) does the same, but if they say 'without built-in', I use the manual swap. For reversing words in a sentence, Split(' ').Reverse().Join(' ') is the cleanest production approach. I'd also validate null/empty upfront."

Anagram Check — "listen" and "silent", "triangle" and "integral"
Easy
💡 Concept
Two strings are anagrams if they contain the SAME characters with the SAME frequency. "listen" = "silent" — same 6 chars, same counts!
Method 1 — Sort and Compare
✅ Method 1: Sort Both Strings and Compare
public static bool IsAnagram(string s1, string s2)
{
    if (s1.Length != s2.Length) return false; // quick exit

    var a1 = s1.ToLower().ToCharArray();
    var a2 = s2.ToLower().ToCharArray();

    Array.Sort(a1);
    Array.Sort(a2);

    return a1.SequenceEqual(a2);
}
// "listen" sorted = "eilnst"
// "silent" sorted = "eilnst" ✅ → true
Time: O(n log n) — sorting
Space: O(n)
Method 2 — Dictionary Frequency Count (Best O(n))
✅ Method 2: Character Frequency Map — Optimal O(n)
public static bool IsAnagramDict(string s1, string s2)
{
    if (s1.Length != s2.Length) return false;

    var freq = new Dictionary<char, int>();

    foreach (char c in s1.ToLower())
        freq[c] = freq.GetValueOrDefault(c) + 1;

    foreach (char c in s2.ToLower())
    {
        if (!freq.ContainsKey(c) || freq[c] == 0)
            return false;
        freq[c]--;
    }
    return true;
}
// "listen" builds: {l:1,i:1,s:1,t:1,e:1,n:1}
// "silent" decrements each — all reach 0 → anagram!
Time: O(n) — two passes
Space: O(1) — max 26 chars
Best: Preferred in interviews!
🎤 Senior Dev Answer:
"Two approaches: Sort-and-compare is O(n log n) — easy to explain. Better is character frequency dictionary — O(n). Build frequency map from s1, then for each char in s2 decrement — if any char missing or count goes negative, not anagram. O(n) time, O(1) space since only 26 possible characters. I'd also do quick-fail on length mismatch first — saves processing."

Count Vowels
public static int CountVowels(string s)
{
    const string vowels = "aeiouAEIOU";
    return s.Count(c => vowels.Contains(c));
}
// "ZenCampus" → 3 (e, a, u)
Most Frequent Character
public static char MostFrequent(string s)
{
    return s.ToLower()
             .GroupBy(c => c)
             .OrderByDescending(g => g.Count())
             .First().Key;
}
// "aabbcca" → 'a' (appears 3 times)
Remove Duplicate Characters (keep first occurrence)
public static string RemoveDuplicateChars(string s)
{
    var seen = new HashSet<char>();
    var sb   = new StringBuilder();

    foreach (char c in s)
        if (seen.Add(c))  // Add returns false if exists
            sb.Append(c);

    return sb.ToString();
}
// "ZenCampus" → "ZenCampus" (all unique here)
// "aabbcc"   → "abc"

Fibonacci Series — Print first N terms
Easy
💡 Fibonacci
0, 1, 1, 2, 3, 5, 8, 13, 21... Each number = sum of previous two. F(0)=0, F(1)=1, F(n) = F(n-1) + F(n-2)
Method 1 — Iterative (Best!)
✅ Method 1: Iterative — O(n) time, O(1) space
public static void PrintFibonacci(int n)
{
    int a = 0, b = 1;

    for (int i = 0; i < n; i++)
    {
        Console.Write($"{a} ");
        (a, b) = (b, a + b); // tuple swap!
    }
}
// n=8 → 0 1 1 2 3 5 8 13

// Return as List
public static List<int> GetFibonacci(int n)
{
    var result = new List<int>();
    int a = 0, b = 1;
    for (int i = 0; i < n; i++, (a, b) = (b, a + b))
        result.Add(a);
    return result;
}
Method 2 — Recursive (shows recursion, but slow!)
Method 2: Recursive — O(2ⁿ) — Demo only, never production
public static int FibRecursive(int n)
{
    if (n <= 1) return n;
    return FibRecursive(n - 1) + FibRecursive(n - 2);
}
// F(40) = ~2 billion calls! Exponential growth
Time: O(2ⁿ) — exponential, bad!
Space: O(n) — call stack
Method 3 — Memoization (Recursive + Cache = DP)
✅ Method 3: Memoization — O(n) with cache (Shows DP knowledge!)
private static Dictionary<int, long> _memo = new();

public static long FibMemo(int n)
{
    if (n <= 1) return n;
    if (_memo.TryGetValue(n, out var cached))
        return cached;

    var result = FibMemo(n - 1) + FibMemo(n - 2);
    _memo[n] = result;
    return result;
}
// F(100) calculated in microseconds — each sub-result cached!
Time: O(n) — each n computed once
Space: O(n) — memo dict
🎤 Senior Dev Answer:
"I present 3 approaches. Iterative is O(n) time O(1) space — best for production. Recursive is clean to understand but O(2ⁿ) — Fib(40) takes seconds! Memoization adds a dictionary cache to recursive — each value computed once, O(n) time and space. In an interview, I'd code iterative, then say 'if they want large values, I'd use memoization or the iterative is already optimal.' I'd also use long instead of int — Fib(93) overflows int!"

Factorial — n! = 1 × 2 × 3 × ... × n
Easy
✅ Iterative + Recursive + Validation
// Iterative — O(n) time, O(1) space
public static long FactorialIterative(int n)
{
    if (n < 0) throw new ArgumentException("n must be ≥ 0");
    long result = 1;
    for (int i = 2; i <= n; i++)
        result *= i;
    return result;
}
// 5! = 120,  10! = 3628800

// Recursive — elegant but O(n) stack
public static long FactorialRecursive(int n)
    => n <= 1 ? 1 : n * FactorialRecursive(n - 1);

// LINQ one-liner (shows language knowledge)
public static long FactorialLinq(int n)
    => Enumerable.Range(1, n)
                  .Aggregate(1L, (acc, x) => acc * x);
🎤 Key point to mention:
"0! = 1 by definition. I'd use long not int — 13! overflows int (2147483647). For very large factorials, I'd use BigInteger. Iterative is preferred over recursive to avoid StackOverflowException for large n."

Prime Number Check + Print all primes up to N (Sieve)
Medium
✅ IsPrime — O(√n) optimization
public static bool IsPrime(int n)
{
    if (n < 2) return false;
    if (n == 2) return true;
    if (n % 2 == 0) return false; // even > 2 = not prime

    // Check up to √n only — if n=100, check up to 10
    for (int i = 3; i * i <= n; i += 2) // skip evens
        if (n % i == 0) return false;

    return true;
}
// IsPrime(17)  → true
// IsPrime(100) → false (100 = 10×10)
Time: O(√n) — why? If n has factor > √n, its pair must be < √n
✅ Sieve of Eratosthenes — All primes up to N (Very impressive!)
public static List<int> SieveOfEratosthenes(int limit)
{
    var isPrime = new bool[limit + 1];
    Array.Fill(isPrime, true);
    isPrime[0] = isPrime[1] = false;

    for (int i = 2; i * i <= limit; i++)
        if (isPrime[i])
            for (int j = i * i; j <= limit; j += i)
                isPrime[j] = false; // mark multiples

    return Enumerable.Range(2, limit - 1)
                      .Where(i => isPrime[i])
                      .ToList();
}
// limit=20 → [2,3,5,7,11,13,17,19]
Time: O(n log log n)
Space: O(n)
Best: Generate ALL primes up to n efficiently
🎤 Senior Dev Answer — Why √n?
"If n is NOT prime, it must have a factor ≤ √n. Because if both factors are >√n, their product would exceed n. So checking up to √n is sufficient. For Sieve — we start marking multiples from i*i (not 2*i) because smaller multiples are already marked by previous primes. This is the most efficient bulk prime generator."

Armstrong Number — 153 = 1³ + 5³ + 3³
Easy 
public static bool IsArmstrong(int n)
{
    string digits = n.ToString();
    int    power  = digits.Length;  // 3 for 153

    int sum = digits
        .Sum(c => (int)Math.Pow(c - '0', power));

    return sum == n;
}
// 153: 1³+5³+3³ = 1+125+27 = 153 ✅
// 370: 3³+7³+0³ = 27+343+0 = 370 ✅
// 371: 3³+7³+1³ = 27+343+1 = 371 ✅
// 407: 4³+0³+7³ = 64+0+343 = 407 ✅

// Print all 3-digit Armstrong numbers
Enumerable.Range(100, 900)
           .Where(IsArmstrong)
           .ToList()
           .ForEach(Console.WriteLine);
// → 153, 370, 371, 407

Find Duplicates in Array — [1,2,3,2,4,3] → [2,3]
Medium
✅ Method 1: HashSet O(n) — Best!
public static List<int> FindDuplicates(int[] arr)
{
    var seen       = new HashSet<int>();
    var duplicates = new HashSet<int>(); // avoid duplicate dups

    foreach (var num in arr)
        if (!seen.Add(num))   // Add returns false = already exists
            duplicates.Add(num);

    return duplicates.ToList();
}
// [1,2,3,2,4,3] → [2,3]
Time: O(n)
Space: O(n)
Method 2: LINQ GroupBy
public static IEnumerable<int> FindDuplicatesLinq(int[] arr)
    => arr.GroupBy(x => x)
           .Where(g => g.Count() > 1)
           .Select(g => g.Key);
🎤 ZenCampus context:
"Same logic applies for finding duplicate roll numbers in a class — HashSet.Add() returns false if already present. For DB-scale duplicates, SQL GROUP BY HAVING COUNT > 1 is more appropriate."

Find Missing Number in 1 to N — [1,2,4,5] → 3
Easy
✅ Method 1: Sum Formula — O(n), O(1) space!
public static int FindMissing(int[] arr, int n)
{
    // Sum of 1..n = n*(n+1)/2
    int expected = n * (n + 1) / 2;
    int actual   = arr.Sum();
    return expected - actual;
}
// n=5, arr=[1,2,4,5]
// expected = 5*6/2 = 15
// actual   = 1+2+4+5 = 12
// missing  = 15-12 = 3 ✅
Time: O(n)
Space: O(1) — no extra structure!
Method 2: XOR (Bit manipulation — Very impressive!)
public static int FindMissingXOR(int[] arr, int n)
{
    int xorFull  = 0;
    int xorArray = 0;

    for (int i = 1; i <= n; i++) xorFull  ^= i;
    foreach (var x in arr)           xorArray ^= x;

    return xorFull ^ xorArray;
    // a ^ a = 0, so matching numbers cancel out
    // Only the missing number remains!
}
🎤 Senior Dev Answer:
"Two approaches: Sum formula is O(n) O(1) — mathematical, elegant. Expected sum - actual sum = missing value. XOR approach is even more elegant — XOR of 1..n with all array elements cancels matched pairs, leaving only the missing number. Both are O(n) time O(1) space. I'd use sum formula in production for clarity, XOR for showing CS fundamentals in interviews."

Two Sum — Find pair that adds up to target
Medium
Method 1: Brute Force O(n²) — Mention but don't code
// O(n²) — nested loop checking all pairs
// arr=[2,7,11,15], target=9 → [0,1] (2+7=9)
// DON'T do this in interview — mention and improve!
✅ Method 2: HashMap O(n) — This is what they want!
public static (int, int) TwoSum(int[] nums, int target)
{
    var seen = new Dictionary<int, int>();
    // key=value, value=index

    for (int i = 0; i < nums.Length; i++)
    {
        int complement = target - nums[i];

        if (seen.TryGetValue(complement, out int j))
            return (j, i);             // found!

        seen[nums[i]] = i;
    }
    return (-1, -1);                // not found
}

// nums=[2,7,11,15], target=9
// i=0: complement=7, not in seen. Add 2→0
// i=1: complement=2, found at index 0! → (0,1) ✅
Time: O(n) — single pass
Space: O(n) — dict
🎤 Senior Dev Answer:
"Classic problem. Brute force is O(n²) — two nested loops. Optimal is HashMap — for each number, calculate complement (target - current), check if complement already seen. One pass, O(n) time. The key insight: instead of looking forward for a pair, look backward using a hash map — 'have I seen the number that would complete this pair?' This is the core pattern for many problems."

FizzBuzz — divisible by 3=Fizz, 5=Buzz, both=FizzBuzz
Easy
✅ Clean, Extendable Solution
public static IEnumerable<string> FizzBuzz(int n)
{
    for (int i = 1; i <= n; i++)
    {
        var result = string.Empty;
        if (i % 3 == 0) result += "Fizz";
        if (i % 5 == 0) result += "Buzz";
        yield return string.IsNullOrEmpty(result)
            ? i.ToString()
            : result;
    }
}
// 1,2,Fizz,4,Buzz,Fizz,7,8,Fizz,Buzz,
// 11,Fizz,13,14,FizzBuzz...
🎤 Why this approach is senior-level:
"I check 3 and 5 SEPARATELY and concatenate — handles FizzBuzz automatically without a special 'divisible by 15' case. It's extendable: add 7='Bazz' by just adding one more if-block. I also use yield return for lazy evaluation — works for large n without building the full list in memory. A junior would write if/else if/else if — mine is O(1) to add new rules."

Second Highest Salary — 6 Different Methods!
Medium
📋 Table: Employees (Id, Name, Salary, DeptId)
Data: 90000, 80000, 70000, 80000 → Second Highest = 80000
Method 1 — OFFSET FETCH (SQL Server — Cleanest!)
✅ Method 1: OFFSET FETCH — Modern SQL Server way
SELECT DISTINCT Salary
FROM   Employees
ORDER BY Salary DESC
OFFSET 1 ROWS FETCH NEXT 1 ROWS ONLY;

-- Skip 1 row (highest), take 1 (second highest)
-- DISTINCT handles duplicate salaries
-- Change OFFSET 1 → OFFSET 2 for 3rd highest!
Method 2 — MAX with NOT IN (Most commonly asked!)
✅ Method 2: MAX excluding the highest
SELECT MAX(Salary) AS SecondHighest
FROM   Employees
WHERE  Salary < (SELECT MAX(Salary) FROM Employees);

-- MAX of all salaries that are LESS than the overall MAX
-- Inner query: finds 90000
-- Outer query: MAX of {80000,70000,80000} = 80000 ✅
Method 3 — DENSE_RANK Window Function (Best for Nth highest!)
✅ Method 3: DENSE_RANK — Works for ANY Nth!
WITH RankedSalaries AS (
    SELECT Salary,
           DENSE_RANK() OVER (ORDER BY Salary DESC)
               AS Rnk
    FROM   Employees
)
SELECT DISTINCT Salary
FROM   RankedSalaries
WHERE  Rnk = 2;   -- change to 3 for 3rd highest!

-- DENSE_RANK: 90000→1, 80000→2, 70000→3
-- (two 80000s both get rank 2 — DENSE_RANK!)
-- WHERE Rnk=2 → 80000 ✅
Best for: Any Nth highest — just change the number
Handles: Duplicate salaries correctly
Method 4 — NOT IN Subquery
Method 4: NOT IN — Easy to understand
SELECT MAX(Salary) AS SecondHighest
FROM   Employees
WHERE  Salary NOT IN (
    SELECT MAX(Salary) FROM Employees
);
Method 5 — TOP with TOP (SQL Server specific)
Method 5: TOP 1 from TOP 2 (Classic SQL Server)
SELECT TOP 1 Salary
FROM (
    SELECT DISTINCT TOP 2 Salary
    FROM   Employees
    ORDER BY Salary DESC
) AS Top2
ORDER BY Salary ASC;

-- Inner: TOP 2 DESC → {90000, 80000}
-- Outer: TOP 1 ASC  → 80000 (smallest of top 2)
Method 6 — NULL safe (returns NULL if no 2nd highest)
✅ Method 6: NULL-safe — Production ready!
SELECT ISNULL(
    (SELECT DISTINCT TOP 1 Salary
     FROM (
         SELECT DISTINCT TOP 2 Salary
         FROM   Employees
         ORDER BY Salary DESC
     ) AS T
     ORDER BY Salary ASC),
    NULL
) AS SecondHighest;

-- If only one unique salary exists → returns NULL
-- Interview trick: always handle edge cases!
🎤 Senior Dev Interview Answer (FULL answer):
"I know 5+ approaches. Let me list them with trade-offs:
1. OFFSET FETCH — cleanest, SQL Server 2012+, O(n log n)
2. MAX subquery — simplest, widely understood
3. DENSE_RANK — most flexible, works for ANY nth salary, handles duplicates correctly. ROW_NUMBER would fail with duplicate salaries — two 80000s would get different ranks!
4. TOP 1 of TOP 2 — classic SQL Server pattern
In production, I'd use DENSE_RANK because it's one parameter change from 2nd to Nth, handles duplicate salaries, and is readable. I'd also add NULL safety in case there's only one unique salary."

Nth Highest Salary — Generic Stored Procedure
Medium
✅ Stored Procedure for Nth Highest (Very Impressive!)
CREATE PROCEDURE usp_GetNthHighestSalary
    @N INT
AS
BEGIN
    WITH RankedSalaries AS (
        SELECT Salary,
               DENSE_RANK() OVER (
                   ORDER BY Salary DESC
               ) AS Rnk
        FROM   Employees
    )
    SELECT DISTINCT
           ISNULL(
               (SELECT DISTINCT Salary
                FROM  RankedSalaries
                WHERE Rnk = @N),
               NULL
           ) AS NthHighestSalary;
END;

-- Usage:
-- EXEC usp_GetNthHighestSalary @N = 1  → 90000
-- EXEC usp_GetNthHighestSalary @N = 2  → 80000
-- EXEC usp_GetNthHighestSalary @N = 99 → NULL

Find Duplicate Records — Students with same RollNo
Easy
✅ Method 1: GROUP BY HAVING COUNT > 1
-- Find duplicate email IDs
SELECT Email, COUNT(*) AS Cnt
FROM   Students
GROUP BY Email
HAVING   COUNT(*) > 1
ORDER BY Cnt DESC;
✅ Method 2: Show FULL duplicate rows (with ROW_NUMBER)
WITH Dupes AS (
    SELECT *,
           ROW_NUMBER() OVER (
               PARTITION BY RollNo, SchoolId
               ORDER BY     Id
           ) AS Rn
    FROM   Students
)
SELECT * FROM Dupes WHERE Rn > 1;

-- Rn=1 = first occurrence (keep)
-- Rn>1 = duplicates (can then DELETE these!)

-- DELETE duplicates keeping first occurrence:
WITH Dupes AS (
    SELECT *,
           ROW_NUMBER() OVER (
               PARTITION BY RollNo, SchoolId
               ORDER BY     Id
           ) AS Rn
    FROM   Students
)
DELETE FROM Dupes WHERE Rn > 1;
🎤 Senior Dev Answer:
"GROUP BY HAVING COUNT > 1 finds which values are duplicated. But to see full rows or DELETE duplicates, ROW_NUMBER OVER PARTITION BY is the correct approach. Partition by the duplicate-key columns, order by Id (to keep the oldest/first). Rn=1 = keep, Rn>1 = duplicate to delete. ZenCampus: same roll number admitted twice in the same school — this query finds and cleans it."

✅ Method 1: CTE with DENSE_RANK per Dept
WITH DeptRanked AS (
    SELECT e.Name, e.Salary,
           d.DeptName,
           DENSE_RANK() OVER (
               PARTITION BY e.DeptId
               ORDER BY     e.Salary DESC
           ) AS Rnk
    FROM   Employees  e
    JOIN   Departments d
        ON d.Id = e.DeptId
)
SELECT DeptName, Name, Salary
FROM   DeptRanked
WHERE  Rnk = 1;   -- top earner per dept (ties included!)

Running Total (Cumulative Sum) — Fee Collection Month-wise
Medium
✅ SUM OVER (ORDER BY) — Running total with window function
-- Monthly fee collection with running total
SELECT
    FORMAT(PaidDate, 'yyyy-MM')  AS Month,
    SUM(Amount)                   AS MonthlyTotal,
    SUM(SUM(Amount)) OVER (
        ORDER BY FORMAT(PaidDate, 'yyyy-MM')
    )                              AS RunningTotal
FROM   Fees
WHERE  IsPaid = 1
GROUP BY FORMAT(PaidDate, 'yyyy-MM')
ORDER BY Month;

-- Output:
-- Month     MonthlyTotal  RunningTotal
-- 2026-01   50000         50000
-- 2026-02   75000         125000
-- 2026-03   60000         185000
🎤 Senior Dev Answer:
"SUM() OVER (ORDER BY) is a window aggregate — it calculates a cumulative sum sorted by the ORDER BY column. The inner SUM(Amount) is the GROUP BY aggregate per month. The outer SUM(SUM(Amount)) OVER ORDER BY adds each month's total cumulatively. ZenCampus uses this for financial dashboards — showing total fee collection growth across the academic year."

Students WITHOUT Fee Records — LEFT JOIN Anti-pattern
Easy
✅ Method 1: LEFT JOIN + WHERE NULL (Anti-Join) — Fastest!
SELECT s.Id, s.Name, s.Class
FROM   Students  s
LEFT JOIN Fees  f
    ON f.StudentId = s.Id
WHERE  f.Id IS NULL  -- no matching fee record!
ORDER BY s.Name;

-- LEFT JOIN returns all students
-- WHERE f.Id IS NULL → only students with NO fees
Method 2: NOT EXISTS (Also good — short-circuits)
SELECT Id, Name, Class
FROM   Students s
WHERE NOT EXISTS (
    SELECT 1
    FROM   Fees
    WHERE  StudentId = s.Id
);

-- NOT EXISTS stops as soon as it finds one match
-- SELECT 1 is intentional — we don't need columns
Method 3: NOT IN (Careful with NULLs!)
SELECT Id, Name
FROM   Students
WHERE  Id NOT IN (
    SELECT DISTINCT StudentId
    FROM   Fees
    WHERE  StudentId IS NOT NULL -- CRITICAL!
);

-- If Fees.StudentId has ANY NULL → NOT IN returns empty!
-- Always add WHERE StudentId IS NOT NULL in subquery
🎤 Senior Dev Answer:
"Three approaches. LEFT JOIN + WHERE NULL = 'anti-join' pattern — most efficient. NOT EXISTS is semantically clearest and short-circuits on first match. NOT IN is dangerous — if the subquery returns even one NULL, the entire result is empty (SQL three-valued logic: TRUE/FALSE/UNKNOWN). Always filter NULLs in NOT IN subqueries. ZenCampus uses anti-join to find students who haven't paid any fees this term."

Top 3 Students by Marks in Each Class
Hard
✅ DENSE_RANK PARTITION BY Class ORDER BY Marks
WITH ClassRanked AS (
    SELECT
        s.Name,
        s.Class,
        e.Marks,
        DENSE_RANK() OVER (
            PARTITION BY s.Class   -- rank WITHIN each class
            ORDER BY     e.Marks DESC
        ) AS ClassRank
    FROM    Students    s
    JOIN    ExamResults e
        ON  e.StudentId = s.Id
    WHERE   e.ExamTerm = 'Term1'
)
SELECT Class, Name, Marks, ClassRank
FROM   ClassRanked
WHERE  ClassRank <= 3
ORDER BY Class, ClassRank;

-- PARTITION BY Class → resets rank per class
-- DENSE_RANK → if two students have 95 marks, both get rank 1
--              next student gets rank 2 (not 3 like ROW_NUMBER)
-- WHERE ClassRank <= 3 → top 3 per class
-- Output:
-- Class  Name    Marks  ClassRank
-- 10A    Guru    98     1
-- 10A    Priya   95     2
-- 10A    Ram     95     2   ← same mark, same rank!
-- 10A    Meena   90     3
-- 10B    Kumar   97     1
-- ...
🎤 Senior Dev Answer:
"PARTITION BY is the key — it resets the rank counter for each class, unlike a global rank. DENSE_RANK vs ROW_NUMBER: if two students share 2nd place with same marks, DENSE_RANK gives both rank 2 and the next gets rank 3. ROW_NUMBER arbitrarily gives one student rank 2 and other rank 3 — unfair! For ZenCampus merit list, DENSE_RANK is the right choice. Change <=3 to <=5 for top 5, or PARTITION BY Class, Section for section-wise ranking."

Year-over-Year Fee Growth — LAG function
Hard
✅ LAG() Window Function — Previous year comparison
WITH YearlyFees AS (
    SELECT
        YEAR(PaidDate)   AS Yr,
        SUM(Amount)     AS TotalFees
    FROM   Fees
    WHERE  IsPaid = 1
    GROUP BY YEAR(PaidDate)
)
SELECT
    Yr,
    TotalFees,
    LAG(TotalFees) OVER (ORDER BY Yr)
        AS PrevYearFees,
    TotalFees -
        LAG(TotalFees) OVER (ORDER BY Yr)
        AS Growth,
    ROUND(
        (TotalFees - LAG(TotalFees) OVER
            (ORDER BY Yr)) * 100.0 /
        NULLIF(LAG(TotalFees) OVER
            (ORDER BY Yr), 0),
    2) AS GrowthPct
FROM   YearlyFees
ORDER BY Yr;

-- NULLIF(...,0) prevents divide-by-zero!
-- Output:
-- Yr    TotalFees  PrevYearFees  Growth   GrowthPct
-- 2024  800000     NULL          NULL     NULL
-- 2025  950000     800000        150000   18.75
-- 2026  1100000    950000        150000   15.79
🎤 Senior Dev Answer:
"LAG(column) OVER (ORDER BY date) gets the previous row's value — perfect for year-over-year comparisons. First year will have NULL (no previous year). For growth percentage, I use NULLIF(prev, 0) to avoid divide-by-zero. LEAD() does the opposite — looks at the next row. ZenCampus management dashboard uses this to show fee collection growth trends across academic years. Same pattern works for month-over-month attendance comparison."
