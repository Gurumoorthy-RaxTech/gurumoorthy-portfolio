🟢 SELECT — Core Syntax & Clauses
Syntax — Full Anatomy
SELECT Statement — Every Clause
-- Execution ORDER (not write order!):
-- FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → TOP/LIMIT

SELECT TOP 10
    s.Id,
    s.Name,
    s.Class,
    COUNT(a.Id)         AS TotalDays,
    SUM(f.Amount)       AS TotalFee,
    AVG(m.Marks)        AS AvgMarks,
    MAX(m.Marks)        AS HighestMark
FROM        Students s
INNER JOIN  Attendance a  ON a.StudentId = s.Id
LEFT  JOIN  Fees f        ON f.StudentId = s.Id
LEFT  JOIN  Marks m       ON m.StudentId = s.Id
WHERE
    s.IsActive  = 1
    AND s.SchoolId = @SchoolId
    AND s.Class    = @Class
GROUP BY
    s.Id, s.Name, s.Class
HAVING
    AVG(m.Marks) > 60
ORDER BY
    AvgMarks DESC;
❓ Interview Q: SQL execution order?
FROM → JOIN → WHERE → GROUP BY → HAVING → SELECT → DISTINCT → ORDER BY → TOP/LIMIT. WHERE-la aggregate use panna mudiyaadu (not yet computed) — HAVING use pannu. Itha therinja interview-la distinguish aagalam!
💡 WHERE vs HAVING: WHERE = row-level filter (before group). HAVING = group-level filter (after group). WHERE AVG(marks) > 60 ← ERROR! HAVING AVG(marks) > 60 ← CORRECT!
Syntax
Subqueries — Scalar · IN · EXISTS
-- Scalar subquery (returns 1 value)
SELECT
    Name,
    (SELECT SUM(Amount)
     FROM Fees
     WHERE StudentId = s.Id) AS TotalFee
FROM Students s;

-- IN subquery
SELECT * FROM Students
WHERE Id IN (
    SELECT StudentId
    FROM   Fees
    WHERE  IsPaid = 0
);

-- EXISTS (faster than IN for large sets)
SELECT * FROM Students s
WHERE EXISTS (
    SELECT 1 FROM Fees f
    WHERE f.StudentId = s.Id
    AND   f.IsPaid = 0
);

-- NOT EXISTS — students with no fees
SELECT * FROM Students s
WHERE NOT EXISTS (
    SELECT 1 FROM Fees f
    WHERE f.StudentId = s.Id
);
❓ Interview Q: IN vs EXISTS — which is faster?
EXISTS = short-circuit — first match kadaichudane stop. IN = full subquery evaluate. Large tables-la EXISTS always faster. ZenCampus unpaid fees check-la EXISTS use panrom.
Syntax
CASE WHEN · COALESCE · ISNULL · NULLIF
-- CASE WHEN — conditional column
SELECT
    Name,
    Marks,
    CASE
        WHEN Marks >= 90 THEN 'A+'
        WHEN Marks >= 75 THEN 'A'
        WHEN Marks >= 60 THEN 'B'
        ELSE                   'C'
    END AS Grade,
    CASE
        WHEN IsPresent = 1 THEN 'Present'
        ELSE                     'Absent'
    END AS Status
FROM Students;

-- COALESCE — first non-null value
SELECT
    COALESCE(Email, Phone, 'No Contact')
    AS ContactInfo
FROM Students;

-- ISNULL — SQL Server specific
SELECT ISNULL(MiddleName, '') FROM Students;

-- NULLIF — returns NULL if equal
SELECT NULLIF(Marks, 0) FROM Exams;
-- marks=0 → NULL (avoid divide by zero)

🔵 JOINs — Every Type with Visual
🎬 Kaaka Muttai (2015): Chinna Kaaka Muttai + Periya Kaaka Muttai — INNER JOIN pannaa only when both exist. LEFT JOIN = Chinna irundha matum paarunga, Periya illalum ok (NULL varum). RIGHT JOIN = Periya matum venum. FULL JOIN = both sides — nobody miss!

Syntax — Core JOINs
INNER · LEFT · RIGHT · FULL OUTER
-- INNER JOIN — only matches
SELECT s.Name, f.Amount, f.PaidDate
FROM        Students  s
INNER JOIN  Fees      f
    ON f.StudentId = s.Id;

-- LEFT JOIN — all students, fee if exists
SELECT s.Name,
       ISNULL(f.Amount, 0) AS FeeAmount
FROM       Students s
LEFT JOIN  Fees     f
    ON f.StudentId = s.Id;

-- Find students with NO fees (anti-join)
SELECT s.Name
FROM       Students s
LEFT JOIN  Fees     f
    ON f.StudentId = s.Id
WHERE f.Id IS NULL;  -- anti-join trick!

-- FULL OUTER JOIN
SELECT s.Name, f.Amount
FROM              Students s
FULL OUTER JOIN   Fees     f
    ON f.StudentId = s.Id;
❓ Interview Q: LEFT JOIN vs INNER JOIN — when to use which?
INNER = only when both sides match (student WITH fees). LEFT = all students even without fees. ZenCampus student list-la — bus fee optional students-ku LEFT JOIN use panrom — INNER use panna bus-less students miss aagiduvaan!

SELF JOIN · 3-Table JOIN · CROSS JOIN

SELECT

🟣 CTE · Window Functions · Ranking
🎬 Vikram Vedha (2017): Story layers — each layer adds new context without losing previous one. CTE = same! Name a subquery, reference it cleanly. Window functions = see each row AND the group simultaneously — like Vedha explaining each story layer!
Syntax — CTE
Common Table Expression — WITH clause
-- Basic CTE
WITH ActiveStudents AS (
    SELECT Id, Name, Class, Marks
    FROM   Students
    WHERE  IsActive = 1
)
SELECT * FROM ActiveStudents
WHERE  Class = '10A';

-- Multiple CTEs
WITH
ActiveStudents AS (
    SELECT Id, Name, Class
    FROM   Students
    WHERE  IsActive = 1
),
FeesPaid AS (
    SELECT StudentId, SUM(Amount) AS Total
    FROM   Fees
    WHERE  IsPaid = 1
    GROUP BY StudentId
)
SELECT a.Name, a.Class, fp.Total
FROM       ActiveStudents a
LEFT JOIN  FeesPaid       fp
    ON fp.StudentId = a.Id;

-- Recursive CTE — org hierarchy
WITH RECURSIVE OrgTree AS (
    -- Anchor: top-level managers
    SELECT Id, Name, ManagerId, 0 AS Level
    FROM   Staff
    WHERE  ManagerId IS NULL

    UNION ALL

    -- Recursive: subordinates
    SELECT s.Id, s.Name, s.ManagerId,
           ot.Level + 1
    FROM        Staff    s
    INNER JOIN  OrgTree  ot
        ON s.ManagerId = ot.Id
)
SELECT * FROM OrgTree ORDER BY Level;
❓ Interview Q: CTE vs Subquery — when to use CTE?
CTE = readable, reusable in same query, recursive possible. Subquery = one-time use. Same query-la CTE-a multiple places reference panalaam. ZenCampus report-la ActiveStudents CTE once define — multiple places reuse panrom.
Syntax — Window Functions
ROW_NUMBER · RANK · DENSE_RANK · NTILE
-- ROW_NUMBER — unique sequential number
SELECT
    Name, Class, Marks,
    ROW_NUMBER() OVER (
        PARTITION BY Class
        ORDER BY     Marks DESC
    ) AS RowNum
FROM Students;

-- RANK — same rank for ties, skips next
SELECT
    Name, Marks,
    RANK() OVER (
        ORDER BY Marks DESC
    ) AS Rank
FROM Students;
-- 95,95,90 → Rank 1,1,3

-- DENSE_RANK — no skips
SELECT
    Name, Marks,
    DENSE_RANK() OVER (
        ORDER BY Marks DESC
    ) AS DenseRank
FROM Students;
-- 95,95,90 → DenseRank 1,1,2

-- NTILE — divide into buckets
SELECT
    Name, Marks,
    NTILE(4) OVER (
        ORDER BY Marks DESC
    ) AS Quartile
FROM Students;
Output — RANK vs DENSE_RANK:
Name | Marks | RANK | DENSE_RANK
--------|-------|------|----------
Guru | 95 | 1 | 1
Priya | 95 | 1 | 1
Siva | 90 | 3 | 2 ← RANK skips 2, DENSE doesn't
Arun | 85 | 4 | 3
❓ Interview Q: RANK vs DENSE_RANK?
RANK = ties share same rank, next rank skipped (1,1,3). DENSE_RANK = ties share same rank, NO skip (1,1,2). School topper list = DENSE_RANK use panrom — "2nd rank" skip aagathu!
Syntax — Window Aggregates
SUM/AVG OVER · LAG · LEAD · FIRST_VALUE
-- Running total per class
SELECT
    Name, Class, FeeAmount,
    SUM(FeeAmount) OVER (
        PARTITION BY Class
        ORDER BY     PaidDate
    ) AS RunningTotal,
    AVG(Marks) OVER (
        PARTITION BY Class
    ) AS ClassAvg
FROM Students;

-- LAG — previous row value
SELECT
    Month, FeeCollected,
    LAG(FeeCollected) OVER (
        ORDER BY Month
    ) AS PrevMonth,
    FeeCollected -
    LAG(FeeCollected) OVER (
        ORDER BY Month
    ) AS MonthlyGrowth
FROM MonthlyFees;

-- LEAD — next row value
SELECT
    ExamDate, Subject,
    LEAD(ExamDate) OVER (
        ORDER BY ExamDate
    ) AS NextExamDate
FROM ExamSchedule;
🏫 ZenCampus — Top 3 per class:
WITH Ranked AS (
    SELECT Name, Class, Marks,
        DENSE_RANK() OVER (
            PARTITION BY Class
            ORDER BY     Marks DESC
        ) AS Rnk
    FROM Students
)
SELECT * FROM Ranked
WHERE Rnk <= 3;
🟠 Stored Procedures · Functions · Triggers
🎬 Baasha (1995) — Manikkam's plan: Manikkam oru plan vechu execute panruvan — once defined, call with parameters, result varum. Stored Procedure = pre-compiled SQL plan, parameters passe, result return!
Syntax
Stored Procedure — Create & Execute
-- Create stored procedure
CREATE PROCEDURE usp_GetStudentsByClass
    @SchoolId   INT,
    @ClassName  NVARCHAR(50),
    @PageNo     INT = 1,       -- default value
    @PageSize   INT = 20
AS
BEGIN
    SET NOCOUNT ON;

    DECLARE @Offset INT =
        (@PageNo - 1) * @PageSize;

    SELECT
        s.Id, s.Name, s.RollNo,
        s.Class, s.Phone
    FROM  Students s
    WHERE s.SchoolId  = @SchoolId
    AND   s.Class     = @ClassName
    AND   s.IsActive  = 1
    ORDER BY s.RollNo
    OFFSET     @Offset   ROWS
    FETCH NEXT @PageSize ROWS ONLY;
END

-- Execute
EXEC usp_GetStudentsByClass
    @SchoolId  = 1,
    @ClassName = N'10A',
    @PageNo    = 2;
🏫 Call from Dapper (C#):
var students = await _db
    .QueryAsync<Student>(
        "usp_GetStudentsByClass",
        new { SchoolId=1,
               ClassName="10A",
               PageNo=1 },
        commandType: CommandType.StoredProcedure);
❓ Interview Q: Stored Proc vs inline SQL?
SP = pre-compiled execution plan (faster 2nd run), network traffic less, security (grant EXEC only), centralised logic. Inline SQL = flexible, ORM-friendly, testable. ZenCampus complex reports-ku SP, CRUD-ku Dapper inline.
Syntax — Advanced SP
OUTPUT Param · TRY-CATCH · Transaction
CREATE PROCEDURE usp_PayFee
    @StudentId  INT,
    @Amount     DECIMAL(10,2),
    @ReceiptNo  INT OUTPUT      -- OUTPUT param
AS
BEGIN
    SET NOCOUNT ON;

    BEGIN TRY
        BEGIN TRANSACTION;

        -- Insert fee record
        INSERT INTO Fees
            (StudentId, Amount, PaidDate)
        VALUES
            (@StudentId, @Amount, GETDATE());

        SET @ReceiptNo = SCOPE_IDENTITY();

        -- Update student balance
        UPDATE Students
        SET    FeeBalance = FeeBalance - @Amount
        WHERE  Id = @StudentId;

        COMMIT TRANSACTION;
    END TRY
    BEGIN CATCH
        ROLLBACK TRANSACTION;
        THROW;  -- re-throw to caller
    END CATCH
END
❓ Interview Q: SCOPE_IDENTITY() vs @@IDENTITY?
SCOPE_IDENTITY() = current scope-la last inserted ID — safe. @@IDENTITY = trigger use panna trigger-la insert aana ID varum — wrong! Always SCOPE_IDENTITY() use pannu.
Syntax
Scalar Function · Table-Valued Function
-- Scalar function — returns 1 value
CREATE FUNCTION dbo.GetAttendancePct
(
    @StudentId INT,
    @Month     INT,
    @Year      INT
)
RETURNS DECIMAL(5,2)
AS
BEGIN
    DECLARE @Pct DECIMAL(5,2);

    SELECT @Pct =
        CAST(SUM(CASE WHEN IsPresent=1
                       THEN 1.0 ELSE 0 END)
             * 100 / COUNT(*) AS DECIMAL(5,2))
    FROM  Attendance
    WHERE StudentId  = @StudentId
    AND   MONTH(Date) = @Month
    AND   YEAR(Date)  = @Year;

    RETURN ISNULL(@Pct, 0);
END

-- Use in SELECT
SELECT
    Name,
    dbo.GetAttendancePct(Id, 6, 2026)
        AS AttendancePct
FROM Students;
⚠️ Trap: Scalar functions in WHERE/SELECT are slow!
Row-by-row execute aagum — N rows = N function calls. Large tables-la avoid. TVF (Table-Valued Function) or inline subquery prefer pannu.
🔴 Indexes · Execution Plan · Query Optimization
🎬 Ratsasan (2018): Detective serial killer-a find panna — Table Scan = every house check (slow!). Index Seek = eyewitness clue use panni direct jump (fast!). Execution Plan = detective's investigation board — show panrom WHERE query time waste aaguthu!
Syntax — Indexes
Clustered · Non-Clustered · Covering Index
-- Clustered Index (Physical sort — 1 per table)
-- Primary Key = auto clustered
CREATE CLUSTERED INDEX IX_Students_Id
    ON Students(Id);

-- Non-Clustered (pointer-based — many per table)
CREATE NONCLUSTERED INDEX
    IX_Students_Class
    ON Students(Class, IsActive);

-- Covering Index (include extra columns)
-- Avoids extra lookup to base table!
CREATE NONCLUSTERED INDEX
    IX_Students_Class_Cover
    ON Students(Class, IsActive)
    INCLUDE (Name, RollNo, Phone);

-- Unique Index
CREATE UNIQUE INDEX UX_Students_RollClass
    ON Students(RollNo, ClassId);

-- Filtered Index (partial table)
CREATE NONCLUSTERED INDEX
    IX_Students_Active
    ON Students(Class)
    WHERE IsActive = 1;
-- Only active students indexed = smaller!

-- Drop index
DROP INDEX IX_Students_Class
    ON Students;
❓ Interview Q: Clustered vs Non-Clustered Index?
Clustered = data physically sorted on disk by this key — 1 per table. Non-clustered = separate B-tree with row pointer — many per table. ZenCampus Students.Id = clustered (PK). Students.Class = non-clustered (frequent filter). Covering index-la INCLUDE column pottucha — Key Lookup avoid aagum!
Concept — Performance
Execution Plan · Table Scan vs Index Seek
-- Check execution plan
SET STATISTICS TIME ON;
SET STATISTICS IO ON;

-- Run your query
SELECT * FROM Students
WHERE Class = '10A';

-- In SSMS: Ctrl+M → Actual Execution Plan
-- OR Ctrl+L → Estimated Plan

-- ❌ BAD — Table Scan (no index)
SELECT * FROM Students
WHERE YEAR(DOB) = 2008;
-- Function on column = index NOT used!

-- ✅ GOOD — Index Seek
SELECT * FROM Students
WHERE DOB BETWEEN
    '2008-01-01' AND '2008-12-31';

-- ❌ LIKE with leading % — no index
WHERE Name LIKE '%Kumar'   -- Scan!

-- ✅ LIKE end % — index used
WHERE Name LIKE 'Guru%'    -- Seek!

-- ❌ Implicit conversion — no index
WHERE StudentId = '100'   -- string vs int!

-- ✅ Correct type
WHERE StudentId = 100
💡 Index Killers — avoid these on WHERE columns:
Function wrap: YEAR(DOB), UPPER(Name), LEN(Code)
Leading wildcard: LIKE '%Kumar'
Implicit conversion: INT column = '100'
OR on different columns (use UNION instead)
Senior Concept
N+1 Problem · Pagination · Query Hints
-- ❌ N+1 Problem
-- 1 query for students + N queries for fees
SELECT * FROM Students;           -- 1 query
-- Then for each student in app code:
SELECT * FROM Fees
    WHERE StudentId = @id;        -- N queries!

-- ✅ Fix — Single JOIN query
SELECT
    s.Id, s.Name,
    f.Amount, f.PaidDate
FROM       Students s
LEFT JOIN  Fees     f
    ON f.StudentId = s.Id;         -- 1 query!

-- Keyset Pagination (faster than OFFSET)
-- ❌ OFFSET gets slow on large pages
SELECT * FROM Students
ORDER BY Id
OFFSET 100000 ROWS FETCH NEXT 20 ROWS ONLY;

-- ✅ Keyset (seek pagination)
SELECT TOP 20 *
FROM  Students
WHERE Id > @LastSeenId
ORDER BY Id;
❓ Interview Q: What is N+1 problem?
1 query returns N students → N more queries for each student's data. 100 students = 101 DB calls! Fix: JOIN or multi-query with IN. ZenCampus attendance report-la ithaye fix panninom — 500ms → 20ms achu!

🔵 Transactions · DDL · String Functions · Dates
Syntax
Transactions — ACID Properties
-- Transaction = all or nothing
BEGIN TRANSACTION;

BEGIN TRY
    INSERT INTO Fees
        (StudentId, Amount)
    VALUES (1, 15000);

    UPDATE Students
    SET    FeeBalance = FeeBalance - 15000
    WHERE  Id = 1;

    COMMIT TRANSACTION;     -- both succeed
END TRY
BEGIN CATCH
    ROLLBACK TRANSACTION;   -- both fail
    SELECT ERROR_MESSAGE() AS ErrMsg;
END CATCH

-- SAVEPOINT — partial rollback
BEGIN TRANSACTION;
    INSERT INTO AuditLog ...;
    SAVE TRANSACTION AfterAudit;

    UPDATE Students ...;
    -- If update fails, rollback to savepoint
    ROLLBACK TRANSACTION AfterAudit;
COMMIT;
❓ Interview Q: ACID properties?
Atomicity = all or nothing. Consistency = valid state before+after. Isolation = concurrent transactions don't interfere. Durability = committed = permanent. ZenCampus fee payment — INSERT + UPDATE must be atomic!
