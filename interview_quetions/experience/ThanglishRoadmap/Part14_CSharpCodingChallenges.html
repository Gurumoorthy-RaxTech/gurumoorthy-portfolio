1. 2nd Largest Number (Difficulty: Easy · Arrays / Single-Pass)
Problem: Write a program that takes user input as a series of integers separated by space and prints the 2nd largest number. Example: input -10 200 9 25 → output 25.
Naive approach na — array ah sort pannitu, arr[length-2] eduthukalam, aana adhu O(n log n). Namba single pass la, O(n) la solve pannalam — iterate panra bodhiye rendu variables vachukanum: largest and secondLargest. Ovvoru number ah largest kitta compare pannalam, adha vida periya na largest ah secondLargest ku push pannitu, pudhu number ah largest ah vekkalam. Illa na, secondLargest kitta compare pannalam — aana duplicate numbers ah ignore panna vendiyadhu muyakiyam, illa na "5 5 5" madhiri input ku wrong answer varum.
using System;

class SecondLargest
{
    static void Main()
    {
        Console.Write("Enter numbers separated by space: ");
        string input = Console.ReadLine();
        string[] parts = input.Split(' ', StringSplitOptions.RemoveEmptyEntries);
        int[] numbers = Array.ConvertAll(parts, int.Parse);

        int? largest = null;
        int? secondLargest = null;

        foreach (int num in numbers)
        {
            if (largest == null || num > largest)
            {
                secondLargest = largest;
                largest = num;
            }
            else if (num != largest && (secondLargest == null || num > secondLargest))
            {
                secondLargest = num;
            }
        }

        if (secondLargest == null)
            Console.WriteLine("No valid 2nd largest (all numbers same, or fewer than 2 distinct values).");
        else
            Console.WriteLine($"2nd largest number: {secondLargest}");
    }
}
🗣️ Interview la eppadi walk-through pannanumI track two nullable ints — largest and secondLargest — while scanning the array once, so this is O(n) time and O(1) extra space, versus O(n log n) for a sort-based approach. The tricky edge case is duplicates: if the same maximum value repeats (like "5 5 5"), there is no valid distinct 2nd largest, so I explicitly guard with num != largest and report that case instead of silently returning a wrong number.
🔁 Follow-up variations they might ask
Q: What if the array has fewer than 2 elements? — secondLargest stays null, so my null-check message handles it gracefully instead of throwing.
Q: Find the Kth largest instead of 2nd? — single-pass two-variable tracking doesn't scale to arbitrary K; I'd switch to a min-heap of size K (O(n log K)) or a sort for simplicity.
Q: What if the input has extra spaces or is comma-separated? — I'd use Split(new[]{' ',','}, StringSplitOptions.RemoveEmptyEntries) to handle both.
2. Swap Two Integers Without an Intermediate Variable (Difficulty: Easy · Bitwise/Arithmetic)
Problem: Swap the values of two integer variables without using a third (temporary) variable.
Rendu classic tricks irukku — arithmetic mattum XOR. Arithmetic la, a+b vachu sum ah store panni, apparam subtract pannitu original values ah extract pannuvom — aana int.MaxValue lagatta values irundha, a+b overflow aagum. XOR trick safe — no overflow, ஏன்னு XOR bit-level operation, magnitude problem illa. Aana real production code la, naan idha use panna maatten — readability ku temp variable or tuple swap use pannuven, idhu interview trick question ku mattum thaan.
using System;

class SwapDemo
{
    static void Main()
    {
        int a = 5, b = 10;

        // Arithmetic trick — CAUTION: a + b can overflow if both are near int.MaxValue
        a = a + b;
        b = a - b;
        a = a - b;
        Console.WriteLine($"Arithmetic swap: a={a}, b={b}");

        // reset for demo
        a = 5; b = 10;

        // XOR trick — no overflow risk, works only for integer types
        a ^= b;
        b ^= a;
        a ^= b;
        Console.WriteLine($"XOR swap: a={a}, b={b}");
    }
}
🗣️ Interview la eppadi walk-through pannanumBoth tricks avoid a third variable in O(1) space, but the arithmetic version can silently overflow if a+b exceeds int.MaxValue, so I'd call out that risk unprompted. The XOR version is safe for integers because it never leaves the representable range, but it only works on integral types and it's genuinely harder to read at a glance — so unless the interviewer specifically bans a temp variable, I'd just say "in real code I'd use (a, b) = (b, a); or a plain temp — premature cleverness here just hurts the next developer reading it."
🔁 Follow-up variations they might ask
Q: Does the XOR trick work if a and b are the same variable (aliased)? — No — XOR-swapping a variable with itself zeroes it out, so you must guard with a reference-equality check first.
Q: Does this work for doubles/floats? — Arithmetic swap technically works with +/- but XOR does not (XOR is bitwise-integer only); for floating point I'd just use a temp or tuple swap.
Q: What's the safest one-liner in modern C#? — (a, b) = (b, a); — tuple deconstruction, zero overflow risk, works for any type, and reads instantly.
3. Same Swap Using Generics (Difficulty: Medium · Generics)
Problem: Write a generic swap method that works for any two numeric variables of the same type (int, double, decimal, etc.), not just int.
Real constraint here — C# doesn't let you do a + b or a - b directly on a generic T, ஏன்னா compiler ku theriyathu T oru number nu. So arithmetic tricks (Q2 la panna madhiri) generic ah panna mudiyathu without extra help. Best real answer — tuple deconstruction swap, (a, b) = (b, a);, idhu ANY type ku work aagum (numeric mattum illa, string, struct, reference type ellame). Bonus ah, .NET 7+ la INumber<T> generic math interface use panni arithmetic-based generic swap kooda kaamikkalam — depth of knowledge kaatanum na mattum idha sollu, aana interview la default answer tuple swap thaan.
using System;
using System.Numerics;

class SwapGeneric
{
    // Best real-world answer — works for ANY type, numeric or not
    static void Swap<T>(ref T a, ref T b)
    {
        (a, b) = (b, a);
    }

    // Bonus / advanced: .NET 7+ generic math — arithmetic-based swap, numeric types only
    static void SwapNumeric<T>(ref T a, ref T b) where T : INumber<T>
    {
        T temp = a - b;
        a = b;
        b = temp + b;
    }

    static void Main()
    {
        int x = 10, y = 20;
        Swap(ref x, ref y);
        Console.WriteLine($"x={x}, y={y}");

        string s1 = "hello", s2 = "world";
        Swap(ref s1, ref s2);
        Console.WriteLine($"s1={s1}, s2={s2}");

        double d1 = 3.5, d2 = 7.25;
        SwapNumeric(ref d1, ref d2);
        Console.WriteLine($"d1={d1}, d2={d2}");
    }
}
🗣️ Interview la eppadi walk-through pannanumI'd lead with the tuple-deconstruction swap — static void Swap<T>(ref T a, ref T b) => (a, b) = (b, a); — because it needs no type constraint at all and works for every type, which is exactly what "generic" should mean. If they push for an arithmetic-only version to test generic math knowledge, I'd bring up .NET 7's INumber<T> constraint, which lets me write real +/- on a generic T, but I'd be upfront that it's more ceremony for zero real benefit over the tuple version.
🔁 Follow-up variations they might ask
Q: Why can't you just write T temp = a; a = b; b = temp; directly? — You can! That works for any T with zero constraints too — it's actually the simplest generic-safe version, tuple swap is just syntactic sugar over the same idea.
Q: What C# version introduced INumber<T>? — .NET 7 / C# 11, part of the "generic math" feature using static abstract interface members.
Q: Can this swap work across two different types, like int and long? — Not directly — ref T forces both arguments to be the exact same type; you'd need an explicit cast or a non-generic overload for mixed types.
4. Number Guessing Game with Cheat Detection (Difficulty: Medium · Binary Search)
Problem: The computer should guess a number between 1 and 1000 that the user is thinking of. The user replies "high", "low", or "correct" after each guess, and the computer narrows down until it finds the number. Bonus: detect if the user is lying / contradicting themselves and print "You're cheating!"
Idhu classic binary search application. Namba oru [low, high] range vechukirom (start: 1 to 1000), ovvoru round la mid-point ah guess panrom. User "high" sonna, actual number guess ah vida chinnadhu nu artham — so high = guess - 1. "low" sonna, low = guess + 1. "correct" na done. Cheat detection — namba already narrow panna range oda boundary la irundhu impossible ah answer kudutha (example: guess already low boundary la irukku, aana "high" nu sonna — adhukku mela number illa nu already theriyum), adha catch pannanum. Simple ah, apply pannadha piragu low > high aana andha range empty aayiduchu nu artham — idhu contradiction indicator.
using System;

class GuessingGame
{
    static void Main()
    {
        int low = 1, high = 1000;
        Console.WriteLine("Think of a number between 1 and 1000. I'll try to guess it!");
        Console.WriteLine("Reply with 'high' (my guess is too high), 'low' (too low), or 'correct'.");

        while (true)
        {
            if (low > high)
            {
                Console.WriteLine("Your answers don't add up — You're cheating!");
                return;
            }

            int guess = low + (high - low) / 2;
            Console.Write($"My guess: {guess}. Your answer (high/low/correct): ");
            string response = Console.ReadLine()?.Trim().ToLower();

            if (response == "correct")
            {
                Console.WriteLine($"Yay! Guessed it — {guess}");
                break;
            }
            else if (response == "high")
            {
                // number should be smaller than guess — but guess is already the lowest
                // possible value in the valid range, so there's nothing smaller left
                if (guess == low)
                {
                    Console.WriteLine("That's impossible given your earlier answers — You're cheating!");
                    return;
                }
                high = guess - 1;
            }
            else if (response == "low")
            {
                if (guess == high)
                {
                    Console.WriteLine("That's impossible given your earlier answers — You're cheating!");
                    return;
                }
                low = guess + 1;
            }
            else
            {
                Console.WriteLine("Please answer with high, low, or correct.");
                continue;
            }

            if (low > high)
            {
                Console.WriteLine("Your answers contradict each other — You're cheating!");
                return;
            }
        }
    }
}
🗣️ Interview la eppadi walk-through pannanumThis is binary search over [low, high], so it takes at most log2(1000) ≈ 10 guesses — O(log n) time. Cheat detection has two layers: first, an immediate boundary check — if the guess already equals low and the user says "high" (or equals high and says "low"), that's logically impossible within the current valid range; second, a defensive low > high check after every update catches any contradiction I might have missed, since a valid range can never invert.
🔁 Follow-up variations they might ask
Q: What if the user changes their mind mid-game and contradicts an earlier answer, not just the current boundary? — The running low/high already encodes every prior answer, so any later response that shrinks the range to empty (low > high) automatically catches historical contradictions too, not just the current one.
Q: How would you make this generic for any range, not just 1–1000? — Pass low/high as parameters or constructor args instead of hardcoding; the algorithm is unchanged.
Q: Worst case, how many guesses does it take? — ceil(log2(1000)) = 10 guesses, because binary search halves the search space every round.
5. Examine a Folder — List Files by Size Ascending (Difficulty: Medium · WinForms / LINQ / IO)
Problem: Build a WinForms screen where the user picks a folder via a dialog, and the app lists every file's name and size, sorted ascending by file size.
FolderBrowserDialog vachu user ah folder select panna vidurom. Apparam Directory.GetFiles() use panni andha folder oda files ellam eduthukanum, LINQ vachu Select(f => new FileInfo(f)) panni FileInfo objects ah build panni, .OrderBy(fi => fi.Length) vachu size ascending order la sort pannanum. Idha ListView la Name + Size subitems ah kaatanum. Simple button-click handler la ellame handle pannalam.
using System;
using System.IO;
using System.Linq;
using System.Windows.Forms;

public partial class FolderSizeForm : Form
{
    private ListView listView1;
    private Button btnBrowse;

    public FolderSizeForm()
    {
        listView1 = new ListView { Dock = DockStyle.Fill, View = View.Details };
        listView1.Columns.Add("File Name", 300);
        listView1.Columns.Add("Size (bytes)", 150);

        btnBrowse = new Button { Text = "Browse Folder...", Dock = DockStyle.Top };
        btnBrowse.Click += BtnBrowse_Click;

        Controls.Add(listView1);
        Controls.Add(btnBrowse);
    }

    private void BtnBrowse_Click(object sender, EventArgs e)
    {
        using var dialog = new FolderBrowserDialog();
        if (dialog.ShowDialog() != DialogResult.OK) return;

        listView1.Items.Clear();

        var files = Directory.GetFiles(dialog.SelectedPath)
                              .Select(f => new FileInfo(f))
                              .OrderBy(fi => fi.Length);

        foreach (var fi in files)
        {
            var item = new ListViewItem(fi.Name);
            item.SubItems.Add(fi.Length.ToString("N0"));
            listView1.Items.Add(item);
        }
    }
}
🗣️ Interview la eppadi walk-through pannanumI use FolderBrowserDialog for the folder picker, then Directory.GetFiles() to enumerate the top-level files, projecting each path into a FileInfo via LINQ so I get both Name and Length in one shot, and OrderBy(fi.Length) does the ascending sort declaratively instead of me hand-rolling a sort algorithm. This is O(n log n) for the sort and O(n) for the file listing, and I clear the ListView first so repeated browsing doesn't append duplicate rows.
🔁 Follow-up variations they might ask
Q: What about subfolders — recursive listing? — Pass SearchOption.AllDirectories as the second argument to Directory.GetFiles(), or use Directory.EnumerateFiles() for better memory behavior on huge trees.
Q: How would you show size in KB/MB instead of raw bytes? — Add a helper that divides Length by 1024/1024² and formats with "N2", or use a converter method that picks the right unit based on magnitude.
Q: What if the folder has thousands of files — will the UI freeze? — Yes, GetFiles() call blocks the UI thread; for large folders I'd move the enumeration to a background Task and marshal results back via Invoke/BeginInvoke, or use EnumerateFiles with streaming updates.
6. Custom NumericTextBox Control (Difficulty: Medium · WinForms Custom Control)
Problem: Build a WinForms control that only accepts positive integers including 0, with a configurable Min/Max range (default 0–9999, settable up to 0–999999).
TextBox ah subclass pannitu, rendu places la validation panna vendum. First, KeyPress event la — user type panra bodhiye, non-digit characters ah reject pannanum (aana Backspace, Delete madhiri control keys ah allow pannanum, illana user ku edit panna mudiyathu). Second, Validating/Leave event la — final value ah Min/Max range kitta check panni, out-of-range aana clamp pannanum. Min, Max, Value properties expose pannanum, so designer la kooda configure panna mudiyum.
using System;
using System.ComponentModel;
using System.Windows.Forms;

public class NumericTextBox : TextBox
{
    private int _min = 0;
    private int _max = 9999;

    [Category("Behavior")]
    public int Min
    {
        get => _min;
        set { _min = value; ClampCurrentValue(); }
    }

    [Category("Behavior")]
    public int Max
    {
        get => _max;
        set { _max = value; ClampCurrentValue(); }
    }

    [Browsable(false)]
    public int Value
    {
        get => int.TryParse(Text, out int v) ? v : Min;
        set => Text = Math.Clamp(value, Min, Max).ToString();
    }

    public NumericTextBox()
    {
        Text = "0";
    }

    protected override void OnKeyPress(KeyPressEventArgs e)
    {
        base.OnKeyPress(e);

        // allow control keys - Backspace, Delete, etc.
        if (char.IsControl(e.KeyChar))
            return;

        // only digits allowed - positive integers only, no minus, no decimal point
        if (!char.IsDigit(e.KeyChar))
        {
            e.Handled = true;
            return;
        }

        // block the keystroke early if the resulting number would exceed Max
        string prospective = Text.Remove(SelectionStart, SelectionLength)
                                  .Insert(SelectionStart, e.KeyChar.ToString());

        if (long.TryParse(prospective, out long prospectiveValue) && prospectiveValue > Max)
        {
            e.Handled = true;
        }
    }

    protected override void OnValidating(CancelEventArgs e)
    {
        base.OnValidating(e);

        if (string.IsNullOrWhiteSpace(Text) || !int.TryParse(Text, out int value))
        {
            Text = Min.ToString();
            return;
        }

        if (value < Min) Text = Min.ToString();
        else if (value > Max) Text = Max.ToString();
    }

    private void ClampCurrentValue()
    {
        if (int.TryParse(Text, out int v))
            Value = v; // setter clamps against Min/Max
    }
}
🗣️ Interview la eppadi walk-through pannanumI split validation into two layers — OnKeyPress does character-level filtering so the user physically can't type a letter or push the value past Max mid-typing, and OnValidating does the final range check on Leave, clamping to Min or Max if the field is empty or out of bounds. Min and Max are Browsable properties so they show up in the designer, and Value is a computed property so the underlying Text stays the single source of truth — I default to 0–9999 but a consumer can widen it up to 0–999999 just by setting Max.
🔁 Follow-up variations they might ask
Q: What about pasting text via Ctrl+V — does KeyPress catch that? — No, paste bypasses KeyPress; I'd also override OnTextChanged or handle the Paste command to strip non-digits from pasted content.
Q: Why override OnValidating instead of just Leave? — Validating is cancelable (CancelEventArgs) so I could block focus from leaving the control entirely if I wanted strict rejection instead of clamping — I chose clamp here for a friendlier UX.
Q: How would you support negative numbers too? — Allow a leading '-' only at position 0 in KeyPress, and change the digit-only check to allow that one exception; Min would then typically go negative too.

7. FizzBuzz (Difficulty: Easy · Loops / LINQ)
Problem: Print numbers 1 to N. For multiples of 3 print "Fizz", multiples of 5 print "Buzz", multiples of both print "FizzBuzz", otherwise print the number itself.
Simple mistake ellarum panradhu — 3 and 5 ah check pannitu, 15 ah check pannama irukradhu, adhu wrong output kudukkum (15 ku "Fizz" mattum varum, "FizzBuzz" varathu). So 15 ah first check pannanum — biggest common divisor combo first, apparam 3, apparam 5. LINQ one-liner la kooda idhe logic ah ternary chain ah eluthalam.
using System;
using System.Linq;

class FizzBuzz
{
    static void Main()
    {
        int n = 30;

        // Imperative version
        for (int i = 1; i <= n; i++)
        {
            if (i % 15 == 0) Console.WriteLine("FizzBuzz");
            else if (i % 3 == 0) Console.WriteLine("Fizz");
            else if (i % 5 == 0) Console.WriteLine("Buzz");
            else Console.WriteLine(i);
        }

        // LINQ one-liner version
        var result = Enumerable.Range(1, n)
            .Select(i => i % 15 == 0 ? "FizzBuzz" : i % 3 == 0 ? "Fizz" : i % 5 == 0 ? "Buzz" : i.ToString());

        Console.WriteLine(string.Join(" ", result));
    }
}
🗣️ Interview la eppadi walk-through pannanumI check divisibility by 15 first, then 3, then 5, so numbers divisible by both never fall into the wrong branch — that ordering is the one thing people trip on. Both versions are O(n) time; the LINQ version is just a declarative rewrite of the same conditional logic using a nested ternary inside Select.
🔁 Follow-up variations they might ask
Q: How would you make it configurable for arbitrary divisor/word pairs? — Use a list of (divisor, word) tuples, build the output string by concatenating words for every divisor that divides i, falling back to i.ToString() if nothing matched.
Q: Can you do it without any modulo operator? — Yes — keep three counters that increment each loop and reset (and print the word) when they hit 3 or 5, avoiding % entirely.
8. Reverse a String (Manual) + Palindrome Check (Difficulty: Easy · Strings / Two-Pointer)
Problem: Reverse a string without using Array.Reverse() or the LINQ .Reverse() method. Then use that to check if a string is a palindrome.
Two-pointer technique — oru pointer string oda start la, vera oru pointer end la vechukanum, rendu characters ah swap pannitu, pointers ah center nokki move pannikitte poganum, left < right varaikkum. Idhu O(n) time, O(1) extra space (in-place on the char array). Palindrome check ku, idhe reverse function ah reuse pannalam — string oda reverse adhe string ah irundha, adhu palindrome.
using System;

class ReverseAndPalindrome
{
    static string ReverseString(string input)
    {
        char[] chars = input.ToCharArray();
        int left = 0, right = chars.Length - 1;

        while (left < right)
        {
            (chars[left], chars[right]) = (chars[right], chars[left]);
            left++;
            right--;
        }

        return new string(chars);
    }

    static bool IsPalindrome(string input)
    {
        string cleaned = input.ToLower();
        return cleaned == ReverseString(cleaned);
    }

    static void Main()
    {
        Console.WriteLine(ReverseString("Gurumoorthy"));   // yhtroomuruG
        Console.WriteLine(IsPalindrome("Malayalam"));       // True
        Console.WriteLine(IsPalindrome("Chennai"));         // False
    }
}
🗣️ Interview la eppadi walk-through pannanumThe two-pointer swap reverses in O(n) time and O(1) extra space beyond the char array copy, since I never allocate a second buffer — I just swap in place and walk the pointers inward. For palindrome checking, I reuse the reverse function and compare against a case-normalized copy, which is O(n) as well and keeps the code DRY instead of writing a separate palindrome loop.
🔁 Follow-up variations they might ask
Q: How would you ignore spaces and punctuation for the palindrome check (e.g. "A man a plan a canal Panama")? — Strip non-alphanumeric characters first with a LINQ Where(char.IsLetterOrDigit) filter before lowering and comparing.
Q: Can you check palindrome without building the reversed string at all? — Yes — two pointers from both ends comparing characters directly and stopping early on a mismatch, which is more efficient since it can short-circuit.
Q: What about Unicode / surrogate pairs (emoji, combined characters)? — char-by-char swapping can break multi-char Unicode grapheme clusters; for correctness with such input I'd use StringInfo.GetTextElementEnumerator to reverse by grapheme, not by char.
9. Find Duplicate Elements in an Array (Difficulty: Easy · LINQ / GroupBy)
Problem: Given an integer array, find which elements appear more than once, and how many times each appears.
LINQ GroupBy use panni, same value oda ella occurrences ah oru group ah pottu, apparam Where(g => g.Count() > 1) vachu one-time-only values ah filter out pannalam. Idhu readable ah irukkum, manual dictionary counting logic eludhardhukku bathil.
using System;
using System.Linq;

class FindDuplicates
{
    static void Main()
    {
        int[] numbers = { 1, 2, 3, 2, 4, 5, 1, 6, 3, 3 };

        var duplicates = numbers
            .GroupBy(n => n)
            .Where(g => g.Count() > 1)
            .Select(g => new { Value = g.Key, Count = g.Count() });

        foreach (var d in duplicates)
            Console.WriteLine($"{d.Value} appears {d.Count} times");
    }
}
🗣️ Interview la eppadi walk-through pannanumGroupBy is O(n) — it does a single pass, bucketing elements by value using an internal hash-based grouping, so it's equivalent in complexity to a manual Dictionary<int,int> counter but far more readable. I project each qualifying group into an anonymous type carrying both the value and its count, so the caller gets everything needed in one query.
🔁 Follow-up variations they might ask
Q: How would you get just the distinct duplicate values, without counts? — numbers.GroupBy(n=>n).Where(g=>g.Count()>1).Select(g=>g.Key).
Q: Can you solve it without GroupBy, using a HashSet? — Yes — keep a "seen" HashSet and a "duplicates" HashSet; if Add() to seen fails (already present), add the value to duplicates — still O(n).
10. Two Sum (Difficulty: Medium · Dictionary / Hashing)
Problem: Given an array of integers and a target value, return the indices of the two numbers that add up to the target.
Naive approach — nested loop, ovvoru element ku vera ella elements oda um sum check pannradhu — O(n²), interview la idha sonna "can you do better?" nu kekka poranga. Better approach — oru Dictionary<int,int> vachukanum (value → index), single pass la, ovvoru number ku "target - number" (complement) already dictionary la irukka nu check pannalam. Irundha, andha rendu indices um answer. Illana, current number ah dictionary la add pannitu next ku poganum. Idhu O(n) time, O(n) space — trade memory for speed.
using System;
using System.Collections.Generic;

class TwoSum
{
    static int[] Solve(int[] nums, int target)
    {
        var seen = new Dictionary<int, int>(); // value -> index

        for (int i = 0; i < nums.Length; i++)
        {
            int complement = target - nums[i];
            if (seen.TryGetValue(complement, out int complementIndex))
                return new[] { complementIndex, i };

            seen[nums[i]] = i;
        }

        throw new InvalidOperationException("No two numbers add up to target.");
    }

    static void Main()
    {
        int[] nums = { 2, 7, 11, 15 };
        var result = Solve(nums, 9);
        Console.WriteLine($"Indices: {result[0]}, {result[1]}"); // 0, 1
    }
}
🗣️ Interview la eppadi walk-through pannanumI do a single pass with a Dictionary mapping value to index — for each number I compute its complement (target minus current value) and check if I've already seen it, giving O(n) time and O(n) space, versus the naive nested-loop's O(n²). This beats sorting-plus-two-pointer too when I need original indices, since sorting would destroy index order and require extra bookkeeping to recover it.
🔁 Follow-up variations they might ask
Q: What if there are multiple valid pairs — return all of them? — Switch from Dictionary<int,int> to Dictionary<int,List<int>> to track every index per value, and collect all matching pairs instead of returning on the first hit.
Q: What if the array is already sorted — is there a lower-memory approach? — Yes — two pointers from both ends moving inward based on whether the current sum is too high or too low, O(n) time and O(1) extra space, but it loses original indices unless you track them separately.
Q: What if no pair sums to target? — My code throws; in production I'd return a nullable/TryGetTwoSum pattern instead of throwing, so callers can handle "not found" without a try-catch.
11. Fibonacci Series — Iterative vs Recursive vs Memoized (Difficulty: Medium · Recursion / DP)
Problem: Write a function to compute the Nth Fibonacci number. Show iterative, naive recursive, and memoized recursive approaches, and compare them.
Iterative version — rendu variables (prev, curr) vachu loop la update pannikitte poganum, O(n) time, O(1) space, idhu best. Naive recursive — fib(n) = fib(n-1) + fib(n-2) nu direct ah eludhinaa, adhu exponential O(2^n) aagum, ஏன்னா same sub-problems ah repeat repeat ah recompute pannum (fib(5) calculate panna fib(3) ah rendu vera branches la calculate pannum). Memoized recursive — same recursive structure, aana already calculated results ah dictionary/cache la store panni, repeat calculation ah avoid pannurom — idhu O(n) aagidum, top-down dynamic programming.
using System;
using System.Collections.Generic;

class Fibonacci
{
    // Iterative - O(n) time, O(1) space - best for most real use cases
    static long Iterative(int n)
    {
        if (n <= 1) return n;
        long prev = 0, curr = 1;
        for (int i = 2; i <= n; i++)
        {
            long next = prev + curr;
            prev = curr;
            curr = next;
        }
        return curr;
    }

    // Naive recursive - O(2^n) time - recomputes the same sub-problems repeatedly
    static long NaiveRecursive(int n)
    {
        if (n <= 1) return n;
        return NaiveRecursive(n - 1) + NaiveRecursive(n - 2);
    }

    // Memoized recursive - O(n) time, O(n) space - top-down DP
    static long MemoizedRecursive(int n, Dictionary<int, long> cache = null)
    {
        cache ??= new Dictionary<int, long>();
        if (n <= 1) return n;
        if (cache.TryGetValue(n, out long cached)) return cached;

        long result = MemoizedRecursive(n - 1, cache) + MemoizedRecursive(n - 2, cache);
        cache[n] = result;
        return result;
    }

    static void Main()
    {
        int n = 10;
        Console.WriteLine(Iterative(n));
        Console.WriteLine(NaiveRecursive(n));
        Console.WriteLine(MemoizedRecursive(n));
    }
}
🗣️ Interview la eppadi walk-through pannanumIterative is my default answer in production — O(n) time, O(1) space, no recursion overhead or stack-depth risk. Naive recursion is O(2^n) because it re-derives overlapping sub-problems from scratch — fib(n-2) gets recomputed as part of both fib(n-1) and directly, and that duplication compounds exponentially. Memoization fixes that by caching each result the first time it's computed, bringing it back down to O(n) time at the cost of O(n) space for the cache plus call-stack depth — I'd mention it specifically when the interviewer wants to see the recursion-to-DP transformation, not as a real-world recommendation over the iterative version.
🔁 Follow-up variations they might ask
Q: What about very large N where long overflows? — Switch to BigInteger from System.Numerics for arbitrary precision.
Q: Can you do it with O(1) space using recursion? — Not with naive tree recursion, but a tail-recursive accumulator version compiles similarly to the iterative loop in some languages — C# doesn't guarantee tail-call optimization though, so I'd stick with the iterative loop for guaranteed O(1) space.
Q: Is there a closed-form formula? — Yes, Binet's formula using the golden ratio gives O(1) time, but it involves floating-point math and loses precision for large n, so it's rarely used in practice.
12. Check if Two Strings Are Anagrams (Difficulty: Easy · Strings / Hashing)
Problem: Given two strings, determine whether they are anagrams of each other (same characters, same frequency, different order), case-insensitive.
Character frequency comparison — first string oda ovvoru character ku count vachu oru Dictionary build pannanum. Apparam second string ah iterate pannumbodhu, ovvoru character ku dictionary la count ah decrement pannikitte poganum. Rendaavadhu string mudinja piragu, dictionary empty ah irundha (allа count ellame zero aayidum), rendu strings um anagram thaan. Length mismatch first check pannitaa early-exit panna mudiyum.
using System;
using System.Collections.Generic;

class AnagramCheck
{
    static bool AreAnagrams(string a, string b)
    {
        a = a.Replace(" ", "").ToLower();
        b = b.Replace(" ", "").ToLower();

        if (a.Length != b.Length) return false;

        var freq = new Dictionary<char, int>();

        foreach (char c in a)
            freq[c] = freq.GetValueOrDefault(c) + 1;

        foreach (char c in b)
        {
            if (!freq.ContainsKey(c)) return false;
            freq[c]--;
            if (freq[c] == 0) freq.Remove(c);
        }

        return freq.Count == 0;
    }

    static void Main()
    {
        Console.WriteLine(AreAnagrams("Listen", "Silent"));   // True
        Console.WriteLine(AreAnagrams("Hello", "World"));     // False
    }
}
🗣️ Interview la eppadi walk-through pannanumI build a frequency dictionary from the first string in O(n), then decrement counts while scanning the second string in O(m) — if any character in the second string isn't in the dictionary, or a count goes negative, they're not anagrams. Overall it's O(n+m) time and O(k) space where k is the alphabet size, which beats a sort-and-compare approach (O(n log n)) for large strings, and I normalize case and strip spaces upfront so "Listen"/"Silent" and phrase-level anagrams both work correctly.
🔁 Follow-up variations they might ask
Q: What if punctuation should also be ignored? — Extend the cleanup step with a Where(char.IsLetterOrDigit) filter before building the frequency map.
Q: Could you solve it with just sorting instead? — Yes — sort both strings' characters and compare equality, O(n log n), simpler code but slower than the O(n) frequency-map approach for large inputs.
Q: How would you check for anagram groups across a whole list of words (like LeetCode's Group Anagrams)? — Use each word's sorted-character signature as a dictionary key and group words by that key with LINQ GroupBy.
13. Remove Duplicates from a List Preserving Order (Difficulty: Easy · Collections / LINQ)
Problem: Given a List<T>, remove duplicate elements while preserving the original order of first occurrence.
.Distinct() LINQ method already preserves the order it saw elements first — internally adhu oru hash set vechu duplicates ah skip pannum, aana yield order maramaathu. Manual approach na, namba explicit ah oru HashSet<T> vechukanum "already seen" track panna, loop pannikitte, seen.Add(item) return true na (first time), result list la add pannuvom. Rendu approach um same O(n) — manual version konjam explicit control kudukkum, example custom equality logic venumna, or duplicate found na edhavadhu side-action pannanum na.
using System;
using System.Collections.Generic;
using System.Linq;

class RemoveDuplicates
{
    static void Main()
    {
        var list = new List<int> { 3, 1, 4, 1, 5, 9, 2, 6, 5, 3 };

        // LINQ Distinct() - preserves first-occurrence order automatically
        var linqResult = list.Distinct().ToList();

        // Manual HashSet approach - same order guarantee, more control if you need
        // extra logic while filtering (logging, custom equality, early break, etc.)
        var seen = new HashSet<int>();
        var manualResult = new List<int>();
        foreach (var item in list)
        {
            if (seen.Add(item))
                manualResult.Add(item);
        }

        Console.WriteLine(string.Join(", ", linqResult));
        Console.WriteLine(string.Join(", ", manualResult));
    }
}
🗣️ Interview la eppadi walk-through pannanumBoth approaches are O(n) time and O(n) space, and both preserve the order of first occurrence because they're built on the same hash-set-backed "have I seen this before" check. I'd default to .Distinct() for clean production code, but reach for the manual HashSet version when I need extra logic in the loop — like logging skipped duplicates, using a custom IEqualityComparer, or bailing out early once I've collected enough unique items.
🔁 Follow-up variations they might ask
Q: How do you dedupe by a specific property of a complex object, like Employee.Id? — list.GroupBy(e => e.Id).Select(g => g.First()), or DistinctBy() available from .NET 6 onward.
Q: Does order matter if the list is later going to be sorted anyway? — No — if a sort follows, plain Distinct() or even a HashSet-then-ToList() without order tracking is simpler and equally correct.
14. Merge Two Sorted Arrays (Difficulty: Medium · Two-Pointer / Arrays)
Problem: Given two already-sorted integer arrays, merge them into a single sorted array — without calling Array.Sort() on the result afterward.
Classic merge-step from merge sort. Rendu pointers vechukanum, oru pointer each array ku. Ovvoru step la, rendu arrays oda current elements ah compare pannitu, chinnadha result array ku push pannitu, andha pointer ah mattum advance pannanum. Oru array mudinja piragu, remaining array oda balance elements ah direct ah copy pannalam (already sorted ah irukkum). Idhu O(n+m) — namba sort pannalam venaam, ஏன்னா input already sorted.
using System;

class MergeSortedArrays
{
    static int[] Merge(int[] a, int[] b)
    {
        int[] result = new int[a.Length + b.Length];
        int i = 0, j = 0, k = 0;

        while (i < a.Length && j < b.Length)
        {
            result[k++] = a[i] <= b[j] ? a[i++] : b[j++];
        }

        while (i < a.Length) result[k++] = a[i++];
        while (j < b.Length) result[k++] = b[j++];

        return result;
    }

    static void Main()
    {
        int[] a = { 1, 3, 5, 7 };
        int[] b = { 2, 4, 6, 8, 10 };

        int[] merged = Merge(a, b);
        Console.WriteLine(string.Join(", ", merged));
    }
}
🗣️ Interview la eppadi walk-through pannanumThis is O(n+m) time and O(n+m) space for the result array — each element from both inputs is visited and placed exactly once, and I deliberately never re-sort the output since the two-pointer merge maintains sorted order by construction. This is literally the merge step from merge sort, which is a natural thing to point out since it shows I understand where the technique comes from.
🔁 Follow-up variations they might ask
Q: What if you need to merge in-place into array A, which has extra trailing capacity (like LeetCode's "Merge Sorted Array")? — Merge from the back — compare the largest remaining elements of both arrays and place them at the end of A working backward, avoiding overwriting unprocessed elements.
Q: How would you merge K sorted arrays, not just 2? — Use a min-heap (PriorityQueue<T,TPriority> in .NET) holding the current head of each array, repeatedly popping the smallest and pushing the next element from that array — O(N log K).
Q: What if the arrays might contain duplicates that need to be removed during merge? — Track the last-added value and skip pushing to result if it equals the value about to be added.
15. Word Frequency Counter (Top 5) (Difficulty: Medium · LINQ / Regex)
Problem: Given a paragraph of text, count occurrences of each word (case-insensitive, ignoring punctuation), and print the top 5 most frequent words.
First, text ah lowercase pannitu, Regex use panni ஒரு "word" nu enna define pannuromo (letters and apostrophes) adha mattum extract pannanum — idhu punctuation (periods, commas) ah automatic ah skip pannidum. Apparam LINQ GroupBy vachu word wise count eduthu, OrderByDescending(count) vachu sort pannitu, Take(5) vachu top 5 mattum eduthukalam.
using System;
using System.Linq;
using System.Text.RegularExpressions;

class WordFrequency
{
    static void Main()
    {
        string text = @"The quick brown fox jumps over the lazy dog.
                         The dog barks, but the fox runs away. The fox is quick!";

        var topWords = Regex.Matches(text.ToLower(), @"[a-z']+")
            .Select(m => m.Value)
            .GroupBy(w => w)
            .Select(g => new { Word = g.Key, Count = g.Count() })
            .OrderByDescending(g => g.Count)
            .Take(5);

        foreach (var w in topWords)
            Console.WriteLine($"{w.Word}: {w.Count}");
    }
}
🗣️ Interview la eppadi walk-through pannanumThe regex [a-z']+ against the lowercased text strips out punctuation and whitespace in one pass, giving me clean word tokens; GroupBy then buckets identical words and OrderByDescending plus Take(5) gets the top results — overall O(n log n) dominated by the sort, where n is the distinct word count, not the total word count. I lowercase before matching so "The" and "the" collapse into the same bucket, satisfying the case-insensitive requirement.
🔁 Follow-up variations they might ask
Q: How would you exclude common stop words like "the", "is", "but"? — Maintain a HashSet<string> of stop words and add a .Where(w => !stopWords.Contains(w)) filter before grouping.
Q: What about hyphenated or apostrophe words like "don't" or "well-known"? — Adjust the regex character class to include the specific punctuation you want preserved inside a word, e.g. [a-z'-]+, and test against real sample text since edge cases vary a lot.
Q: How would this scale to a very large file (gigabytes of text)? — Stream the file line-by-line instead of loading it all into one string, and use a Dictionary<string,int> accumulator across lines instead of building one giant MatchCollection in memory; only materialize the top-5 sort at the very end.
