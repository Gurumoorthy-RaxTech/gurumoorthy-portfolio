1A C# Language Fundamentals
1. Value Type vs Reference Type (Stack vs Heap)
Value type (int, bool, struct) na data direct ah stack la irukkum — oru variable ah vera variable ku assign pannina, andha value full ah copy aagum, rendும் independent aagi irukkும். Reference type (class, object, array) na — actual data heap la irukkும், variable la konjam "address" mattும் irukkும். So rendு variable-um same object ah point pannும், oru place la change pannina, vera variable la kூடa andha change theriyum. Analogy: value type — Xerox copy kudukurathு madhiri (original untouched); reference type — same house key rendு perukku kudukurathு madhiri (rendு perும் ஒரே house ku thaan pogும்).
🗣️ Interview la eppadi sollanumValue types like int, bool, and struct are stored on the stack and copied by value — each variable owns its own independent copy. Reference types like classes are stored on the heap, and the variable just holds a reference to that memory address — so assigning one reference variable to another means both point to the same underlying object.
🔁 Follow-up Qs they might fire next
Q: struct ah reference type madhiri behave panna mudiyuma? — Struct default ah value type; class-oda semantics venum na straight ah class use pannanum, struct-a wrapper class kulla vachukalam.
Q: string oru reference type-dhaan, aana value type madhiri behave pannுdhu, en? — String immutable, so every "change" pudhu object create pannும், adhанால் value-type feel varும் aana adhு heap-la irukka reference type thaan.
Q: Boxing idhoda eppadi related? — Value type ah object/interface type ku assign pannுmbodhu adhு heap ku "box" aagும் — adhுthaan next topic.
2. Boxing & Unboxing
Value type (int, struct) ah object type ah convert pannுрathு boxing — stack la irukka data ah heap ku eduthु poi wrap pannுрோம். Unboxing na thirும்ba andha object-a value type ku convert pannுрathு. Idhு expensive operation — edhуkки na heap allocation + type check venum. Analogy: cash (value) ah envelope (object) la potு kudukurathу boxing; envelope open panni cash edukurathу unboxing — rendுmе extra step, time waste.
🗣️ Interview la eppadi sollanumBoxing implicitly converts a value type into a reference type by wrapping it on the heap — like assigning an int to an object variable. Unboxing extracts it back. I avoid this in hot paths — for example using generic collections like List<int> instead of the old ArrayList, since ArrayList boxes every value type it stores.
🔁 Follow-up Qs they might fire next
Q: ArrayList vs List<int> difference? — ArrayList stores object, so adding an int boxes it every time; List<int> is generic, no boxing, faster.
Q: Boxing performance ah eppadi affect pannும்? — Extra heap allocation + GC pressure, loop kulla thousands of times panna noticeable slowdown varும்.
Q: Enum-um boxing aaguma? — Aama, enum value type thaan, object/interface ku assign pannுmbodhu box aagும்.
3. string vs StringBuilder
string immutable — oru string ah modify pannina, existing object change aagathு, pudhu string object heap la create aagும். Loop kulla str += x panna panna, ovvoru iteration-um pudhு object create aagும் — memory waste, slow. StringBuilder mutable — same internal buffer ah reuse pannikittu append pannும், every time pudhு object venam. Analogy: string — ovvoru correction ku pudhу notebook vaangura madhiri; StringBuilder — same notebook la pencil vachु erase panni edit pannுрा madhiri.
🗣️ Interview la eppadi sollanumStrings are immutable in C#, so every concatenation creates a new string object on the heap. Inside a loop with many concatenations that gets expensive. StringBuilder maintains a mutable internal buffer and appends in place, so I always switch to it when building strings inside loops — like generating a large report or a log message with a lot of dynamic content.
🔁 Follow-up Qs they might fire next
Q: Eppo string use pannanum, eppo StringBuilder? — Fixed, one-time concatenation ku string pothum; loop or dynamic building ku StringBuilder.
Q: string interpolation ($"...") vera option-a? — Small, fixed number of concatenations ku fine, aana loop iterations athigama irundha StringBuilder thaan correct choice.
Q: StringBuilder capacity nu edhу? — Initial buffer size define panna mudiyும், resize operations kammi aagி performance innும் improve aagும்.
4. ref vs out vs in parameters
Normal ah parameter pass pannுmbodhu, value copy pass aagும் (changes method kulla mattும் irukkும்). ref na — variable ah reference ah pass pannுрோம், method kulla change pannina outside kூடa reflect aagும் — aana call panna munnadi variable ku value already assign pannirukkanum. out na — method thaan compulsory ah value assign pannanum (multiple values "return" panna use pannுрோம்), call panna munnadi initialize pannанум nu kidaiyathு. in na — ref madhiri reference ah pass pannும் aana method kulla adhа modify panna mudiyathு — periya struct pass panna, copy avoid panna use pannுрோம்.
🗣️ Interview la eppadi sollanumref passes a variable by reference for both reading and writing, and the caller has to initialize it first. out is similar, but the method is required to assign a value before returning — I've used it in TryParse-style patterns where I need a success flag plus an output value. in passes a read-only reference, which is useful for large structs where I want to avoid the copy cost without letting the method modify it.
🔁 Follow-up Qs they might fire next
Q: int.TryParse la out eppadi use aagும்? — bool TryParse(string s, out int result) — success/failure return pannitு, actual parsed value ah out parameter la kudukкும்.
Q: ref return nu edhу? — Method oru variable ah reference ah return pannலாம், caller andha memory location ah directly modify panna mudiyும் — rare, advanced use case.
Q: in use pannாma vittha enna aagும்? — Periya struct normal ah pass pannina, full copy aagும் every call — in use pannina copy avoid pannலாம்.
5. static vs instance members
static member na class ku ஒரே copy mattும் irukkும் — object create pannாма directly ClassName.Member nu access pannலாம், ella instances-um idha share pannும். instance member na ovvoru object ku separate ah copy irukkும் — object create pannitு thaan access pannanum. Analogy: static — college oda "college name" madhiri, ella students ku ஒரே பெயர்; instance — ovvoru student oda "roll number" madhiri, ovvoruthுkkும் thani.
🗣️ Interview la eppadi sollanumStatic members belong to the type itself and are shared across every instance — like a utility method or a shared counter. Instance members belong to each object separately, so every object holds its own copy of that state. I use static classes for stateless helpers, like a logging utility, and instance members for anything that carries per-object state, like a Student entity's properties.
🔁 Follow-up Qs they might fire next
Q: static constructor eppo run aagும்? — First time andha class access panna mudhல் moment la, automatic ah once mattும் run aagும் — object create panna venam.
Q: static class edhуkaga fully static ah irukanum? — Instance create panna venam nu design pannுmbodhu (utility/helper classes) — object create panna mudiyாma poga vendு static class use pannுрோம்.
Q: DI container la static state problematic-a? — Aama, static mutable state thread-safe illana, multiple requests same field ah touch pannina race condition varும் — so DI la Singleton preferred, static mutable state avoid pannுрோம்.
6. sealed / readonly / const differences
sealed — oru class ah vera class inherit pannாma block pannுрathу (design decision lock panna, example already GetHashCode override pannina class ah innும் override aagாma thadukka). readonly — field ah constructor la mattும் assign pannலாம், apparam change panna mudiyathு, aana value runtime la decide pannலாம் (example: config value DB-la irundhு run time la set aagலாம்). const — compile time ku thaan fix pannанும், value already theriyanum (example: Pi = 3.14), runtime la change panna mudியவே mudியாthу, implicitly static.
🗣️ Interview la eppadi sollanumsealed prevents a class from being further inherited — I use it when I want to lock down a design decision. readonly fields can be set once, either at declaration or in the constructor, and the value can come from runtime logic. const is stricter — the value must be known at compile time, and it's implicitly static, shared across the whole app.
🔁 Follow-up Qs they might fire next
Q: readonly vs const — eppo edhу use pannanum? — Compile-time already theriyum value na const (like PI); runtime-la calculate/config-based value na readonly (like connection string).
Q: sealed method nu edhу? — Base class la virtual method-a override pannitு, appuram andha override-a innும் override aagாma "sealed override" nu podலாம்.
Q: static readonly vs const — enna difference? — const inline aagும் compile time la itself; static readonly runtime la assign aagும், so recompile pannாma dependent DLL update panna mudiyும் without breaking callers.
7. Nullable types & null-coalescing (?? / ??=) & null-conditional (?.)
Value type (int, bool, DateTime) default ah null aaga mudியாthு. int? (Nullable<int>) use pannina, value type kூடa null store panna mudியும் (example: optional DateOfBirth field). ?? operator na — left side null na, right side value use pannு nu sollுрோம் (default value kudukрathу). ??= na — left side null na mattும் assign pannு (already value irundha touch pannாdhு). ?. na — object null na, exception throw pannாma null return pannு (member call pannுмa munnadi manual null check thevaiyilla).
🗣️ Interview la eppadi sollanumNullable value types like int? let a value type hold null, useful for optional database fields. The null-coalescing operator ?? gives a fallback value when the left side is null, and ??= assigns only when the variable is currently null. The null-conditional operator ?. short-circuits to null instead of throwing a NullReferenceException when calling a member on a possibly-null object — I use this a lot navigating nested objects from an API response.
🔁 Follow-up Qs they might fire next
Q: string?.Length ?? 0 eppadi work aagும்? — string null na, ?. null return pannும், apparam ?? andha null-a 0 ku fallback pannும் — safe way to get length without an explicit null check.
Q: Nullable reference types (C# 8+) nu edhу? — Compiler warning kudukкும் variable null aaga chance irundha, nullable annotations context enable pannitு compile-time null safety improve pannலாம்.
Q: HasValue and Value properties eppadi use pannுрோம்? — if(x.HasValue) { var v = x.Value; } — null check pannitு actual value access pannுрathуkки, aana ?? use pannுрathе cleaner.
1B Generics, Delegates & Functional C#
8. Generics (why & how)
Generics illama na, ovvoru data type ku separate ah class/method eluthanum (IntList, StringList...) — code duplicate aagும். Generics vachு, ஒரே class/method ah, edhு type venалும் use panna vidலாம், type ah compile time la decide pannலாம். List<T> namba List<Student>, List<int> nu edhу type venалும் use pannலாம் — type safety kூடa kidaikkும் (wrong type add panna compile error).
🗣️ Interview la eppadi sollanumGenerics let me write a class or method once and reuse it for any data type, with compile-time type safety — instead of writing a separate collection or method per type, or falling back to object and losing type checking. I use a generic repository pattern, like Repository<T>, so the same CRUD logic works across Student, Fee, and Attendance entities in my microservices.
🔁 Follow-up Qs they might fire next
Q: Generic constraints nu edhу? — where T : class or where T : IEntity nu sollitு, T edhு type venалும் aagாdhு, oru specific base type/interface implement pannанும் nu restrict pannுрோம்.
Q: Generic method example sollுngа? — T Max<T>(T a, T b) where T : IComparable<T> — edhு type venалும் compare panni max return pannும்.
Q: Generics vs object type — enna difference? — object use pannina boxing + casting venும், runtime error risk irukkும்; generics compile time la type check aagும், value type ku boxing um venам்.
9. Delegates
Delegate na — oru method ah "point" pannுрa variable, function pointer madhiri nu nenachikonga. Method ah namba directly call pannாма, delegate variable la vachு, appuram andha delegate ah call pannலாம் — same signature irukka edhு method ah venалும் assign pannலாம். Idhு vachு, "enna method run aagும்" nu runtime la decide panna mudiyும் — plug-and-play madhiri.
🗣️ Interview la eppadi sollanumA delegate is a type-safe reference to a method with a matching signature — it lets me pass a method around like a variable and invoke it later, or swap which method runs at runtime. This is the foundation for callbacks, and it's what events, Func, and Action are built on top of internally.
🔁 Follow-up Qs they might fire next
Q: Multicast delegate nu edhу? — Delegate variable oru method mattும் illама, += vechு multiple methods ah chain panna mudiyும், invoke pannுmbodhu ella methods-um sequence ah call aagும்.
Q: Delegate vs interface — eppo edhу use pannanum? — Single method behavior venும் na delegate simple; multiple related methods/state venும் na interface better.
Q: Delegate real project la eppadi use pannirukeenga? — Callback pattern la — async operation complete aana apparam call pannанும் nu oru delegate pass pannலாம், event handler wire panna kூடa delegate thaan base.
10. Events vs Delegates
Event na, delegate mela build panna oru "restricted" version. Delegate ah namba class outside la irundhு direct ah invoke pannலாம், = vachு replace kூடa pannலாம். Event na, outside class andha event ah subscribe (+=) pannலாம் mattум், direct ah invoke panna mudியாthு — class kulла irundhு mattும் trigger panna mudியும். Analogy: delegate oru phone number madhiri — yaru venалும் call pannலாம்; event oru "notification subscribe" madhiri — neenga subscribe pannலாம், aana andha company thaan eppo notification anுppanum nu decide pannும், neenga trigger panna mudியாthு.
🗣️ Interview la eppadi sollanumAn event is built on top of a delegate but restricts access — external code can only subscribe or unsubscribe with += and -=, it can't invoke the event directly or overwrite the subscriber list. That encapsulation is the whole point — I use events for things like OnOrderCreated, where multiple parts of the app need to react, but only the publisher decides when it actually fires.
🔁 Follow-up Qs they might fire next
Q: EventHandler built-in delegate eppadi use pannuvom? — public event EventHandler<OrderEventArgs> OrderCreated; — standard pattern, sender + custom EventArgs pass pannலாம்.
Q: Event illама delegate mattум் use pannina enna problem varும்? — Outside class andha delegate ah = vechு overwrite pannிடலாம், existing subscribers ellам் poidும் — event idhа thadukkும்.
Q: Events multithreaded environment la safe-a? — Not automatically — invoke panna munnadi null check pannанும் (?.Invoke()), race condition possibility irukkு.
11. Func / Action / Predicate
Idhу ellам் built-in generic delegates — namba custom delegate type define panna vendам். Func<T,TResult> — value edukkும், value return pannும் (example: Func<int,int,int> add = (a,b)=>a+b). Action<T> — value edukkும், aana return pannaathு (void). Predicate<T> — value edukkும், bool return pannும் (condition check panna, example Where clause kulla).
🗣️ Interview la eppadi sollanumFunc, Action, and Predicate are pre-built generic delegate types so I don't have to define my own delegate for every scenario. Func returns a value, Action returns void, and Predicate specifically returns a bool for a condition check — LINQ methods like Where use a Predicate-shaped Func internally under the hood.
🔁 Follow-up Qs they might fire next
Q: Func<int,bool> and Predicate<int> — same thing-a? — Almost same signature, aana Predicate<T> specifically meant for boolean conditions; LINQ mostly uses Func internally for consistency.
Q: Action epdi use pannirukeenga real project la? — Callback pass panna — processing complete aana apparam Action<Result> onComplete ah invoke pannுрோம், return value venам்.
Q: Multiple parameters Func la eppadi kudukрathு? — Func<int,int,int,int> nu 3 input types + last one return type, max 16 parameters varaikkும் support pannும்.
12. Lambda expressions
Lambda na — oru chinna, name illама irukka function — (parameters) => expression/body nu eludhுрோம். Namba delegate/Func/Action ku method separate ah define pannама, inline ah short ah eludhalாம். Example: students.Where(s => s.Age > 18) — idhу oru lambda, "s" parameter, "s.Age>18" thaan body/condition.
🗣️ Interview la eppadi sollanumA lambda is an anonymous, inline function — a short way to write a delegate without declaring a separate named method. I use lambdas constantly with LINQ, like students.Where(s => s.Marks > 80), and for quick inline callbacks where writing a whole separate method would be overkill.
🔁 Follow-up Qs they might fire next
Q: Expression-bodied lambda vs statement lambda — difference? — x => x*2 single expression; x => { var y=x*2; return y; } multi-statement body with braces.
Q: Lambda captured variable (closure) nu edhу? — Lambda outside irukka local variable ah "capture" pannikkும், andha variable oda current value ah reference vechு use pannும் — loop variable capture panna careful ah irukkanum.
Q: Lambda vs local function — enna difference? — Local function named, performance konjам் better, recursive ah easy ah call pannலாம்; lambda anonymous, delegate/Func type ku assign pannுрோம்.
13. Extension methods
Existing class ah (namba source code access illatha, example string, List<T> — .NET framework classes) modify pannама, adhуkку pudhу method "add" pannுрathу thaan extension method. static class kulla static method eludhi, first parameter mun this keyword podanum. Appuram andha type oda object mela, andha method ah normal instance method madhiri call pannலாம்.
🗣️ Interview la eppadi sollanumExtension methods let me add new functionality to an existing type without modifying its source or inheriting from it — I write a static method in a static class with 'this' before the first parameter, and it shows up as if it were an instance method on that type. LINQ itself is built entirely with extension methods on IEnumerable<T> — that's how Where, Select, and OrderBy work on any collection.
🔁 Follow-up Qs they might fire next
Q: Extension method example neenga eludhi irukeenga-a? — Aama, example string.IsValidEmail() nu oru extension method eludhi, common validation logic ah multiple places la reuse pannுрோம்.
Q: Extension method vs instance method — priority edhуkku? — Same signature irundha, actual instance method already class-la irundha adhу thaan win pannும், extension method call aagாdhு.
Q: Extension method static class-a irukkanum nu edhуkку compulsory? — It needs to be a stateless static method callable without an object — a regular instance method can't act as an extension.
1C LINQ, Async, Exceptions & Modern C#
14. LINQ basics
LINQ (Language Integrated Query) na, C# code kullaye SQL madhiri query eludhуra mechanism — Where, Select, OrderBy, GroupBy, Join ellам் method ah namba use pannலாம் any collection (list, array, DB via EF Core) mela. Rendு syntax irukkு — method syntax (students.Where(s=>s.Age>18)) and query syntax (from s in students where s.Age>18 select s) — rendும் same result, method syntax thaan mostly use pannுрோம்.
🗣️ Interview la eppadi sollanumLINQ lets me write query operations — filtering, projecting, grouping, joining — directly in C# with a consistent syntax, whether the source is an in-memory list, an array, or a database through EF Core. I mostly use method syntax with lambdas, like students.Where(s => s.Age > 18).OrderBy(s => s.Name), because it reads naturally and chains well.
🔁 Follow-up Qs they might fire next
Q: GroupBy example sollungа? — students.GroupBy(s => s.ClassId) — ovvoru class ku students ah group pannitு, ovvoru group kulla aggregate (count, average) calculate pannலாம்.
Q: Join eppadi LINQ la pannுрோம்? — students.Join(fees, s=>s.Id, f=>f.StudentId, (s,f)=>new{...}) — SQL inner join madhiri, rendு collections ah common key vechு combine pannுрோம்.
Q: LINQ performance impact edhachு irukka? — In-memory collections ku fine, but IQueryable mela over-chaining pannina complex SQL generate aagi slow aaga chance irukkு — execution plan check pannanum.
15. Deferred vs Immediate execution in LINQ
Deferred execution na — namba query define panna udane execute aagаdhு, andha query variable ah actual ah iterate pannுmbodhu (foreach, ToList(), Count() etc call pannுmbodhu) thaan execute aagும். Where, Select ellам் deferred. Immediate execution na — udane execute aagi result return pannும், example ToList(), ToArray(), Count(), First(). Idhу theriyама irundha bug varும் — query define pannitு, appuram underlying data change pannitு, appuram iterate pannina, latest data thaan varும் — stale-a nenaichikkalam, aana adhу illa.
🗣️ Interview la eppadi sollanumDeferred execution means the query isn't run when I define it — it only executes when I actually enumerate it, like with a foreach or ToList(). That matters because if the underlying data changes between defining the query and enumerating it, the query picks up the latest data at execution time. Methods like ToList(), Count(), and First() force immediate execution and materialize the result right away.
🔁 Follow-up Qs they might fire next
Q: Idhу enna bug create pannалам் real project la? — Query define pannitு, munnadi data modify pannitு, appuram loop la use pannina expected-ку vera result varும் — so DB call related query ah ToList() vachு freeze pannitு use pannanum.
Q: Multiple times iterate pannina enna aagும் deferred query ah? — Ovvoru iteration-um query thirும்ba execute aagும் — DB query na, multiple round trips aagும், so ToList() vechு cache pannikanum.
Q: yield return deferred execution ku eppadi related? — yield return use panra iterator methods kூடa lazily execute aagும், ovvoru element mela call panna thaan next element generate aagும்.
16. IEnumerable vs IQueryable
IEnumerable — in-memory collection mela iterate panna use pannுрோம், filter/where panna methods "client side" (memory kulla) execute aagும். IQueryable — EF Core madhiri ORM la use pannுmbodhu, query ah "expression tree" ah build pannikittu poi, final ah namba ToList()/enumerate pannுmbodhu mattум், andha whole query ah SQL ah convert panni DB kitta anுppuм் — so filtering DB level la nadakkும், best performance.
🗣️ Interview la eppadi sollanumIEnumerable executes queries in memory — if I call .Where() on an IEnumerable from a database source, it pulls all the data first and filters client-side. IQueryable builds an expression tree and defers translation to SQL until the query is actually executed, so the filtering happens in the database. That's a common mistake I watch for in code reviews — accidentally converting to a List too early before adding filters, which pulls the entire table into memory.
🔁 Follow-up Qs they might fire next
Q: Real bug example sollungа idhу vechு? — dbContext.Students.ToList().Where(s=>s.Age>18) na ella students-um first memory ku vandhuduchu, appuram filter aagுdhு. dbContext.Students.Where(s=>s.Age>18).ToList() na filter DB level la nadakkும் — correct way.
Q: IQueryable eppo IEnumerable ah maaруm்? — .ToList(), .AsEnumerable(), or foreach start panna udane, andha point la irundhு query execute aagi results kidaikkும்.
Q: Dapper la IQueryable irukka? — Illa, Dapper raw SQL query eludhi direct ah result mapping pannும், IQueryable concept illa — adhу EF Core specific.
17. async/await and Task vs Thread
async/await na — I/O operations (DB call, API call, file read) nadakkுmbodhu, andha thread ah block pannама, free ah vittு vera velai panna vidுрathу. await keyword vecha bodhu, "idhу neram aagum, wait pannама continue pannு" nu compiler ku sollுрோம். Task na — "future ah complete aagappoura velai" nu represent pannуra object, Thread na — actual OS-level execution unit. async/await Task use pannும், aana necessary ah pudhу Thread create pannாthு — I/O bound velaikkு, thread pool thread ah temporarily use pannிட்டு, wait panra podhu andha thread ah release pannitு vidும்.
🗣️ Interview la eppadi sollanumasync/await frees up the calling thread while waiting on an I/O-bound operation like a database call or an HTTP request — the thread pool thread is released back to the pool during the wait and picked up again once the operation completes. Task represents that asynchronous operation and its eventual result, whereas Thread is an actual OS-level unit of execution. This is exactly what let me support 1,000+ concurrent users in my ERP project without exhausting threads.
🔁 Follow-up Qs they might fire next
Q: Task.Run() vs async/await — difference? — Task.Run offloads CPU-bound work to a thread pool thread explicitly; async/await for I/O-bound work doesn't need a dedicated thread at all while waiting.
Q: async void eppo use pannanum? — Almost never — only for top-level event handlers; everywhere else async Task, since async void can't be awaited and unhandled exceptions can crash the app.
Q: ConfigureAwait(false) nu edhу? — Continuation ah original context (like UI thread) ku thirும்ba varanum nu venам் na use pannுрோм் — library code la nalladhு, ASP.NET Core la vithyasam kammi aana still good practice.
18. try/catch/finally + custom exceptions
try block la risky code (DB call, file access, external API) vachுkuvom. catch block la andha exception ah handle pannுрோம் — specific exception type ah first catch pannanum (general Exception ah last la mattум்). finally block — exception vandhalும் varaadhalும், always execute aagும் (resource cleanup, connection close panna use pannுрோம்). Custom exception na — built-in exceptions (ArgumentException etc) pothaadhu na, namba oda business-specific exception class create pannுрோம் (example: InsufficientBalanceException, OtpExpiredException) — Exception ah inherit pannitு.
🗣️ Interview la eppadi sollanumI keep try blocks tight around the risky operation, catch specific exception types first before a general catch, and use finally for cleanup that must always run, like closing a connection. For business rule violations I create custom exceptions inheriting from Exception — like OtpExpiredException in the bank locker project — so the global exception middleware can catch them specifically and return a meaningful error response instead of a generic 500.
🔁 Follow-up Qs they might fire next
Q: Custom exception epdi define pannuvinga? — public class OtpExpiredException : Exception { public OtpExpiredException(string msg) : base(msg) {} } — constructors properly chain pannanum, extra properties (like OtpId) add pannலாம்.
Q: catch(Exception) mattум் use pannina enna problem? — Ellame same ah handle pannirும், specific error type ku specific action edukka mudiyathு, debugging kashtaма aagும் — so specific first, general last nu order pannanum.
Q: finally vs using block — enna difference? — using automatic ah Dispose() call pannும் IDisposable objects ku (cleaner syntax); finally manual ah edhு venалும் cleanup code eludhalாm், more generic.
19. Indexers & Properties (get/set/init)
Property na — field ah direct ah expose pannама, get/set mூlама controlled access kudukрathу (validation, logic add panna mudiyும்). Auto-property na { get; set; } short-cut. init na (C# 9+) — object creation time la mattум் value set panna anுматikkும், appuram immutable aagidum — object initializer syntax la mattும் set panna mudiyும். Indexer na — class ah array madhiri [] vechு access panna vidுрathу — this[int index] nu define pannுрோம், custom collection class ku useful.
🗣️ Interview la eppadi sollanumProperties wrap a field with get/set accessors so I can add validation or logic instead of exposing raw fields — auto-properties are the shorthand when there's no extra logic needed. init accessors, from C# 9, let a property be set only during object initialization and become immutable afterward, which I like for DTOs I want read-only after construction. Indexers let a class be accessed with array-like syntax using this[] — I've used this for a custom cache wrapper class.
🔁 Follow-up Qs they might fire next
Q: init vs readonly property — difference? — readonly field can only be set in constructor code; init property can be set via object initializer syntax too ({ Name = "X" }), giving more flexible immutable construction.
Q: private set vs get-only property? — private set allows the class itself to modify it internally after construction; get-only (no setter) truly can't change after the constructor assigns it.
Q: Indexer example real project la? — Custom cache wrapper class la this[string key] define pannitு, cache["StudentList"] madhiri dictionary access syntax kudukalam.
20. Records vs Classes / pattern matching (switch expressions)
Record (C# 9+) na — immutable, data-focused types ku design panna oru pudhу type. Class-la namba manual ah Equals(), GetHashCode(), ToString() override pannanum value-based comparison ku; record automatic ah idhа ellам் generate pannikkும் — rendு record same property values irundha "equal" nu consider pannும் (reference illa, value comparison). with expression vechு, oru record ah copy pannitு, konjam property mattும் change panna mudiyும் (non-destructive mutation). Pattern matching / switch expression na — traditional switch-case ku modern, concise alternative — type check + value check + condition ellame ஒரே expression la eludhalாம்.
🗣️ Interview la eppadi sollanumRecords are immutable, value-based types — two records with the same property values are considered equal automatically, without me overriding Equals or GetHashCode myself, and I can create a modified copy using a with expression. I use records for DTOs and API response models where I want that immutability and value equality. Switch expressions let me write concise conditional logic, like mapping an enum status to a response message, in a single expression instead of a verbose switch-case block.
🔁 Follow-up Qs they might fire next
Q: record vs record struct — difference? — record is reference type (heap), record struct (C# 10+) is value type (stack) with the same value-equality behavior — pick record struct for small, frequently copied data.
Q: with expression example sollungа? — var updated = student with { Name = "New Name" }; — existing record ah base vechு, oru property mattும் change panni pudhу immutable copy create pannும்.
Q: switch expression example real code la? — var message = status switch { OrderStatus.Pending => "Waiting", OrderStatus.Shipped => "On the way", _ => "Unknown" }; — namba report/dashboard status label mapping ku use pannirukom.
