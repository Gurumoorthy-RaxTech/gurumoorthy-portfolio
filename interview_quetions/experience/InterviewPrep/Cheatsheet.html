Quick facts

  • Routing: attribute routing ([Route("api/[controller]")]) preferred for Web API; conventional routing for MVC views.
  • Model validation: [ApiController] auto-returns 400 on invalid ModelState.
  • Filters order: Authorization → Resource → Action → Exception → Result.
  • Razor Pages = page-focused (one class per page); MVC = controller-centric, better for multi-action APIs/views mix.
  • Versioning: URL segment (/api/v1/) or header-based via Microsoft.AspNetCore.Mvc.Versioning.
  • Swagger/OpenAPI: auto-generates docs via AddSwaggerGen() — critical for microservices contract sharing.
  • WCF vs Web API: WCF = SOAP/config-heavy, multi-protocol; Web API = lightweight REST/JSON over HTTP — used WCF only on one legacy integration.

Core patterns to name-drop

  • Database-per-service — each microservice (admissions, billing, attendance, payroll) owns its own schema.
  • API Gateway (Ocelot) — single entry point: routing, JWT validation, rate limiting, load balancing, request aggregation.
  • Circuit breaker / resilience — Polly for retries + circuit breaking when a downstream service is slow/down.
  • Correlation ID — propagate a request ID across services for tracing in centralized logs.
  • Saga / eventual consistency — used for multi-service transactions like admission→billing.
  • Chatty communication → mitigated with response aggregation at the gateway.

SQL Server quick facts

  • Clustered index = table sorted by key (1 per table). Non-clustered = separate structure with pointer, can have many.
  • Covering index = includes all columns a query needs → avoids key lookup.
  • Fix N+1: batch queries / use Include() in EF or a single JOIN query in Dapper.
  • Deadlocks: caused by two txns locking resources in reverse order → fix with consistent access order, shorter transactions, READ COMMITTED SNAPSHOT.
  • Always use parameterized queries — never string-concatenate SQL (SQL injection).
  • Execution plan: look for Table Scan (bad, add index) vs Index Seek (good).
  • My 40% win = added missing non-clustered indexes on FK/filter columns + converted N+1 loops to stored procs + added in-memory caching for lookup tables.

🍃 MongoDB

  • Used for: unstructured/high-volume data — activity logs, real-time analytics, dashboard feeds in Zen Campus.
  • Document model = BSON, schema-flexible; embed for 1:few tight relations, reference for 1:many/large collections.
  • Indexing: createIndex() on frequently filtered fields; compound indexes for multi-field queries.
  • Aggregation pipeline: $match → $group → $sort → $project for analytics rollups.
  • Polyglot persistence: SQL Server = transactional/relational (billing, admissions); MongoDB = flexible/high-volume (logs, dashboards).
  • CAP theorem: MongoDB tunes toward CP or AP via write/read concern settings.

🔐 JWT / RBAC / Security

JWT structure

header.payload.signature
{alg, typ}.{sub, roles, exp, iat}.HMACSHA256(base64(header)+"."+base64(payload), secret)
  • Access token = short-lived (15-60 min); Refresh token = long-lived, stored securely, used to mint new access tokens.
  • RBAC in ASP.NET Core: [Authorize(Roles = "Admin,Staff")] or policy-based [Authorize(Policy = "CanEditBilling")].
  • Gateway-level auth: Ocelot validates JWT once at the gateway so downstream services trust the token (or re-validate for defense-in-depth).
  • Prevent SQL injection: parameterized queries / stored procs, never raw string concatenation.
  • Enforce HTTPS/TLS everywhere; HSTS in production.
  • Store JWT in httpOnly secure cookie or memory (avoid localStorage for XSS resistance).
  • Audit trail logging on the OTP system — every generate/deliver/validate/expire event logged with timestamp + actor.

🔌 TCP Socket / OTP / SMS Gateway

  • TCP = connection-oriented, reliable, ordered (used for locker hardware — correctness matters more than speed).
  • .NET Socket/TcpListener/TcpClient classes; async via SendAsync/ReceiveAsync.
  • OTP lifecycle: Generate (secure random, 6-digit) → Deliver (SMS gateway API) → Validate (compare + expiry check) → Expire (TTL ~2-5 min, audit-logged).
  • Rate-limit OTP attempts (e.g. 3 tries) to block brute force; lock/cooldown after threshold.
  • Sub-3-second SMS delivery achieved via async dispatch + gateway API with fast provider SLA.
  • Keep-alive + reconnection logic handles intermittent socket drops from locker hardware in the field.

📈 Performance / Logging / Production Issues

  • Centralized exception-handling middleware: app.UseExceptionHandler() or custom middleware catching unhandled exceptions, logging structured error + correlation ID, returning consistent error response.
  • Structured logging: ILogger<T> / Serilog, log as JSON with request ID, user, timestamp, severity.
  • In-memory caching (IMemoryCache) for lookup/reference data (fee structures, class lists) — biggest lever in the 40% perf win.
  • 35% incident-resolution win = centralized logs + correlation IDs meant no more grepping across servers — root cause found from one dashboard.
  • Production outage triage order: check health/monitoring dashboard → check recent deployment → check logs by correlation ID → isolate service → mitigate (rollback/scale/restart) → root-cause after.
  • Memory leak diagnosis: dotnet-counters / dotnet-trace, watch Gen2 heap growth, check for undisposed connections/event handler leaks.
  • Thread pool starvation: caused by blocking async code (.Result/.Wait()) — always await end-to-end.

🌟 STAR Answers — Top Achievements (speak these out loud once before the call)

40% query time reduction: Situation — Zen Campus reports were timing out under load. Task — mine was to optimize the data-access layer. Action — profiled slow queries, added targeted non-clustered/covering indexes, converted hot-path inline queries to stored procs via Dapper, added in-memory caching for reference data. Result — 40% cut in query execution time, reports that took 4-5 seconds now return in under 2.
35% incident resolution improvement: Situation — production bugs took hours to trace across microservices with scattered logs. Task — build a way to find root cause fast. Action — designed centralized logging + global exception-handling middleware with correlation IDs threaded through the Ocelot gateway and every downstream service. Result — resolution time dropped 35%, on-call debugging became a single dashboard search instead of SSH-ing into multiple servers.
Zero unauthorized access (bank locker OTP): Situation — a bank needed a secure, auditable locker access system. Task — build MFA with OTP over SMS integrated with physical locker hardware over TCP. Action — designed the full OTP lifecycle with expiry + attempt rate-limiting, encrypted the TCP channel, added full audit trail logging in SQL Server. Result — zero unauthorized access incidents post-deployment, sub-3-second SMS delivery, 50% faster OTP validation after Dapper tuning.
Microservices at scale (1,000+ users): Situation — Zen Campus needed to serve 1,000+ concurrent users across many modules without one module's load affecting another. Task — architect for independent scalability. Action — split into microservices per domain (admissions, attendance, billing, payroll, inventory), fronted by Ocelot for routing/load balancing/rate limiting, containerized each with Docker for consistent deploys. Result — services scale and deploy independently; a spike in attendance-module traffic during morning check-in doesn't degrade billing.
