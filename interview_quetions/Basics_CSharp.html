❓ Nullable Types — Handling Missing Data

// Nullable value type — int? means "int or null"
int? studentScore = null;    // exam not yet taken
decimal? discount = null;   // no discount applied

// Check before use
if (studentScore.HasValue)
    Console.WriteLine($"Score: {studentScore.Value}");

// Null coalescing — default if null
int score = studentScore ?? 0;          // 0 if null
string phone = student?.Phone ?? "N/A"; // "N/A" if student or phone is null

// Null conditional — safe navigation
string? upperPhone = student?.Phone?.ToUpper(); // null if student or Phone is null

// Null coalescing assignment (C# 8)
student.Phone ??= "0000000000"; // assign only if currently null

// Nullable reference types (C# 8+) — enabled in modern projects
string  name  = "Basha";   // cannot be null (warning if assigned null)
string? email = null;     // explicitly allowed to be null

// Pattern matching null check
if (student is not null)
    Console.WriteLine(student.Name);

⚙️ Methods — Modifiers and Patterns

public void Print(string msg) { Console.WriteLine(msg); }

GetStudents(schoolId: 1, size: 50, page: 2);

{
    student = _db.Students.Find(id);

public decimal Sum(params decimal[] amounts)
    => amounts.Sum(); // can call with any number: Sum(100, 200, 300)

public static class StringExtensions
{
    public static string ToTitleCase(this string s)

🎯 Delegates, Func, Action, Lambda

// Delegate = type that holds reference to a method
public delegate decimal FeeCalculator(int studentId);

// Built-in generic delegates (use these, don't define custom)
// Func<T, TResult> — takes input, returns output
Func<int, decimal> calcFee = (id) => id * 1000m;
decimal fee = calcFee(5);  // 5000

// Action<T> — takes input, returns void
Action<string> log = (msg) => Console.WriteLine($"[LOG] {msg}");

// Predicate<T> — takes input, returns bool
Predicate<Student> isActive = (s) => s.IsActive;

// Lambda expressions — inline anonymous methods
var activeStudents = students.Where(s => s.IsActive).ToList();
var sorted = students.OrderBy(s => s.Name).ToList();
var names = students.Select(s => s.Name).ToList();
var totalFee = students.Sum(s => s.FeeAmount);

🎯 10 Interview Scenarios

1
Why use decimal instead of double for fee amounts?Types

double uses binary floating-point (IEEE 754) — cannot represent 0.1 exactly. 0.1 + 0.2 = 0.30000000000000004 in double. decimal uses base-10 representation — exact for financial calculations. decimal is slower than double but precision is mandatory for money. In ZenCampus: fee amount is decimal. Attendance percentage is double (0.01 precision fine). Rule: money/financial = decimal. Scientific/engineering = double.

2
What is the difference between == and .Equals() for strings?Strings

For strings in C#: == compares content (value), not reference. "Basha" == "Basha" is true. This is because string overloads the == operator. .Equals() also compares content by default. Difference: .Equals() has overload for StringComparison. "BASHA".Equals("basha", StringComparison.OrdinalIgnoreCase) — case-insensitive. Use Equals when case-insensitive comparison needed. For null safety: use == (Equals throws NullReferenceException if left side is null).

3
What is string immutability? Why use StringBuilder?Strings

String is immutable — once created, cannot be modified. "Basha" + "Kumar" creates a NEW string object, discards both originals. In a loop concatenating 1,000 strings: 999 intermediate string objects created and garbage collected. StringBuilder uses a buffer, appends without creating new objects. Rule: 3+ concatenations in a loop → StringBuilder. Simple "Hello " + name → string interpolation fine. For ZenCampus report with 1,000 student rows: StringBuilder, not string +.

4
What is the difference between var and dynamic in C#?Types

var: type inferred at COMPILE TIME. var student = new Student() — compiler knows it's Student. Fully type-safe. IntelliSense works. No runtime overhead. dynamic: type resolved at RUNTIME. dynamic obj = GetSomething() — type unknown at compile time. No IntelliSense. Runtime errors possible. Use var always for local variables. Use dynamic only when working with COM interop, reflection, or deserialized JSON without a model. ZenCampus: var everywhere, dynamic never.

5
Explain the difference between out and ref parameters.Methods

ref: variable must be initialized BEFORE calling the method. Method can read AND write. out: variable doesn't need initialization before call. Method MUST assign a value before returning. Usage: ref = pass existing value that method may modify. out = declare a variable that method will fill. TryParse pattern uses out: int.TryParse("123", out int result). Prefer returning a tuple or using nullable return instead of out parameters in modern C#.

6
What is a lambda expression?Lambda

Lambda: anonymous function written inline. s => s.IsActive is equivalent to bool IsActive(Student s) { return s.IsActive; }. Syntax: (parameters) => expression or (parameters) => { statements; }. Used with LINQ, delegates, Func/Action. Closure: lambda can capture variables from outer scope (be careful with loop variables). students.Where(s => s.SchoolId == currentSchoolId) — captures currentSchoolId.

9
What are generics? Why use List<T> instead of ArrayList?Generics

Generics: write type-safe, reusable code with a type parameter T. List<Student> only accepts Student objects — compile-time type safety. ArrayList accepts any object — runtime cast errors, boxing overhead for value types. Generic method: T GetById<T>(int id) works for any type. Generic class: Repository<T> where T : class. Benefits: type safety, no boxing, IntelliSense, code reuse. Use generics for collections, repositories, result types.

10
What is boxing and unboxing?Types

Boxing: converting value type (int) to reference type (object). Wraps value in heap-allocated object. int x = 5; object boxed = x; — memory allocation on heap. Unboxing: extracting value back. int unboxed = (int)boxed; — explicit cast needed. Performance cost: boxing = heap allocation + copy. Unboxing = type check + copy. Avoid boxing in hot paths. Generics eliminate need for boxing (List<int> vs ArrayList). With modern C#: boxing is rare if you use generics correctly.
