⚡ Async / Await — Non-Blocking C#
🎬 Enthiran (2010) — Chitti: Chitti oru nanosecond-la 1000 calculations parallel panuvan. async/await = same logic — oru task wait panna, thread block aagaadu — parallel-a other work continue aagum. Task.WhenAll() = Chitti's multi-threaded brain!
Syntax — Foundation
async / await — Basic Pattern
// async method — returns Task or Task<T>
public async Task<Student> GetStudentAsync(int id)
{
    // await = release thread until DB responds
    var student = await _repo
                        .GetByIdAsync(id);

    if (student is null)
        throw new KeyNotFoundException(
            $"Student {id} not found");

    return student;
}

// void async — fire and forget (avoid in APIs!)
public async void SendNotification(string msg)
{
    await _sms.SendAsync(msg);
}

// Return types summary
async Task          DoWorkAsync()  { } // no return value
async Task<int>    CountAsync()  { return 5; }
async ValueTask<int> FastAsync()  { return 5; }
❓ Interview Q: What does await actually do?
await = method-a pause panni current thread-a release pannum — thread pool-ku return aagum. DB response varum podhu, execution resume aagum (same or different thread). Thread block aagaadu = scalability!
⚠️ Trap: async void — never use in APIs!
Exception catch panna mudiyaadu. Fire-and-forget ok, but background jobs-ku Task return + _ = Task.Run(...) use pannu.
Syntax — Parallel Await
Task.WhenAll · Task.WhenAny · Task.Run
// Task.WhenAll — parallel execution
public async Task<StudentDashboard>
    GetDashboardAsync(int studentId)
{
    // All 3 start simultaneously!
    var studentTask    = _repo.GetStudentAsync(studentId);
    var attendanceTask = _att.GetSummaryAsync(studentId);
    var feesTask       = _fee.GetBalanceAsync(studentId);

    // Wait for ALL to complete
    await Task.WhenAll(studentTask,
                        attendanceTask,
                        feesTask);

    return new StudentDashboard
    {
        Student    = studentTask.Result,
        Attendance = attendanceTask.Result,
        FeeBalance = feesTask.Result
    };
}

// Task.WhenAny — first one wins
var first = await Task.WhenAny(
    TryPrimaryServer(),
    TryFallbackServer());
var result = await first;

// Task.Run — CPU-bound work off UI thread
var report = await Task.Run(() =>
    GeneratePdfReport(data));
🏫 ZenCampus — Dashboard 3x faster:
Sequential: 300ms + 200ms + 150ms = 650ms
Task.WhenAll: max(300, 200, 150) = 300ms only!
❓ Interview Q: WhenAll vs WhenAny?
WhenAll = all complete aana mattum continue. WhenAny = first one complete aana continue. Timeout pattern-ku WhenAny + delay task use panrom.
Syntax — Cancellation
CancellationToken — Stop Mid-Flight
// Controller — token auto-injected
[HttpGet("{id}")]
public async Task<IActionResult> Get(
    int id,
    CancellationToken ct)   // ← auto from HTTP
{
    var student = await _repo
                        .GetByIdAsync(id, ct);
    return Ok(student);
}

// Repository — pass token to Dapper
public async Task<Student> GetByIdAsync(
    int id, CancellationToken ct)
{
    return await _db.QueryFirstOrDefaultAsync<Student>(
        new CommandDefinition(
            "SELECT * FROM Students WHERE Id=@Id",
            new { Id = id },
            cancellationToken: ct));
}

// Manual cancel after timeout
using var cts =
    new CancellationTokenSource(TimeSpan
        .FromSeconds(5));
await DoWorkAsync(cts.Token);
❓ Interview Q: Why CancellationToken?
User browser close panna / request cancel panna — DB query still runs waste aagum. CancellationToken = request cancelled-a DB query-um cancel aagum. Resource save! ZenCampus report generation-la use panrom.
Syntax — Advanced
ConfigureAwait · Deadlock Prevention
// ConfigureAwait(false) — library code
public async Task<string> FetchDataAsync()
{
    var data = await _httpClient
        .GetStringAsync(url)
        .ConfigureAwait(false); // ← no sync ctx
    return data;
}

// ❌ DEADLOCK — never do in async methods
var result = GetStudentAsync(1).Result;   // blocks!
var result = GetStudentAsync(1).Wait();   // blocks!

// ✅ CORRECT — always await
var result = await GetStudentAsync(1);
💡 Rule: Web API / ASP.NET Core-la synchronisation context illai — ConfigureAwait(false) optional. But class library code ezhuthum podhu always ConfigureAwait(false) use pannu — caller context block aagathu!
❓ Interview Q: How does async deadlock happen?
.Result or .Wait() = current thread block. await = same thread-la resume aaga wait. Thread blocked = deadlock! Fix: async all the way — top to bottom await use pannu.

✅ Async — Do vs Don't

✅ DO THIS
await GetStudentAsync(id);
await Task.WhenAll(t1, t2);
async Task<T> MethodAsync()
.ConfigureAwait(false) // in libs
❌ NEVER DO
GetStudentAsync(id).Result
GetStudentAsync(id).Wait()
async void Method()  // in APIs
// missing await = fire & forget bug
🟣 LINQ — Language Integrated Query
🎬 Vikram (2022) — Rolex: Rolex — precise, chain-la move. LINQ-um same — data-a chain-la filter, transform, group, sort. One clean pipeline — no messy loops! .Where().Select().GroupBy().OrderBy() = Rolex watch gears!
Syntax — Core Operators
Where · Select · OrderBy · First · Count
var students = await _repo.GetAllAsync();

// WHERE — filter
var active = students
    .Where(s => s.IsActive);

// SELECT — project / transform

    .Select(s => s.Name);

// SELECT to anonymous type

    .Select(s => new {
        s.Name, s.Class, s.RollNo });

// ORDER BY
var sorted = students
    .OrderBy(s => s.Name);           // asc

    .OrderByDescending(s => s.Marks); // desc

// FIRST / SINGLE / DEFAULT

❓ Interview Q: Any() vs Count() — which is better?
Any() = first match kadaichudane stop. Count() = full collection traverse. Performance-a "exists check" panna Any() always use pannu. ZenCampus-la active student check = students.Any(s => s.IsActive).

Senior-Level Concept
IQueryable vs IEnumerable — The Critical Difference
// IEnumerable — filter in MEMORY
IEnumerable<Student> q1 = _dbContext
    .Students             // loads ALL rows!
    .AsEnumerable()
    .Where(s => s.IsActive);

// IQueryable — filter in DATABASE (SQL)
IQueryable<Student> q2 = _dbContext
    .Students
    .Where(s => s.IsActive);  // SQL WHERE

// Deferred execution — not run yet!
IQueryable<Student> query = _dbContext
    .Students
    .Where(s => s.IsActive);

// Add more filters — still no DB call
query = query.OrderBy(s => s.Name);

// ToList() / First() = EXECUTES query
var list = await query.ToListAsync();
❓ Interview Q: IQueryable vs IEnumerable — explain with example.
IEnumerable = DB-la ALL rows load panni memory-la filter. IQueryable = filter-a SQL-la translate panni DB-la execute. 50,000 students irundha — IEnumerable = all 50k load! IQueryable = WHERE clause SQL-la = only matching rows. ZenCampus-la always IQueryable + deferred execution use panrom.
💡 Deferred Execution: Query define panna execute aagaadu. .ToList(), .First(), foreach — ithellam trigger aana matum DB-ku poidum. Itha therinja interview gold!

Syntax — Query Style
LINQ Query Syntax vs Method Syntax
// ✅ Method syntax (preferred, more powerful)

    .Where(s => s.IsActive && s.Marks > 60)
    .OrderBy(s => s.Marks)
    .Select(s => new { s.Name, s.Marks })
    .ToList();

❓ Interview Q: When is LINQ executed?

🟢 Generics — Type-Safe Reusable Code
🎬 Kamal Hassan — Dasavathaaram (2008): One actor, 10 different roles — same skill, different character. Generics = same code, works for ANY type. Write once, use with Student, Staff, Fee, Report — type-safe at compile time!
Syntax
Generic Class · Generic Method
// Generic Repository — works for ANY entity
public class Repository<T> where T : class
{
    private readonly IDbConnection _db;

    public async Task<T?> GetByIdAsync(int id)
        => await _db.QueryFirstOrDefaultAsync<T>(
               $"SELECT * FROM {typeof(T).Name}s WHERE Id=@Id",
               new { Id = id });

    public async Task<List<T>> GetAllAsync()
        => (await _db.QueryAsync<T>(
               $"SELECT * FROM {typeof(T).Name}s"))
               .ToList();
}

// Usage — same repo, different types
var studentRepo = new Repository<Student>();
var staffRepo   = new Repository<Staff>();
var feeRepo     = new Repository<Fee>();

// Generic method
public T ParseConfig<T>(string json)
    => JsonSerializer.Deserialize<T>(json)!;

// Call
var cfg = ParseConfig<SchoolConfig>(json);
❓ Interview Q: What are Generics? Why use them?
Generics = type parameter <T> — compile-time type safety with code reuse. Without generics: separate repo for Student, Staff, Fee. With generics: one Repository<T> handles all. Casting illai, boxing illai = performance better!
Syntax — Constraints
Generic Constraints · ApiResponse<T>
// Constraints — restrict what T can be
where T : class           // reference type
where T : struct          // value type
where T : new()           // has default ctor
where T : IEntity         // implements interface
where T : BaseEntity      // inherits class
where T : BaseEntity, new() // multiple

// ZenCampus — Wrapper API response
public class ApiResponse<T>
{
    public bool   Success { get; set; }
    public T?     Data    { get; set; }
    public string Message { get; set; }

    public static ApiResponse<T> Ok(T data)
        => new() { Success=true, Data=data };

    public static ApiResponse<T> Fail(string msg)
        => new() { Success=false, Message=msg };
}

// Usage in controller
return Ok(ApiResponse<Student>.Ok(student));
return BadRequest(ApiResponse<Student>
    .Fail("Student not found"));

// { success:true, data:{...}, message:"" }

🟠 Delegates · Func · Action · Events · Lambda
🎬 96 (2018) — Ram: Ram always waiting for Janu's callback. Delegate = same concept — "call me when done". You pass a method as a parameter — called back when the event fires. Func, Action = modern shorthand for delegates!
Syntax — Foundation
Delegate · Named + Anonymous
// Delegate — type for a method signature
public delegate decimal
    FeeCalculatorDelegate(Student s);

// Named method
public decimal CalcTermFee(Student s)
    => 15000m;

// Assign + invoke
FeeCalculatorDelegate calc = CalcTermFee;
decimal fee = calc(student);

// Anonymous method (older)
FeeCalculatorDelegate anon =
    delegate(Student s) { return 20000m; };

// Lambda (modern — prefer this)
FeeCalculatorDelegate lambda =
    s => 20000m;

// Multicast — chain methods
FeeCalculatorDelegate multi =
    CalcTermFee;
multi += CalcBusFee;  // both called
❓ Interview Q: What is a delegate?
Delegate = type-safe function pointer. Method-a variable-la store panni later call panalaam. Callback patterns, event handlers, strategy pattern-la use panrom. ZenCampus-la fee calculation strategy panna delegate use panrom.
Syntax — Built-in Delegates
Func · Action · Predicate
// Func — has return value
// Func<input1, input2, ..., output>
Func<int, string>       f1 = id  => $"ID:{id}";
Func<int, int, int>    f2 = (a, b) => a + b;
Func<Student, decimal> calcFee =
    s => s.IsHosteller ? 45000m : 15000m;

// Action — no return value (void)
Action<string>    log    = msg =>
    Console.WriteLine(msg);
Action<Student>  notify = s   =>
    _sms.Send(s.Phone, "Fee due");

// Predicate — bool return
Predicate<Student> isActive =
    s => s.IsActive;
bool result = isActive(student);

// Passing Func as parameter
public List<Student> Filter(
    List<Student> all,
    Func<Student, bool> condition)
    => all.Where(condition).ToList();

// Call with lambda
var active = Filter(students,
    s => s.IsActive && s.Marks > 50);
🏫 ZenCampus — Fee strategy via Func:
Func<Student, decimal> strategy =
    student.Category switch
    {
        "Hosteller" => s => 45000m,
        "Bus"       => s => 20000m,
        _           => s => 15000m
    };

decimal fee = strategy(student);
❓ Interview Q: Func vs Action vs Predicate?
Func = return value varum (last type = return). Action = return value illai (void). Predicate = bool return (same as Func<T,bool> but older). LINQ Where = Func<T,bool> internally!
Syntax — Events
Events — Publisher / Subscriber
// Publisher — raises event
public class FeeService
{
    // Event declaration
    public event EventHandler<FeeEventArgs>?
        FeePaid;

    public void ProcessPayment(
        Student s, decimal amount)
    {
        // ... payment logic ...

        // Raise event (notify subscribers)
        FeePaid?.Invoke(this,
            new FeeEventArgs {
                Student = s,
                Amount  = amount
            });
    }
}

// EventArgs
public class FeeEventArgs : EventArgs
{
    public Student Student { get; set; }
    public decimal Amount  { get; set; }
}

// Subscriber — listens to event
_feeService.FeePaid += (sender, e) =>
{
    _sms.Send(e.Student.Phone,
        $"₹{e.Amount} received. Thank you!");
    _audit.Log("Fee paid", e.Student.Id);
};
❓ Interview Q: Delegate vs Event?
Both use same mechanism. Event = restricted delegate — subscribers can only += / -=, cannot invoke or reassign (=). Encapsulated pub-sub pattern. ZenCampus FeePaid event-la SMS + audit both subscribe panranga — decoupled!
Syntax — Lambda
Lambda Expressions — Arrow Functions
// Expression lambda — single expression
Func<int, int> square = x => x * x;

// Statement lambda — multiple lines
Func<Student, string> grade = s =>
{
    if (s.Marks >= 90) return "A+";
    if (s.Marks >= 75) return "A";
    return "B";
};

// Closure — captures outer variable
int minMarks = 60;
Func<Student, bool> passed =
    s => s.Marks >= minMarks; // captures minMarks

// Lambda in LINQ
var toppers = students
    .Where(s => s.Marks > 90)
    .Select(s => new { s.Name, s.Marks })
    .OrderByDescending(s => s.Marks)
    .Take(3)
    .ToList();
❓ Interview Q: What is a closure?
Lambda outer scope variables capture pannum — closure. minMarks lambda-la inside use pannuvu — variable lifetime lambda-kaaga extend aagum. Loop-la closure trap: loop variable capture panna last value matum use aagum!
