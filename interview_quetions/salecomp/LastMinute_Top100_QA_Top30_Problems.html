1. C# & .NET Core (Q1–15)

1.NET Framework vs .NET Core/.NET 5+?
.NET Core is cross-platform, open-source, faster, modular. .NET 5+ unified Framework and Core into one yearly-released platform.
2What does async/await actually do?
Compiler turns the method into a state machine; await frees the thread while waiting instead of blocking it. Continuation resumes on a pool thread when the awaited task completes.
3Task vs Thread?
Task is a higher-level unit of work run on the thread pool, supports return values/composition. Thread is a raw, expensive OS thread — rarely created directly today.
4Why avoid async void?
Caller can't await it or catch its exceptions — an unhandled exception crashes the process. Only acceptable for top-level UI event handlers.
5Value type vs reference type?
Value types (struct, int) hold data directly, usually on the stack. Reference types (class, string) live on the heap; the variable just holds a pointer.
6Abstract class vs interface?
Abstract class can hold state/fields and is single-inherited. Interface is a pure contract (default methods allowed since C# 8), multiple can be implemented.
7What is dependency injection?
Classes depend on abstractions (interfaces), and a container supplies concrete implementations at runtime. Enables loose coupling and easy unit testing via mocks.
83 DI lifetimes in ASP.NET Core?
Transient (new every request), Scoped (one per HTTP request), Singleton (one for app lifetime). Injecting Scoped into Singleton causes the "captive dependency" bug.
9What are generics, and why use them?
Type-safe, reusable code (List<T>) without boxing or runtime casts. Compiler generates specialized code per concrete type used.
10Delegate vs Func vs Action?
Delegate is a custom-signature function pointer type. Func returns a value, Action returns void — both are pre-built generic delegates.
11What is a record type?
Reference type with built-in value-based equality and a `with` expression for non-destructive mutation. Ideal for immutable DTOs.
12throw; vs throw ex;?
`throw;` preserves the original stack trace. `throw ex;` resets it to the current line, losing the real failure location — always use `throw;`.
13lock vs SemaphoreSlim?
lock allows only one thread, no async support (can't await inside it). SemaphoreSlim allows N concurrent callers and supports WaitAsync().
14GC generations — how does the .NET GC work?
Gen 0 (new, frequent, cheap collections), Gen 1 (buffer), Gen 2 (long-lived, rare but expensive). Most objects die in Gen 0.
15SOLID — say it in one line each?
Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion. Enforced daily via small classes and interface-based design.

2. ASP.NET Core & REST APIs (Q16–27)

16What is middleware in ASP.NET Core?
A chain of components each handling the request/response, calling `next()` to pass control along. Order matters — e.g. Authentication before Authorization.
17How do you version a REST API?
URL segment (/api/v1/...), query string, or custom header — URL segment is most common and explicit. Keep old versions running until clients migrate.
18Key HTTP status codes to know?
200 OK, 201 Created, 400 BadRequest, 401 Unauthorized, 403 Forbidden, 404 NotFound, 409 Conflict, 500 ServerError. Return the most specific one, not always 200/500.
19How do you secure a Web API?
JWT Bearer tokens validated via AddJwtBearer, `[Authorize]` on controllers/actions. Role/policy-based authorization for fine-grained access.
20Model validation in ASP.NET Core?
DataAnnotations (`[Required]`, `[MaxLength]`) auto-checked via ModelState, or FluentValidation for complex rules. Return 400 with error details when invalid.
21How do you handle CORS?
`AddCors` policy specifying allowed origins/methods/headers, then `app.UseCors(policyName)`. Never use AllowAnyOrigin with AllowCredentials together.
22Web API controller vs MVC controller?
Web API (`ControllerBase`) returns data (JSON/XML) for clients/SPAs. MVC (`Controller`) returns Views/HTML for server-rendered pages.
23How do you implement pagination in an API?
Accept pageNumber/pageSize query params, use `.Skip().Take()` in the query. Return total count in a header or wrapper object for client-side paging UI.
24What is Swagger/OpenAPI for?
Auto-generates interactive API documentation and a testable UI from your controllers/attributes. Essential for a lead to enforce consistent, discoverable contracts.
25How do you design REST endpoints for a Student/Fee module?
Resource-based nouns: GET/POST /api/students, GET/PUT /api/students/{id}/fees. Use plural nouns, HTTP verbs for actions, not verbs in the URL.
26What is idempotency, and why does PUT need it but POST doesn't?
Idempotent means calling it multiple times has the same effect as once. PUT (full replace) is naturally idempotent; POST (create) is not unless you add a dedupe key.
27How do you implement rate limiting?
Built-in `AddRateLimiter` middleware (.NET 7+) with fixed/sliding window or token bucket policies. Returns 429 TooManyRequests when the limit is exceeded.

3. Entity Framework Core (Q28–37)

28Code-First vs Database-First?
Code-First: classes define the schema, migrations generate the DB. Database-First: scaffold classes from an existing DB — better for a legacy DB you don't own the schema design of.
29What are EF Core migrations?
Versioned, incremental scripts that evolve the DB schema to match your model changes. Apply in production carefully — review the generated SQL first.
30AsNoTracking() — when to use it?
Skips EF's change-tracking overhead for read-only queries, faster and less memory. Never use it if you plan to modify and SaveChanges() the entities.
31What is the N+1 query problem?
Looping over parent rows triggers one extra query per row for lazy-loaded related data. Fix with `.Include()`/`.ThenInclude()` for eager loading.
32How do you handle optimistic concurrency in EF Core?
Add a `[Timestamp]`/RowVersion column; EF checks it on SaveChanges and throws `DbUpdateConcurrencyException` on conflict. Catch it and reload/merge or prompt the user.
33Find() vs FirstOrDefault()?
Find() checks the local change-tracker cache first before hitting the DB, keyed by primary key only. FirstOrDefault() always queries and supports any filter condition.
34How do you improve EF Core query performance?
Project only needed columns (`.Select()`), use AsNoTracking, avoid N+1, add proper indexes, consider split queries for multiple Includes. Profile with the generated SQL, not guesswork.
35How do you implement soft delete in EF Core?
Add an `IsDeleted` flag and a global query filter (`HasQueryFilter`) so deleted rows are auto-excluded everywhere. Override delete logic to set the flag instead of removing the row.
36DbContext lifetime in ASP.NET Core?
Registered Scoped by default — one instance per HTTP request, not thread-safe for concurrent use. Never inject it into a Singleton directly; use IServiceScopeFactory instead.
37IEnumerable vs IQueryable in EF Core queries?
IQueryable builds an expression tree translated to SQL by EF, deferred until enumerated. Switching to IEnumerable too early (e.g. early ToList()) pulls data into memory before further filtering.

4. MSSQL / SQL Server (Q38–49)

38Clustered vs non-clustered index?
Clustered index physically sorts the table data by the key — only one per table. Non-clustered is a separate structure with pointers back to the data — many allowed per table.
39What is a covering index?
A non-clustered index that includes all columns a query needs, so SQL Server never touches the base table. Fastest possible read path for that specific query.
40How do you diagnose a slow query?
Check the Actual Execution Plan for table scans, missing index warnings, and high-cost operators. Add a targeted index or rewrite the query (avoid SELECT *, functions on indexed columns).
41What are SQL Server transaction isolation levels?
Read Uncommitted, Read Committed (default), Repeatable Read, Serializable, Snapshot. Higher isolation = more consistency but more locking/blocking.
42What causes a deadlock and how do you prevent it?
Two transactions lock resources in opposite order and wait on each other forever. Always access tables/rows in the same consistent order across the app to avoid it.
43WHERE vs HAVING?
WHERE filters rows before grouping/aggregation. HAVING filters groups after GROUP BY has been applied (e.g. HAVING COUNT(*) > 5).
44What is a CTE, and when do you use a recursive one?
A Common Table Expression is a named temporary result set for one query, improves readability. Recursive CTEs handle hierarchical data like org charts or category trees.
45Normalization vs denormalization?
Normalization reduces redundancy via related tables (better writes/integrity). Denormalization duplicates data for faster reads — used in reporting tables when read speed matters more.
46What are window functions (ROW_NUMBER, RANK)?
Compute a value across a set of rows related to the current row without collapsing them, using OVER/PARTITION BY. Common for "top N per group" or running totals.
47How do you migrate a large production table with minimal downtime?
Add new columns as nullable first, backfill in batches, then add constraints — avoid one giant blocking ALTER. Use online index rebuilds where supported.
48Stored procedure vs view?
A view is a saved SELECT statement, queryable like a table but can't contain business logic/branching. A stored procedure can contain logic, parameters, and multiple statements/transactions.
49How do you index a table with heavy read AND write load?
Every index speeds reads but slows writes (each insert/update must maintain it) — so index only columns actually used in WHERE/JOIN/ORDER BY. Balance count and width of indexes against write throughput needs.

5. ADO.NET (Q50–55)

50When would you use raw ADO.NET over EF Core?
For maximum performance on hot-path queries, complex stored-procedure-heavy legacy DBs, or bulk operations. EF Core adds convenience but some overhead vs raw ADO.NET.
51Connected vs disconnected architecture?
Connected (SqlDataReader) keeps the connection open, forward-only streaming, fastest. Disconnected (DataSet/DataAdapter) loads into memory then closes the connection, editable offline.
52How do you prevent SQL injection in ADO.NET?
Always use parameterized queries (`@Param` + `cmd.Parameters`), never string concatenation. The DB treats parameter values as data, never as executable SQL syntax.
53How do you call a stored procedure with an OUTPUT parameter?
Set `cmd.CommandType = StoredProcedure`, add a parameter with `Direction = ParameterDirection.Output`. Read its `.Value` after `ExecuteNonQuery()`/`ExecuteReader()` completes.
54What is connection pooling and how do you avoid exhausting it?
ADO.NET reuses physical connections per connection string instead of opening new ones each time. Always wrap SqlConnection in `using` so it's returned to the pool promptly.
55How do you bulk insert 100,000 rows efficiently?
Use `SqlBulkCopy` to stream all rows in one round-trip via the TDS bulk protocol. Far faster than looping individual INSERT statements one row at a time.

6. WinForms (Q56–60)

56How do you update UI from a background thread safely?
Use `async`/`await` (resumes on UI thread automatically) or `Control.Invoke`/`BeginInvoke`. Directly touching a control from a non-UI thread throws "Cross-thread operation not valid".
57What is data binding in WinForms (BindingSource)?
BindingSource decouples the UI grid/controls from the actual data source, supporting filtering/sorting/navigation. Changes in the data source can auto-refresh bound controls.
58How do you validate input in a WinForms form?
Use the `Validating` event per control plus `ErrorProvider` for inline error icons/messages. Block form submission (`e.Cancel = true`) until all required fields pass.
59How do you avoid freezing the UI during a long operation?
Run the work with `await Task.Run(...)` instead of blocking the UI thread synchronously. Show a progress indicator and re-enable controls once the awaited task completes.
60How do you deploy a WinForms app?
ClickOnce for easy auto-updating installs, or an MSI/Setup project for controlled enterprise deployment. Modern option: publish as a self-contained single-file exe.

7. HTML / CSS / JavaScript (Q61–70)

61Explain the CSS box model.
Every element = content + padding + border + margin, stacked outward. `box-sizing: border-box` makes width include padding/border for easier layout math.
62let vs const vs var?
`var` is function-scoped and hoisted (legacy, avoid). `let`/`const` are block-scoped; `const` can't be reassigned but objects/arrays it holds are still mutable.
63What is a closure in JavaScript?
A function that "remembers" variables from its outer scope even after that outer function has returned. Commonly used for private state or event-handler-specific data.
64What is the JavaScript event loop?
Single call stack runs sync code; async callbacks queue up (microtasks like Promises run before macrotasks like setTimeout). The loop pulls from queues only when the stack is empty.
65Flexbox vs Grid — when to use each?
Flexbox is best for one-dimensional layouts (a row or column of items). Grid is best for two-dimensional layouts (rows AND columns together, like a dashboard).
66== vs === in JavaScript?
`==` compares with type coercion (1 == "1" is true). `===` compares value AND type strictly — always prefer `===` to avoid subtle bugs.
67What is a Promise, and async/await in JS?
A Promise represents a future value from an async operation (pending/fulfilled/rejected). `async/await` is syntax sugar over Promises, making async code read like synchronous code.
68What is event bubbling in the DOM?
An event fired on a child element propagates upward through its ancestors unless stopped. `event.stopPropagation()` halts it; used for event delegation on a parent container.
69Responsive design — how do you achieve it?
Media queries adapt CSS to screen size, combined with relative units (%, rem, vw) and Flexbox/Grid. Mobile-first CSS (base styles for small screens, override for larger) is the standard approach.
70How would you optimize a web portal's frontend performance?
Bundle/minify JS-CSS, lazy-load images/routes, cache static assets with proper headers. Reduce unnecessary re-renders and large third-party script payloads.

8. Angular / React (Q71–80)

71Angular vs React — how do you decide?
Angular is a full opinionated framework (DI, routing, forms built in) — good for large structured teams. React is a lighter UI library, more flexible, faster to start, needs more third-party choices.
72What are React hooks (useState/useEffect)?
`useState` adds local state to a function component. `useEffect` runs side effects (API calls, subscriptions) after render, with a dependency array controlling when it re-runs.
73How does Angular's dependency injection compare to ASP.NET Core's?
Both use a container to inject dependencies via constructors, following the same DI principle. Angular's is provided at module/component level; ASP.NET Core's is provided at app startup (Transient/Scoped/Singleton).
74How do you call a REST API from Angular/React?
Angular uses `HttpClient` with Observables (`.subscribe()`); React uses `fetch`/`axios` with Promises or async/await. Always handle errors (catchError / try-catch) instead of assuming success.
75Two-way binding (Angular) vs one-way data flow (React)?
Angular's `[(ngModel)]` auto-syncs the UI and model both directions. React only flows data down via props; the child must call a passed-in function to notify the parent of changes.
76How do you manage state in a large Angular/React app?
NgRx (Angular) or Redux/Context API (React) centralize state instead of passing props/services everywhere. Use it when many unrelated components need the same shared data.
77What is a React/Angular component lifecycle?
React: mount → render → update → unmount, hooked via useEffect. Angular: ngOnInit → ngOnChanges → ngOnDestroy, via lifecycle interfaces.
78What is lazy loading in Angular/React?
Load a module/route's code only when the user actually navigates to it, not upfront. Reduces initial bundle size and speeds up first page load.
79What is a directive in Angular?
A class that adds behavior to DOM elements — structural (`*ngIf`, `*ngFor` — change DOM structure) or attribute (`ngClass` — change appearance/behavior). Custom directives encapsulate reusable DOM logic.
80Virtual DOM in React — why does it matter?
React keeps an in-memory copy of the DOM, diffs it against the previous version, and updates only the changed real DOM nodes. Much faster than re-rendering the whole page on every state change.

9. Azure / AWS Cloud (Q81–88)

81How would you host a .NET Core Web API on Azure?
Azure App Service — easiest, managed PaaS, auto-scaling, deployment slots. AKS/Kubernetes only if you need container orchestration at real scale.
82What are App Service deployment slots?
Separate staging environments (same App Service) to deploy and warm up a new version before swapping it into production. Swap is near-instant with easy rollback if issues appear.
83How do you manage secrets securely in Azure?
Azure Key Vault stores connection strings/API keys, accessed via Managed Identity — no secrets in appsettings.json. Rotate secrets centrally without redeploying code.
84IaaS vs PaaS vs SaaS?
IaaS = raw VMs, you manage everything (Azure VM). PaaS = platform managed for you, you deploy code (App Service). SaaS = fully managed application (Office 365).
85Azure SQL Database vs SQL Server on a VM?
Azure SQL Database is fully managed (patching, backups, scaling handled for you). SQL Server on a VM gives full OS/DB control but you manage everything yourself.
86AWS EC2/S3/RDS/Lambda — Azure equivalents?
EC2≈Azure VM, S3≈Blob Storage, RDS≈Azure SQL/Database services, Lambda≈Azure Functions. Same cloud concepts, different vendor names.
87What is Azure Functions/serverless, and when to use it?
Run small pieces of code on-demand without managing servers, billed per execution. Good for event-driven tasks (queue trigger, timer job) not needing an always-on app.
88How do you monitor a production app in Azure?
Application Insights for request tracing, exceptions, and performance metrics. Set up alerts on error rate/response time thresholds to catch issues before users report them.

10. DevOps / CI-CD (Q89–95)

89How would you design a CI/CD pipeline for a .NET Core app?
CI: build → run unit tests → code analysis on every commit/PR. CD: deploy to Dev → QA → Staging → Prod, each gated by approvals and passing tests.
90What is a YAML pipeline in Azure DevOps?
Pipeline-as-code defining build/test/deploy stages in a versioned YAML file alongside the source. More maintainable and reviewable than the old classic UI-based pipelines.
91Blue-green vs canary deployment?
Blue-green switches all traffic at once between two identical environments — instant rollback. Canary rolls out to a small % of users first, gradually increasing if healthy.
92How do you roll back a failed production deployment?
Swap back to the previous deployment slot (Azure) or redeploy the last known-good build/artifact. Automated rollback triggers on health-check/smoke-test failure post-deploy.
93What is Docker, and why containerize a .NET Core app?
Packages the app plus its exact runtime/dependencies into one portable image. Guarantees "works on my machine" also works identically in prod.
94What is Kubernetes, and do you always need it?
Orchestrates, scales, and self-heals many containers across a cluster. Overkill for a simple app — App Service/simple container hosting is enough unless you truly need multi-service orchestration at scale.
95What is Infrastructure as Code (Terraform/ARM/Bicep)?
Define cloud infrastructure in version-controlled config files instead of manual portal clicks. Enables repeatable, reviewable, disaster-recoverable environment setup.

11. Leadership & Analytical (Q96–100)

96How do you review a junior developer's PR effectively?
Check correctness and design first, then style — explain the "why" behind each comment, not just "change this". Approve incrementally so small good PRs aren't blocked by nitpicks.
97How do you handle a technical disagreement with a teammate?
Discuss tradeoffs with data/prototypes rather than opinions, and time-box the debate. If still split, the person who owns the long-term maintenance usually gets the final call.
98How do you handle a production incident end-to-end?
Stabilize first (rollback/mitigate) before root-causing — restore service, then investigate. Follow up with a blameless post-mortem and a concrete prevention action item.
99How do you approach technical debt with stakeholders?
Frame it in business terms — cost of delay, risk of outage, slower future delivery — not just "the code is messy". Propose small incremental payoffs bundled with feature work, not a big-bang rewrite ask.
100Why should we hire you for this lead/senior role?
Lead with concrete outcomes — a system you owned end-to-end, a team you mentored, a production issue you resolved. Tie it directly back to their JD (.NET Core/MSSQL/leadership) in one tight closing sentence.

1Reverse a string without using built-in Reverse(). 
static string Reverse(string s)
{
    var arr = s.ToCharArray();
    int i = 0, j = arr.Length - 1;
    while (i < j) { (arr[i], arr[j]) = (arr[j], arr[i]); i++; j--; }
    return new string(arr);
}

4Find the second highest number in an array.

5Check if two strings are anagrams. 
static bool IsAnagram(string a, string b) =>

SELECT * FROM OrgChart;

30Analytical: A production report is randomly slow — walk through your debugging process.
Reproduce with the actual slow parameters, check the SQL execution plan for scans/missing indexes, check for N+1 queries in the app layer, check server CPU/memory/blocking at that time via monitoring, then fix the root cause (index, query rewrite, or caching) — not just increase the timeout.
