DatabaseInitializer.cs — Code-First only

public static void EnsureCreated()
{
    string connectionString = ConfigurationManager.ConnectionStrings["SchoolDb"].ConnectionString;

    const string sql = @"
        IF NOT EXISTS (SELECT * FROM sys.tables WHERE name = 'Students')
        BEGIN
            CREATE TABLE Students (
                Id INT IDENTITY(1,1) PRIMARY KEY,
                Name NVARCHAR(100) NOT NULL,
                ClassName NVARCHAR(50) NULL,
                Age INT NOT NULL,
                Email NVARCHAR(150) NULL
            );
        END";

    using (var conn = new SqlConnection(connectionString))
    using (var cmd = new SqlCommand(sql, conn))
    {
        conn.Open();
        cmd.ExecuteNonQuery();
    }
}

IF NOT EXISTS (SELECT * FROM sys.tables WHERE name = 'Students') — idhu safety check, table already irundhaalum error varaadhu, andha CREATE TABLE skip pannidum. ExecuteNonQuery() use pannurom, aeandha DDL statement-ku result rows edhuvum varadhu.

▶️ How to Run

Database-First

  1. Open SSMS, connect to 192.168.1.52 with the SchoolCloud login.
  2. Open and run Run_This_First.sql against the SchoolTest database — creates the Students table + 3 sample rows.
  3. Open SchoolApp_DatabaseFirst.sln in Visual Studio.
  4. Press F5 — the grid should show the 3 sample students immediately.

Code-First

  1. Open SchoolApp_CodeFirst.sln in Visual Studio — no SQL script needed first.
  2. Press F5 — DatabaseInitializer.EnsureCreated() creates the Students table automatically if it's not already there.
  3. Grid starts empty (unless you already ran the other project against the same DB) — use the Add button to create your first student.

🛠️ Troubleshooting

"A network-related or instance-specific error..." — can't reach 192.168.1.52. Check you're on the same network/VPN as that SQL Server, and that its firewall allows port 1433.
"Login failed for user 'SchoolCloud'" — password typo, or SQL Authentication isn't enabled on that server (it might be Windows-Auth-only) — confirm with whoever manages that server.
"A connection was successfully established... but then an error occurred during the pre-login handshake" / SSL/TLS errors — usually fixed by keeping TrustServerCertificate=True (already set) — if it still fails, the SQL Server's "Force Encryption" setting may need checking with the DBA.
"Invalid object name 'Students'" on the Database-First project — you forgot to run Run_This_First.sql first, or ran it against a different database than SchoolTest.

🎯 Interview Scenarios

1
Why is running DatabaseInitializer.EnsureCreated() on every app startup risky in production?Architecture

If multiple instances of the app start at the same time (e.g., several client machines launching simultaneously), they could all run the CREATE TABLE check concurrently, causing race conditions or lock contention. More importantly, in a real production system you generally want schema changes reviewed and applied in a controlled deployment step (like EF Core migrations run deliberately, or a DBA-reviewed script) — not silently triggered by whichever client happens to run first. This sample uses it for convenience in a small desktop app; for a shared multi-user system, Database-First (schema owned and deployed separately) is usually safer.

2
Why does this project use ADO.NET directly instead of Entity Framework, given EF Core has built-in migrations?Trade-off

The role's stated focus is ADO.NET and WinForms specifically — showing raw ADO.NET fluency (SqlConnection/SqlCommand, parameterized queries, manual mapping) demonstrates the fundamentals EF abstracts away. It's also lighter-weight for a small desktop tool with one simple table, avoids adding EF as a dependency, and gives full control over the exact SQL executed — useful when the app needs to run against an existing, DBA-managed database where hand-tuned queries matter more than rapid CRUD scaffolding.

3
What would you change to make this app support multiple users editing at once safely?Concurrency

Right now, Update() blindly overwrites by Id with no concurrency check — "last save wins." To fix: add a RowVersion/timestamp column to Students, include it in the UPDATE's WHERE clause (WHERE Id=@Id AND RowVersion=@RowVersion), and check ExecuteNonQuery()'s return value — if 0 rows were affected, someone else changed the row first, so show a "record was modified, please refresh" message instead of silently succeeding.

4
The search box calls the database on every keystroke (TextChanged). Is that a problem, and how would you fix it?Performance

For a small local table it's harmless, but at scale (thousands of rows, remote/slow DB link) it means one query per keystroke — wasteful and can make the UI feel laggy if the network round-trip is slow. Fix: debounce with a short Timer (e.g., 300ms) that resets on every keystroke and only fires the actual search after typing pauses, or filter an already-loaded in-memory list/DataTable via BindingSource.Filter instead of hitting the DB each time.

5
How would you extend Insert + a related table (e.g., Fees) to be atomic, like the ADO.NET guide's transaction example?Transactions

Open one connection, call BeginTransaction(), pass that transaction object into every SqlCommand used for both the Student insert and the Fee insert, and only Commit() after both succeed — Rollback() in a catch block if either fails. See the ADO.NET guide's Concept #6 for the full pattern — it's a direct drop-in extension of this repository's Insert() method.
