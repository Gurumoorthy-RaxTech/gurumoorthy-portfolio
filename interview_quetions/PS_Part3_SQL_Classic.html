Second Highest Salary — 6 Different Methods!
Medium
📋 Table: Employees (Id, Name, Salary, DeptId)
Data: 90000, 80000, 70000, 80000 → Answer: 80000
Method 1 — OFFSET FETCH (Modern SQL Server)
✅ Method 1: OFFSET FETCH — Cleanest
SELECT DISTINCT Salary
FROM   Employees
ORDER BY Salary DESC
OFFSET 1 ROWS FETCH NEXT 1 ROWS ONLY;
-- DISTINCT: handles duplicate salaries
-- OFFSET 1: skip the highest
-- FETCH 1:  take one (= 2nd highest)
Method 2 — MAX excluding highest (Most popular in interviews)
✅ Method 2: MAX with subquery
SELECT MAX(Salary) AS SecondHighest
FROM   Employees
WHERE  Salary < (SELECT MAX(Salary) FROM Employees);
-- Inner: MAX=90000
-- Outer: MAX of all <90000 → 80000 ✅
Method 3 — DENSE_RANK (Best for generalizing to Nth)
✅ Method 3: DENSE_RANK — Works for ANY Nth!
WITH Ranked AS (
    SELECT Salary,
           DENSE_RANK() OVER
               (ORDER BY Salary DESC) AS Rnk
    FROM   Employees
)
SELECT DISTINCT Salary
FROM   Ranked
WHERE  Rnk = 2;    -- change for Nth!
-- 90000→Rnk1, 80000→Rnk2 (both 80000s!), 70000→Rnk3
Method 4 — NOT IN Subquery
Method 4: NOT IN
SELECT MAX(Salary)
FROM   Employees
WHERE  Salary NOT IN
           (SELECT MAX(Salary) FROM Employees);
Method 5 — TOP 1 from TOP 2
Method 5: TOP 1 from TOP 2 (Classic SQL Server)
SELECT TOP 1 Salary
FROM (
    SELECT DISTINCT TOP 2 Salary
    FROM   Employees
    ORDER BY Salary DESC
) T
ORDER BY Salary ASC;
-- Inner: {90000,80000}. Outer: smallest = 80000
Method 6 — NULL-safe production version
✅ Method 6: NULL safe (returns NULL if no 2nd salary)
SELECT ISNULL(
    (SELECT DISTINCT TOP 1 Salary
     FROM (
         SELECT DISTINCT TOP 2 Salary
         FROM   Employees
         ORDER BY Salary DESC
     ) T
     ORDER BY Salary ASC),
    NULL
) AS SecondHighest;
-- If only 1 unique salary → returns NULL gracefully
🎤 Complete Senior Dev Answer
"I present 5+ approaches for this classic question:
1. OFFSET FETCH — cleanest syntax, SQL Server 2012+
2. MAX subquery — simplest, universally understood
3. DENSE_RANK — most flexible: change '2' to any N for Nth salary. Also handles ties correctly — two employees with same salary both get same rank
4. NOT IN — simple but watch out: if salary column has NULLs, NOT IN returns empty set!
5. TOP 1 from TOP 2 — classic SQL Server pattern

Production recommendation: DENSE_RANK because (a) generalizes to Nth without rewrite, (b) handles duplicate salaries correctly, (c) explicit about intent. Also add ISNULL wrapper for NULL safety when table has only one distinct salary."

Nth Highest Salary — Generic Stored Procedure with @N
Medium
CREATE PROCEDURE usp_GetNthHighestSalary
    @N INT
AS
BEGIN
    SET NOCOUNT ON;

    IF @N < 1
    BEGIN
        RAISERROR('N must be >= 1', 16, 1);
        RETURN;
    END;

    WITH Ranked AS (
        SELECT Salary,
               DENSE_RANK() OVER
                   (ORDER BY Salary DESC) AS Rnk
        FROM   Employees
    )
    SELECT DISTINCT
           ISNULL(
               (SELECT DISTINCT Salary
                FROM  Ranked
                WHERE Rnk = @N),
               NULL
           ) AS NthHighest;
END;

-- EXEC usp_GetNthHighestSalary @N = 1   → 90000
-- EXEC usp_GetNthHighestSalary @N = 2   → 80000
-- EXEC usp_GetNthHighestSalary @N = 3   → 70000
-- EXEC usp_GetNthHighestSalary @N = 99  → NULL
🎤 Key Points
"SET NOCOUNT ON prevents 'rows affected' messages from cluttering result sets. Input validation with RAISERROR before proceeding. ISNULL handles when N exceeds distinct salary count. In SQL Server, stored procedures are preferred for parameterized repeated queries — execution plan cached, security via EXECUTE permission (not direct table access)."

Find Duplicate Records in Students Table
Easy
✅ Method 1: GROUP BY HAVING COUNT > 1
-- Find duplicate phone numbers
SELECT   PhoneNo,
         COUNT(*) AS OccurrenceCount
FROM     Students
GROUP BY PhoneNo
HAVING   COUNT(*) > 1
ORDER BY OccurrenceCount DESC;

-- Find duplicates on multiple columns
-- (same student admitted twice in same school)
SELECT   RollNo, SchoolId,
         COUNT(*) AS Cnt
FROM     Students
GROUP BY RollNo, SchoolId
HAVING   COUNT(*) > 1;
✅ Method 2: Show FULL duplicate rows with ROW_NUMBER
WITH Dups AS (
    SELECT *,
           ROW_NUMBER() OVER (
               PARTITION BY RollNo, SchoolId
               ORDER BY     Id
           ) AS Rn
    FROM   Students
)
SELECT * FROM Dups
WHERE  Rn > 1;  -- Rn=1 is "original", Rn>1 are copies
🎤 Experienced Answer
"GROUP BY HAVING finds WHICH values are duplicated. But to see the full duplicate rows (all columns), use ROW_NUMBER PARTITION BY the duplicate key columns. Rn=1 is the first occurrence (keep), Rn>1 are duplicates. This pattern is also used for deletion. ZenCampus: prevents a student being admitted twice, catches data entry errors."

Delete Duplicate Rows — Keep only one record
Medium
-- Safe pattern: SELECT first, then DELETE

-- Step 1: Preview what will be deleted
WITH Dups AS (
    SELECT *,
           ROW_NUMBER() OVER (
               PARTITION BY RollNo, SchoolId
               ORDER BY     CreatedDate ASC
           ) AS Rn   -- order by date keeps OLDEST
    FROM   Students
)
SELECT * FROM Dups WHERE Rn > 1; -- preview

-- Step 2: Delete after confirming
WITH Dups AS (
    SELECT *,
           ROW_NUMBER() OVER (
               PARTITION BY RollNo, SchoolId
               ORDER BY     CreatedDate ASC
           ) AS Rn
    FROM   Students
)
DELETE FROM Dups WHERE Rn > 1;
-- SQL Server allows DELETE from CTE — deletes from base table!
🎤 Experienced Answer — Safety First!
"ALWAYS preview with SELECT before running DELETE — change only the final statement. ORDER BY in PARTITION BY controls which copy to KEEP: ORDER BY Id ASC keeps oldest (smallest Id), ORDER BY CreatedDate DESC keeps newest. SQL Server allows DELETE directly from a CTE — it targets the underlying base table. Wrap in a transaction (BEGIN TRAN ... COMMIT/ROLLBACK) for extra safety in production."

Employees Earning Above Department Average
Medium
✅ Method 1: CTE with AVG per Dept
WITH DeptAvg AS (
    SELECT   DeptId,
             AVG(Salary * 1.0) AS AvgSalary
    FROM     Employees
    GROUP BY DeptId
)
SELECT e.Name, e.Salary, e.DeptId,
       ROUND(da.AvgSalary, 2) AS DeptAvg
FROM   Employees  e
JOIN   DeptAvg    da
    ON da.DeptId = e.DeptId
WHERE  e.Salary > da.AvgSalary
ORDER BY e.DeptId, e.Salary DESC;

-- Salary*1.0: force decimal division (int/int = int in SQL!)
Method 2: Window AVG (no join needed)
SELECT Name, Salary, DeptId,
       AVG(Salary * 1.0) OVER
           (PARTITION BY DeptId) AS DeptAvg
FROM   Employees
WHERE  Salary >
    (SELECT AVG(Salary) FROM Employees e2
     WHERE e2.DeptId = Employees.DeptId);

-- Cannot filter on window function in WHERE — use CTE method above
-- Window version: shows the average alongside each row (for display)
🎤 Key Trap — Integer Division!
"In SQL, integer ÷ integer = integer (5/2 = 2, not 2.5!). Always use Salary * 1.0 or CAST(Salary AS DECIMAL) to force floating-point division. Critical: cannot use window functions in WHERE clause — window functions evaluated AFTER WHERE. Use CTE or subquery to pre-compute department averages, then JOIN/filter."

Top 3 Students by Marks Per Class — PARTITION BY
Hard
WITH ClassRanked AS (
    SELECT
        s.Name,
        s.Class,
        s.Section,
        e.Marks,
        DENSE_RANK() OVER (
            PARTITION BY s.Class   -- reset rank per class
            ORDER BY     e.Marks DESC
        ) AS ClassRank
    FROM     Students    s
    JOIN     ExamResults e
        ON   e.StudentId = s.Id
    WHERE    e.AcademicYear = '2025-26'
    AND      e.Term         = 'Term1'
)
SELECT   Class, ClassRank, Name, Marks
FROM     ClassRanked
WHERE    ClassRank <= 3
ORDER BY Class, ClassRank, Name;

-- ROW_NUMBER vs RANK vs DENSE_RANK for ties:
-- Two students with 95 marks, then one with 90:
-- ROW_NUMBER:  1, 2, 3  (arbitrary tie-break — UNFAIR!)
-- RANK:        1, 1, 3  (gap after tie — position 2 unused)
-- DENSE_RANK:  1, 1, 2  (no gap — FAIR for merit lists!)

-- Change ClassRank <= 3 to <= 5 for top 5
-- Change PARTITION BY Class to PARTITION BY Class,Section for section-wise
🎤 ROW_NUMBER vs RANK vs DENSE_RANK — Must know this!
"PARTITION BY resets the rank counter for each partition (class). This is the key difference from just ORDER BY which ranks globally. DENSE_RANK is the right choice for merit lists — if two students share 2nd place, both get rank 2 and the next gets rank 3 (no gaps). ROW_NUMBER would arbitrarily decide who is 2nd and 3rd — unfair. RANK gives gaps (1,1,3) — position 2 is 'unused'."

Running Total (Cumulative Sum) — Month-wise Fee Collection
Medium
WITH Monthly AS (
    SELECT
        FORMAT(PaidDate, 'yyyy-MM') AS Month,
        SUM(Amount)                  AS MonthlyTotal
    FROM     Fees
    WHERE    IsPaid = 1
    GROUP BY FORMAT(PaidDate, 'yyyy-MM')
)
SELECT
    Month,
    MonthlyTotal,
    SUM(MonthlyTotal) OVER (
        ORDER BY Month
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS RunningTotal
FROM   Monthly
ORDER BY Month;

-- Month    MonthlyTotal  RunningTotal
-- 2026-01  50000         50000
-- 2026-02  75000         125000
-- 2026-03  60000         185000
-- 2026-04  80000         265000
🎤 Experienced Answer
"SUM() OVER (ORDER BY) is a window aggregate — adds all values from first row up to current row (ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW — this is the default when you specify ORDER BY). The inner GROUP BY collapses daily payments to monthly totals. ZenCampus finance dashboard: shows how total fee collection grows month-over-month through the academic year. Add PARTITION BY SchoolId for per-school running totals."

Students Without Fee Records — LEFT JOIN Anti-Join Pattern
Easy
✅ Method 1: LEFT JOIN + WHERE NULL (Fastest)
SELECT   s.Id, s.Name, s.Class,
         s.AdmissionNo
FROM     Students  s
LEFT JOIN   Fees  f
    ON   f.StudentId = s.Id
WHERE    f.Id IS NULL   -- no fee record at all
ORDER BY s.Class, s.Name;
Method 2: NOT EXISTS (Semantic, short-circuits)
SELECT Id, Name, Class
FROM   Students s
WHERE NOT EXISTS (
    SELECT 1    -- SELECT 1 is intentional (no columns needed)
    FROM   Fees
    WHERE  StudentId = s.Id
);
⚠️ Method 3: NOT IN — DANGEROUS with NULLs!
SELECT Id, Name
FROM   Students
WHERE  Id NOT IN (
    SELECT StudentId
    FROM   Fees
    WHERE  StudentId IS NOT NULL  -- CRITICAL!
);
-- WARNING: If ANY StudentId is NULL in Fees,
-- NOT IN returns ZERO rows! SQL NULL logic (UNKNOWN).
-- Always add WHERE StudentId IS NOT NULL
🎤 Experienced Answer — NULL Trap!
"Three approaches. LEFT JOIN + WHERE NULL is the 'anti-join' pattern — fastest in most cases. NOT EXISTS is semantically clearest and short-circuits on first match. NOT IN is the dangerous one — if the subquery returns even ONE NULL, the entire NOT IN result is empty! SQL three-valued logic: any comparison with NULL = UNKNOWN, and WHERE UNKNOWN = nothing. Always add WHERE StudentId IS NOT NULL in NOT IN subqueries. I always prefer LEFT JOIN or NOT EXISTS in production."

Self Join — Show Employee with their Manager
Medium
-- Employees table has ManagerId (FK to same table)
SELECT
    e.Name          AS EmployeeName,
    e.Salary        AS EmployeeSalary,
    ISNULL(m.Name, '[CEO]') AS ManagerName
FROM      Employees e
LEFT JOIN Employees m       -- same table, different alias
    ON    m.Id  = e.ManagerId
ORDER BY  e.Name;

-- LEFT JOIN: CEO has no manager (ManagerId=NULL) → shows as [CEO]
-- INNER JOIN would exclude the CEO from results

-- Find employees earning MORE than their manager
SELECT e.Name AS Employee,
       e.Salary,
       m.Name   AS Manager,
       m.Salary AS ManagerSalary
FROM   Employees e
JOIN   Employees m
    ON m.Id = e.ManagerId
WHERE  e.Salary > m.Salary;
🎤 Experienced Answer
"Self join joins a table to itself using different aliases — same physical table, two logical roles. 'e' = employee, 'm' = manager. LEFT JOIN includes root nodes (CEO with no manager). Common in org charts, category hierarchies, parent-child relationships. ZenCampus: class teacher → subject teacher → HOD hierarchy all in one Staff table."

Find Consecutive Attendance Streaks — Islands and Gaps
Hard
-- Attendance: StudentId, AttendanceDate, IsPresent
-- Find longest consecutive present streak per student
WITH Numbered AS (
    SELECT
        StudentId,
        AttendanceDate,
        ROW_NUMBER() OVER (
            PARTITION BY StudentId
            ORDER BY     AttendanceDate
        ) AS Rn
    FROM   Attendance
    WHERE  IsPresent = 1  -- only present days
),
Islands AS (
    SELECT
        StudentId,
        AttendanceDate,
        -- Key trick: date - row_number = constant for consecutive dates!
        DATEADD(day, -Rn, AttendanceDate) AS GroupDate
    FROM   Numbered
)
SELECT
    StudentId,
    MIN(AttendanceDate) AS StreakStart,
    MAX(AttendanceDate) AS StreakEnd,
    COUNT(*)            AS ConsecutiveDays
FROM     Islands
GROUP BY StudentId, GroupDate
ORDER BY ConsecutiveDays DESC;
🎤 The Island Trick — Key insight!
"The island-and-gaps technique: Date - RowNumber = constant for consecutive dates (because both increment by 1 each day). If a day is missing, the date jumps by 2 but row number by 1, so the result changes — new 'island' starts. GROUP BY this constant 'GroupDate' groups each consecutive streak. Brilliant O(n log n) solution. ZenCampus: find students with 100% attendance streaks for awards."

Pivot Table — Class-wise Subject Marks as Columns
Hard
-- Input: StudentId, Subject, Marks
-- Output: StudentId | Maths | Science | English

-- Static PIVOT (known subjects)
SELECT StudentId, Maths, Science, English
FROM (
    SELECT StudentId, Subject, Marks
    FROM   ExamResults
) AS src
PIVOT (
    MAX(Marks)
    FOR Subject IN ([Maths], [Science], [English])
) AS pvt;

-- CONDITIONAL AGGREGATION (works in all DBs, more flexible)
SELECT
    StudentId,
    MAX(CASE WHEN Subject = 'Maths'   THEN Marks END) AS Maths,
    MAX(CASE WHEN Subject = 'Science' THEN Marks END) AS Science,
    MAX(CASE WHEN Subject = 'English' THEN Marks END) AS English
FROM     ExamResults
GROUP BY StudentId;
🎤 PIVOT vs CASE WHEN
"SQL Server PIVOT syntax is concise but rigid — you must know column values at query-write time. CASE WHEN approach works in all databases (MySQL, PostgreSQL, Oracle) and is more readable. For dynamic subjects (unknown at query time), use Dynamic SQL — build the column list as a string, then EXEC. ZenCampus: report cards with all subjects as columns use this pivot pattern."

Year-over-Year Growth using LAG() Window Function
Hard
WITH YearlyRevenue AS (
    SELECT
        YEAR(PaidDate)  AS Yr,
        SUM(Amount)    AS TotalFees
    FROM     Fees
    WHERE    IsPaid = 1
    GROUP BY YEAR(PaidDate)
)
SELECT
    Yr,
    TotalFees,
    LAG(TotalFees, 1, 0) OVER
        (ORDER BY Yr)     AS PrevYearFees,
    TotalFees
        - LAG(TotalFees, 1, 0) OVER
            (ORDER BY Yr) AS GrowthAmt,
    ROUND(
        (TotalFees
            - LAG(TotalFees) OVER(ORDER BY Yr))
        * 100.0
        / NULLIF(LAG(TotalFees) OVER(ORDER BY Yr), 0),
    2) AS GrowthPct
FROM   YearlyRevenue
ORDER BY Yr;

-- Yr    TotalFees  PrevYearFees  GrowthAmt  GrowthPct
-- 2024  800000     0             800000     NULL
-- 2025  950000     800000        150000     18.75
-- 2026  1100000    950000        150000     15.79

-- LAG(col, 1, 0): 1=look 1 row back, 0=default if no prev row
-- NULLIF(val, 0): returns NULL if val=0, prevents divide-by-zero!
🎤 Experienced Answer
"LAG(column, N, default) gets the value N rows before the current row. First row returns default (I use 0). LEAD() does opposite — looks forward. NULLIF(expr, 0) is the elegant divide-by-zero guard — returns NULL instead of error, and NULL in arithmetic produces NULL (not error). ZenCampus management uses this for annual revenue growth KPIs. Same pattern for month-over-month, or PARTITION BY SchoolId for per-school trends."

Recursive CTE — Organization Hierarchy (CEO to staff)
Hard
WITH OrgHierarchy AS (
    -- Anchor: top level (no manager)
    SELECT  Id, Name, ManagerId, Designation,
            0              AS Level,
            Name           AS Path
    FROM    Staff
    WHERE   ManagerId IS NULL  -- root (Principal)

    UNION ALL

    -- Recursive: join children to their parent
    SELECT  s.Id, s.Name, s.ManagerId, s.Designation,
            h.Level + 1,
            h.Path + ' → ' + s.Name
    FROM    Staff          s
    JOIN    OrgHierarchy   h
        ON  h.Id = s.ManagerId
)
SELECT
    Level,
    REPLICATE('  ', Level) + Name AS IndentedName,
    Designation,
    Path
FROM   OrgHierarchy
ORDER BY Path;

-- Output:
-- Level  IndentedName          Designation
-- 0      Principal             Principal
-- 1        HOD-Science         Head of Dept
-- 2          Teacher-Phy       Physics Teacher
-- 2          Teacher-Chem      Chemistry Teacher
-- 1        HOD-Maths           Head of Dept
🎤 Experienced Answer
"Recursive CTE has two parts: ANCHOR (base case — root nodes, no parent) and RECURSIVE MEMBER (joins children to parent, increments level). SQL Server adds MAXRECURSION option to prevent infinite loops (circular references). REPLICATE(' ',Level) indents each level visually. Path column traces the full chain. ZenCampus: renders org chart, calculates leave approval hierarchy, shows who reports to whom."

Find Gaps in Roll Number Sequence
Medium
-- Find missing roll numbers in a class
-- Roll numbers should be 1,2,3,...N with no gaps

WITH Expected AS (
    SELECT  MIN(RollNo)                     AS StartRoll,
            MAX(RollNo)                     AS EndRoll
    FROM    Students WHERE ClassId = 5
),
Numbers AS (
    SELECT TOP (SELECT EndRoll-StartRoll+1 FROM Expected)
           ROW_NUMBER() OVER (ORDER BY (SELECT NULL))
             + (SELECT StartRoll-1 FROM Expected) AS Num
    FROM    sys.objects a
    CROSS JOIN sys.objects b
)
SELECT Num AS MissingRollNo
FROM   Numbers
WHERE  Num NOT IN
    (SELECT RollNo FROM Students
     WHERE  ClassId = 5
     AND    RollNo IS NOT NULL);

-- ZenCampus: detect missing roll numbers in a class
-- Same pattern for missing invoice numbers, ticket IDs
🎤 Key Pattern
"Generate a complete sequence (using row_number over sys.objects cross join — a tally table trick), then LEFT JOIN or NOT IN to find what's missing. Alternative: use LAG() to compare each roll number with the previous — if gap > 1, the numbers in between are missing. ZenCampus: audit roll number assignment, find invoice gaps for compliance."

First Fee Payment per Student — ROW_NUMBER Pattern
Medium
-- Find each student's FIRST fee payment
WITH FirstPayment AS (
    SELECT
        StudentId,
        Amount,
        PaidDate,
        ReceiptNo,
        ROW_NUMBER() OVER (
            PARTITION BY StudentId
            ORDER BY     PaidDate ASC
        ) AS Rn
    FROM   Fees
    WHERE  IsPaid = 1
)
SELECT
    s.Name,
    fp.Amount   AS FirstPaymentAmt,
    fp.PaidDate AS FirstPaymentDate,
    fp.ReceiptNo
FROM   FirstPayment fp
JOIN   Students     s
    ON s.Id = fp.StudentId
WHERE  fp.Rn = 1   -- only first payment
ORDER BY fp.PaidDate;

-- Change Rn = 1 to Rn = 2 for second payment
-- Change ASC to DESC for LATEST payment
🎤 Experienced Answer
"PARTITION BY StudentId + ORDER BY PaidDate ASC + ROW_NUMBER gives sequential numbers within each student's payments. Rn=1 = first. This is far better than a correlated subquery (SELECT MIN(PaidDate)...) because it returns ALL columns of the first row, not just the date. Classic pattern for 'first purchase', 'last login', 'most recent order'."

Monthly Admission Trend with Moving Average
Hard
WITH MonthlyAdm AS (
    SELECT
        FORMAT(AdmissionDate, 'yyyy-MM') AS Mon,
        COUNT(*)                           AS NewStudents
    FROM     Students
    WHERE    YEAR(AdmissionDate) = 2026
    GROUP BY FORMAT(AdmissionDate, 'yyyy-MM')
)
SELECT
    Mon,
    NewStudents,
    SUM(NewStudents) OVER
        (ORDER BY Mon)             AS CumulativeTotal,
    AVG(NewStudents * 1.0) OVER (
        ORDER BY Mon
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    )                              AS MovingAvg3Month
FROM   MonthlyAdm
ORDER BY Mon;

-- Mon      NewStudents  Cumulative  MovingAvg3
-- 2026-01  120          120         120.00
-- 2026-02  85           205         102.50
-- 2026-03  140          345         115.00  ← avg of 120,85,140
-- 2026-04  90           435         105.00  ← avg of 85,140,90

-- ROWS BETWEEN 2 PRECEDING AND CURRENT ROW = 3-month window
-- Change 2 to 5 for 6-month moving average
🎤 Experienced Answer — Window Frame!
"ROWS BETWEEN N PRECEDING AND CURRENT ROW defines the window frame size. This gives a rolling 3-month average — smooths out seasonal spikes. CumulativeTotal uses default frame (UNBOUNDED PRECEDING to CURRENT ROW). Moving average is a common business analytics pattern — used in finance, attendance, enrollment trends. ZenCampus management dashboard: shows if admission numbers are growing or declining, identifies seasonal patterns."
