📦 Azure Blob Storage — Student Documents

// NuGet: Azure.Storage.Blobs

public class AzureBlobStorageService(BlobServiceClient _blobClient) : IFileStorageService
{
    private const string StudentDocsContainer = "student-documents";
    private const string PhotosContainer = "student-photos";

    public async Task<string> UploadStudentPhotoAsync(
        int studentId, Stream photoStream, string contentType,
        CancellationToken ct = default)
    {
        var container = _blobClient.GetBlobContainerClient(PhotosContainer);
        await container.CreateIfNotExistsAsync(cancellationToken: ct);

        var blobName = $"students/{studentId}/photo_{DateTime.UtcNow:yyyyMMdd}.jpg";
        var blobClient = container.GetBlobClient(blobName);

        await blobClient.UploadAsync(photoStream, new BlobUploadOptions
        {
            HttpHeaders = new BlobHttpHeaders { ContentType = contentType },
            Tags = new Dictionary<string, string>
            {
                ["studentId"] = studentId.ToString(),
                ["uploadedBy"] = "ZenCampus"
            }
        }, ct);

        return blobClient.Uri.ToString();
    }

    public async Task<string> GenerateSasUriAsync(string blobName, int expiryMinutes = 30)
    {
        var blobClient = _blobClient
            .GetBlobContainerClient(PhotosContainer)
            .GetBlobClient(blobName);

        var sasUri = blobClient.GenerateSasUri(BlobSasPermissions.Read,
            DateTimeOffset.UtcNow.AddMinutes(expiryMinutes));
        return sasUri.ToString();
    }
}

🔐 Key Vault + Managed Identity — Zero Credentials

// Program.cs — Azure Key Vault configuration provider
using Azure.Identity;

if (builder.Environment.IsProduction())
{
    // DefaultAzureCredential tries: Managed Identity → VS credentials → CLI credentials
    var keyVaultUri = new Uri(builder.Configuration["KeyVault:Uri"]!);
    builder.Configuration.AddAzureKeyVault(keyVaultUri, new DefaultAzureCredential());
}

// Key Vault stores:
// ZenCampusDb--ConnectionString   → SqlServer connection string
// Jwt--SecretKey                  → JWT signing key
// Msg91--ApiKey                   → SMS service API key
// BlobStorage--ConnectionString   → Azure Storage connection string

// Access like normal configuration — Key Vault is transparent
var connStr = builder.Configuration.GetConnectionString("ZenCampusDb"); // from Key Vault

// Registration with DefaultAzureCredential (no passwords in code!)
builder.Services.AddSingleton(sp => new BlobServiceClient(
    new Uri($"https://{accountName}.blob.core.windows.net"),
    new DefaultAzureCredential()));  // Managed Identity in Azure, VS/CLI in dev
Managed Identity = Zero-credential architecture. App Service gets a system-assigned identity. That identity is granted "Key Vault Secrets User" role. No passwords, no connection strings in config files, no rotation headaches. If credentials leak — there are no credentials to leak.

🎯 10 Interview Scenarios

1
What is Managed Identity and why is it the best approach for connecting ZenCampus to Azure SQL?Core

Managed Identity: Azure-assigned identity for App Service (like a service account). Grant that identity "db_datareader/datawriter" role in Azure SQL. Connection string becomes "Server=...;Authentication=Active Directory Managed Identity". No username/password. No rotation. If app is compromised, attacker can't steal credentials — there are none. Works seamlessly with DefaultAzureCredential in code.

2
How does Azure Key Vault integrate with ASP.NET Core configuration?Code

AddAzureKeyVault() adds Key Vault as a configuration source. Secrets use double-dash naming (ZenCampusDb--ConnectionString = ConnectionStrings:ZenCampusDb). On startup, app reads all accessible secrets and merges them into IConfiguration. Code uses GetConnectionString() normally — doesn't know or care that it came from Key Vault. No code changes needed.

3
What are Azure App Service deployment slots?Concept

Named deployments within the same App Service (e.g., "staging", "canary"). Each slot has its own URL, settings, and can run a different version. Deploy ZenCampus v2 to staging, test with real traffic, then swap staging↔production (instant, zero-downtime). If issues: swap back immediately. Slot-specific settings (like pointing staging to test DB) don't swap — production keeps its production connection string.

4
How would you store and retrieve student photos in Azure Blob Storage?Code

Container "student-photos" set to private. Upload via BlobClient.UploadAsync with ContentType set. Generate time-limited SAS URI for display (30-minute expiry). Never make containers public — student photos are private. Organize: students/{studentId}/photo_20260624.jpg. Set lifecycle policy: archive after 1 year, delete after 7 years. Use CDN for frequently accessed photos.

5
What is DefaultAzureCredential and how does it work in different environments?Concept

DefaultAzureCredential tries multiple auth methods in order: Environment variables → Workload identity → Managed Identity → Visual Studio credentials → Azure CLI credentials → Browser. In Azure: Managed Identity is picked automatically. On dev machine: Visual Studio or CLI credentials. Same code works everywhere — no environment-specific auth branches. One credential class for all environments.

6
Explain Azure Elastic Pool for a multi-school ZenCampus setup.Architecture

Problem: 200 schools, each with own Azure SQL database = 200 × min DTUs = expensive. Elastic Pool: pool of shared resources (DTUs/vCores) that databases draw from as needed. School exam time = that school's DB spikes, borrows from pool. At night all databases quiet, pool is mostly idle. One pool for all schools vs 200 independent databases = 60–80% cost reduction.

7
What does Application Insights automatically capture without any code changes?Concept

Auto-captured: HTTP requests (URL, method, response code, duration), dependency calls (SQL, HTTP, Redis), exceptions and stack traces, page views (if JS SDK added), performance counters (CPU, memory), request failures, slow requests. For ZenCampus: every /api/students GET, every SQL query duration, every exception in fee calculation — all captured with zero custom code. Useful for production diagnostics.

8
How would you configure auto-scaling for ZenCampus during exam season?Architecture

App Service Standard tier: set auto-scale rules. Scale out when CPU > 70% for 5 minutes (add instance). Scale in when CPU < 30% for 15 minutes (remove instance). Set min instances: 2 (for HA), max: 10. For exam season: schedule-based scaling — automatically add 5 instances during exam hours. Session affinity: disabled (stateless app). Pair with Azure SQL elastic pool scaling.

9
Why should you use SAS tokens for blob access instead of making containers public?Security

Public container = anyone with URL can access student photos — GDPR/privacy violation. SAS (Shared Access Signature): time-limited URL (e.g., 30 minutes), specific permissions (read only), specific blob or container. After expiry, URL is useless. Can be revoked via policy. For ZenCampus: generate fresh SAS when parent views child's photo. URL expires before it could be shared or abused.

10
What is the difference between App Service Plan and App Service?Concept

App Service Plan: the VM farm (compute resources — size, scale, region, OS). You pay for the plan regardless of how many apps run on it. App Service: the application running on that plan. You can host multiple App Services (ZenCampus API, ZenCampus Admin, ZenCampus Parent portal) on one plan to share the cost. Plan = infrastructure. App Service = app deployed on that infrastructure.
