🔵 C# — Loops, Arrays, Collections, Exception Handling
🎬 Baasha (1995): Manikkam — quiet panra, but sollana confirm panruvan. C# — oru discipline-a follow panra language. Every loop, every check — purposeful, no waste!
Syntax
for Loop
// Classic for loop
for (int i = 0; i < 10; i++)
{
    Console.WriteLine(i);
}

// Reverse loop
for (int i = 9; i >= 0; i--)
{
    Console.WriteLine(i);
}
🏫 ZenCampus Example:
// Print roll numbers 1–40
for (int roll = 1; roll <= 40; roll++)
{
    Console.WriteLine($"Roll: {roll}");
}
❓ Interview Q: When do you use for vs foreach?
for = index venum podhu (modify/reverse). foreach = just read panna. Performance same — readability matter!
Syntax
foreach Loop
var students = new List<string>
    { "Ram", "Sita", "Guna" };

foreach (var student in students)
{
    Console.WriteLine(student);
}
🏫 ZenCampus Example:
// Mark attendance for each student
foreach (var s in students)
{
    s.IsPresent = true;
    await _repo.SaveAttendanceAsync(s);
}
⚠️ Trap: foreach-la modify pannaadhe!
Collection modify panna InvalidOperationException varum. ToList() eduthu loop pannu.
Syntax
while & do-while
// while — condition true-a check first
int count = 0;
while (count < 5)
{
    Console.WriteLine(count);
    count++;
}

// do-while — at least once execute
int page = 1;
do
{
    var data = FetchPage(page++);
} while (data.HasMore);
🏫 ZenCampus Example:
// Retry fee payment 3 times
int attempts = 0;
bool success = false;
while (attempts < 3 && !success)
{
    success = await _payment.ProcessAsync();
    attempts++;
}
❓ Interview Q: while vs do-while?
while = condition check first (0 times possible). do-while = at least once execute. Retry logic-ku do-while perfect.
Syntax
Arrays & Collections
// Array — fixed size
int[] marks = { 90, 85, 78, 92 };
int[] arr   = new int[5];

// List — dynamic size
var names = new List<string>();
names.Add("Guru");
names.Remove("Guru");
names.Contains("Guru");

// Dictionary — key/value
var dict = new Dictionary<int, string>();
dict[1]  = "Student A";
dict.TryGetValue(1, out var val);
🏫 ZenCampus Example:
// Class → Student list mapping
var classStudents =
  new Dictionary<string, List<Student>>();
classStudents["10A"] = await _repo
                       .GetByClassAsync("10A");
❓ Interview Q: List vs Array?
Array = fixed size, faster access. List = dynamic, flexible. 90% of time List use panrom. ZenCampus student list = List<Student>.
Syntax
Exception Handling — try / catch / finally
try
{
    var result = await _service.GetStudentAsync(id);
    return Ok(result);
}
catch (KeyNotFoundException ex)
{
    // Specific exception first
    return NotFound(ex.Message);
}
catch (SqlException ex)
{
    _logger.LogError(ex, "DB error");
    return StatusCode(500);
}
catch (Exception ex)
{
    // Generic catch — last always
    throw; // re-throw keeps stack trace
}
finally
{
    // Always runs — cleanup here
    _connection.Close();
}
🏫 ZenCampus — Global Exception Middleware:
// Program.cs
app.UseExceptionHandler(err =>
{
    err.Run(async ctx =>
    {
        ctx.Response.StatusCode = 500;
        await ctx.Response.WriteAsJsonAsync(
            new { error = "Something went wrong" });
    });
});
❓ Interview Q: throw vs throw ex — difference?
throw = original stack trace preserve. throw ex = stack trace reset (bad!). Always use bare throw to re-throw.
⚠️ Trap: catch (Exception ex) { } — empty catch
Exception swallow aagum — bug hide aagum. Always log or re-throw!
Syntax
Null Handling — ?. ?? ??=
// Null conditional — ?.
string? name = student?.Name;
int?    age  = student?.Profile?.Age;

// Null coalescing — ??
string display = name ?? "Unknown";

// Null coalescing assignment — ??=
student.Email ??= "no-email@school.com";

// Null check
if (student is null)
    throw new KeyNotFoundException();

// Pattern matching with null
if (student is not null)
    Process(student);
🏫 ZenCampus Example:
// Safe chaining — no NullReferenceException
var busRoute = student?.Transport?
                              .Route?.Name
               ?? "No Bus Assigned";
❓ Interview Q: ?? vs ??= difference?
?? = return right side if left is null (read-only). ??= = assign right side IF variable is null. Interview-la ithaye sollu with example!
Syntax
String Operations — Interpolation, StringBuilder
// String interpolation (prefer this)
string msg = $"Student {student.Name} — Roll {student.RollNo}";

// Verbatim string
string path = @"C:\Reports\2026";

// String methods
name.ToUpper();
name.Trim();
name.Contains("Guru");
name.StartsWith("G");
name.Replace("old", "new");
name.Split(',');

// StringBuilder — loop-la concatenation
var sb = new StringBuilder();
foreach (var s in students)
    sb.AppendLine(s.Name);
string result = sb.ToString();
🏫 ZenCampus Example:
// SMS message builder
var sb = new StringBuilder();
foreach (var fee in pendingFees)
    sb.AppendLine(
        $"{fee.StudentName}: ₹{fee.Amount} due");
_sms.Send(parentPhone, sb.ToString());
❓ Interview Q: string vs StringBuilder?
string = immutable (each + creates new object). Loop-la string+ = memory waste. StringBuilder = mutable buffer — loop-la always use pannu.
Syntax
var, const, readonly, switch
// var — type inferred at compile time
var students = new List<Student>();
var count    = 42;

// const — compile-time constant
const int MaxStudents = 60;

// readonly — runtime constant
private readonly IStudentRepo _repo;

// switch expression (C# 8+)
string grade = marks switch
{
    >= 90            => "A+",
    >= 75 and < 90  => "A",
    >= 60 and < 75  => "B",
    _                => "C"
};
🏫 ZenCampus Example:
// Fee category switch
decimal fee = student.Category switch
{
    "Day Scholar"  => 15000m,
    "Hosteller"    => 45000m,
    "Bus"          => 20000m,
    _              => 15000m
};
❓ Interview Q: const vs readonly?
const = compile-time, only primitives/strings, implicitly static. readonly = runtime, any type, can be set in constructor. DI-inject panna readonly use panrom.
