Find All Duplicate Elements in an Array
Easy
✅ Method 1: HashSet — O(n) time, O(n) space
public static List<int> FindDuplicates(int[] arr)
{
    var seen  = new HashSet<int>();
    var dupes = new HashSet<int>();
    foreach (var n in arr)
        if (!seen.Add(n)) dupes.Add(n);
    return dupes.ToList();
}
// [1,2,3,2,4,3,5] → [2,3]
Method 2: LINQ GroupBy
var dupes = arr
    .GroupBy(x => x)
    .Where(g => g.Count() > 1)
    .Select(g => g.Key)
    .ToList();
Bonus: Count frequency of each element
var freq = arr
    .GroupBy(x => x)
    .ToDictionary(g => g.Key, g => g.Count());
// [1,2,2,3,3,3] → {1:1, 2:2, 3:3}
🎤 Experienced Answer
"HashSet.Add() returns false if element already exists — elegant way to detect duplicates. Using two HashSets: 'seen' tracks all, 'dupes' tracks duplicates (second HashSet prevents reporting the same duplicate multiple times if it appears 3+ times). LINQ GroupBy is clean for functional style. ZenCampus: find students enrolled twice in same class."

Find Missing Number in 1..N Array
Easy
✅ Sum Formula — O(n) O(1)
public static int FindMissing(int[] arr, int n)
{
    // Sum of 1..n = n*(n+1)/2
    int expected = n * (n + 1) / 2;
    return expected - arr.Sum();
}
// n=5, [1,2,4,5] → expected=15, sum=12, missing=3
XOR Trick — even more elegant!
public static int FindMissingXOR(int[] arr, int n)
{
    int xor = 0;
    for (int i=1; i<=n; i++) xor ^= i;
    foreach (var x in arr)         xor ^= x;
    return xor; // a^a=0, so pairs cancel, missing remains
}
// Avoids int overflow risk with sum for large n
🎤 Experienced Answer
"Two approaches. Sum formula: n*(n+1)/2 is the Gaussian formula for sum 1..n. Missing = expected - actual sum. Simple, O(1) space. XOR approach: x^x = 0, so XOR all 1..n with all array elements — matching pairs cancel, only the missing number remains. XOR is safer for very large n where sum might overflow long. I'd mention both and let them choose."

Two Sum — find indices of pair adding to target
Medium
✅ HashMap — O(n) time, O(n) space
public static (int, int) TwoSum(int[] nums, int target)
{
    var map = new Dictionary<int,int>(); // val → index
    for (int i=0; i<nums.Length; i++)
    {
        int complement = target - nums[i];
        if (map.TryGetValue(complement, out int j))
            return (j, i);
        map[nums[i]] = i;
    }
    return (-1,-1);
}
// nums=[2,7,11,15], target=9
// i=0: complement=7, not seen. map={2:0}
// i=1: complement=2, found at 0! → (0,1) ✅
Three Sum — extension (sorted + two pointer)
public static List<(int,int,int)> ThreeSum(int[] nums)
{
    var result = new List<(int,int,int)>();
    Array.Sort(nums);
    for (int i=0; i<nums.Length-2; i++)
    {
        int l=i+1, r=nums.Length-1;
        while (l<r)
        {
            int s=nums[i]+nums[l]+nums[r];
            if      (s==0) { result.Add((nums[i],nums[l],nums[r])); l++; r--; }
            else if (s<0) l++;
            else           r--;
        }
    }
    return result;
}
🎤 Experienced Answer
"Classic problem. Brute force is O(n²) — nested loops. Optimal: HashMap stores {value → index}. For each number, compute complement (target - current), check if it's already in map. Single pass! The key insight: don't look forward for the pair, look backward using the hash map. ThreeSum extends this: sort first, then fix one element and use two-pointer for the remaining pair — O(n²) total."

Maximum Subarray Sum — Kadane's Algorithm
Medium
💡 Problem
Given [-2,1,-3,4,-1,2,1,-5,4] → max subarray is [4,-1,2,1] = 6
✅ Kadane's Algorithm — O(n) O(1)
public static int MaxSubarraySum(int[] nums)
{
    int maxSoFar    = nums[0];
    int maxEndingHere = nums[0];

    for (int i=1; i<nums.Length; i++)
    {
        // extend current subarray OR start fresh
        maxEndingHere = Math.Max(nums[i],
                         maxEndingHere + nums[i]);
        maxSoFar = Math.Max(maxSoFar, maxEndingHere);
    }
    return maxSoFar;
}
// [-2,1,-3,4,-1,2,1,-5,4] → 6
// Track: maxEndingHere = -2,1,-2,4,3,5,6,1,5
//        maxSoFar      = -2,1, 1,4,4,5,6,6,6
🎤 Experienced Answer — Kadane's insight
"At each position, decide: extend the previous subarray OR start a new subarray from current element. If maxEndingHere goes negative, it's better to start fresh — negative prefix only hurts the sum. O(n) single pass, O(1) space. This is Dynamic Programming — current decision based on previous optimal state. Variant: also track start/end indices to return the actual subarray."

Bubble Sort + Selection Sort + Insertion Sort
Easy
Bubble Sort — O(n²) with early exit optimization
public static void BubbleSort(int[] arr)
{
    int n = arr.Length;
    for (int i=0; i<n-1; i++)
    {
        bool swapped = false;
        for (int j=0; j<n-i-1; j++)
            if (arr[j] > arr[j+1])
            {
                (arr[j], arr[j+1]) = (arr[j+1], arr[j]);
                swapped = true;
            }
        if (!swapped) break; // already sorted!
    }
}
Insertion Sort — Best for nearly sorted data
public static void InsertionSort(int[] arr)
{
    for (int i=1; i<arr.Length; i++)
    {
        int key = arr[i];
        int j   = i - 1;
        while (j >= 0 && arr[j] > key)
        {
            arr[j+1] = arr[j];
            j--;
        }
        arr[j+1] = key;
    }
}
// Best case O(n) when nearly sorted — great for real data!
🎤 Comparison Table
"Bubble Sort: O(n²) worst, O(n) best (with swap flag). Each pass 'bubbles' the largest to end. Insertion Sort: O(n²) worst, O(n) best — insert each element into its correct position in the sorted left part. Best for nearly-sorted data — faster in practice. For interviews, code Bubble Sort but mention C#'s Array.Sort() uses an introsort (quicksort + heapsort + insertion sort hybrid) — O(n log n) average."

Binary Search — Iterative + Recursive
Medium
✅ Iterative — No stack overflow risk
public static int BinarySearch(int[] arr, int target)
{
    int low = 0, high = arr.Length - 1;
    while (low <= high)
    {
        // Avoid overflow: mid = low + (high-low)/2
        int mid = low + (high - low) / 2;
        if      (arr[mid] == target) return mid;
        else if (arr[mid] <  target) low  = mid + 1;
        else                            high = mid - 1;
    }
    return -1; // not found
}
// arr=[1,3,5,7,9,11], target=7 → index 3
// Steps: mid=5(index 2)→too small, mid=9(4)→too big, mid=7(3)✅
🎤 Critical Bug to mention
"Common bug: mid = (low + high) / 2 can INTEGER OVERFLOW when both low and high are near Int.MaxValue. Correct: mid = low + (high - low) / 2. Array MUST be sorted. O(log n) — halves search space each step. Log₂(1 million) = ~20 steps. Prefer Array.BinarySearch() in production."

Implement a Stack using Array — Push, Pop, Peek
Medium
public class MyStack<T>
{
    private T[] _data;
    private int _top = -1;

    public MyStack(int capacity = 16)
        => _data = new T[capacity];

    public bool IsEmpty => _top == -1;
    public int  Count   => _top + 1;

    public void Push(T item)
    {
        if (_top == _data.Length-1) Resize();
        _data[++_top] = item;
    }
    public T Pop()
    {
        if (IsEmpty) throw new InvalidOperationException("Empty");
        var item = _data[_top];
        _data[_top--] = default!; // GC friendly
        return item;
    }
    public T Peek()
    {
        if (IsEmpty) throw new InvalidOperationException("Empty");
        return _data[_top];
    }
    private void Resize()
    {
        var bigger = new T[_data.Length * 2];
        Array.Copy(_data, bigger, _data.Length);
        _data = bigger;
    }
}
🎤 Experienced Answer
"Generic Stack using array with dynamic resize (double when full, like List internally). _top=-1 means empty. Push increments top first, Pop reads then decrements. Setting popped slot to default! allows GC to collect objects (important for reference types). Resize doubles capacity — amortized O(1) push. This is essentially how .NET's Stack works internally."

Implement Queue using Two Stacks
Hard
💡 Famous Interview Problem
Stack = LIFO, Queue = FIFO. Use two stacks to simulate queue behavior.
public class MyQueue<T>
{
    private Stack<T> _inbox  = new();
    private Stack<T> _outbox = new();

    public void Enqueue(T item)
        => _inbox.Push(item);  // always push to inbox

    public T Dequeue()
    {
        Refill();
        return _outbox.Pop();
    }
    public T Peek() { Refill(); return _outbox.Peek(); }

    private void Refill()
    {
        if (_outbox.Count > 0) return; // outbox not empty
        if (_inbox.Count == 0)
            throw new InvalidOperationException("Empty");
        // Transfer inbox → outbox (reverses order = FIFO!)
        while (_inbox.Count > 0)
            _outbox.Push(_inbox.Pop());
    }
}
// Enqueue: 1,2,3
// Dequeue: 1 (not 3!) ← two stacks reverse the reversal!
🎤 Key Insight
"Transfer only when outbox is empty — amortized O(1) per operation. Inbox is for writing, outbox is for reading. When reading: if outbox empty, pour inbox into outbox (reverses order). A single element moves at most twice (once in, once out). Total O(n) for n operations = amortized O(1) each. This is the 'lazy evaluation' principle — defer work until necessary."

Balanced Brackets — Valid Parentheses
Medium
public static bool IsBalanced(string s)
{
    var stack = new Stack<char>();
    var pairs = new Dictionary<char,char>
    {
        { ')', '(' }, { ']', '[' }, { '}', '{' }
    };

    foreach (char c in s)
    {
        if ("([{".Contains(c))
            stack.Push(c);
        else if (pairs.ContainsKey(c))
        {
            if (stack.Count==0 || stack.Pop() != pairs[c])
                return false;
        }
    }
    return stack.Count == 0; // stack must be empty
}
// "()[]{}"  → true
// "([)]"    → false
// "{[()]}"  → true
// "((("     → false (stack not empty)
🎤 Experienced Answer
"Stack is the natural data structure — push open brackets, pop and verify when closing bracket encountered. The pairs dictionary maps closing to opening. Two error conditions: closing bracket when stack empty (no matching open), or top of stack doesn't match. Final check: stack must be empty (all opens were closed). ZenCampus: same logic validates JSON/XML template expressions."

Group Anagrams Together — ["eat","tea","tan","ate","nat","bat"]
Medium
public static List<List<string>> GroupAnagrams(string[] words)
{
    var map = new Dictionary<string, List<string>>();

    foreach (var word in words)
    {
        // Sorted word is the canonical key
        var key = new string(word.OrderBy(c=>c).ToArray());
        if (!map.ContainsKey(key))
            map[key] = new List<string>();
        map[key].Add(word);
    }
    return map.Values.ToList();
}
// ["eat","tea","ate"] → sorted key = "aet"
// ["tan","nat"]       → sorted key = "ant"
// ["bat"]             → sorted key = "abt"
// Output: [["eat","tea","ate"],["tan","nat"],["bat"]]
🎤 Key Insight
"Sorted string is the canonical key — all anagrams sort to the same string ('eat', 'tea', 'ate' all sort to 'aet'). Group by this key. O(n * k log k) where k is avg word length. LINQ version: words.GroupBy(w => new string(w.OrderBy(c=>c).ToArray()))"

LINQ — GroupBy Aggregations on ZenCampus Data
Medium
Student Fee Analysis with LINQ
var students = new[] {
    new { Name="Guru",  Class="10A", Marks=90 },
    new { Name="Priya", Class="10A", Marks=85 },
    new { Name="Kumar", Class="10B", Marks=78 },
    new { Name="Meena", Class="10B", Marks=92 },
};

// Average marks per class
var classAvg = students
    .GroupBy(s => s.Class)
    .Select(g => new {
        Class   = g.Key,
        Average = g.Average(s => s.Marks),
        TopStudent = g.MaxBy(s => s.Marks).Name
    });
// 10A: Avg=87.5, Top=Guru
// 10B: Avg=85.0, Top=Meena

// Students above class average (complex LINQ)
var aboveAvg = from s in students
               let avg = students
                   .Where(x => x.Class == s.Class)
                   .Average(x => x.Marks)
               where s.Marks > avg
               select new { s.Name, s.Marks, s.Class };
🎤 Experienced Answer
"GroupBy in LINQ groups elements by a key — each group is IGrouping. Chaining aggregates (Average, Max, Count, Sum) on each group is the standard BI query pattern. let keyword in query syntax creates a computed sub-expression. MaxBy() is .NET 6+ — before that use OrderByDescending().First(). ZenCampus reporting module uses exactly this pattern for class-wise analytics."

Flatten Nested List — SelectMany + Recursive
Medium
// Simple: List of Lists → flat list
var nested = new List<List<int>> {
    new { 1,2,3 }, new { 4,5 }, new { 6,7,8 }
};
var flat = nested.SelectMany(x => x).ToList();
// → [1,2,3,4,5,6,7,8]

// Deeply nested: custom recursive flatten
static IEnumerable<int> FlattenDeep(
    IEnumerable<object> source)
{
    foreach (var item in source)
    {
        if (item is int n)
            yield return n;
        else if (item is IEnumerable<object> inner)
            foreach (var x in FlattenDeep(inner))
                yield return x;
    }
}
// .NET 6+: IEnumerable.Chunk(3) splits into batches of 3!
🎤 Key Point
"SelectMany is LINQ's flat-map operator — it projects each element to a sequence and flattens the results. Equivalent to two nested foreach loops. For ZenCampus: flatten class-wise student lists into one report list. Recursive flatten uses pattern matching (is int n, is IEnumerable) — C# 7+ switch expression or is-patterns make this clean."

Generic Repository Pattern — T constraints
Medium
// Generic base repository — ZenCampus pattern
public interface IRepository<T> where T : class
{
    Task<T?>         GetByIdAsync(int id);
    Task<IEnumerable<T>> GetAllAsync();
    Task             AddAsync(T entity);
    Task             UpdateAsync(T entity);
    Task             DeleteAsync(int id);
}

public class GenericRepository<T> : IRepository<T>
    where T : class, IEntity  // must have Id
{
    protected readonly AppDbContext _ctx;
    public GenericRepository(AppDbContext ctx)
        => _ctx = ctx;

    public async Task<T?> GetByIdAsync(int id)
        => await _ctx.Set<T>().FindAsync(id);

    public async Task<IEnumerable<T>> GetAllAsync()
        => await _ctx.Set<T>().ToListAsync();

    public async Task AddAsync(T entity)
    {
        await _ctx.Set<T>().AddAsync(entity);
        await _ctx.SaveChangesAsync();
    }
}
🎤 Experienced Answer
"Generic Repository reduces boilerplate — DbContext.Set() returns the DbSet for any entity type. The 'where T : class' constraint prevents value types. 'where T : class, IEntity' adds a custom constraint ensuring all entities have an Id. In ZenCampus, Student, Staff, and Fee repositories all extend this — Add/Update/Delete/GetAll implemented once, entity-specific queries in child repositories."

Singleton Pattern — Thread-safe implementation
Medium
❌ Not thread-safe (wrong)
// WRONG — two threads can both find _instance==null!
public class BadSingleton
{
    private static BadSingleton _instance;
    public static BadSingleton Instance
        => _instance ??= new BadSingleton();
}
✅ Thread-safe — Lazy
public sealed class AppConfig
{
    private static readonly Lazy<AppConfig> _lazy
        = new(() => new AppConfig());

    public static AppConfig Instance
        => _lazy.Value;

    private AppConfig() { }  // private constructor

    public string ApiBaseUrl { get; set; }
        = "https://api.zencampus.com";
}
// Usage: AppConfig.Instance.ApiBaseUrl
// Lazy is thread-safe by default
🎤 Experienced Answer
"Lazy is the modern C# way — thread-safe by default, lazy (created on first access), and clean. sealed prevents subclassing which could break the singleton guarantee. private constructor prevents external instantiation. Note: in ASP.NET Core, prefer DI container with AddSingleton() — IoC containers handle thread-safe singleton lifecycle, easier to test."

Builder Pattern — Construct complex StudentAdmission object
Hard
public class Student
{
    public string Name    { get; init; }
    public string Class   { get; init; }
    public string Section { get; init; }
    public string Contact { get; init; }
    public bool   HasTransport { get; init; }
    public bool   HasHostel    { get; init; }
}

public class StudentBuilder
{
    private string _name, _class, _section, _contact;
    private bool   _transport, _hostel;

    public StudentBuilder WithName(string n)    { _name      = n; return this; }
    public StudentBuilder InClass(string c)    { _class     = c; return this; }
    public StudentBuilder InSection(string s)  { _section   = s; return this; }
    public StudentBuilder WithContact(string p){ _contact   = p; return this; }
    public StudentBuilder WithTransport()       { _transport = true; return this; }
    public StudentBuilder WithHostel()          { _hostel    = true; return this; }

    public Student Build()
    {
        if (string.IsNullOrEmpty(_name))
            throw new InvalidOperationException("Name required");
        return new Student {
            Name         = _name,
            Class        = _class,
            Section      = _section,
            Contact      = _contact,
            HasTransport = _transport,
            HasHostel    = _hostel
        };
    }
}

// Fluent usage
var student = new StudentBuilder()
    .WithName("Gurumoorthy")
    .InClass("10A")
    .InSection("A")
    .WithTransport()
    .Build();
🎤 Experienced Answer
"Builder separates construction from representation. Each setter returns 'this' for method chaining — fluent API. Build() validates required fields and constructs the immutable object. Prevents 'telescoping constructor' problem where you need 8 overloads for 8 optional params. In C#, record types with required properties + object initializers reduce need for Builder, but Builder is still valuable for complex validation logic during construction."
