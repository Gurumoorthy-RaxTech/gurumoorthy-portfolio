2 Key Vault + Managed Identity — Zero Credential Code
⛔ Real Incident — Never put secrets in code/config!

Developer committed appsettings.json with real DB connection string to GitHub. GitHub secret scanner caught it — but 6 minutes exposed. Azure SQL firewall blocked external access (lucky). Key Vault + Managed Identity = credentials never exist in code OR config files.

🎬 Dasavathaaram — Govind keeps the vial safe through 800 years. Key Vault = same. Secrets locked, only Managed Identity (the rightful owner) can access. No password needed — Azure handles the authentication!

Managed Identity = Azure gives the app an identity (like a service account). AKS pod says "I am ZenCampus StudentService" — Azure verifies via its own systems — Key Vault grants access. Zero passwords. Zero rotation headaches. Zero leaked secrets.

3 Azure Blob Storage — Student Photos, PDFs, Receipts
🎬 Mersal — Hospital stores records of every patient safely. Blob Storage = same. Student photos, fee receipts, report cards — stored safely, retrieved with SAS token (limited-time access).

ZenCampus stores: student photos (200KB avg × 50,000 = 10GB), fee receipt PDFs (monthly), report cards, attendance export Excel files. Azure Blob = cheap, durable, CDN-ready. SAS token = give parent 1-hour download link for their child's receipt only. No direct storage access!

// BlobStorageService — ZenCampus implementation
public class AzureBlobService : IBlobService
{
    private readonly BlobServiceClient _client;

    public async Task<string> UploadStudentPhotoAsync(
        int studentId, Stream photo, string contentType)
    {
        var container = _client.GetBlobContainerClient("student-photos");
        await container.CreateIfNotExistsAsync(PublicAccessType.None); // private!

        var blobName = $"school/{_currentUser.SchoolId}/students/{studentId}/photo.jpg";
        var blob     = container.GetBlobClient(blobName);

        await blob.UploadAsync(photo, new BlobUploadOptions
        {
            HttpHeaders = new BlobHttpHeaders { ContentType = contentType },
            Metadata    = new Dictionary<string, string>
                          { ["studentId"] = studentId.ToString() }
        });

        return blobName; // store path in DB, not full URL
    }

    // SAS token — time-limited, operation-limited URL for secure download
    public Uri GeneratePhotoSasUrl(string blobName, TimeSpan expiry)
    {
        var container = _client.GetBlobContainerClient("student-photos");
        var blob      = container.GetBlobClient(blobName);

        return blob.GenerateSasUri(new BlobSasBuilder
        {
            BlobContainerName = "student-photos",
            BlobName          = blobName,
            Resource          = "b",                     // blob (not container)
            ExpiresOn         = DateTimeOffset.UtcNow + expiry,
            Permissions       = BlobSasPermissions.Read  // read only!
        });
        // Returns: https://zencampusstorage.blob.core.windows.net/student-photos/...?sv=...&se=...&sig=...
        // Valid for 'expiry' only, read-only. Parent can download receipt, not modify!
    }

    // Upload fee receipt PDF (generated by Hangfire background job)
    public async Task<string> UploadReceiptAsync(int receiptId, byte[] pdf)
    {
        var blobName  = $"receipts/{DateTime.UtcNow:yyyy/MM}/{receiptId}.pdf";
        var container = _client.GetBlobContainerClient("fee-receipts");
        var blob      = container.GetBlobClient(blobName);
        await blob.UploadAsync(new BinaryData(pdf), overwrite: true);
        return blobName;
    }
}

4 Azure Service Bus — Reliable Event Messaging
// NuGet: Azure.Messaging.ServiceBus
// Topics = pub/sub (multiple subscribers). Queues = point-to-point.

// Publisher — FeeService (after fee collected)
public class AzureServiceBusPublisher
{
    private readonly ServiceBusClient _client;

    public async Task PublishFeePaidAsync(FeePaidEvent evt)
    {
        await using var sender   = _client.CreateSender("fee-events"); // topic name
        var message                = new ServiceBusMessage(
            BinaryData.FromObjectAsJson(evt))
        {
            MessageId            = evt.EventId.ToString(), // dedup key!
            Subject              = "fee.paid",
            CorrelationId        = evt.ReceiptNo,
            TimeToLive           = TimeSpan.FromDays(7),    // auto-expire after 7 days
            ApplicationProperties = { ["SchoolId"] = evt.SchoolId }
        };
        await sender.SendMessageAsync(message);
    }
}

// Subscriber — NotificationService (processes fee events)
public class FeePaidProcessor : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken ct)
    {
        await using var processor = _client.CreateProcessor(
            topicName:        "fee-events",
            subscriptionName: "notification-service",
            new ServiceBusProcessorOptions
            {
                MaxConcurrentCalls       = 5,     // 5 messages in parallel
                AutoCompleteMessages     = false, // manual ACK — control retry!
                MaxAutoLockRenewalDuration = TimeSpan.FromMinutes(5)
            });

        processor.ProcessMessageAsync += async args =>
        {
            var evt = args.Message.Body.ToObjectFromJson<FeePaidEvent>();

            try
            {
                await _sms.SendReceiptAsync(evt.ParentPhone, evt.ReceiptNo);
                await args.CompleteMessageAsync(args.Message); // ACK — remove from queue
            }
            catch (Exception ex)
            {
                _log.LogError(ex, "SMS failed for {Receipt}", evt.ReceiptNo);

                // Abandon — message goes back to queue for retry
                if (args.Message.DeliveryCount >= 5)
                    await args.DeadLetterMessageAsync(args.Message,
                        "MaxRetriesExceeded", ex.Message); // → dead letter queue
                else
                    await args.AbandonMessageAsync(args.Message); // retry later
            }
        };

        processor.ProcessErrorAsync += args =>
        {
            _log.LogError(args.Exception, "Service Bus error");
            return Task.CompletedTask;
        };

        await processor.StartProcessingAsync(ct);
        await Task.Delay(Timeout.Infinite, ct);
        await processor.StopProcessingAsync();
    }
}
💡 Service Bus: Queue vs Topic

Queue — point-to-point. One sender, one receiver. fee-receipt-pdf-queue → only PDFWorker processes it.
Topic + Subscription — pub/sub. One sender, MULTIPLE receivers. fee-events topic → NotificationService subscription (SMS) + FeeAuditService subscription (audit log) + DashboardService (invalidate cache). All receive same message independently.

// Custom properties on every request — critical for multi-tenant debugging!
public class TelemetryEnrichmentMiddleware(RequestDelegate next, TelemetryClient telemetry)
{
    public async Task InvokeAsync(HttpContext ctx)
    {
        var schoolId = ctx.User.FindFirst("schoolId")?.Value;
        var role     = ctx.User.FindFirst(ClaimTypes.Role)?.Value;

        using var op = telemetry.StartOperation<RequestTelemetry>("Request");
        op.Telemetry.Properties["SchoolId"] = schoolId;
        op.Telemetry.Properties["Role"]     = role;

        await next(ctx);
    }
}

    - name: Build and push

    - run: |
        kubectl set image deployment/student-service \
            student-service=${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ needs.build-push.outputs.image-tag }} \
            -n zencampus

// API endpoint — validates parent owns this student
[HttpGet("students/{studentId}/report-card")]
[Authorize(Roles = "Parent")]
public async Task<IActionResult> GetReportCard(int studentId)
{
    var parentId = int.Parse(User.FindFirst(ClaimTypes.NameIdentifier)!.Value);

    // Verify: this parent is linked to this student!
    var isParent = await _studentRepo.IsParentOfStudentAsync(parentId, studentId);
    if (!isParent) return Forbid();  // 403 — not your child!

    var blobPath = await _studentRepo.GetReportCardPathAsync(studentId);
    if (blobPath is null) return NotFound();

    // Generate SAS URL — 1 hour, read-only, this blob only
    var sasUrl = _blobService.GeneratePhotoSasUrl(blobPath, TimeSpan.FromHours(1));

    return Ok(new { DownloadUrl = sasUrl.ToString(), ExpiresIn = "1 hour" });
    // Parent app: redirect browser to SAS URL → download from Azure directly
    // Server not involved in download — CDN handles it!
    // After 1 hour: URL expired. New request needed. Audit trail maintained.
}
// Security: JWT auth → parent identity. IsParentOfStudent → data access. SAS → time-limited.

// Step 1: Read dead letter messages (don't delete — investigate first!)
await using var dlqReceiver = _client.CreateReceiver(
    "fee-events",
    "notification-service",
    new ServiceBusReceiverOptions
    {
        SubQueue = SubQueue.DeadLetter   // read from DLQ!
    });

var dlqMessages = await dlqReceiver.PeekMessagesAsync(10);
foreach (var msg in dlqMessages)
{
    _log.LogInformation(
        "DLQ: {Reason} — {Body}",
        msg.DeadLetterReason,     // "MaxRetriesExceeded" or "SmsGatewayDown"
        msg.Body.ToString());
}
// Root cause: SMS gateway had 503 for 2 hours.
// All messages hit retry limit (5) → dead letter

// Step 2: Fix root cause (SMS gateway recovered)
// Step 3: Reprocess DLQ — resend to original topic
await using var sender = _client.CreateSender("fee-events");
var msg = await dlqReceiver.ReceiveMessageAsync();

while (msg != null)
{
    // Repost to main topic
    await sender.SendMessageAsync(new ServiceBusMessage(msg.Body));
    await dlqReceiver.CompleteMessageAsync(msg); // remove from DLQ
    msg = await dlqReceiver.ReceiveMessageAsync();
}
// All 500 messages reprocessed → 500 parents get SMS!

Q1: DefaultAzureCredential — local development-la எப்படி authenticate ஆகும்?

A) Reads from appsettings.json credentials section
B) Uses 'az login' session (AzureCliCredential) or Visual Studio logged-in account

✅ DefaultAzureCredential tries multiple methods in order. Locally: VisualStudioCredential (VS logged-in account) or AzureCliCredential (az login). In AKS: WorkloadIdentityCredential. In App Service: ManagedIdentityCredential. Same code works everywhere! No credentials hardcoded.
❌ DefaultAzureCredential tries multiple auth methods in sequence. Locally = AzureCliCredential (uses 'az login' session) or VisualStudioCredential. In AKS = WorkloadIdentityCredential. Same code — different environments authenticate differently, zero credentials in code!

Q2: Azure Blob Storage SAS token — main purpose?

A) Encrypt the blob content
B) Grant time-limited, operation-limited access to a specific blob without exposing storage account keys
C) Speed up blob downloads via CDN
D) Compress blobs automatically
✅ SAS = Shared Access Signature. Time-limited (expires in 1 hour), operation-limited (read only, not write/delete), resource-limited (this blob, not entire container). Parent gets download link for their child's receipt — link expires, they can't access other blobs, and storage keys are never exposed!
❌ SAS = Shared Access Signature — a signed URL with built-in permissions and expiry. No storage key needed by the caller. Parent's download link: expires in 1 hour (ExpiresOn), read-only (BlobSasPermissions.Read), this blob only (BlobName set). Security in depth!

Q3: Service Bus message delivery count 5 reached — message goes where?

A) Message is permanently deleted
B) Automatically moved to Dead Letter Queue (DLQ)
C) Resets delivery count and starts again
D) Sent back to the original publisher
✅ After MaxDeliveryCount (default 10, can set lower) retries → message auto-moved to Dead Letter Queue. DLQ = separate queue, messages stay there until explicitly processed or expire. Monitor DLQ! Zero DLQ messages = healthy system. 500 DLQ = 500 parents waiting for SMS!
❌ Dead Letter Queue (DLQ)! After max delivery count reached, Service Bus automatically moves message to DLQ. Message stays there indefinitely (or until TTL). You can read/reprocess from DLQ. Alert on DLQ count > 0 — means messages are failing to process!

Q4: Application Insights-la custom dimension add பண்ணணும் — why?

A) Required for Application Insights to work
B) Improves query performance in KQL
C) Enables filtering telemetry by business dimensions (SchoolId, Role) — find which school/user caused an error
D) Reduces telemetry storage costs
✅ Multi-tenant app — without SchoolId in telemetry, you see "API is slow" but not "API is slow for School 42 only." Custom dimensions = business context on every trace. Filter by SchoolId → find it's only one school's data causing slowness → targeted fix. Priceless for production debugging!
❌ Custom dimensions = business context on telemetry! Without it: "fee API is slow" → check everything. With SchoolId custom dimension: "fee API slow" → filter by SchoolId → School 42 p95=4200ms, others 320ms → targeted fix. Multi-tenant app MUST add tenant identifier to every telemetry item!
