🟡 ES6+ JavaScript Fundamentals
🎬 Kaadhal (2004) — Selvam & Anitha: Old-style var romance = unpredictable, leaks everywhere (function scope only). Modern let/const = boundaries respected, block-scoped, no hoisting surprises. Write modern JS — always let/const!
Syntax — Variables
var · let · const · Hoisting · TDZ
/* ── var — AVOID in modern JS ─────── */
var x = 10;          // function scoped
var x = 20;          // re-declaration: OK! (bug!)

// HOISTING — declaration lifted to top
console.log(y);    // undefined (not ReferenceError!)
var y = 5;

/* ── let — block scoped ─────────── */
let count = 0;
count = 1;          // reassign: OK

if (true) {
  let inner = 'block';
}
// console.log(inner) → ReferenceError ✅

// TDZ — Temporal Dead Zone
console.log(z);    // ReferenceError ✅
let z = 5;

/* ── const — block scoped, no reassign */
const PI = 3.14;
// PI = 3 → TypeError ✅

// const object — contents CAN change!
const student = { name: 'Guru' };
student.name = 'Ram';   // OK — property change
// student = {} → TypeError (reassign) ✅

// const array — items CAN change!
const fees = [1000, 2000];
fees.push(3000);     // OK ✅
// fees = [] → TypeError ✅
❓ Interview Q: var vs let vs const?
var = function scope, hoisted to undefined, re-declarable — avoid! let = block scope, TDZ, reassignable. const = block scope, TDZ, no reassign (but object contents mutable). Interview rule: always const first, let if reassignment needed, never var!

Arrow Functions · Template Literals · Destructuring

// Regular function

// Arrow — concise

      console.log(this.amount); // ✅ this = class

  }
}

❓ Interview Q: Arrow function vs regular function?
1. Syntax shorter. 2. No own 'this' — inherits from lexical scope. 3. Cannot be used as constructor (no new). 4. No arguments object. 5. No prototype. ZenCampus-la setTimeout callbacks, array methods — always arrow. Class methods that need their own 'this' — regular function!
Syntax — Modern JS
Spread · Rest · Optional Chaining · Nullish
/* ── SPREAD OPERATOR (...) ─────── */
// Array spread — clone/merge
const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5];    // [1,2,3,4,5]
const clone = [...arr1];          // shallow clone

// Object spread — merge/override
const base    = { name:'Guru', class:'9A' };
const updated = { ...base, class:'10A' };
// { name:'Guru', class:'10A' }

// Spread in function call
const nums = [3, 1, 4, 1, 5];
Math.max(...nums);  // 5

/* ── REST PARAMETER ─────────────── */
const sum = (...nums) =>
  nums.reduce((a, b) => a + b, 0);
sum(1, 2, 3, 4);  // 10

const first = (head, ...tail) => head;
first(10, 20, 30); // 10, tail=[20,30]

/* ── OPTIONAL CHAINING (?.) ─────── */
const student = {
  name: 'Guru',
  address: { city: 'Chennai' }
};

// Without ?. — throws if address is null
student.address.city   // 'Chennai'
student.phone?.mobile // undefined (not error!)

// Method optional chain
student.getName?.()     // undefined if no method
arr?.[0]               // array index optional

/* ── NULLISH COALESCING (??) ─────── */
const name = student.name ?? 'Unknown';
// ?? only null/undefined → default
// || also catches 0, '', false (bug risk!)

const count = data.count ?? 0;
// If count is 0 → keeps 0 (|| would replace!)

/* ── LOGICAL ASSIGNMENT ─────────── */
a ??= 'default'; // assign if null/undefined
b ||= 'fallback';// assign if falsy
c &&= newVal;    // assign if truthy
❓ Interview Q: ?? vs ||?
|| = falsy check (0, '', false, null, undefined all trigger default). ?? = nullish check (ONLY null/undefined triggers default). ZenCampus fee count = 0 → || wrongly replaces 0 with default! Always use ?? for null/undefined checks!
Syntax — Classes + Modules
ES6 Class · extends · static · Modules
/* ── ES6 CLASS ───────────────────── */
class Person {
  // Class field (modern)
  #id;                     // private field
  name;

  constructor(id, name) {
    this.#id   = id;
    this.name = name;
  }

  // Getter
  get id() { return this.#id; }

  // Instance method
  greet() {
    return `Hi, I'm ${this.name}`;
  }

  // Static — no instance needed
  static create(id, name) {
    return new Person(id, name);
  }
}

// Inheritance
class Student extends Person {
  rollNo;

  constructor(id, name, rollNo) {
    super(id, name);           // MUST call super
    this.rollNo = rollNo;
  }

  // Override parent method
  greet() {
    return `${super.greet()}, Roll: ${this.rollNo}`;
  }
}

const s = new Student(1, 'Guru', '10');
s.greet(); // "Hi, I'm Guru, Roll: 10"
s instanceof Student; // true
s instanceof Person;  // true

🏫 ZenCampus Event Delegation Pattern:
Student list has 1000 rows. Don't add 1000 click listeners! One listener on the table parent with e.target.closest('.student-row') — works for dynamically added rows too!
❓ Interview Q: What is event delegation?
Attaching ONE event listener to a PARENT element to handle events from multiple CHILDREN via event bubbling. Benefits: 1. Memory efficient (1 listener vs 1000), 2. Works for dynamically added elements, 3. Simpler code. target.closest() finds the relevant ancestor!
🔵 Promises · Async/Await · fetch · Error Handling
🎬 Mudhalvan (1999) — Arjun (CM for a day): Arjun makes a promise — "I'll fix the city in 24 hours." He waits for responses (API calls = officers acting). Some fulfill (resolve), some fail (reject). He doesn't freeze — he handles each async result. Promises + async/await = JavaScript doing the same!
Syntax — Promises
Promise · then/catch · Promise.all · race
/* ── CREATE PROMISE ─────────────── */
const fetchFee = (studentId) =>
  new Promise((resolve, reject) => {
    setTimeout(() => {
      if (studentId > 0)
        resolve({ amount: 5000 });  // success
      else
        reject(new Error('Invalid ID'));
    }, 1000);
  });

/* ── CONSUME PROMISE ─────────────── */
fetchFee(42)
  .then(data  => console.log(data))
  .catch(err   => console.error(err))
  .finally(() => setLoading(false));

/* ── PROMISE.ALL — parallel, all or none */
const [students, fees, attendance] =
  await Promise.all([
    fetchStudents(),
    fetchFees(),
    fetchAttendance()
  ]);
// ALL must resolve — one fails = all fail

/* ── PROMISE.ALLSETTLED — all, partial ok */
const results = await Promise.allSettled([
  fetchStudents(),
  fetchFees()
]);
results.forEach(r => {
  if (r.status === 'fulfilled')
    console.log(r.value);
  else
    console.error(r.reason);
});

/* ── PROMISE.RACE — first settles wins */
const result = await Promise.race([
  fetchFromServer1(),
  fetchFromServer2()     // faster wins!
]);

/* ── PROMISE.ANY — first RESOLVED wins */
const res = await Promise.any([
  cdn1(), cdn2(), cdn3()  // ignores rejects
]);
❓ Interview Q: Promise.all vs Promise.allSettled?
Promise.all = ALL must resolve, one rejection = entire promise rejects. allSettled = waits for ALL to finish regardless, gives status:'fulfilled'/'rejected' per promise. ZenCampus dashboard: fetch students+fees+attendance parallel — allSettled if we want partial data even if one API fails!

async/await · fetch · Error Handling Pattern
/* ── ASYNC/AWAIT ─────────────────── */
const loadDashboard = async () => {
  setLoading(true);
  try {
    // Sequential (when B needs A's result)
    const student = await getStudent(42);
    const fees    = await getFees(student.id);

/* ── FETCH API ───────────────────── */
const fetchStudents = async (className) => {
  const response = await fetch(
    `/api/v1/students?class=${className}`,
    {
      method:  'GET',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${getToken()}`
      }
    }
  );

  // IMPORTANT: fetch doesn't throw on 4xx/5xx!
  if (!response.ok) {
    throw new Error(
      `HTTP ${response.status}: ${response.statusText}`
    );
  }

  return response.json();
};

// POST with body
const admitStudent = async (data) => {
  const res = await fetch('/api/v1/students', {
    method:  'POST',
    headers: { 'Content-Type': 'application/json' },
    body:    JSON.stringify(data)
  });
  if (!res.ok) throw new Error(await res.text());
  return res.json();
};
⚠️ Biggest Fetch Trap — 404 doesn't throw!
fetch() only rejects on NETWORK error (offline). HTTP 400/404/500 = response.ok is false but no exception! Always check if (!response.ok) throw new Error() manually!
🟣 Closures · Scope · IIFE · Module Pattern
🎬 Vinnaithaandi Varuvaayaa (2010) — Karthik & Jessie: Karthik remembers every moment even after Jessie is gone — closure of memory. JavaScript closures = inner function REMEMBERS outer function's variables even after outer function returns. "En peyar Jessie!" — the reference is closed over, lives forever!
Syntax — Closures
Closure · Counter · Private Variables
/* ── BASIC CLOSURE ────────────────── */
function makeCounter(start = 0) {
  let count = start;  // outer variable

  return {
    increment: () => ++count,
    decrement: () => --count,
    value:     () => count
  };
  // Inner functions CLOSE OVER count —
  // count lives as long as counter object lives!
}

const counter = makeCounter(10);
counter.increment(); // 11
counter.increment(); // 12
counter.value();    // 12 — private variable!

/* ── ZenCampus: Private API Store ── */
function createFeeStore(initialBalance) {
  let balance      = initialBalance; // private!
  const transactions = [];

  return {
    pay(amount) {
      if (amount > balance)
        throw new Error('Insufficient balance');
      balance -= amount;
      transactions.push({ amount, type:'pay' });
    },
    credit(amount) {
      balance += amount;
      transactions.push({ amount, type:'credit' });
    },
    getBalance: () => balance,
    getHistory: () => [...transactions] // copy
  };
}

/* ── CLASSIC CLOSURE BUG ──────────── */
// Bug — all print 3
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 1000);
}
// ↑ var i is shared — all see final i=3

// Fix 1 — use let (block scoped)
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 1000);
}
// ↑ prints 0, 1, 2 ✅

// Fix 2 — IIFE
for (var i = 0; i < 3; i++) {
  ((n) => {
    setTimeout(() =>
      console.log(n), 1000);
  })(i);
}
❓ Interview Q: What is a closure?
A closure is a function that remembers variables from its outer scope even after that outer function has returned. Inner function + outer scope variables = closure. Use cases: private variables, factory functions, memoization, event handlers with captured data. VTV movie hook: Karthik remembers Jessie — inner function remembers outer variables!

/* ── call, apply, bind ────────────── */
function greet(greeting, punct) {
  return `${greeting}, ${this.name}${punct}`;
}
const student = { name: 'Guru' };

// call — invoke immediately, args listed
greet.call(student, 'Hi', '!');
// "Hi, Guru!"

// apply — invoke immediately, args array
greet.apply(student, ['Hello', '.']);
// "Hello, Guru."

// bind — returns NEW function with this fixed
const boundGreet = greet.bind(student);
boundGreet('Hey', '?');
// "Hey, Guru?" — this always = student

/* ── SCOPE CHAIN ─────────────────── */
const schoolName = 'ZenCampus'; // global

function outer() {
  const grade = '10A';     // outer scope

  function inner() {
    const name = 'Guru';   // inner scope
    // Can access: name ✅ grade ✅ schoolName ✅
    console.log(`${name} ${grade} ${schoolName}`);
  }
  // Can access: grade ✅ schoolName ✅
  // Cannot access: name ❌
  inner();
}
💡 call/apply/bind memory trick:
call = comma-separated args (c for comma)
apply = array args (a for array)
bind = borrow function, returns new function (b for borrow)
🟢 Array Methods — map · filter · reduce · find · flat
🎬 Mersal (2017) — Vijay triple role: Three completely different characters, each with a specific purpose. Array methods = same: map (transform every item), filter (select specific items), reduce (calculate one result from all). Each a specialist, chain them together = powerful pipeline!
Syntax — Core Methods
map · filter · reduce · find · findIndex
const students = [
  { id:1, name:'Guru',  class:'10A', fee:5000, paid:true  },
  { id:2, name:'Ram',   class:'10B', fee:4500, paid:false },
  { id:3, name:'Priya', class:'10A', fee:5000, paid:true  },
];

/* ── map — transform each item ──── */
const names = students.map(s => s.name);
// ['Guru', 'Ram', 'Priya']

const cards = students.map(s =>
  `<div class="card">${s.name}</div>`
).join('');  // HTML string from array

/* ── filter — keep matching items ── */
const unpaid  = students.filter(s => !s.paid);
const class10A = students.filter(s => s.class === '10A');

/* ── reduce — reduce to single value ─ */
const totalFee = students
  .reduce((sum, s) => sum + s.fee, 0);
// 14500

// Group by class
const byClass = students
  .reduce((acc, s) => {
    (acc[s.class] ??= []).push(s);
    return acc;
  }, {});
// { '10A': [...], '10B': [...] }

/* ── find — first matching item ──── */
const guru = students.find(s => s.id === 1);
// { id:1, name:'Guru', ... }

/* ── findIndex ───────────────────── */
const idx = students.findIndex(s => s.id === 2);
// 1

/* ── some / every ────────────────── */
const anyUnpaid  = students.some(s => !s.paid); // true
const allPaid    = students.every(s => s.paid); // false

/* ── includes / indexOf ──────────── */
[1,2,3].includes(2);   // true
[1,2,3].indexOf(2);   // 1
❓ Interview Q: map vs forEach?
map = RETURNS new array (non-mutating). forEach = runs function, returns undefined. Use map when you need a transformed array. Use forEach for side effects (logging, DOM updates). ZenCampus: students.map(s => s.name) for name list. students.forEach(s => renderCard(s)) for DOM!

⚠️ sort() mutates original array!
Always clone first: [...arr].sort() or arr.slice().sort(). Default sort converts to string — [10, 2, 1].sort() = [1, 10, 2]! Always provide comparator: .sort((a,b) => a - b)!

OOP Pillars

  • Encapsulation → private fields, public methods

Async / LINQ / Generics

Delegates / Events

SQL Core

  • LAG/LEAD → previous/next row value
  • Covering index → INCLUDE non-key cols, no lookup
  • Keyset pagination → WHERE Id > @lastId faster than OFFSET
  • N+1 problem → JOIN in 1 query vs loop queries

Dapper / Microservices

  • Dapper vs EF → raw SQL fast vs ORM migrations
  • Multi-map → JOIN → nested C# objects, splitOn
  • QueryMultiple → 3 result sets, 1 round-trip
  • Dapper transaction → pass tx to every Execute
  • IHttpClientFactory → pool reuse, no socket exhaustion
  • API Gateway → single entry: auth+route+ratelimit

JWT / Auth

  • JWT structure → Header.Payload.Signature Base64
  • Access vs Refresh → 15min vs 7days, revocable
  • Claims → userId, role, schoolId in payload
  • Payload security → NOT encrypted — no sensitive data!
  • Role vs Policy → role=string, policy=custom rules
  • Secret key → env/Key Vault, never appsettings

HTML / CSS

  • Semantic HTML → meaning for SEO + accessibility
  • Specificity → inline 1000, ID 100, class 10, tag 1
  • box-sizing border-box → padding included in width
  • Flexbox vs Grid → 1D components vs 2D page layout
  • Mobile-first → min-width adds, not max-width removes
  • Animate what → transform+opacity only (GPU)

JavaScript Fundamentals

  • var vs let vs const → func/block/block, const first!
  • == vs === → coercion vs strict type+value
  • ?? vs || → ??=null/undef only, ||=falsy
  • Arrow vs function → no own this, no arguments
  • Closure → inner fn remembers outer scope
  • Hoisting → var=undefined, let/const=TDZ error

JavaScript Async / DOM

  • Promise.all vs allSettled → all-or-none vs partial results
  • fetch error check → if(!res.ok) throw (404 not throws!)
  • Event delegation → parent listener, e.target.closest()
  • textContent vs innerHTML → safe vs XSS risk
  • call/apply/bind → comma/array/returns-fn
  • DOMContentLoaded vs load → DOM ready vs all assets ready

Array Methods

  • map → transform, returns new array
  • filter → select matching, returns new array
  • reduce → single value from array
  • find vs filter → first match vs all matches
  • sort mutation → [...arr].sort() — clone first!
  • flat/flatMap → nested arrays → flat, then map
