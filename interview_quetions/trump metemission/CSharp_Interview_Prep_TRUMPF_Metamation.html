Written Test

Full worked solutions for the six challenges on the paper your colleague shared. Console apps target .NET (any recent version) unless noted.

CHALLENGE 1

Print the 2nd largest of space-separated integers

Single pass, O(n), tracks the top two values without sorting. n != largest guards against a duplicate of the max being mistaken for a distinct second value.

using System;
using System.Linq;

class SecondLargest
{
    static void Main()
    {
        Console.Write("Enter integers separated by space: ");
        var numbers = Console.ReadLine()
            .Split(' ', StringSplitOptions.RemoveEmptyEntries)
            .Select(int.Parse)
            .ToArray();

        int largest = int.MinValue, second = int.MinValue;
        foreach (var n in numbers)
        {
            if (n > largest)
            {
                second = largest;
                largest = n;
            }
            else if (n > second && n != largest)
            {
                second = n;
            }
        }
        Console.WriteLine($"Second largest: {second}");
    }
}

Check: input -10 200 9 25 → largest 200, second 25. Matches the worked example on the sheet.

One-line LINQ alternative for a shorter answer if asked: numbers.Distinct().OrderByDescending(x => x).Skip(1).First(). It's O(n log n) and throws if there aren't 2 distinct values — mention both trade-offs out loud, interviewers like that.
CHALLENGE 2

Swap two integers without an intermediate variable

Arithmetic trick — works for int, but a + b can overflow for values near int.MaxValue:

static void Swap(ref int a, ref int b)
{
    a = a + b;
    b = a - b;
    a = a - b;
}

Overflow-safe alternative — bitwise XOR:

static void Swap(ref int a, ref int b)
{
    a ^= b;
    b ^= a;
    a ^= b;
}
Say out loud: XOR-swap breaks if a and b are the same variable (aliased) — it zeroes it out. Flag that as a known caveat; interviewers who ask this question are usually listening for it.
CHALLENGE 3

Same swap, using generics — for any numeric type

Arithmetic and XOR don't compile against a generic T — the compiler has no idea it's numeric. The clean, idiomatic C# answer is tuple deconstruction, which needs no explicit temp in your source (the compiler manages storage under the hood, but nothing you write is a "variable"):

static void Swap<T>(ref T a, ref T b)
{
    (a, b) = (b, a);
}

// usage
int x = 5, y = 9;
Swap(ref x, ref y);

double p = 1.5, q = 3.25;
Swap(ref p, ref q);

If the interviewer explicitly disallows tuple sugar, fall back to Interlocked.Exchange shape or constrain T to a numeric interface (INumber<T> in .NET 7+) and reuse the arithmetic trick — but lead with the tuple version; it's what a working C# dev reaches for.

CHALLENGE 4

Number-guessing game — computer guesses, and detects cheating

This is binary search dressed up as a game. Maintain the range [low, high] that is still consistent with every answer given so far. Guess the midpoint; narrow the range based on the reply. Cheat detection is free: if the range ever becomes empty (low > high), the user's answers contradict each other — no number satisfies them all.

using System;

class GuessingGame
{
    static void Main()
    {
        int low = 1, high = 1000, attempts = 0;
        Console.WriteLine("Think of a number 1-1000. I'll guess it.");

        while (true)
        {
            if (low > high)
            {
                Console.WriteLine("Impossible — your answers contradict each other. You're cheating!");
                return;
            }

            int guess = low + (high - low) / 2;
            attempts++;
            Console.Write($"Attempt {attempts}: is it {guess}? (high/low/correct): ");
            string reply = Console.ReadLine()?.Trim().ToLower();

            switch (reply)
            {
                case "correct":
                    Console.WriteLine($"Got it in {attempts} attempts!");
                    return;
                case "high":   // secret is lower than guess
                    high = guess - 1;
                    break;
                case "low":    // secret is higher than guess
                    low = guess + 1;
                    break;
                default:
                    Console.WriteLine("Please answer high, low, or correct.");
                    attempts--;
                    break;
            }
        }
    }
}

Fewest attempts: binary search over 1000 values needs at most ⌈log₂ 1000⌉ = 10 guesses — say this number out loud, it's exactly the "bonus points" the sheet is fishing for.

CHALLENGE 5

Browse a folder, list files by ascending size (WinForms + LINQ)

using System;
using System.IO;
using System.Linq;
using System.Windows.Forms;

public class MainForm : Form
{
    readonly Button _browse = new() { Text = "Browse Folder...", Dock = DockStyle.Top };
    readonly ListView _list = new() { View = View.Details, Dock = DockStyle.Fill };

    public MainForm()
    {
        Text = "Folder File-Size Browser"; Width = 600; Height = 420;
        _list.Columns.Add("File Name", 360);
        _list.Columns.Add("Size (bytes)", 140);
        Controls.Add(_list);
        Controls.Add(_browse);
        _browse.Click += (s, e) => LoadFolder();
    }

    void LoadFolder()
    {
        using var dlg = new FolderBrowserDialog();
        if (dlg.ShowDialog() != DialogResult.OK) return;

        _list.Items.Clear();
        var rows = new DirectoryInfo(dlg.SelectedPath)
            .GetFiles()
            .OrderBy(f => f.Length)                     // LINQ: ascending by size
            .Select(f => new ListViewItem(new[] { f.Name, f.Length.ToString("N0") }));

        _list.Items.AddRange(rows.ToArray());
    }

    [STAThread]
    static void Main() => Application.Run(new MainForm());
}
WPF variant: same LINQ query, bind the result to an ObservableCollection<FileInfo> exposed by a view-model and render it in a DataGrid — keeps file-system access out of the view, which is the point of MVVM (see section C).
CHALLENGE 6

NumericTextBox custom control — digits only, configurable range

A WPF UserControl wrapping a TextBox, exposing MinValue/MaxValue/Value as dependency properties so it's bindable and settable from XAML like any built-in control. Default range is 0–9999 per the spec; consumers can widen it up to 999999.

// NumericTextBox.xaml.cs
public partial class NumericTextBox : UserControl
{
    public static readonly DependencyProperty MinValueProperty =
        DependencyProperty.Register(nameof(MinValue), typeof(int), typeof(NumericTextBox),
            new PropertyMetadata(0));

    public static readonly DependencyProperty MaxValueProperty =
        DependencyProperty.Register(nameof(MaxValue), typeof(int), typeof(NumericTextBox),
            new PropertyMetadata(9999));

    public static readonly DependencyProperty ValueProperty =
        DependencyProperty.Register(nameof(Value), typeof(int), typeof(NumericTextBox),
            new FrameworkPropertyMetadata(0, FrameworkPropertyMetadataOptions.BindsTwoWayByDefault, OnValueChanged));

    public int MinValue { get => (int)GetValue(MinValueProperty); set => SetValue(MinValueProperty, value); }
    public int MaxValue { get => (int)GetValue(MaxValueProperty); set => SetValue(MaxValueProperty, value); }
    public int Value    { get => (int)GetValue(ValueProperty);    set => SetValue(ValueProperty, value); }

    public NumericTextBox()
    {
        InitializeComponent();
        InnerTextBox.Text = Value.ToString();
        DataObject.AddPastingHandler(InnerTextBox, OnPaste);
    }

    // blocks any non-digit keystroke
    void InnerTextBox_PreviewTextInput(object sender, TextCompositionEventArgs e)
        => e.Handled = !e.Text.All(char.IsDigit);

    // PreviewTextInput doesn't fire for paste — handle it separately
    void OnPaste(object sender, DataObjectPastingEventArgs e)
    {
        var text = e.DataObject.GetData(typeof(string)) as string;
        if (text is null || !text.All(char.IsDigit)) e.CancelCommand();
    }

    void InnerTextBox_TextChanged(object sender, TextChangedEventArgs e)
    {
        if (int.TryParse(InnerTextBox.Text, out int typed))
        {
            int clamped = Math.Clamp(typed, MinValue, MaxValue);
            Value = clamped;
            if (clamped != typed)
            {
                InnerTextBox.Text = clamped.ToString();
                InnerTextBox.CaretIndex = InnerTextBox.Text.Length;
            }
        }
        else if (string.IsNullOrEmpty(InnerTextBox.Text))
        {
            Value = MinValue;
        }
    }

    static void OnValueChanged(DependencyObject d, DependencyPropertyChangedEventArgs e)
    {
        var ctrl = (NumericTextBox)d;
        if (ctrl.InnerTextBox.Text != e.NewValue.ToString())
            ctrl.InnerTextBox.Text = e.NewValue.ToString();
    }
}

XAML usage, any range up to the required maximum:

<controls:NumericTextBox MinValue="0" MaxValue="999999" Value="{Binding Quantity}" />
Talking points if pushed further: this is functionally a poor man's Spinner/NumericUpDown; mention IntegerUpDown from the WPF Toolkit as the "don't reinvent it in production" answer, and that a WinForms version would override OnKeyPress instead of PreviewTextInput.

Same format as the written test: the classic warm-up problems interviewers reach for when they want to see you think, not just recall. Say the time/space complexity out loud for each — that's usually the real thing being scored.

PRACTICE 1

Reverse a string in place — no Array.Reverse, no .Reverse()

Two pointers closing in from both ends, O(n) time, O(1) extra space (on a char[]; strings themselves are immutable in C#, so you reverse a mutable copy):

static string ReverseString(string input)
{
    var chars = input.ToCharArray();
    int left = 0, right = chars.Length - 1;
    while (left < right)
    {
        (chars[left], chars[right]) = (chars[right], chars[left]);
        left++;
        right--;
    }
    return new string(chars);
}
PRACTICE 2

Check if a string is a palindrome — case- and punctuation-insensitive

Interviewers usually mean "A man, a plan, a canal: Panama" should count — filter to letters/digits before comparing:

static bool IsPalindrome(string input)
{
    var cleaned = new string(input.Where(char.IsLetterOrDigit).ToArray()).ToLowerInvariant();
    int left = 0, right = cleaned.Length - 1;
    while (left < right)
    {
        if (cleaned[left] != cleaned[right]) return false;
        left++;
        right--;
    }
    return true;
}
PRACTICE 3

Check if two strings are anagrams of each other

Count character frequency once, compare — O(n), avoids the O(n log n) cost of sorting both strings:

static bool AreAnagrams(string a, string b)
{
    a = a.Replace(" ", "").ToLowerInvariant();
    b = b.Replace(" ", "").ToLowerInvariant();
    if (a.Length != b.Length) return false;

    var counts = new Dictionary<char, int>();
    foreach (var c in a) counts[c] = counts.GetValueOrDefault(c) + 1;
    foreach (var c in b)
    {
        if (!counts.ContainsKey(c) || --counts[c] < 0) return false;
    }
    return true;
}
PRACTICE 4

Count word frequency in a block of text

static Dictionary<string, int> WordFrequency(string text)
{
    return text.Split(new[] { ' ', '\n', '\r', '\t', '.', ',' }, StringSplitOptions.RemoveEmptyEntries)
        .GroupBy(w => w.ToLowerInvariant())
        .OrderByDescending(g => g.Count())
        .ToDictionary(g => g.Key, g => g.Count());
}
LINQ's GroupBy here replaces what would otherwise be a manual Dictionary<string,int> with a hand-rolled loop — mention you know both and picked LINQ for readability.
PRACTICE 5

FizzBuzz — the filter question, know it cold

for (int i = 1; i <= 100; i++)
{
    if (i % 15 == 0) Console.WriteLine("FizzBuzz");
    else if (i % 3 == 0) Console.WriteLine("Fizz");
    else if (i % 5 == 0) Console.WriteLine("Buzz");

PRACTICE 6

Find all duplicate values in an array

LINQ one-liner for the interview room, plus the manual O(n) version so you can explain what it's doing underneath:

// LINQ
var duplicates = numbers.GroupBy(n => n).Where(g => g.Count() > 1).Select(g => g.Key);

// manual O(n) with a HashSet
static IEnumerable<int> FindDuplicates(int[] numbers)
{
    var seen = new HashSet<int>();
    var dupes = new HashSet<int>();
    foreach (var n in numbers)
    {
        if (!seen.Add(n)) dupes.Add(n);
    }
    return dupes;
}
PRACTICE 7

Find the missing number in an array containing 1..N with one gap

The expected sum of 1..N is N*(N+1)/2 — subtract the actual sum and what's left is the missing number. O(n) time, O(1) space, no sorting needed:

static int FindMissing(int[] numbers, int n)
{
    int expectedSum = n * (n + 1) / 2;
    int actualSum = numbers.Sum();
    return expectedSum - actualSum;
}
If asked for the XOR variant (safer against int overflow on very large N): XOR all numbers 1..N with all array elements — every pair cancels except the missing one.
PRACTICE 8

Two Sum — return the indices of two numbers that add up to a target

Naive nested loop is O(n²). One pass with a dictionary mapping value → index gets it to O(n):

static int[] TwoSum(int[] numbers, int target)
{
    var seen = new Dictionary<int, int>(); // value -> index
    for (int i = 0; i < numbers.Length; i++)
    {
        int complement = target - numbers[i];
        if (seen.TryGetValue(complement, out int j))
            return new[] { j, i };
        seen[numbers[i]] = i;
    }
    throw new ArgumentException("No two numbers sum to target");
}
PRACTICE 9

Fibonacci — naive recursion vs memoized vs iterative

Lead with why naive recursion is bad (exponential re-computation), then show the fix:

// naive: O(2^n) — recomputes the same sub-values over and over
static long FibNaive(int n) => n <= 1 ? n : FibNaive(n - 1) + FibNaive(n - 2);

// memoized: O(n) time, O(n) space
static long FibMemo(int n, Dictionary<int, long> cache = null)
{
    cache ??= new();
    if (n <= 1) return n;
    if (cache.TryGetValue(n, out var hit)) return hit;
    var result = FibMemo(n - 1, cache) + FibMemo(n - 2, cache);
    cache[n] = result;
    return result;
}

// iterative: O(n) time, O(1) space — the answer to lead with in an interview
static long FibIterative(int n)
{
    if (n <= 1) return n;
    long a = 0, b = 1;
    for (int i = 2; i <= n; i++)
    {
        (a, b) = (b, a + b);
    }
    return b;
}
PRACTICE 10

Factorial — including N large enough to overflow long

// standard recursive version — fine up to ~20!
static long Factorial(int n) => n <= 1 ? 1 : n * Factorial(n - 1);

// 21! already overflows long — use BigInteger for arbitrary N
using System.Numerics;

static BigInteger FactorialBig(int n)
{
    BigInteger result = 1;
    for (int i = 2; i <= n; i++) result *= i;
    return result;
}

Knowing long.MaxValue is roughly 9.2×10¹⁸ and that 21! already exceeds it is the kind of detail that separates "wrote code that compiles" from "understands what it computes."

PRACTICE 11

Check if a number is prime — efficiently

Only need to test divisors up to √n — if nothing up to there divides it, nothing above does either:

static bool IsPrime(int n)
{
    if (n < 2) return false;
    if (n == 2) return true;
    if (n % 2 == 0) return false;

    for (int i = 3; i * i <= n; i += 2)
    {
        if (n % i == 0) return false;
    }
    return true;
}
PRACTICE 12

Implement a sort by hand — bubble sort, then quicksort's partition

Nobody wants you to out-perform Array.Sort — this checks you understand what O(n²) vs O(n log n) actually looks like in code:

// bubble sort — O(n^2), the "I understand the naive baseline" answer
static void BubbleSort(int[] arr)
{
    for (int i = 0; i < arr.Length - 1; i++)
        for (int j = 0; j < arr.Length - i - 1; j++)
            if (arr[j] > arr[j + 1])
                (arr[j], arr[j + 1]) = (arr[j + 1], arr[j]);
}

// quicksort — average O(n log n), the "I understand divide-and-conquer" answer
static void QuickSort(int[] arr, int low, int high)
{
    if (low >= high) return;
    int pivot = arr[high], i = low - 1;
    for (int j = low; j < high; j++)
    {
        if (arr[j] < pivot)
        {
            i++;
            (arr[i], arr[j]) = (arr[j], arr[i]);
        }
    }
    (arr[i + 1], arr[high]) = (arr[high], arr[i + 1]);
    QuickSort(arr, low, i);
    QuickSort(arr, i + 2, high);
}
PRACTICE 13

Check if brackets in an expression are balanced

Classic stack problem — push openers, pop-and-match on closers:

static bool IsBalanced(string expr)
{
    var stack = new Stack<char>();
    var pairs = new Dictionary<char, char> { [')'] = '(', [']'] = '[', ['}'] = '{' };

    foreach (var c in expr)
    {
        if (c == '(' || c == '[' || c == '{')
            stack.Push(c);
        else if (pairs.ContainsKey(c))
        {
            if (stack.Count == 0 || stack.Pop() != pairs[c]) return false;
        }
    }
    return stack.Count == 0;
}
PRACTICE 14

Implement a generic Stack<T> from scratch (array-backed)

Ties generics, arrays, and basic data-structure design together — a common "show me you understand what's under System.Collections.Generic" question:

public class MyStack<T>
{
    private T[] _items = new T[4];
    private int _count;

    public int Count => _count;

    public void Push(T item)
    {
        if (_count == _items.Length)
            Array.Resize(ref _items, _items.Length * 2);
        _items[_count++] = item;
    }

    public T Pop()
    {
        if (_count == 0) throw new InvalidOperationException("Stack is empty");
        return _items[--_count];
    }

    public T Peek()
    {
        if (_count == 0) throw new InvalidOperationException("Stack is empty");
        return _items[_count - 1];
    }
}

Mention the amortized O(1) push: doubling the backing array on overflow means the average cost per push stays constant even though a resize itself is O(n).

PRACTICE 17

Find the first non-repeated character in a string

Two passes, O(n): count frequencies, then scan again in original order for the first count of 1 (a Dictionary alone won't preserve order reliably enough to skip the second pass):

static char? FirstNonRepeatedChar(string s)
{
    var counts = new Dictionary<char, int>();
    foreach (var c in s) counts[c] = counts.GetValueOrDefault(c) + 1;
    foreach (var c in s)
        if (counts[c] == 1) return c;
    return null;
}
PRACTICE 18

String compression — "aabcccccaaa" → "a2b1c5a3"

static string Compress(string s)
{
    if (string.IsNullOrEmpty(s)) return s;

    var sb = new StringBuilder();
    int count = 1;
    for (int i = 1; i <= s.Length; i++)
    {
        if (i < s.Length && s[i] == s[i - 1])
        {
            count++;
        }
        else
        {
            sb.Append(s[i - 1]).Append(count);
            count = 1;
        }
    }
    return sb.ToString();
}

O(n) time; StringBuilder avoids the O(n²) blowup of repeated string concatenation (see section B, Q18).

PRACTICE 19

Rotate an array right by k positions — in place

The reversal trick gets this to O(n) time, O(1) extra space, no second array needed: reverse the whole array, then reverse each of the two segments the rotation point splits it into.

static void RotateRight(int[] arr, int k)
{
    int n = arr.Length;
    k %= n;
    Reverse(arr, 0, n - 1);
    Reverse(arr, 0, k - 1);
    Reverse(arr, k, n - 1);
}

static void Reverse(int[] arr, int start, int end)
{
    while (start < end)
    {
        (arr[start], arr[end]) = (arr[end], arr[start]);
        start++;
        end--;
    }
}
PRACTICE 20

Maximum subarray sum — Kadane's algorithm

The naive approach checks every subarray in O(n²) or O(n³). Kadane's runs a single pass, tracking the best sum ending at each position:

static int MaxSubArraySum(int[] nums)
{
    int maxSoFar = nums[0], maxEndingHere = nums[0];
    for (int i = 1; i < nums.Length; i++)
    {
        maxEndingHere = Math.Max(nums[i], maxEndingHere + nums[i]);
        maxSoFar = Math.Max(maxSoFar, maxEndingHere);
    }
    return maxSoFar;
}
PRACTICE 21

Remove duplicates from a sorted array, in place

Because it's sorted, duplicates are always adjacent — a single write-pointer walk does it without a HashSet:

static int RemoveDuplicates(int[] nums)
{
    if (nums.Length == 0) return 0;

    int writeIndex = 1;
    for (int i = 1; i < nums.Length; i++)
    {
        if (nums[i] != nums[writeIndex - 1])
        {
            nums[writeIndex] = nums[i];
            writeIndex++;
        }
    }
    return writeIndex; // new logical length; nums[0..writeIndex) is the deduped array
}
PRACTICE 22

Binary search, written by hand (not Array.BinarySearch)

static int BinarySearch(int[] sorted, int target)
{
    int low = 0, high = sorted.Length - 1;
    while (low <= high)
    {
        int mid = low + (high - low) / 2; // avoids overflow vs (low+high)/2
        if (sorted[mid] == target) return mid;
        else if (sorted[mid] < target) low = mid + 1;
        else high = mid - 1;
    }
    return -1;
}

O(log n). This is the same halving idea as the guessing-game challenge earlier — worth pointing that out if the interviewer asks both.

PRACTICE 23

Detect a cycle in a linked list — Floyd's tortoise and hare

public class Node
{
    public int Value;
    public Node Next;
}

static bool HasCycle(Node head)
{
    Node slow = head, fast = head;
    while (fast != null && fast.Next != null)
    {
        slow = slow.Next;
        fast = fast.Next.Next;
        if (slow == fast) return true; // fast lapped slow — only possible inside a loop
    }
    return false;
}

O(n) time, O(1) space — no HashSet of visited nodes needed, which is the naive but memory-heavier alternative worth mentioning as the fallback.

PRACTICE 24

Find the middle node of a linked list in a single pass

Same two-pointer idea as the cycle check — fast moves twice as fast as slow, so when fast reaches the end, slow is at the midpoint:

static Node FindMiddle(Node head)
{
    Node slow = head, fast = head;
    while (fast != null && fast.Next != null)
    {
        slow = slow.Next;
        fast = fast.Next.Next;
    }
    return slow;
}
PRACTICE 25

Implement a Queue using two Stacks

Push always goes to _in. Dequeue drains _in into _out (reversing order) only when _out is empty — amortized O(1) per operation, not O(n) every time:

public class QueueFromStacks<T>
{
    private Stack<T> _in = new();
    private Stack<T> _out = new();

    public void Enqueue(T item) => _in.Push(item);

    public T Dequeue()
    {
        if (_out.Count == 0)
        {
            while (_in.Count > 0) _out.Push(_in.Pop());
        }
        if (_out.Count == 0) throw new InvalidOperationException("Queue is empty");
        return _out.Pop();
    }
}
PRACTICE 26

Build a minimal generic LinkedList<T> — node type plus traversal

public class LinkedListNode<T>
{
    public T Value;
    public LinkedListNode<T> Next;
    public LinkedListNode(T value) => Value = value;
}

public class SimpleLinkedList<T>
{
    private LinkedListNode<T> _head;
    private LinkedListNode<T> _tail;

    public void Add(T value)
    {
        var node = new LinkedListNode<T>(value);
        if (_head == null) _head = _tail = node;
        else { _tail.Next = node; _tail = node; }
    }

    public IEnumerable<T> Traverse()
    {
        var current = _head;
        while (current != null)
        {
            yield return current.Value;
            current = current.Next;
        }
    }
}

Add is O(1) because _tail is tracked separately — without it, appending would mean walking the whole list every time.

    // default order: highest salary first
    public int CompareTo(Employee other) => other.Salary.CompareTo(Salary);

public class ByNameComparer : IComparer<Employee>

Make a custom type work in a foreach loop

Implement IEnumerable<T> and use yield return — the compiler builds the state machine for you, so you never write an explicit IEnumerator class by hand:

public class Range : IEnumerable<int>
{
    private readonly int _start, _end;
    public Range(int start, int end) { _start = start; _end = end; }

    public IEnumerator<int> GetEnumerator()
    {
        for (int i = _start; i <= _end; i++)
            yield return i;
    }

}

PRACTICE 31

Deep copy vs shallow copy — show the difference in code

MemberwiseClone() (and a naive hand-written Clone) copies value-type fields fully but only copies the reference for any field that's itself a reference type — both objects end up pointing at the same nested object:

public class Address { public string City; }

public class Person
{
    public string Name;
    public Address Address;

    // shallow: clone shares the SAME Address instance as the original
    public Person ShallowClone() => new Person { Name = Name, Address = Address };

    // deep: clone gets its own independent Address
    public Person DeepClone() => new Person { Name = Name, Address = new Address { City = Address.City } };
}

Prove it: mutate clone.Address.City after a shallow clone and the original's Address.City changes too — that's the bug this question is testing for.

PRACTICE 32

Check if a number is a power of two — bitwise, O(1)

A power of two has exactly one bit set (0100, 1000, ...). Subtracting 1 flips every bit up to and including that one — ANDing the two together always gives zero, for any power of two and only for a power of two:

static bool IsPowerOfTwo(int n) => n > 0 && (n & (n - 1)) == 0;
PRACTICE 33

Count the number of set bits in an integer (Hamming weight)

Brian Kernighan's trick: n & (n - 1) clears the lowest set bit each time, so the loop runs once per set bit rather than once per bit position:

static int CountSetBits(int n)
{
    int count = 0;
    while (n != 0)
    {
        n &= (n - 1);
        count++;
    }
    return count;
}
PRACTICE 34

Find the intersection of two arrays

static int[] Intersect(int[] a, int[] b)
{
    var setA = new HashSet<int>(a);
    return b.Where(setA.Contains).Distinct().ToArray();
}

Building the HashSet once makes each lookup O(1), so the whole thing is O(n + m) instead of the O(n·m) a nested-loop version would cost.

PRACTICE 36

Producer-consumer with BlockingCollection<T>

The concurrency question — one thread produces, another consumes, without either of you hand-rolling locks around a shared queue:

using System.Collections.Concurrent;

var queue = new BlockingCollection<int>(boundedCapacity: 10);

var producer = Task.Run(() =>
{
    for (int i = 0; i < 100; i++) queue.Add(i);
    queue.CompleteAdding();
});

var consumer = Task.Run(() =>
{
    foreach (var item in queue.GetConsumingEnumerable())
        Console.WriteLine($"Processing {item}");
});

Task.WaitAll(producer, consumer);

boundedCapacity makes Add block once the queue is full, throttling a fast producer to match a slower consumer — that back-pressure is the whole reason to reach for this over a plain Queue<T> with manual locking.

A · C# & OOP Fundamentals

The "basic skills" line in the JD is literally "programming and problem solving, .NET/C#, OOP design" — this section is the core of the face-to-face.

1. Value types vs reference types — what's the practical difference?

Value types (struct, int, bool, enum) hold their data directly and get copied on assignment/passing. Reference types (class, arrays, strings, delegates) hold a pointer to data on the heap — assignment copies the reference, not the data. Practical consequence: mutate a struct you passed by value and the caller's copy is untouched; mutate a class instance and every reference sees it.

2. What is boxing/unboxing, and why does it matter for performance?

Boxing wraps a value type in an object on the heap (e.g. assigning an int to an object); unboxing casts it back. Every box is a heap allocation plus GC pressure — costly in a tight loop. Generics (List<int> vs old-style ArrayList) exist largely to avoid this.

3. Walk through the four OOP pillars with a C# example each.
  • Encapsulation — private fields exposed via properties, so invariants can be enforced (a Machine class rejecting a negative FeedRate).
  • Abstraction — an IExporter interface hides whether you're writing DXF or G-code.
  • Inheritance — LaserCuttingJob : MachiningJob reuses shared scheduling logic.
  • Polymorphism — a list of MachiningJob where each Run() call dispatches to the correct override at runtime.
4. Abstract class vs interface — when do you reach for which?

Abstract class: shared state and partial implementation, single inheritance only, models an "is-a" hierarchy. Interface: a contract with no state, a type can implement many, models "can-do" capability. Since C# 8, interfaces can carry default method bodies, narrowing the gap — but state still belongs in a base class.

5. Method overloading vs overriding — compile-time vs runtime polymorphism.

Overloading: same name, different signature, resolved by the compiler at compile time. Overriding: virtual/abstract base member replaced by a derived override, resolved at runtime via the object's actual type (v-table dispatch).

6. What do virtual, override, new, and sealed do on a method?

virtual marks a method as overridable; override replaces it polymorphically; new hides the base member instead of overriding it (no polymorphism — call through a base reference and you get the base version); sealed on an override stops further derived classes from overriding it again.

7. struct vs class — beyond "one's a value type."

Structs are cheap for small, immutable, short-lived data (a Point3D in a geometry kernel) — no heap allocation, no GC. They can't have a parameterless constructor pre-C# 10 in the way classes do, can't be null unless wrapped in Nullable<T>, and large structs are expensive to copy — that copy cost is the usual reason to switch to a class.

8. Auto-implemented properties vs fields — why not just use public fields?

Properties let you add validation, computed logic, or a breakpoint-friendly setter later without breaking binary/source compatibility with callers. public int Speed { get; set; } compiles to a backing field plus get/set methods — free encapsulation with field-like syntax.

9. readonly vs const.

const is a compile-time constant, inlined at every call site, must be a primitive/string, implicitly static. readonly is assigned once at runtime (in the declaration or constructor), can be instance-level, and can hold any type including one computed at startup.

10. == vs .Equals() vs ReferenceEquals.

ReferenceEquals always compares identity. == is operator-overloadable — value types compare by value, reference types by identity unless the class overloads it (like string, which compares content). .Equals() is virtual and is what you override to define custom value equality; overriding it without also overriding GetHashCode() breaks dictionaries and hash sets.

11. How do you handle exceptions correctly in C#? What's the difference between throw and throw ex?

Catch only what you can meaningfully handle; use finally (or better, using) for cleanup; create specific exception types instead of throwing generic Exception. throw; inside a catch re-throws preserving the original stack trace; throw ex; resets the stack trace to the current frame, hiding where it actually failed — always prefer bare throw;.

12. Delegates vs events, and where do Func/Action/Predicate fit in?

A delegate is a type-safe function pointer; an event is a delegate field with restricted access (outsiders can +=/-= but not invoke or reassign it directly) — the standard pattern for publish/subscribe, like a UI button's Click. Func<T,TResult>, Action<T>, and Predicate<T> are just pre-defined generic delegate shapes so you don't declare your own delegate type for common cases.

B · .NET & CLR

13. What is the CLR, and what does JIT compilation actually do?

The Common Language Runtime hosts execution: memory management, GC, type safety, exception handling. C# compiles to IL (intermediate language), and the JIT compiles that IL to native machine code the first time a method runs on the current CPU — trading a bit of startup cost for code tuned to the actual hardware.

14. Explain GC generations, and when you actually need IDisposable.

Gen 0 holds new, short-lived objects and is collected most often and cheaply; survivors promote to Gen 1, then Gen 2 for long-lived objects, collected less frequently since scanning is more expensive. GC only manages memory — it doesn't know about unmanaged resources (file handles, DB connections, GDI handles). Implement IDisposable to release those deterministically, and always consume it via using so Dispose() runs even on an exception.

15. IEnumerable vs IEnumerator vs IQueryable.

IEnumerable<T> exposes GetEnumerator() — "this can be iterated." IEnumerator<T> is the cursor doing the iterating (MoveNext/Current). IQueryable<T> extends IEnumerable<T> but builds an expression tree instead of running immediately — that's how EF Core translates a LINQ query into SQL before anything executes.

16. What is deferred execution in LINQ, and why does it trip people up?

Where/Select/OrderBy build a query but don't run it until enumerated (foreach, .ToList(), .Count()). Trap: if the underlying collection changes between defining the query and enumerating it, you'll see the mutated data — or a InvalidOperationException for "collection modified." Force immediate evaluation with .ToList()/.ToArray() when you need a snapshot.

17. async/await — what problem does it solve, and how is it different from spinning up a new thread?

It frees a thread while waiting on I/O (disk, network, DB) instead of blocking it — the thread returns to the pool and picks up other work, then resumes your method via a continuation when the awaited task completes. It's not automatically parallel — await Task.Delay(1000) uses zero extra threads while waiting. Use Task.Run only to offload genuine CPU-bound work onto a background thread.

18. Why is string immutable, and when should you use StringBuilder instead?

Every apparent mutation (+=, .Replace()) allocates a new string, since the original is fixed once created — that's what makes strings safe to share and hash. In a loop building up text, that means N allocations; StringBuilder mutates an internal buffer in place, so use it once you're concatenating more than a handful of times.

19. Why use generics instead of object, and what do constraints like where T : IComparable<T> buy you?

Generics give compile-time type safety and avoid boxing/casting that object-based code needs. A constraint restricts what T can be so the compiler lets you call members on it — where T : IComparable<T> lets a generic sort method call CompareTo without knowing the concrete type.

20. .NET Framework vs .NET Core / modern .NET vs .NET Standard — do you need to know the difference today?

.NET Framework is the legacy Windows-only runtime; .NET Core (now unified simply as ".NET 5" onward) is the cross-platform, actively developed runtime; .NET Standard was a compatibility spec for sharing libraries across both, largely superseded now that Framework is in maintenance mode. Worth knowing which one a WPF/WinForms desktop product targets, since older CAD/CAM codebases are frequently still on Framework.

C · WPF / WinForms — GUI Programming

Explicit bonus skill in the JD, and challenges 5 & 6 above are this in disguise.

21. How does data binding work in WPF at a mechanical level?

A binding wires a target property (e.g. a TextBox.Text) to a source property via a path, resolved through the element's DataContext at runtime by reflection (or compiled bindings in newer WPF). For the UI to react to source changes, the source must raise INotifyPropertyChanged; for two-way sync, the binding's Mode must be TwoWay.

22. Explain MVVM and why WPF leans on it so heavily.

Model (data/business logic) → ViewModel (state + commands, no UI types) → View (XAML, binds to the ViewModel). It exists because WPF's binding engine makes it nearly free, and it keeps UI logic unit-testable — you can test a ViewModel's behavior with no window ever created.

23. What's a DependencyProperty, and why not just use a normal CLR property?

It plugs into WPF's property system, which gives you binding, styling/templating, animation, and value coercion/inheritance for free — none of that works on a plain auto-property. The cost is boilerplate: a static DependencyProperty field plus a CLR wrapper (exactly the pattern in the NumericTextBox answer above).

24. WPF vs WinForms — how do you actually choose?

WinForms wraps GDI+/native Win32 controls — simple, fast to prototype, pixel-based layout, no real data-binding story. WPF renders itself via DirectX with a resolution-independent vector layout system, rich styling/templating/animation, and first-class binding — better for complex, evolving UI, at the cost of a steeper learning curve. For a CAD/CAM tool with dense property panels and live previews, WPF is the natural fit; a quick internal utility often isn't worth the overhead.

25. How do you restrict a TextBox to numeric-only input, end to end (not just keystrokes)?

Filter PreviewTextInput for keystrokes, but that alone misses paste (Ctrl+V) and drag-drop text — those bypass TextInput entirely. Also handle DataObject.AddPastingHandler to validate/cancel a paste, and validate again in the bound property setter as the last line of defense. See the full NumericTextBox solution above.

26. What's an ICommand, and why not just wire a Click event handler?

ICommand (Execute/CanExecute) lets the ViewModel expose an action a View binds to via Command="{Binding SaveCommand}", so the button's enabled state and behavior come from testable ViewModel logic rather than code-behind — the MVVM-friendly replacement for an event handler.

27. Why does a bound WPF control not update, and how do you fix it?

Usually the source class doesn't implement INotifyPropertyChanged, or the setter changes the backing field without firing PropertyChanged. Fix: raise PropertyChanged(nameof(Property)) in every setter (or use [ObservableProperty] source generators in modern MVVM toolkits). Second most common cause: binding to a field instead of a property — WPF bindings require a property.

D · SQL & Database

28. INNER vs LEFT vs RIGHT vs FULL JOIN — with a one-line example.

INNER JOIN keeps only matching rows on both sides; LEFT JOIN keeps every row from the left table, with NULLs where there's no right match; RIGHT JOIN is the mirror; FULL JOIN keeps unmatched rows from both sides.

SELECT o.Id, c.Name
FROM Orders o
LEFT JOIN Customers c ON o.CustomerId = c.Id;
29. What is normalization, and when would you deliberately denormalize?

1NF: atomic columns, no repeating groups. 2NF: every non-key column depends on the whole primary key (matters for composite keys). 3NF: no column depends on another non-key column (removes transitive dependency). Denormalize deliberately for read-heavy reporting tables where join cost outweighs the duplication/consistency risk.

30. What does a database index actually speed up, and what does it cost?

An index (typically a B-tree) lets the engine find rows by a lookup instead of a full table scan — huge win for WHERE/JOIN/ORDER BY on the indexed column. Cost: extra storage, and every INSERT/UPDATE/DELETE has to maintain the index too — over-indexing slows writes.

31. WHERE vs HAVING.

WHERE filters individual rows before grouping/aggregation happens. HAVING filters groups after GROUP BY — use it when the filter condition itself is an aggregate, like HAVING COUNT(*) > 5.

32. Primary key vs foreign key vs unique constraint.

Primary key uniquely identifies a row and disallows NULL; a table has exactly one (possibly composite). Foreign key enforces referential integrity, pointing to another table's primary/unique key. Unique constraint enforces no-duplicates on a column but, unlike a primary key, allows one NULL and a table can have several.

33. How do you prevent SQL injection from C#?

Never concatenate user input into SQL text. Use parameterized queries (SqlCommand.Parameters.Add) or an ORM like Entity Framework, which parameterizes automatically. Parameters ensure user input is always treated as data, never as executable SQL syntax.

using var cmd = new SqlCommand("SELECT * FROM Users WHERE Email = @email", conn);
cmd.Parameters.AddWithValue("@email", userInput);

E · Git (and Mercurial)

34. git merge vs git rebase — what's the actual difference in history?

merge creates a new commit joining two histories, preserving exactly what happened, but the log gets a merge-commit graph. rebase replays your commits on top of the target branch one by one, producing a linear history — cleaner to read, but it rewrites commit hashes, so never rebase a branch someone else has already pulled.

35. How do you resolve a merge conflict?

Git marks the conflicting region in the file with <<<<<<</=======/>>>>>>> markers. Edit the file to the correct final content, remove the markers, git add the file, then continue the merge/rebase. For anything non-trivial, open the diff in a merge tool rather than eyeballing raw markers.

36. The JD lists Mercurial alongside Git — what's the conceptual overlap?

Both are distributed VCS: every clone has the full history, commits form a DAG, and you can work fully offline. Differences are mostly branching model and UX — Mercurial models a branch as a permanent named attribute on commits (plus lightweight "bookmarks" as a Git-branch analogue), while Git branches are just movable pointers. If you know Git well, say so plainly and note the concepts transfer even if the commands don't.

37. Describe your day-to-day Git workflow.

Feature branch off main/develop, small focused commits with descriptive messages, rebase or merge latest target branch in before opening a PR, code review, squash or merge on approval. Mention you keep commits atomic enough that git bisect or a revert doesn't take down unrelated work — that's the real signal of Git maturity.

F · Unit Testing

38. What makes a good unit test?

Arrange–Act–Assert structure; tests one behavior; fast, isolated, repeatable, and self-verifying (FIRST). No hidden dependency on execution order, system clock, network, or file system — those belong behind an abstraction you can fake.

39. Unit test vs integration test — where's the line?

A unit test exercises one class/method in isolation, dependencies faked or mocked, no I/O. An integration test exercises multiple real components together — e.g. your repository class against an actual database — to verify they cooperate correctly. You want far more of the former than the latter; unit tests should catch most regressions cheaply.

40. What is mocking, and how would you test a class with Moq?

A mock is a fake implementation of a dependency (usually via an interface) that lets you control its behavior and assert how it was called, so the test isolates just the class under test.

var repo = new Mock<IJobRepository>();
repo.Setup(r => r.GetById(1)).Returns(new Job { Id = 1 });

var service = new JobService(repo.Object);
var job = service.Load(1);

Assert.Equal(1, job.Id);
repo.Verify(r => r.GetById(1), Times.Once);
41. What is code coverage, and why isn't 100% coverage the goal?

Coverage measures which lines/branches executed during tests — it tells you what wasn't tested, but says nothing about whether the assertions are meaningful. A test that calls a method with no assertions can hit 100% coverage and catch zero bugs. Use coverage to find untested gaps, not as a quality target in itself.

G · C++ & Problem Solving

Bonus skill, but a CAD/CAM shop's geometry kernel is plausibly C++ underneath a C# UI — expect at least one comparison question.

42. Key differences between C# and C++ you'd actually run into.

Memory: C++ is manual (new/delete, RAII, smart pointers) vs C#'s garbage collector. Pointers: raw pointer arithmetic is normal in C++, restricted to unsafe blocks in C#. Generics vs templates: C++ templates are compile-time code generation (duck-typed), C# generics are a first-class CLR construct with runtime type identity. Compilation: C++ compiles to native machine code directly; C# compiles to IL, JITted at runtime.

43. Where would C++ still make sense in a CAD/CAM product that's mostly C#?

Performance-critical geometry/kernel code — mesh processing, toolpath computation, nesting algorithms on large parts — where deterministic, allocation-free, low-level memory control matters. The common pattern: a native C++ computation core wrapped and exposed to a C# WPF UI via P/Invoke or C++/CLI.

44. Why does Big-O matter when your input is "a sheet-metal part's geometry file"?

Because real parts can have thousands of entities, and an O(n²) nearest-neighbor or intersection check that's fine on a toy part becomes visibly slow — or hangs — on a dense production drawing. Know the complexity of the collection operations you reach for (e.g. List.Contains is O(n); a HashSet lookup is O(1)) before it's a customer-facing performance bug.

45. How would you track down a memory leak in a long-running desktop app?

In managed code it's almost always an event handler or subscription that's never unsubscribed, keeping an object alive via a reference chain the GC can't break. Take heap snapshots (dotMemory, Visual Studio's memory profiler, or dotnet-gcdump) at two points in time and diff retained object counts to find what's growing; check for missing -= unsubscribes and undisposed IDisposables.

46. Quick coding check: reverse a singly linked list, iteratively.

O(n) time, O(1) space — walk the list once, flipping each Next pointer as you go.

public Node Reverse(Node head)
{
    Node prev = null;
    while (head != null)
    {
        Node next = head.Next;
        head.Next = prev;
        prev = head;
        head = next;
    }
    return prev;
}

Semantic structural tags (<header>, <nav>, <section>, <article>, <footer>) instead of generic <div> soup, native <audio>/<video>/<canvas> without plugins, new form input types (email, date, number, range) with built-in browser validation, and a simplified doctype: <!DOCTYPE html>.

Screen readers and search engines understand the page's structure (accessibility + SEO), and it's more self-documenting for other developers reading the markup. Functionally a <div> and a <section> render the same by default — the difference is entirely meaning, not layout.

55. localStorage vs sessionStorage vs cookies — quick difference?

    else if (n < second && n != smallest) { second = n; }

60. Recursively compute the total size of a folder, including all subfolders.

Natural follow-up to Challenge 5 — tests recursion, not just LINQ:

static long FolderSize(string path)
{
    var dir = new DirectoryInfo(path);
    long total = dir.GetFiles().Sum(f => f.Length);
    foreach (var sub in dir.GetDirectories())
        total += FolderSize(sub.FullName); // recurse into each subfolder

63. OOP design check: model Shape → Circle/Rectangle with polymorphic Area().

The JD lists "Object oriented design" as a basic skill by name — a live whiteboard version of this is a realistic ask:

abstract class Shape
{
    public abstract double Area();
}
class Circle : Shape
{
    public double Radius { get; set; }
    public override double Area() => Math.PI * Radius * Radius;
}
class Rectangle : Shape
{
    public double Width { get; set; }, Height { get; set; }
    public override double Area() => Width * Height;
}

67. Merge two already-sorted arrays into one sorted array (no full re-sort).

Classic two-pointer merge — O(n+m), the same building block as merge sort's combine step:

static int[] Merge(int[] a, int[] b)
{
    var result = new int[a.Length + b.Length];
    int i = 0, j = 0, k = 0;
    while (i < a.Length && j < b.Length)
        result[k++] = a[i] <= b[j] ? a[i++] : b[j++];
    while (i < a.Length) result[k++] = a[i++];
    while (j < b.Length) result[k++] = b[j++];
    return result;
}
