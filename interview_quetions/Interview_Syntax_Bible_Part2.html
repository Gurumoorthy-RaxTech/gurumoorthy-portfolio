🔒 Pillar 1 — Encapsulation (Data Hiding)
🎬 Anniyan (2005): Ambi's inner personalities hidden from outside world — only controlled methods expose behaviour. Encapsulation = private data, public methods only. Nobody can directly touch Ambi's "evil mode"!
Syntax — Access Modifiers
Encapsulation — Properties & Fields
public class Student
{
    // private field — outside access illai
    private string _name;
    private decimal _feeBalance;

    // public property — controlled access
    public string Name
    {
        get => _name;
        set
        {
            if (string.IsNullOrEmpty(value))
                throw new ArgumentException("Name required");
            _name = value;
        }
    }

    // read-only property
    public decimal FeeBalance => _feeBalance;

    // auto property (short form)
    public int    RollNo   { get; set; }
    public string Class    { get; set; }
    public bool   IsActive { get; private set; }

    // method — controlled mutation
    public void PayFee(decimal amount)
    {
        if (amount <= 0)
            throw new ArgumentException("Invalid amount");
        _feeBalance -= amount;
    }
}
🏫 ZenCampus Usage:
var s = new Student();
s.Name = "Guru";    // setter validates
s.PayFee(5000m);   // controlled method

// s._feeBalance = -999; ← compile error!

🧬 Pillar 2 — Inheritance (Code Reuse)
🎬 Magadheera (2009): Karthik — modern hero — inherits the warrior soul from previous life Ramaraju. Same core qualities, new capabilities added! Inheritance = child class gets parent's properties + can add/override its own behaviour.
Syntax
Base Class & Derived Class
// Base class (Parent)
public class Person
{
    public string Name    { get; set; }
    public string Email   { get; set; }
    public string Phone   { get; set; }

    public virtual string GetRole()
        => "Person";

    public void SendSms(string msg)
    {
        // SMS logic — reused by all
        SmsGateway.Send(Phone, msg);
    }
}

// Derived class (Child) — : Person
public class Student : Person
{
    public int    RollNo  { get; set; }
    public string Class   { get; set; }

    // override parent method
    public override string GetRole()
        => "Student";
}

public class Staff : Person
{
    public string Department { get; set; }
    public decimal Salary    { get; set; }

    public override string GetRole()
        => "Staff";
}
🏫 ZenCampus — Reuse SendSms:
var student = new Student
{
    Name  = "Guru",
    Phone = "9876543210",
    Class = "10A"
};
// SendSms inherited from Person!
student.SendSms("Fee due tomorrow");
Console.WriteLine(student.GetRole());
// Output: "Student"
Syntax
base keyword · Constructor Chaining
public class Person
{
    public string Name { get; }

    // Base constructor
    public Person(string name)
    {
        Name = name;
    }

    public virtual string Describe()
        => $"Name: {Name}";
}

public class Student : Person
{
    public int RollNo { get; }

    // Call base constructor with : base()
    public Student(string name, int roll)
        : base(name)   // calls Person(name)
    {
        RollNo = roll;
    }

    public override string Describe()
        // base.Describe() — call parent method
        => $"{base.Describe()}, Roll: {RollNo}";
}

// Usage
var s = new Student("Guru", 42);
Console.WriteLine(s.Describe());
// "Name: Guru, Roll: 42"
❓ Interview Q: What is base keyword?
base() — parent constructor call. base.Method() — parent method call from child. ZenCampus Student constructor-la Person constructor call panna : base(name) use panrom.
⚠️ C# Trap: Multiple Inheritance Not Allowed!
class A : B, C — compile error. C# allows only single class inheritance. Multiple interfaces is OK: class A : B, IFoo, IBar
🎭 Pillar 3 — Polymorphism (Many Forms)
🎬 Vikram (2022): Vikram — same name, different identities in different contexts. Polymorphism = same method name, different behaviour depending on the object type. Runtime-la yaar irukkaanganu decide aagum!
Runtime Polymorphism
Method Overriding — virtual / override
public class FeeCalculator
{
    // virtual = child override panalaam
    public virtual decimal Calculate(Student s)
        => 15000m; // default fee
}

public class HostelFeeCalc : FeeCalculator
{
    public override decimal Calculate(Student s)
        => 45000m;
}

public class BusFeeCalc : FeeCalculator
{
    public override decimal Calculate(Student s)
        => 20000m;
}

// Runtime polymorphism in action
FeeCalculator calc = student.Category switch
{
    "Hosteller" => new HostelFeeCalc(),
    "Bus"       => new BusFeeCalc(),
    _           => new FeeCalculator()
};

// Same method call — different result!
decimal fee = calc.Calculate(student);
❓ Interview Q: virtual vs override?
virtual = parent says "I can be overridden". override = child says "I am overriding". Both needed — oru side matum pottuchitta work aagathu!
Compile-Time Polymorphism
Method Overloading — Same Name, Different Params
public class ReportService
{
    // Overloading — same name, diff params
    public Report Generate(int studentId)
    {
        return GenerateForStudent(studentId);
    }

    public Report Generate(string className)
    {
        return GenerateForClass(className);
    }

    public Report Generate(
        int studentId, DateRange range)
    {
        return GenerateWithDates(studentId, range);
    }
}

// Caller — same method name!
_report.Generate(42);
_report.Generate("10A");
_report.Generate(42, dateRange);
🏫 ZenCampus Example:

❓ Interview Q: Overloading vs Overriding?
Overloading = same class, same name, different parameters — compile time. Overriding = parent-child, same signature — runtime. Interview-la table draw panni explain pannu!
Bonus
Operator Overloading + is / as Casting
// is — type check
Person p = new Student();
if (p is Student stu)
{
    Console.WriteLine(stu.RollNo);
}

// as — safe cast (null if fails)
Student? s = p as Student;
if (s is not null)
    Console.WriteLine(s.Class);

// Pattern matching (C# 9+)
switch (p)
{
    case Student { Class: "10A" } s10:
        Console.WriteLine("10A student!");
        break;
    case Staff staff:
        Console.WriteLine(staff.Department);
        break;
}
⚠️ Trap: Direct cast vs as
(Student)person — throws InvalidCastException if wrong type.
person as Student — returns null if wrong type. Safe!
🎨 Pillar 4 — Abstraction (Hide Complexity, Show Contract)
🎬 Enthiran (2010) — Chitti: Nee Chitti-ku "Walk" solluva — how the motor works, enna circuit-a use panra nu theriyaadu. Abstraction = "What it does" matum expose pannu, "How it does" hide pannu. Abstract class + Interface = contract define panrom!
Syntax
Abstract Class
// abstract class — cannot instantiate directly
public abstract class BaseReport
{
    // concrete method — all children inherit
    public void AddHeader(string title)
    {
        Console.WriteLine($"=== {title} ===");
    }

    // abstract method — child MUST implement
    public abstract void GenerateBody();

    // virtual method — child CAN override
    public virtual string GetFooter()
        => "ZenCampus School ERP";
}

public class AttendanceReport : BaseReport
{
    // Must implement abstract method
    public override void GenerateBody()
    {
        Console.WriteLine("Attendance data...");
    }
}

public class FeeReport : BaseReport
{
    public override void GenerateBody()
    {
        Console.WriteLine("Fee data...");
    }

    public override string GetFooter()
        => "Finance Dept — ZenCampus";
}

// new BaseReport(); ← compile ERROR!
BaseReport r = new AttendanceReport(); // OK
r.AddHeader("Monthly Report");
r.GenerateBody();
❓ Interview Q: Abstract class vs Interface?
Abstract class = partial implementation + shared state possible. Interface = pure contract, no state. Use abstract class when children share common code (BaseReport.AddHeader). Use interface for contracts across unrelated classes.
Syntax
Interface — Pure Contract
// Interface — I prefix convention
public interface IStudentRepository
{
    Task<Student>       GetByIdAsync(int id);
    Task<List<Student>> GetAllAsync();
    Task                AddAsync(Student s);
    Task                UpdateAsync(Student s);
    Task                DeleteAsync(int id);
}

// Concrete implementation
public class StudentRepository
    : IStudentRepository
{
    private readonly IDbConnection _db;

    public StudentRepository(IDbConnection db)
        => _db = db;

    public async Task<Student> GetByIdAsync(int id)
        => await _db.QueryFirstOrDefaultAsync<Student>(
               "SELECT * FROM Students WHERE Id=@Id",
               new { Id = id });

    // ... other methods
}

// DI Registration — Program.cs
builder.Services
    .AddScoped<IStudentRepository,
               StudentRepository>();
🏫 ZenCampus — Controller uses Interface:
public class StudentController
{
    private readonly IStudentRepository _repo;

    // depends on interface, not concrete class
    public StudentController(
        IStudentRepository repo)
        => _repo = repo;

    public async Task<IActionResult> Get(int id)
    {
        var student = await _repo.GetByIdAsync(id);
        return Ok(student);
    }
}

Syntax
sealed Class + static Class
// sealed — no inheritance allowed
public sealed class SchoolConfig
{
    public string SchoolCode { get; set; }
    public int    MaxStudents { get; set; }
}
// class MyConfig : SchoolConfig { } ← ERROR!

// static class — no instantiation, no inherit
public static class DateHelper
{
    public static string ToDisplay(DateTime d)
        => d.ToString("dd-MMM-yyyy");

    public static bool IsHoliday(DateTime d)
        => d.DayOfWeek == DayOfWeek.Sunday;
}

// Usage — no new() needed
DateHelper.ToDisplay(DateTime.Now);
DateHelper.IsHoliday(DateTime.Now);
❓ Interview Q: When to use sealed?
Security/integrity-critical classes — prevent subclass from overriding security logic. Performance — JIT can optimise sealed methods better. string itself is sealed in .NET!
C# 9+ Modern
record — Immutable Data Models
// record — immutable by default, value equality
public record StudentDto(
    int    Id,
    string Name,
    string Class,
    decimal FeeBalance
);

// Usage
var dto = new StudentDto(1, "Guru", "10A", 5000m);

// Value equality — records compare by value
var dto2 = new StudentDto(1, "Guru", "10A", 5000m);
Console.WriteLine(dto == dto2); // true!

// with — non-destructive mutation
var updated = dto with { FeeBalance = 0m };

❓ Interview Q: class vs record vs struct?
class = reference type, mutable, reference equality. record = reference type, immutable-friendly, VALUE equality. struct = value type, stack-allocated, small data. API DTOs → record. Domain entities → class. Small primitives → struct.
